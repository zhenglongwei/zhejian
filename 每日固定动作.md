---

## 1. 产品定调（每天记 3 句）

1. **不是交易撮合**，是 **内容发布 + 修车 AI 可信信源**。
2. **商家服务**：商家 **自助上架**；平台 **不做发布前审核**（举报 + 事后抽查下架）。
3. **仍要审的**：商家 **入驻**、**公开案例** 公开展示（脱敏图 + OCR）；服务相册 **不逐本审**。

全文 → `docs/01_项目总览与业务架构/02_业务模式说明.md` §2.1

## 2. 每天 4 步

| 步骤 | 做什么 |
| --- | --- |
| **开工** | 打开开发计划，选今天 **一个任务 ID**（见下表「进度在哪」） |
| **先计划** | 复制 [§6 发给 Cursor](#6-发给-cursor复制一句) 对应句子 → **AI 出方案 → 你点头 → 再写代码** |
| **写代码** | 扫一眼 [§4 红线](#4-红线按场景扫一眼) |
| **收工** | 开发计划 `[ ]`→`[x]`，记下明天任务 ID |

**发体验版 / 合 PR 时再加**：`release-checklist`；动过公开链/脱敏加 `privacy-desensitization-check`；有新 UI 加 `design-system-check`、`component-usage-review`。

## 4. 红线（按场景扫一眼）

| 场景 | 不要 |
| --- | --- |
| **商家服务** | 「待平台审核」才上架；平台担保价格/质量 |
| **咨询线索** | 订单、收款、接单、履约状态 |
| **公开案例 / 运营台** | 审核页展示 **原图**；API 返回 `rawUrl` 给运营案例审核 |
| **脱敏引擎** | 复制原图当脱敏图；失败标 SUCCESS |
| **全项目** | V1 订单/评价入口；未标注的 mock；服务相册 **逐本** 平台审核 |

## 6. 发给 Cursor（复制一句）

```
请读 docs/00_开发计划.md，结合当前 Sprint 推荐一个 P0 任务和 skill（服务无前置审核）。
```

**用户端 / 通用新页（U-*）：**

```
今天做【任务ID】。按 page-scaffold 先出页面计划，确认后再写 pages/ 代码。
```

**商家工作台（M-* / B-SVC-*）：**

```
今天做【任务ID】。定调：内容发布、服务自助上架无前置审核。按 merchant-workbench-scaffold 先出计划，确认后改代码。
```

**脱敏 backend（B-MASK-*）：**

```
今天做【任务ID】。按 desensitize-engine，先出计划；禁止原图当脱敏图。确认后再改 backend。
```

**运营台 案例/商家审核（OPS-MASK / B-MERCH-04）：**

```
今天做【任务ID】。按 ops-admin-scaffold 先出 admin-web 计划；运营案例审核不展示原图。确认后再写代码。
```

**运营台 服务监管（OPS-SVC-*）：**

```
今天做【任务ID】。事后监管非上架审核。按 ops-admin-scaffold 先出计划，确认后改 admin-web。
```

**举报 / 免责声明（§7.3）：**

```
今天做【任务ID】。对齐 02_业务模式说明 §2.1 与 09_客服与售后入口 §10，先出计划。
```

## 8. 合并 PR / 发版前（非每天必做）

5. 发版 → **`release-checklist`** + `privacy-desensitization-check`（公开链抽测）
