## **2. 模块定位**

「我的」是用户端个人服务中心，承载：

1. 登录与手机号绑定；
2. **我的咨询/预约**、**我的服务相册**、**我的公开发布**（历史名「我的公开授权」）；
3. 收藏公开案例与门店（P1，**仅供浏览参考**，非平台推荐）；
4. 车辆信息（P1 可选；P0 可在咨询表单填车型）；
5. 帮助反馈、隐私设置、账号安全；
6. 商家工作台 / 平台运营工作台入口（按权限展示）。

## **3.1 开发确认口径（2026-06-24 · UI-ALB-F）**

与 `02_工具入口首页.md` 历史规格冲突时，**以本节为准**。

### **3.1.1 入口与扫码**

| 项 | 规则 |
| --- | --- |
| App 内扫码 | **不做** 专门扫码按钮；依赖 **微信原生「扫一扫」** 打开门店码/分享卡片 |
| 单页说明 | 扫码与使用说明迁至 **设置 → 关于辙见**（`pages/mine/settings/about`）；空态不堆长文案 |
| 深链 | 扫码/分享仍直达相册详情（现有 `onLoad` 逻辑） |

### **3.1.2 信息隔离与 H5**

| 用户态 | H5 内容站出口 | 最近查看 |
| --- | --- | --- |
| 未登录 + 公域冷启动 + 无相册绑定 | ✅ 条件展示（`utils/tool-entry-context.js` · `shouldShowH5PublicCaseLink`） | 不展示 |
| 已登录 + 无相册绑定 | ✅ 空态主按钮「查看支持相册的商家 →」（H5 门店列表；非小程序内逛店） | 不展示 |
| 已登录 + 有相册绑定 | ❌ 不展示（降低跨店浏览，突出单店工具属性） | ❌ 不展示 |

### **3.1.3 荣誉鼓励（非 V1 收益中心 · 修订 2026-07-18）**

| 项 | 规则 |
| --- | --- |
| 定位 | 鼓励车主 **发布脱敏案例到公开网站** 与合规分享；用语为 **帮助同城车主、透明记录** 等荣誉/心理向 |
| 与「不做收益」关系 | **分享链接/卡片/按钮** 不得带现金、红包、分享有奖；平台 **不** 做物质分享奖励 |
| 门店线下 | 物质感谢由门店自行处理，平台不承诺、不结算、不在产品内诱导 |
| 呈现 | 弱卡片/说明条；**禁止** 首屏 warning 渐变 +「累计收益（元）—」占位 |
| 明细页 | `pages/mine/earnings/index` 若保留，仅规则说明；**不得** 做成分享领钱入口 |

### **3.1.4 待办与角标**

- 首屏 **待办聚合** 一块展示；无待办时 **整块不渲染**  
- 待办仅含：相册有更新、待发布到公开网站（**不含**站内消息条数）

### **3.1.5 微信订阅消息授权**

| 时机 | 行为 |
| --- | --- |
| **设置页（主入口）** | 登录后顶部 `ui-message-subscribe-banner` · 点击「开启微信通知」→ `requestSubscribeMessage` |
| **发布到公开网站提交后** | 相册列表/详情确认发布 → `requestUserNotificationSubscribe('authorize')`（实现事件名可暂保留） |
| **首页** | **不展示**合规说明条；**不展示**站内消息列表入口 |
| **合规说明** | 迁至 **设置页页脚** +「关于辙见」 |

微信一次性订阅：**每次用户点击授权 = 1 条发送额度**；额度用尽后需在设置中再次授权。

## **8. 我的公开发布**（原「我的公开授权」）

展示用户已提交发布或处理中的公开案例：

| 字段 | 说明 |
| --- | --- |
| 关联相册/案例 | 脱敏预览图 |
| 发布状态 | 待预览 / 已确认发布 / 已拒绝 / 已撤回 |
| 审核状态 | 待审核 / 已通过 / 已驳回 / 已下线 |
| 操作 | 查看服务相册、撤回发布（**含已在公开网站的案例**） |

**列表 Tag（Phase 1）**：**公开相册** / **审核中** / **私密相册**（与「我的服务相册」列表口径一致；不展示开发向「已授权」字样）。

### **8.1 撤回发布规则**

用户在 **任何时候** 均可撤回已发起的公开发布，**不受审核状态限制**（含待审核、已审核通过并已上公开网站）。

| 审核状态 | 是否可撤回 | 说明 |
| --- | --- | --- |
| 待审核 | ✅ | 审核队列中移除，案例不上网 |
| 已通过 / 已在公开网站 | ✅ | 公开案例立即下线（H5 noindex / 404，移出 sitemap） |
| 已驳回 | — | 未公开，无需撤回 |
| 已撤回 | — | 已从列表移除或展示「已撤回」 |

撤回后：私密服务相册仍保留；已产生的传播数据保留后台记录；搜索引擎缓存清除存在延迟（见 `10_隐私协议与授权规则.md` §10.2）。

规则详见 `docs/04_维修过程相册/03_服务相册流程.md` §7.5 与 `10_隐私协议与授权规则.md` §10。

## **14. 账号注销**

### **14.1 前置条件**

1. 无进行中的隐私/投诉工单（平台定义）；
2. 无未处理的授权争议（可选校验）；
3. 用户确认不可恢复。

**删除 V1.0 条件**：无进行中订单、无待结算收益。

### **14.2 注销后**

1. 账号不可用；
2. 咨询记录、相册查看权限按法规与隐私政策保留或匿名化；
3. 已公开且已授权案例按规则处理（可下线或匿名化）；
4. 本地登录态清除。

提示文案须说明：平台不提供交易与资金托管，注销不涉及退款处理。

## **18. P0 验收标准**

1. 未登录可浏览公开内容（H5 条件出口），个人数据入口触发登录（`tool-login-copy` 统一文案）；
2. 支持微信登录与手机号绑定；
3. 单页展示：**相册摘要 Hero**、分组 Nav Cell（授权/消息/车辆/设置/帮助/客服/商家）；
4. **不展示**订单、评价入口；**不做** App 内扫码按钮；
5. 荣誉鼓励为弱提示，**无** 首屏金额占位与 warning 钱包大卡、**无** 分享领钱；
6. 服务相册摘要 + 列表可进入详情；摘要为纵向 `ui-album-card`；
7. 可查看发布状态并支持撤回（**含已审核通过并已在公开网站的案例**）；
8. 联系客服 **非占位**；「关于辙见」含「微信扫一扫获取相册」说明；
9. 已登录有相册绑定 **不展示** H5 内容站出口；已登录无相册空态主按钮进 H5 门店列表；
10. 待办聚合：有待办时一块展示，**无**「暂无待处理」占位 pill；
11. 页脚合规说明 **唯一** 展示位；Nav Cell **禁止** 整行功能色底。
