## **2. 模块定位**

### **2.1 一句话**

小程序首页 = **服务相册工具入口**，不是平台发现页。

### **2.2 用户感知**

| 场景 | 用户应理解为 |
| --- | --- |
| 微信搜索首次进入 | 「这是一个查看门店为我创建的维修服务记录的工具」 |
| 商家分享相册/门店链接进入 | 「这是 XX 门店给我的相册/门店页」（单店上下文，见 DS-A-09～11） |
| 想找公开案例 / 比选门店 | 「应去 H5 内容站或公众号」，**不在** 小程序内完成 |

### **2.3 与旧首页的差异**

| 维度 | 旧 `02_首页与服务发现` | 本 PRD（工具入口） |
| --- | --- | --- |
| 定位 | GEO 引流 + 本地发现 | 相册查看 + 扫码/相册码入口 |
| 数据源 | `GET /user/home` 聚合推荐 | 本地最近访问 + H5 条件；相册摘要见「我的」Tab |
| Tab | 五 Tab（首页/服务/案例/门店/我的） | **首页 + 我的**（可选「相册」快捷 Tab） |
| 公域出口 | 小程序内浏览案例/门店 | 引导至 **H5 内容站** |

## **5. 首页整体结构**

> **职责边界（V1.1）**：首页 = **工具台**（获取/继续查看相册）；个人数据与相册摘要 = **「我的」Tab**（见 `12_我的页面与账户体系.md` §3）。

```text
工具 Hero（主标题 / 副标题；合规说明见「我的」页脚，首页不重复）
主操作区
  - 扫描门店二维码（primary）
  - 输入相册码（secondary；已登录时可单独占一行）
次操作区（仅未登录）
  - 我的服务相册 → LoginSheet（album 语义，见 constants/tool-login-copy.js）
最近查看（localStorage 有记录时，仅 1 条）
站外出口（条件展示）
  - 「想了解公开维修案例？前往辙见内容站」→ web-view 或复制 H5 链接
     （公域搜索冷启动 + 无相册绑定 + 无商家扫码史，见 utils/tool-entry-context.js）
底部 Tab：首页 | 我的
```

**不在首页展示（V1.1 迁出）**：

- 已登录用户的「我的服务记录」列表/空态 → **「我的」Tab 摘要区**
- 商家工作台、使用说明、联系客服 → **「我的」Tab 菜单**
- 页脚 `ComplianceNotice` → **仅「我的」Tab**

**模块优先级**（自上而下）：主操作 > 次操作（未登录）> 最近查看 > 站外出口。

## **6. 模块规格**

### **6.1 工具 Hero**

| 字段 | 规则 |
| --- | --- |
| 主标题 | **服务相册**（与 Phase1 口径一致；可用「维修服务记录」副语义，禁止旧品牌「维修档案/透明维修」） |
| 副标题 | 查看门店为你创建的维修服务记录 |
| 说明条 | **不在 Hero 内嵌**；`HOME_PLATFORM_IDENTITY` 仅在 **「我的」页脚** 展示（避免双 Tab 重复） |
| 禁止 | 促销 Banner、城市切换、搜索框跳转全站搜索、Hero 内嵌精选案例/门店 |

**推荐文案（P0）**：

- 身份说明（`HOME_PLATFORM_IDENTITY` 工具版，实现时写入 `constants/compliance-copy.js`）：  
  「辙见提供服务相册查看与咨询预约工具。实际维修、报价、收款与售后由门店线下提供和承担。」

### **6.2 主操作 · 扫描门店二维码**

| 项 | 规则 |
| --- | --- |
| 交互 | 点击 → `wx.scanCode` |
| 解析 | 支持门店码、相册邀请码、相册认领码（与 `packageMerchant/pages/album/invite`、`pages/album/claim` 现有链路对齐） |
| 成功 | 跳转对应相册详情 / 门店页 / 认领页 |
| 失败 | Toast「无法识别该二维码」；不 fallback 到首页推荐 |

实现任务：**DS-A-03**。

### **6.3 主操作 · 输入相册码**

| 项 | 规则 |
| --- | --- |
| 交互 | 点击 → 打开相册码输入（独立页 `pages/album/enter-code` **或** 首页 BottomSheet，二选一，P0 至少一种） |
| 校验 | 非空；格式错误友好提示 |
| 成功 | 跳转 `pages/album/detail`（或认领流程） |
| 禁止 | 输入框旁不出现「搜索全站案例/门店」 |

实现任务：**DS-A-04**。

### **6.4 次操作 · 我的服务相册（仅未登录）**

| 状态 | 展示 |
| --- | --- |
| 未登录 | 快捷入口 + `TOOL_GUEST_ALBUM_HINT` → `LoginSheet`（`bindContext=album`）；登录成功 **switchTab 至「我的」** |
| 已登录 | **不展示**该快捷入口；查看记录在「我的 → 我的服务记录摘要 / 我的服务相册」 |

### **6.5 最近查看（P1 · DS-A-05 ✅）**

| 项 | 规则 |
| --- | --- |
| 数据源 | `localStorage` / `wx.setStorage`；字段 `storeId`、`albumId`、`storeName`、`visitedAt` |
| 展示 | **仅 1 条**最近记录；文案示例：「最近查看：XX汽修 · 继续查看」 |
| 空态 | **不展示**该模块（不占位） |
| 禁止 | 展示多条、跨店推荐列表 |

### **6.6 我的服务记录（已登录）— 已迁至「我的」Tab**

| 项 | 规则 |
| --- | --- |
| 展示位置 | **`pages/mine/index`** 用户信息区下方（摘要 1～3 条 +「查看全部」+ 待授权角标） |
| 数据源 | `GET /user/mine/summary`（`recentAlbums` + 角标；**不再**单独请求 service-albums 列表） |
| 首页 | **不得**再展示相册列表或空态卡片 |

实现：`pages/mine/index.*` + `ui-album-card`（**DS-A-15**）。

### **6.7 站外出口 · H5 内容站（P1 · DS-A-06 ✅）**

| 项 | 规则 |
| --- | --- |
| 文案 | 「想了解公开维修案例？前往辙见内容站」 |
| 行为 | 复制链接 `https://geo.simplewin.cn/`（或业务常量 `constants/h5-links.js`）+ 提示浏览器打开；**或** 企业主体允许时使用 `web-view` |
| 说明 | 公开案例、门店列表、GEO 专题在 **站外** 浏览；与 Phase1 §4.5 授权公示渠道一致 |

### **6.8 辅助入口 — 已迁至「我的」Tab**

| 入口 | 规则 |
| --- | --- |
| 我是商家 | **不在首页**；「我的 → 商家工作台」 |
| 联系客服 | **不在首页**；「我的 → 联系客服」→ `utils/support-contact.js`（**DS-A-16**） |
| 使用说明 | **不在首页**；「我的 → 使用说明与帮助」→ `constants/help-content.js` |

## **7. 禁止模块（红线）**

以下模块 **不得** 出现在工具入口首页（设计体系 + 08 计划 §4.2）：

- 附近门店、精选案例、推荐服务、GEO 专题横滑
- 城市定位切换（用于发现）；小程序不采集用户实时位置（**DS-A-08**：仅商家 `chooseLocation` 选点）
- 跨店搜索入口（原 `ui-search-bar` 跳转 `pages/search`）
- 六宫格服务分类入口（原 `HOME_SERVICE_ENTRIES`）
- 事故车重点入口卡片（引导去 H5 或深链，不在冷启动首页主推）
- 任何「浏览全部门店 / 全部案例 / 查看更多门店」文案或按钮
- 调用 `GET /user/home` 渲染推荐模块

## **14. P0 验收标准**

1. 微信搜索冷启动：见 **§5 工具首页**，**无** 门店/案例/GEO 推荐模块。  
2. 可扫码 / 输入相册码进入有效链路。  
3. **未登录**「我的服务相册」→ LoginSheet → 登录后进入「我的」Tab。  
4. **已登录**首页 **无** 相册列表/空态/「我的服务相册」快捷格。  
5. Tab 仅 **首页 + 我的**。  
6. H5 内容站出口（条件展示）可用。  
7. 商家/客服/帮助 **不在首页**，在「我的」菜单。  
8. 分享深链不被工具首页拦截。  
9. 文案通过 §2.4 自检。
