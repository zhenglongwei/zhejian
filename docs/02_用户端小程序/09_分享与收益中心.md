## 2. 模块定位

分享与收益中心用于承接平台内容传播、用户推荐和奖励结算。

核心目标：

1. 支持用户分享服务、门店、公开案例；
2. 通过合规方式记录传播归因；
3. 为用户展示收益、奖励和记录；
4. 支持公开案例传播；
5. 支持推荐新用户或订单转化奖励；
6. 避免诱导分享、诱导点击、诱导传播等违规风险；
7. 建立可追踪、可审核、可风控的收益体系。

本模块必须遵守以下原则：

1. 不在分享文案中展示诱导奖励；
2. 不使用“点击得钱”“转发赚钱”“拉人头赚钱”等表达；
3. 不强制用户分享；
4. 不以分享作为使用核心功能的条件；
5. 不诱导用户分享到群；
6. 不诱导好友助力；
7. 不做多级分销；
8. 不做无限层级返佣；
9. 收益规则必须透明；
10. 收益发放必须可追溯。

## **9. 传播归因**

### **9.1 归因目标**

记录用户分享后带来的：

1. 页面访问；
2. 新用户注册；
3. 服务收藏；
4. 门店咨询；
5. 预约提交；
6. 订单支付；
7. 订单完成；
8. 公开案例浏览。

### **9.2 分享参数**

分享路径建议携带：

```
share_code
source_user_id，加密或短码
content_type
content_id
channel
timestamp

```

示例：

```
/pages/case/detail?id=case_001&share_code=abc123

```

### **9.3 归因规则**

P0 建议支持单层归因：

```
A 分享 → B 访问/注册/下单 → A 获得对应奖励

```

不支持：

```
A 分享 → B 分享 → C 下单 → A、B 多级分佣

```

### **9.4 归因有效期**

建议：


| 行为       | 有效期     |
| -------- | ------- |
| 浏览归因     | 7 天     |
| 注册归因     | 15 天    |
| 下单归因     | 30 天    |
| 完成订单奖励归因 | 30-60 天 |


具体以平台规则为准。

### **9.5 归因优先级**

如用户被多个分享来源触达，可采用：

1. 最近一次有效分享；
2. 首次有效分享；
3. 内容类型优先；
4. 平台活动优先。

P0 建议：

```
最近一次有效分享归因。

```

9. 符合平台奖励规则。

2. 维修档案生成；
3. 用户授权公开；
4. 平台完成脱敏；
5. 案例审核通过；
6. 案例质量达标；
7. 不存在售后争议；
8. 不存在虚假内容；

## **15. 提现与使用**

### **15.1 使用方式**

平台可配置：

1. 提现到微信零钱；
2. 抵扣订单金额；
3. 兑换优惠券；
4. 兑换积分；
5. 仅作为平台余额使用。

P0 建议优先支持：

```
订单抵扣 或 优惠券发放

```

提现涉及更高财务、税务、合规和风控要求，可放 P1。

### **15.2 提现条件**

如支持提现，需满足：

1. 实名认证，视规则；
2. 达到最低提现金额；
3. 收益为可用状态；
4. 账号无风控限制；
5. 符合微信支付和平台规则；
6. 税务处理符合要求；
7. 不涉及争议订单。

### **15.3 最低提现金额**

示例：

```
满 ¥10 可提现

```

### **15.4 提现状态**


| 状态                 | 说明    |
| ------------------ | ----- |
| withdraw_pending   | 提现申请中 |
| withdraw_reviewing | 审核中   |
| withdraw_success   | 提现成功  |
| withdraw_failed    | 提现失败  |
| withdraw_rejected  | 提现拒绝  |


### **15.5 提现提示**

```
提现申请提交后，实际到账时间以微信支付及银行处理时间为准。

```

## **17. 退款与收益回收**

### **17.1 回收场景**

收益可被取消或回收：

1. 推荐订单取消；
2. 推荐订单退款；
3. 推荐订单未完成；
4. 用户虚假下单；
5. 用户刷单；
6. 商家刷单；
7. 内容违规下架；
8. 评价删除或审核不通过；
9. 公开案例撤回或下架；
10. 平台风控判定异常。

### **17.2 回收规则**


| 收益状态  | 处理            |
| ----- | ------------- |
| 待结算   | 直接取消          |
| 可用未使用 | 扣回            |
| 已抵扣   | 产生负向记录或限制后续收益 |
| 已提现   | 进入人工处理/追偿，视规则 |
| 冻结中   | 审核后决定         |

## **18. 风控规则**

### **18.1 风控场景**

1. 同设备多账号互邀；
2. 同手机号段批量注册；
3. 同一支付账户多账号；
4. 同一车辆重复异常下单；
5. 短时间大量分享转化；
6. 异常退款率；
7. 订单完成后立即退款或售后；
8. 虚假评价；
9. 虚假公开案例；
10. 商家与用户合谋刷单；
11. 使用脚本刷浏览；
12. 非自然传播流量。

### **18.2 风控处理**

1. 收益冻结；
2. 延迟结算；
3. 人工审核；
4. 取消收益；
5. 限制提现；
6. 限制分享归因；
7. 账号处罚；
8. 商家处罚；
9. 黑名单处理。

## **21. 与公开案例的关系**

公开案例是适合分享传播的核心内容。

流程：

```
订单完成
→ 维修档案生成
→ 用户授权公开
→ 平台脱敏审核
→ 生成公开案例
→ 用户可分享案例
→ 平台记录传播归因
→ 符合条件发放奖励

```

注意：

1. 私密维修相册不可分享；
2. 公开案例必须脱敏；
3. 公开案例不展示实际支付金额；
4. 事故车案例不得展示线上最终报价；
5. 分享文案不得诱导奖励。

## **22. 与评价奖励的关系**

评价奖励和分享收益需区分。


| 类型     | 触发条件        | 是否依赖分享 |
| ------ | ----------- | ------ |
| 评价奖励   | 提交真实评价并审核通过 | 否      |
| 公开案例奖励 | 授权公开且审核通过   | 否/可选   |
| 分享收益   | 分享后产生有效转化   | 是      |


不得将评价奖励包装为：

```
好评返现

```

不得将分享收益包装为：

```
拉人头返佣

```

2. 不得诱导点击；
3. 不得诱导好友助力；

8. 不得分享私密订单和相册；
9. 公开内容必须审核和脱敏；
10. 收益规则必须可查看；
11. 提现规则必须清晰；
12. 风控处理必须留痕；
13. 退款后收益需按规则取消或回收；

## **30. 验收标准**

1. 用户可分享服务、门店和公开案例；
2. 私密订单不可分享；
3. 私密维修相册不可分享；
4. 分享路径带有效分享参数；
5. 被分享用户访问后可记录归因；
6. 收益中心可展示累计、可用、待结算收益；
7. 收益明细状态准确；
8. 订单退款后相关收益可取消或回收；
9. 风控命中后收益可冻结；
10. 分享文案不出现诱导奖励表达；
11. 不支持多级分销；
12. 公开案例必须审核脱敏后才能分享；
13. 提现或使用规则可查看；
14. 被邀请用户隐私脱敏展示；
15. 收益规则清晰可查。
