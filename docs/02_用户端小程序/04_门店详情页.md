## **2. 模块定位**

门店详情是用户选择维修承接方的核心页面。

目标：

1. 展示经审核的门店真实信息与资质；
2. 展示擅长项目与服务方案（参考价格）；
3. 展示脱敏公开案例与服务相册能力；
4. 引导 **电话 / 导航 / 提交咨询 / 预约到店**；
5. 避免未经审核的品牌授权、资质与宣传误导用户。

信任信号优先级：**资质 + 脱敏案例 + 服务相册能力 + 价格说明 + 营业态/新鲜度 + 已审核能力墙**，而非交易评分。

**卷十一补充**：活店感（营业态、临时休息、内容新鲜度）与能力感（技师、设备、擅长品牌、暂不承接、授权有效期）。见 `docs/03_商家端/07_门店数据资产A_B开发计划.md`。

## **8. 门店基础信息**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 门店名称 | 是 | |
| 资质标签 | 是 | |
| 平台审核状态 | 是 | 已审核/认证中 |
| 地址 | 是 | |
| 距离 | 否 | 授权后 |
| 营业时间 | 是 | |
| 联系电话 | 是 | 可虚拟号，点击拨打 |
| 公开案例数 | 否 | |
| 支持服务相册 | 否 | 标签 |
| 品牌授权 | 否 | 审核通过后 |

### **8.1 营业状态**

| 状态 | 文案 | 计算口径（卷十一） |
| --- | --- | --- |
| open | 营业中 | 在营业时段内，且不在临时休息日，且门店未暂停/下线 |
| closed | 休息中 | 当日不在营业时段，或命中临时休息 |
| holiday | 节假日休息 | 临时休息注明节假日，或运营标记 |
| suspended | 暂停预约 | 运营/商家暂停预约（可浏览） |
| offline | 暂不可预约 | 未上线或下架 |

**禁止**：将数据库门店上线态（如 `ACTIVE`）直接展示为「营业中」。筛选「营业中」须与实时 `businessStatus === open` 一致。

不可预约时：

```text
该门店当前暂不可预约，你可以查看其他门店或先浏览案例。
```

### **8.2 内容新鲜度（卷十一）**

详情页可展示一句新鲜度说明（有数据才展示），例如：

```text
最近公开案例：2026-07-10 · 资料最近核实：2026-07-12
```

数据来自平台统计的最近公开案例时间、能力/资质最近审核通过时间，**不得**由商家手填「今日更新」类口号。

## **10. 门店资质**

### **10.1 须展示**

营业执照认证、维修资质类型、门头/工位真实性审核、平台审核状态、地址。

### **10.2 资质字段**

| 字段 | 说明 |
| --- | --- |
| 营业执照状态 | 已认证/认证中/未认证 |
| 维修资质类型 | 一类/二类/三类/专项 |
| 资质有效期 | 须维护；过期隐藏认证标签并降推荐（见 §19） |
| 门店真实性审核 | 已审核 |
| 品牌授权 | 可选，须凭证 + **有效期** |

### **10.3 品牌授权**

非入驻必要条件；仅审核通过且在有效期内展示「品牌授权：已认证」。未通过或过期时不展示「无品牌授权」，也不展示已过期授权。

### **10.4 禁止宣传**

未经审核的原厂授权、4S 同等、官方指定、全网最低、保证修复、保险指定（无证明）等。

## **11. 擅长项目**

标签示例：钣喷修复、事故车、刹车系统、小保养、新能源检测。

**数据来源（MVP）**：商家填写 + 平台审核；可参考公开案例数量做展示排序。

**删除 V1.0**：历史订单统计、用户评价标签。

点击 → 该门店服务列表 / 服务详情 / 案例筛选（本店）。

### **11.1 擅长品牌与暂不承接（卷十一）**

| 字段 | 说明 |
| --- | --- |
| 擅长品牌 | 商家维护，即时展示；须真实，虚假走举报 |
| 暂不承接 | 短标签或短句（如「暂不承接大型货车」），即时展示；空则不展示区块 |

### **11.2 技师与设备能力墙（卷十一）**

| 区块 | 展示条件 | 字段 |
| --- | --- | --- |
| 技师公示卡 | **平台审核通过**后才展示 | 称呼、主修方向、年限、资质标签 |
| 设备/场标签 | **平台审核通过**后才展示 | 如烤房、四轮定位、诊断电脑、新能源工位等 |

未审核、已驳回、已过期授权：**不展示**对应认证或能力条目，不得用灰态暗示「未认证」。

品牌授权须带有效期；过期自动取消「品牌授权」展示。

## **14. 服务相册能力说明（P0 简版）**

若门店启用服务相册，在详情页展示说明条：

```text
本门店支持服务相册，线下维修后可记录检测、施工、完工等阶段（需门店创建并关联你的手机号）。
```

不展示其他用户的私密相册；仅说明能力与公开案例/相册规则。用户已有相册从「我的服务相册」进入。

**硬规则（卷十一）**：**进行中相册不得出现在门店公开面**（详情、列表、分享、H5、Feed）。可公示内容仅限已完工且用户已授权（或商家历史案例）并审核通过的案例。

透明度中的「过程资料齐全度」为已完工相册的聚合比例，**不代表**可浏览他人进行中过程，不得外链私密相册。

---

## **14.1 透明度指标（卷十一加固）**

替代交易评价的信任表达。有公开案例时才展示总分与分项百分数（诚实空态）。

建议分项：公开案例数、过程资料齐全度（已完工聚合）、可预约服务完整度、资质核验（含有效期）、**内容新鲜度**；能力墙完善度可轻量计入。

**咨询响应**（近 7 日线索已联系比例）：仅商家数据看板与透明度**内部维**，**不对用户端 / H5 展示**具体数字或「响应很快」口号。

## **15. 用户评价（V2.0 不做）**

V2.0 **不展示**门店交易评价列表、星级汇总、六维评分。

信任替代：资质、脱敏案例、服务相册能力、价格透明说明。

P1（可选，非 P0）：独立「过程反馈」轻量问卷（无现金、无公开星级榜），若做须单独 PRD，**不在**门店详情主结构占位。

**删除 V1.0**：评价区、评分计算、评价 API、无评价空态引导「完成服务后可评价」。

## **16. 咨询/预约入口（底栏）**

### **16.1 默认底栏**

```text
联系门店 | 导航 | 预约到店 / 提交咨询
```

从服务详情带门店进入时，主按钮可为「选择该门店并咨询」（跳转 `05_咨询与到店预约.md`，预填）。

### **16.2 按场景**

| 场景 | 行为 |
| --- | --- |
| 标准服务 | 提交咨询/预约到店 |
| 复杂维修 | 预约到店检测表单 |
| 事故车 | 预约检测 + 勾选到店报价条款 |

**禁止**：订单确认页、微信支付、立即下单。

### **16.3 登录**

浏览门店无需登录。提交咨询/预约、收藏（P1）须登录并绑定手机号。

**标题示例**：

## **19. 异常与边界**

| 场景 | 处理 |
| --- | --- |
| 下架 | 列表不展示；深链提示不可查看/不可预约 |
| 冻结 | 同下架，不推荐 |
| 资质过期 | 隐藏认证标签、暂停推荐、可暂停预约 |
| 无案例 | 「该门店暂无公开案例」→ 引导看服务或联系门店 |
| 暂停营业 | 「该门店当前暂停营业，暂不可预约。」 |

## **20. 权限与隐私**

定位仅用于距离、排序、导航辅助，不公开用户位置。

商家联系人身份证、后台资料不对用户展示。

公开案例与图片须脱敏；分享仅脱敏已审核内容。

---

## **21. 合规要求**

**资质**：真实有效，过期降级。**宣传**：禁止虚假评分（V2.0 不展示评分）、虚假案例数、事故车线上精准报价。**案例**：来源明确、区间价、无支付金额。**平台身份**（详情页可见）：

```text
平台展示门店信息与脱敏案例，不提供线上最终报价，不参与线下维修、收款与售后。实际服务由你与门店线下确认。
```

3. 擅长项目 + 服务方案（参考价）；
4. 公开脱敏案例；
5. 服务相册能力说明（若启用）；
6. 咨询/预约底栏，无支付路径；
7. 下架/冻结/暂停/资质过期处理；
8. 分享无奖励文案；
9. **无**交易评价区与评分。

---

## **26. P1 优化**

地图找店、在线咨询、门店收藏/对比、预约日历、视频介绍、配件品牌、保险合作展示、轻量过程反馈（若单独立项）、到店打卡。

**卷十一一期已覆盖**：技师介绍、设备/场标签、擅长品牌、暂不承接、营业态与新鲜度（见 §8、§11、§14.1）。  
**卷十一二期第一批**：已审设备实景图并入环境轮播（§7）；资质/授权过期仅影响排序与运营催办，详情页不展示羞辱文案。可约档期仍属后续二期。

## **27. P0 验收标准**

1. 可进入列表与详情，可按服务筛门店；
2. 展示名称、地址、营业时间、资质状态；
3. 未审核资质不展示认证标签；
4. 定位授权后可展示距离，拒绝定位仍可浏览；
5. 可导航、拨打电话；
6. 可查看服务方案并进入服务详情或咨询表单；
7. 案例已脱敏、来源标签正确，无实际支付金额；
8. 事故车相关服务无线上最终价；
9. 下架/冻结门店不出现在列表且不可预约；
10. 底栏无下单/支付按钮；
11. 无交易评价模块；
12. 分享卡片无诱导奖励；
13. 异常态文案明确。
