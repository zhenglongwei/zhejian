# **《咨询与到店预约》**

> **当前版本：V2.0**

---

## **1. 文档信息**

| 项目 | 内容 |
| --- | --- |
| 所属模块 | 用户端小程序 |
| 对应主文档 | `01_用户端小程序PRD V2.0.md` |
| 文档目标 | 定义用户向门店提交咨询/预约线索的流程、字段与合规提示 |

---

## **2. 流程**

```text
用户查看服务 / 案例 / 门店
→ 点击联系门店 / 预约到店
→ 填写咨询信息
→ 提交给门店
→ 门店线下联系用户
→ 用户自行决定是否到店、是否维修
```

---

## **3. 表单字段**

| 字段 | 必填 |
| --- | --- |
| 服务类型 | 是 |
| 门店 | 是 |
| 手机号 | 是 |
| 车辆品牌/车系 | 建议 |
| 问题描述 | 建议 |
| 上传图片 | 否 |
| 期望到店时间 | 否 |
| 同意将信息发送给门店 | 是（勾选） |

---

## **4. 提交前提示**

提交按钮上方：

> 平台将把你的咨询信息发送给所选门店，便于门店与你联系。平台不提供线上最终报价，不参与线下维修服务、收款、发票、质保和售后。实际维修方案和费用需你与门店线下确认。

用户须勾选：

> 我已知晓实际维修服务由门店线下提供，平台仅提供信息展示和咨询转接服务。

---

## **5. 事故车特殊说明**

> 事故车维修无法仅凭线上图片准确报价。平台不提供线上最终报价。实际维修方案和费用需门店检测、拆检或定损后与你线下确认。你可以先查看类似案例，了解常见维修项目和费用影响因素。

须勾选：我已知晓事故车需到店检测后由门店报价。

---

## **6. 按钮文案**

推荐：联系门店、预约到店、咨询报价、提交咨询。

禁止：立即购买、提交订单、在线支付、确认下单。
