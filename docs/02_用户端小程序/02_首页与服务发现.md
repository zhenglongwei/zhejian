# 《首页与服务发现 PRD》

---

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 文档名称 | 首页与服务发现 PRD |
| 所属模块 | 用户端小程序 |
| 当前版本 | V1.0 |
| 对应主文档 | 01_用户端小程序PRD.md |
| 适用端 | 微信小程序用户端 |
| 首发城市 | 杭州 |
| 页面范围 | 首页、搜索、服务发现、案例推荐、门店推荐、事故车入口 |
| 文档目标 | 细化用户进入小程序后的首页结构、服务发现路径、搜索规则、推荐展示规则和合规要求 |

---

## 2. 模块定位

首页与服务发现模块是用户端小程序的第一入口，负责完成以下目标：

1. 让用户快速理解平台定位；
2. 引导用户找到需要的维修服务；
3. 通过真实案例和门店信息建立信任；
4. 为标准服务下单、复杂维修预约、事故车检测预约提供入口；
5. 为案例、门店、服务详情页导流；
6. 支持杭州本地服务发现；
7. 避免线上误导报价，尤其是事故车维修场景。

首页不是单纯的信息展示页，而是用户端核心交易链路的起点。

---

## 3. 与主 PRD 的关系

本文档是 `01_用户端小程序PRD.md` 中首页、服务发现、搜索、案例推荐、门店推荐相关内容的细化说明。

如本文档与主 PRD 存在冲突：

```text
以 01_用户端小程序PRD.md 的产品方向为准；
以本文档的页面字段、交互规则和异常处理作为开发落地依据。

```

---

## **4. 用户目标**

用户进入首页后，需要完成以下典型任务：

1. 了解平台是否可信；
2. 快速找到自己需要的维修服务；
3. 查看类似维修案例；
4. 查看附近或推荐门店；
5. 判断服务价格是否透明；
6. 判断是否可以线上下单或需要到店检测；
7. 对事故车、钣喷、复杂维修有正确预期；
8. 进入下单、预约、咨询或案例浏览流程。

---

## **5. 首页整体结构**

首页建议采用以下结构：

```
顶部定位城市
搜索框
平台主标语
核心服务入口
事故车/钣喷重点入口
附近门店
精选案例
透明维修说明
用户保障说明
底部导航

```

页面从上到下优先级：

1. 城市与搜索；
2. 平台定位；
3. 高频服务入口；
4. 事故车/钣喷重点入口；
5. 附近门店；
6. 精选案例；
7. 平台说明与保障。

## **6. 页面入口**

用户可通过以下方式进入首页：

1. 打开小程序默认进入；
2. 底部导航点击“首页”；
3. 从分享卡片进入后点击返回首页；
4. H5 页面跳转小程序首页；
5. 服务通知跳转小程序首页；
6. 搜索结果为空时返回首页；
7. 订单、案例、门店页面点击平台 Logo 或首页入口。

## **7. 首页顶部区域**

### **7.1 城市定位**

#### **7.1.1 展示内容**

首页顶部展示当前服务城市：

```
杭州

```

MVP 阶段默认城市为杭州。

#### **7.1.2 定位规则**


| 场景       | 处理方式               |
| -------- | ------------------ |
| 用户首次进入   | 可请求定位授权            |
| 用户同意定位   | 获取当前位置，判断是否在杭州服务范围 |
| 用户拒绝定位   | 默认展示杭州             |
| 用户定位在杭州  | 正常展示杭州服务           |
| 用户定位不在杭州 | 提示当前服务暂以杭州为主       |
| 定位失败     | 默认杭州，不阻断使用         |


#### **7.1.3 提示文案**

用户不在杭州时，提示：

```
当前服务暂以杭州为主，你仍可浏览案例和服务信息。

```

定位失败时，提示：

```
定位失败，已为你默认选择杭州。

```

用户拒绝定位时，不强制再次弹窗，可在城市区域提供“重新定位”。

#### **7.1.4 MVP 限制**

MVP 不开放多城市切换。

城市选择页可以暂不做，点击城市时可展示说明：

```
当前平台首发服务城市为杭州，更多城市陆续开放。

```

6. 事故车相关搜索结果不得展示线上最终价；

## **11. 事故车/钣喷重点入口**

### **11.1 模块定位**

事故车和钣喷服务是平台重要的服务发现入口，但必须避免线上承诺最终维修价格。

该模块目标是：

1. 引导用户预约门店检测；
2. 展示类似案例；
3. 说明价格影响因素；
4. 明确事故车不能线上最终报价；
5. 降低用户对复杂维修价格的误解。

### **11.2 展示文案**

推荐主文案：

```
事故车维修预约
不线上估价，到店检测后报价

```

副文案：

```
可查看类似案例，了解维修方案和费用影响因素

```

### **11.3 按钮**


| 按钮       | 跳转         |
| -------- | ---------- |
| 查看事故车案例  | 案例列表，筛选事故车 |
| 预约门店检测   | 事故车预约流程    |
| 查看价格影响因素 | 事故车说明页，可选  |


### **11.4 必须展示的提示**

在模块内或点击进入后必须展示：

```
事故车维修无法仅凭线上图片准确报价，实际维修方案和费用需门店检测或拆检后确认。

```

### **11.5 禁止展示内容**

事故车入口禁止展示：

1. 线上一口价；
2. 最低价承诺；
3. 保证报价；
4. “上传照片立即精准报价”；
5. “全网最低”；
6. “原厂授权”虚假宣传；
7. 具体用户订单金额。

## **12. 附近门店模块**

### **12.1 模块目标**

通过展示本地可信门店，帮助用户选择维修服务承接方。

### **12.2 展示条件**

首页默认展示杭州门店。

排序可综合：

1. 用户距离；
2. 门店评分；
3. 资质认证；
4. 案例数量；
5. 透明维修能力；
6. 服务匹配度；
7. 平台运营推荐。

MVP 可先使用运营推荐 + 距离排序。

### **12.3 门店卡片字段**


| 字段     | 是否必填 | 说明            |
| ------ | ---- | ------------- |
| 门店名称   | 是    | 商家后台提交，平台审核   |
| 门头图    | 是    | 已审核图片         |
| 地址     | 是    | 门店地址          |
| 距离     | 否    | 用户授权定位后展示     |
| 营业时间   | 是    | 如 09:00-18:00 |
| 资质标签   | 是    | 一类/二类/三类/专项   |
| 擅长项目   | 否    | 如钣喷、保养、事故车    |
| 案例数量   | 否    | 公开案例数量        |
| 评分     | 否    | 用户评价得分        |
| 透明维修标签 | 否    | 如“支持维修相册”     |
| 品牌授权   | 否    | 有则展示，无则不展示    |


### **12.4 门店卡片点击**

点击门店卡片进入：

```
门店详情页

```

### **12.5 门店快捷操作**

可选操作：

1. 查看门店；
2. 导航；
3. 电话咨询；
4. 预约服务。

MVP 建议首页卡片只保留“查看门店”，避免操作过重。

### **12.6 空状态**

无可展示门店时：

```
当前暂无可预约门店
你可以先查看维修案例，或联系客服了解服务开放情况。

```

### **12.7 门店展示限制**

以下门店不得在首页展示：

1. 未通过平台审核；
2. 已冻结；
3. 已下架；
4. 资质过期；
5. 服务能力不匹配；
6. 被风控限制推荐；
7. 存在严重投诉未处理。

## **13. 精选案例模块**

### **13.1 模块目标**

精选案例用于建立用户信任，让用户在下单前看到真实维修过程和参考方案。

### **13.2 案例来源**

可展示：

1. 平台订单案例；
2. 用户授权公开案例；
3. 商家历史案例，需商家责任确认；
4. 平台审核通过的优质案例。

### **13.3 案例卡片字段**


| 字段   | 是否必填 | 说明             |
| ---- | ---- | -------------- |
| 主图   | 是    | 必须为脱敏图         |
| 标题   | 是    | 车型 + 服务项目      |
| 服务项目 | 是    | 如钣喷修复          |
| 维修摘要 | 是    | 简短描述           |
| 价格区间 | 是    | 不展示实际支付金额      |
| 门店名称 | 否    | 关联门店           |
| 标签   | 是    | 已脱敏/已审核/平台订单案例 |
| 发布时间 | 否    | 可选             |
| 浏览量  | 否    | 可选             |


### **13.4 案例卡片点击**

点击进入：

```
小程序内案例详情页

```

### **13.5 案例排序**

MVP 可采用运营推荐。

后续可综合：

1. 案例质量；
2. 图片完整度；
3. 服务项目热度；
4. 车型常见度；
5. 用户浏览量；
6. 门店资质；
7. 更新时间；
8. 是否平台订单案例。

### **13.6 案例展示限制**

首页精选案例必须满足：

1. 已审核通过；
2. 图片已脱敏；
3. 文本无隐私；
4. 不展示用户手机号；
5. 不展示车牌；
6. 不展示 VIN；
7. 不展示身份证、行驶证；
8. 不展示实际支付金额；
9. 不包含夸大宣传；
10. 事故车案例仅展示参考区间和影响因素；
11. H5 收录状态不影响小程序内展示，但公开页需遵守 H5 规则。

### **13.7 价格展示规则**

案例卡片只展示价格区间：

```
参考区间：¥300-¥1200

```

或：

```
到店检测后确认

```

禁止展示：

```
实际支付 ¥xxx
成交价 ¥xxx
最低只要 ¥xxx

```

### **13.8 空状态**

无精选案例时，隐藏模块或展示：

```
精选案例正在整理中
你可以先查看服务项目或附近门店。

```

MVP 推荐直接隐藏模块，避免首页显得空。

## **14. 透明维修说明模块**

### **14.1 模块目标**

解释平台的核心价值：

```
修车过程看得见，每次维修有档案

```

### **14.2 展示内容**

建议展示 3-4 个说明点：

1. 维修过程相册；
2. 平台订单生成维修档案；
3. 案例公开前脱敏审核；
4. 用户可评价和申请售后。

### **14.3 推荐文案**

```
维修过程看得见
门店可上传检测、施工、完工等过程图片，你可以在订单中查看维修进度。

每次维修有档案
平台订单会沉淀为维修档案，方便后续复查、维权、卖车和再次维修参考。

公开案例先脱敏
公开展示前会隐藏车牌、人脸、VIN、手机号等隐私信息。

```

### **14.4 点击行为**

可点击进入：

```
透明维修说明页

```

MVP 可以不单独做详情页，点击后弹窗展示说明。

## **15. 用户保障说明模块**

## **19. 权限与隐私**

### **19.1 定位权限**

定位权限仅用于：

1. 计算附近门店距离；
2. 优化门店排序；
3. 判断是否在服务城市。

不得用于：

1. 未经授权的精准营销；
2. 对外展示用户位置；
3. 写入公开案例；
4. 分享页面透出用户位置。

### **19.2 搜索隐私**

用户搜索关键词中可能包含车牌、手机号、VIN 等敏感信息。

系统应：

1. 避免在前台回显敏感完整信息；
2. 搜索日志中敏感信息应脱敏或限制保存；
3. 不将敏感搜索词用于公开推荐；
4. 不基于敏感词生成公开内容。

### **19.3 案例隐私**

首页所有案例必须使用：

```
脱敏图
脱敏文本
价格区间
审核通过状态

```

不得展示：

1. 用户手机号；
2. 用户姓名；
3. 车牌；
4. VIN；
5. 身份证；
6. 行驶证；
7. 保险单号；
8. 详细住址；
9. 未脱敏人脸；
10. 原始订单金额。

## **20. 合规要求**

### **20.1 事故车合规**

事故车相关入口必须遵守：

```
不线上承诺最终价；
不展示一口价；
不承诺上传照片即可精准报价；
实际维修方案和费用以门店检测或拆检结果为准。

```

### **20.2 门店宣传合规**

门店不得展示未经审核的：

1. 品牌授权；
2. 原厂认证；
3. 官方合作；
4. 全网最低价；
5. 保证修复；
6. 夸大资质。

### **20.3 案例展示合规**

首页案例必须：

1. 用户授权或商家责任确认；
2. 图片已脱敏；
3. 文本已审核；
4. 价格以区间展示；
5. 明确案例来源；
6. 不展示隐私信息。

## **23. 前端展示规则**

### **23.1 图片**

1. 门店图必须使用审核通过图片；
2. 案例图必须使用脱敏图；
3. 图片加载失败展示默认占位图；
4. 禁止前端直接展示原图地址；
5. 图片 URL 应使用受控 CDN 或对象存储访问地址。

### **23.2 文案**

1. 不夸大服务能力；
2. 不使用“最低价”“保证修好”等绝对化表达；
3. 事故车不展示确定报价；
4. 分享、点赞、浏览奖励不在首页公开诱导展示；
5. 奖励相关入口可存在于“我的/收益中心”，首页不作为诱导分享入口。

### **23.3 跳转**

所有跳转页面需要校验目标状态：


| 目标    | 校验         |
| ----- | ---------- |
| 服务    | 是否上架       |
| 门店    | 是否正常营业/未冻结 |
| 案例    | 是否审核通过/未下架 |
| 事故车预约 | 服务是否开放     |
| 搜索结果  | 是否可展示      |

## **24. 异常与边界场景**

### **24.1 用户拒绝定位**

处理：

1. 首页正常展示；
2. 默认杭州；
3. 门店距离不展示；
4. 提供重新授权入口。

### **24.2 服务入口下架**

处理：

1. 首页不展示该入口；
2. 如用户通过缓存点击，提示：

```
该服务暂不可用，请查看其他服务。

```

### **24.3 门店被下架**

处理：

1. 首页不展示；
2. 已打开详情页时提示：

```
该门店暂不可预约。

```

### **24.4 案例被下架**

处理：

1. 首页不展示；
2. 已打开时提示：

```
该案例暂不可查看。

```

### **24.5 搜索命中敏感信息**

如果用户搜索车牌、手机号、VIN 等敏感内容：

1. 不做公开联想推荐；
2. 不返回非本人隐私内容；
3. 可提示：

```
未找到相关公开内容。

```

### **24.6 首页接口部分失败**

处理：

1. 服务入口失败：首页可重试；
2. 门店失败：隐藏门店模块；
3. 案例失败：隐藏案例模块；
4. 平台说明失败：展示默认文案。

## **28. 验收标准**

### **28.1 首页基础验收**

1. 用户打开小程序可进入首页；
2. 首页默认展示杭州；
3. 用户拒绝定位不影响首页使用；
4. 首页展示平台主标语；
5. 首页展示核心服务入口；
6. 首页展示事故车入口；
7. 首页展示推荐门店；
8. 首页展示精选案例；
9. 底部导航首页 Tab 正常；
10. 首页接口失败时有兜底状态。

### **28.2 搜索验收**

1. 用户可点击搜索框；
2. 用户可搜索服务；
3. 用户可搜索门店；
4. 用户可搜索公开案例；
5. 搜索无结果时展示空状态；
6. 搜索结果点击可进入对应详情；
7. 未审核或下架内容不出现在搜索结果；
8. 搜索结果不展示隐私信息。

### **28.3 服务入口验收**

1. 核心服务入口可正常展示；
2. 点击服务入口进入对应服务列表或详情；
3. 下架服务不展示；
4. 即将开放服务不可下单；
5. 事故车入口不展示线上最终价。

### **28.4 门店推荐验收**

1. 推荐门店只展示审核通过门店；
2. 门店卡片字段展示正确；
3. 用户授权定位后可展示距离；
4. 用户拒绝定位时不展示距离；
5. 点击门店进入门店详情；
6. 冻结或下架门店不展示。

### **28.5 案例推荐验收**

1. 精选案例只展示审核通过案例；
2. 案例主图为脱敏图；
3. 案例不展示实际支付金额；
4. 案例不展示车牌、手机号、VIN；
5. 点击案例进入案例详情；
6. 下架案例不展示。

### **28.6 合规验收**

1. 首页不出现诱导分享奖励文案；
2. 首页不出现浏览有奖、点赞有奖、点击有奖文案；
3. 事故车不承诺线上最终报价；
4. 门店宣传不出现未经审核品牌授权；
5. 首页公开案例符合隐私脱敏要求；
6. 首页风控拦截不导致页面整体不可用。
