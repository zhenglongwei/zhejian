## 2. 模块定位

下单与支付流程是用户端交易闭环的核心模块，负责承接：

1. 标准服务下单；
2. 套餐服务下单；
3. 复杂维修预约；
4. 事故车到店检测预约；
5. 微信支付；
6. 商家接单前后的订单状态流转；
7. 用户取消订单；
8. 退款申请入口；
9. 异常订单处理。

本模块必须同时支持两类交易：

```text
确定价格订单：用户线上确认价格并支付。
不确定价格预约：用户提交预约，到店检测或拆检后确认方案和费用。

```

## **4. 下单入口**

用户可从以下入口进入下单或预约流程：

1. 服务详情页点击“立即下单”；
2. 服务详情页点击“预约到店检测”；
3. 门店详情页点击“预约服务”；
4. 门店服务卡片点击“下单/预约”；
5. 案例详情页点击“预约类似服务”；
6. 订单详情页点击“再次下单”；
7. 首页事故车入口点击“预约门店检测”；
8. 搜索结果进入服务后下单；
9. H5 页面跳转小程序下单。

## **5. 订单类型**

### **5.1 订单类型枚举**


| 类型                     | 说明      | 是否线上支付     | 是否需要门店确认 |
| ---------------------- | ------- | ---------- | -------- |
| standard_order         | 标准服务订单  | 是          | 可选       |
| package_order          | 套餐订单    | 是          | 可选       |
| inspection_booking     | 到店检测预约  | 否/可收检测费    | 是        |
| complex_repair_booking | 复杂维修预约  | 否/可收订金     | 是        |
| accident_booking       | 事故车检测预约 | 否/可收检测费/订金 | 是        |
| offline_quote_order    | 到店报价后订单 | 可选         | 是        |


### **5.2 标准服务订单**

适用：

1. 小保养；
2. 电瓶更换；
3. 轮胎安装；
4. 标准检测；
5. 明确套餐价格的服务。

特点：

```
用户可在线确认服务、门店、时间、车辆和价格，并发起支付。

```

### **5.3 复杂维修预约**

适用：

1. 钣喷修复；
2. 底盘异响；
3. 空调维修；
4. 发动机/变速箱异常；
5. 需要检测后才能确认价格的维修。

特点：

```
用户先提交预约，实际方案和费用以门店检测结果为准。

```

### **5.4 事故车检测预约**

适用：

1. 事故车维修；
2. 保险定损后维修；
3. 结构件损伤；
4. 安全气囊相关损伤；
5. 新能源电池包或高压系统相关风险；
6. 需要拆检后确认方案的重大维修。

特点：

```
不得在线承诺最终维修价格。
必须明确提示需到店检测或拆检后确认方案和费用。

```

## **6. 标准下单流程**

### **6.1 流程图**

```
选择服务
→ 选择套餐，可选
→ 选择门店
→ 选择车辆
→ 选择预约时间
→ 填写联系人

→ 微信支付
→ 支付成功
→ 等待商家接单/确认

```

### **6.2 前置条件**

用户提交标准订单前必须满足：

1. 已选择服务；
2. 已选择门店；
3. 已选择预约时间；
4. 已选择或填写车辆；
5. 已绑定手机号；
6. 服务处于可预约状态；
7. 门店处于可预约状态；
8. 服务价格有效；
9. 用户已确认服务包含和不包含内容。

## **8. 事故车预约流程**

### **8.1 流程图**

```
进入事故车预约
→ 阅读事故车报价说明
→ 选择门店
→ 填写事故情况
→ 上传车辆损伤照片，可选
→ 填写是否保险定损
→ 选择预约时间
→ 填写联系人
→ 勾选知晓到店检测后报价
→ 提交预约

```

### **8.2 必须展示的说明**

事故车预约页必须展示：

```
事故车维修无法仅凭线上图片准确报价。
平台不提供线上最终报价。
实际维修方案和费用需门店检测或拆检后确认。
如涉及保险定损、结构件、安全气囊、新能源高压系统等情况，可能需要更长检测时间。

```

### **8.3 必须勾选项**

提交前必须勾选：

```
我已知晓事故车维修需到店检测或拆检后确认维修方案和费用。

```

未勾选时提示：

```
请先确认已知晓事故车检测报价规则。

```

### **8.4 禁止能力**

事故车预约流程禁止：

1. 展示线上最终价；
2. 展示固定维修金额；
3. 承诺照片精准报价；
4. 使用“最低价”“保证修好”等绝对化表达；
5. 引导用户误解预约即锁定最终费用。

4. 用户未确认时不得开始新增维修；
5. 新增费用应形成补差价订单或线下确认记录。

## **11. 优惠与奖励抵扣**

### **11.1 MVP 处理**

P0 可暂不支持优惠券，或仅预留字段。

如支持奖励抵扣，必须明确：

1. 哪些奖励可抵扣；
2. 抵扣上限；
3. 是否可提现与抵扣互斥；
4. 退款时奖励如何退回。

### **11.2 抵扣展示**

```
奖励抵扣：-¥10

```

不得在下单页诱导分享：

```
分享后立减
找好友点赞减免

```

奖励来源和规则在奖励中心展示，不在订单页做诱导分享表达。

## **12. 支付流程**

### **12.1 支付方式**

MVP 支持：

```
微信支付
```

```
用户点击提交订单
```

### **12.3 支付状态**


| 状态               | 说明     |
| ---------------- | ------ |
| unpaid           | 待支付    |

| paid             | 已支付    |
| pay_failed       | 支付失败   |
| pay_cancelled    | 用户取消支付 |

| refunded         | 已退款    |

提示：

```

```

按钮：

```
重新支付
取消订单
查看订单

```

### **12.5 支付超时**

待支付订单超时时间建议：

```
15 分钟

```

1. 订单自动关闭；
2. 释放预约时间；
3. 用户需重新下单；
4. 不保留价格锁定，除非业务另有配置。

## **14. 订单取消**

### **14.1 用户可取消状态**

用户可取消：

1. 待支付；
2. 待商家接单；
3. 待门店确认；
4. 已预约但未到店，具体按取消规则；
5. 商家建议改约时。

### **14.2 不可直接取消状态**

以下状态不允许直接取消：

1. 维修中；
2. 已完工待确认；
3. 已完成；
4. 售后处理中；
5. 已退款；
6. 商家已采购专用配件，需人工处理。

### **14.3 取消原因**

用户取消时选择：

1. 不想修了；
2. 价格不合适；
3. 时间不方便；
4. 选错服务；
5. 选错门店；
6. 门店联系不上；
7. 已在线下处理；
8. 其他。

### **14.4 取消后处理**


| 场景        | 处理           |
| --------- | ------------ |
| 未支付取消     | 订单关闭         |
| 已支付未接单取消  | 自动发起退款       |
| 已接单未服务取消  | 按规则退款或人工审核   |
| 已到店/已检测取消 | 检测费是否退还按规则处理 |
| 已采购配件     | 进入人工审核       |

## **18. 隐私与脱敏**

### **18.1 订单隐私**

订单信息不得在公开页面展示。

### **18.2 车辆隐私**

车牌、VIN、行驶证、保险单等信息不得公开展示。

### **18.3 图片隐私**

用户上传的事故车或维修照片：

1. 默认仅用于门店判断和订单服务；
2. 不得直接作为公开案例；
3. 公开前必须用户授权；
4. 公开前必须平台脱敏审核。

### **18.4 授权提示**

订单页可展示：

```
你上传的车辆照片仅用于本次服务沟通。未经授权不会公开展示。

```

## **19. 异常与边界场景**

### **19.1 服务下架**

提交时校验服务状态。

提示：

```
该服务暂不可预约，请选择其他服务。

```

### **19.2 门店不可预约**

提示：

```
该门店当前暂不可预约，请选择其他门店。

```

### **19.3 预约时间被占用**

提示：

```
该时间段已被预约，请选择其他时间。

```

### **19.4 价格变动**

如果进入订单确认页后价格发生变化：

```
服务价格已更新，请重新确认后提交订单。

```

### **19.5 支付回调延迟**

前端展示：

```
支付结果确认中，请稍后查看订单状态。

```

后端以微信支付回调和主动查询结果为准。

### **19.6 重复提交**

后端必须防止重复创建订单和重复支付。

### **19.7 用户退出支付**

订单保留待支付状态，用户可重新支付或取消。

---

## **20. 合规要求**

1. 事故车不得线上承诺最终价；
2. 到店检测后报价类服务必须明确提示；
3. 标准套餐必须明确包含和不包含内容；
4. 不得默认收取未确认费用；
5. 新增维修项目必须用户确认；
6. 用户上传图片不得未经授权公开；
7. 订单页不得出现诱导分享奖励文案；
8. 支付金额、退款金额必须准确展示；
9. 退款规则必须可查看；
10. 不得将预约表述为最终成交承诺。

## **25. 验收标准**

1. 用户可从服务页进入订单确认页；
2. 用户可从门店页进入预约流程；
3. 标准服务可正常提交并支付；
4. 复杂维修可提交预约；
5. 事故车预约必须展示检测后报价提示；
6. 未绑定手机号不可提交订单；
7. 服务下架时不可提交；
8. 门店不可预约时不可提交；
9. 支付成功后订单状态正确；
10. 支付失败后可重新支付；
11. 用户可取消符合条件的订单；
12. 用户可申请符合条件的退款；
13. 用户上传图片不会自动公开；
14. 订单页无诱导分享奖励文案。
