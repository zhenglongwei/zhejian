## 2. 模块定位

评价与奖励领取模块负责承接订单完成后的用户反馈和内容沉淀。

核心目标：

1. 鼓励用户发表真实服务评价；
2. 为门店、服务、案例形成可信内容；
3. 支持用户上传真实维修体验图片；
4. 建立评价审核机制；
5. 在符合规则时发放评价奖励；
6. 避免诱导好评、虚假评价和违规推广；
7. 为分享收益中心提供合规内容基础。

## **4. 评价入口**

### **4.1 入口位置**

评价入口可出现在：

1. 订单详情页；
2. 订单列表卡片；
3. 完工确认成功页；
4. 消息通知；
5. 我的评价；
6. 维修档案页；
7. 维修相册页，作为辅助入口。

### **4.2 展示条件**

评价入口展示条件：

1. 订单状态为已完成；
2. 用户为订单本人；
3. 当前订单尚未评价；
4. 订单未被风控限制；
5. 订单未处于严重售后争议中，或平台允许评价；
6. 服务类型支持评价。

### **4.3 不展示评价入口场景**

1. 订单未完成；
2. 订单已取消；
3. 订单已退款且未实际服务；
4. 用户非订单本人；
5. 评价已提交；
6. 订单存在作弊风险；
7. 平台规则限制评价。

## **5. 评价流程**

### **5.1 标准流程**

```
订单完成
→ 用户点击去评价
→ 进入评价提交页
→ 选择评分
→ 填写文字评价
→ 上传图片，可选
→ 选择标签，可选
→ 勾选真实评价承诺
→ 提交评价
→ 平台审核
→ 审核通过
→ 评价展示
→ 符合条件则发放奖励

```

### **5.2 异常流程**

```
评价提交
→ 审核不通过
→ 用户可查看原因
→ 可修改后重新提交，按规则

```

或：

```
评价提交
→ 风控命中
→ 进入人工审核
→ 审核后决定展示与奖励

```

## **7. 评分规则**

### **7.1 评分维度**

P0 建议支持总评分，P1 支持多维度评分。

P0：

```
总体评分：1-5 星

```

P1：

```
服务态度
维修质量
价格透明
交付效率
环境体验

```

### **7.2 星级含义**


| 星级  | 含义    |
| --- | ----- |
| 5 星 | 非常满意  |
| 4 星 | 满意    |
| 3 星 | 一般    |
| 2 星 | 不满意   |
| 1 星 | 非常不满意 |


### **7.3 必填规则**

总体评分必填。

未选择评分时提示：

```
请选择本次服务评分。

```

## **8. 文字评价**

### **8.1 字数限制**

建议：

```
最少 10 字
最多 500 字

```

P0 可设置：

```
最少 5 字
最多 300 字

```

### **8.2 输入提示**

```
请分享本次真实维修体验，例如服务态度、维修过程、价格透明度、交付效果等。

```

### **8.3 禁止内容**

文字评价不得包含：

1. 辱骂、人身攻击；
2. 色情、暴力、违法内容；
3. 联系方式；
4. 广告引流；
5. 虚假宣传；
6. 恶意诋毁；
7. 涉及隐私的完整车牌、手机号、VIN；
8. 诱导交易；
9. 与订单无关内容；
10. 平台认定的其他违规内容。

### **9.1 正向标签**

低评分时可展示：

等待较久

价格不清晰
维修效果不满意
交付延期

### **9.3 标签规则**

1. 标签非必填；
2. 可多选；
3. 标签需与评分联动；
4. 低分评价不得只展示正向标签；
5. 标签不得替代文字评价审核。

## **13. 评价奖励**

### **13.1 奖励定位**

评价奖励用于鼓励用户提交真实、有效、有参考价值的服务评价。

奖励不得用于诱导好评。

### **13.2 奖励发放原则**

奖励发放应满足：

1. 订单真实完成；
2. 用户为订单本人；
3. 评价内容真实有效；
4. 评价通过平台审核；
5. 不要求必须好评；
6. 不以 5 星为发放条件；
7. 不以修改差评为发放条件；
8. 不得诱导用户虚假评价。

### **13.3 奖励类型**


| 类型             | 说明       |
| -------------- | -------- |
| balance_reward | 奖励余额     |
| coupon_reward  | 优惠券      |
| points_reward  | 积分       |
| cash_reward    | 现金红包，需合规 |
| no_reward      | 无奖励      |


P0 建议：

```
奖励余额或优惠券二选一。

```

### **13.4 奖励金额展示**

评价页可展示：

```
提交真实评价并通过审核后，可获得平台评价奖励。

```

如金额固定，可展示：

```
评价通过审核后可获得 ¥X 奖励。

```

注意不得展示：

```
五星好评返现
好评返现
晒图好评得红包
给好评立返

```

## **15. 评价审核**

### **15.1 审核方式**

可采用：

1. 机器审核；
2. 人工审核；
3. 机器初审 + 人工复审；
4. 风控抽检。

### **15.2 审核结果**


| 状态              | 说明    |
| --------------- | ----- |
| review_pending  | 待审核   |
| review_approved | 审核通过  |
| review_rejected | 审核不通过 |
| review_hidden   | 已隐藏   |
| review_deleted  | 已删除   |
| review_risk     | 风控审核中 |


### **15.3 审核通过**

审核通过后：

1. 评价可展示在门店页；
2. 可展示在服务页；
3. 可展示在用户维修档案内；
4. 可计入门店评分；
5. 符合条件则发放奖励。

### **15.4 审核不通过**

原因可包括：

1. 内容违规；
2. 与订单无关；
3. 含隐私信息；
4. 含广告引流；
5. 疑似虚假评价；
6. 图片违规；
7. 重复评价；
8. 恶意评价。

提示示例：

```
评价未通过审核，请修改违规内容后重新提交。

```

## **20. 合规要求**

1. 不得诱导好评；
2. 不得以五星好评作为奖励条件；
3. 不得要求用户修改差评后发奖励；
4. 奖励规则必须清晰可查；
5. 评价展示前必须审核；
6. 评价图片必须审核；
7. 评价不得泄露隐私；
8. 匿名评价仍需保留后台追溯；
9. 虚假评价应进入风控；
10. 商家不得用评价返现绕过平台规则。

---

## **21. 风控规则**

### **21.1 风控触发场景**

1. 同一用户短时间多笔评价；
2. 同一设备多账号评价；
3. 评价内容高度重复；
4. 订单异常低价或虚假完成；
5. 商家集中刷评；
6. 用户与商家存在异常关联；
7. 评价图片重复使用；
8. 评价内容与服务不符；
9. IP、设备、手机号异常；
10. 奖励领取频率异常。

### **21.2 风控处理**

1. 延迟发放奖励；
2. 人工审核；
3. 不展示评价；
4. 取消奖励；
5. 限制账号评价；
6. 商家风控标记；
7. 平台处罚。

## **29. 验收标准**

1. 完成订单后用户可进入评价页；
2. 未完成订单不可评价；
3. 非订单本人不可评价；
4. 同一订单不可重复评价；
5. 评分必填；
6. 文字评价需满足字数要求；
7. 图片上传有数量和格式限制；
8. 评价提交后进入审核状态；
9. 审核通过后评价可展示；
10. 审核不通过时用户可看到原因；
11. 奖励不以五星好评为条件；
12. 不出现“好评返现”等违规文案；
13. 奖励发放记录可查看；
14. 匿名评价前台匿名展示；
15. 评价内容和图片不得泄露隐私。
