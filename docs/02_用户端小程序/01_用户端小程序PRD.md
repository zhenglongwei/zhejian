---

## **2. 用户端产品目标**

用户端小程序的核心目标不是简单让用户“买一个汽修服务”，而是帮助用户完成三个动作：

1. **看得懂**
  - 看懂服务项目；
  - 看懂价格区间；
  - 看懂类似案例；
  - 看懂影响维修费用的因素。
2. **敢选择**
  - 通过门店资质、真实案例、维修相册、评价建立信任；
  - 降低用户对修车被坑、价格不透明、过程不可见的担忧。
3. **有档案**
  - 每次平台订单自动生成维修档案；
  - 用户可以查看维修过程；
  - 后续可用于复查、维权、卖车、保养提醒、再次维修参考。

---

## **3. 用户端核心闭环**

```
用户进入小程序
→ 浏览服务/案例/门店
→ 添加车辆
→ 选择服务
→ 选择门店
→ 下单或预约
→ 商家接单
→ 到店维修
→ 查看维修相册
→ 确认完工
→ 完成评价
→ 获得评价奖励
→ 查看脱敏预览
→ 决定是否公开案例
→ 分享脱敏案例/档案

```

# **5. 首页 PRD**

## **5.1 页面目标**

首页承担三个任务：

1. 告诉用户平台是“透明维修服务平台”；
2. 引导用户进入高优先级服务；
3. 通过真实案例和门店建立信任。

---

## **5.2 首页结构**

建议首页模块：

```
顶部定位城市
搜索框
平台主标语
核心服务入口
事故车/钣喷重点入口
附近门店
精选案例
透明维修说明
用户保障说明

```

---

## **5.3 顶部定位城市**

### **功能说明**

显示当前城市，MVP 默认杭州。

字段：

- 当前城市：杭州；
- 用户定位，可选；
- 城市切换，MVP 可暂不开放多城市。

### **交互规则**

- 首次进入可请求定位；
- 用户拒绝定位时默认杭州；
- 若用户定位不在杭州，可提示：“当前服务暂以杭州为主”。

---

## **5.4 搜索框**

### **搜索范围**

支持搜索：

1. 服务项目；
2. 门店名称；
3. 车型；
4. 故障关键词；
5. 案例关键词。

### **MVP 可先实现**

- 关键词模糊搜索服务；
- 关键词模糊搜索门店；
- 关键词模糊搜索公开案例标题。

## **6.4 价格展示规则**

### **一口价/套餐价服务**

适用于：

- 小保养；
- 电瓶；
- 轮胎；
- 部分标准检测。

展示：

```
¥299 起
或
套餐价 ¥399

```

需要明确包含：

- 配件；
- 工时；
- 是否含税，MVP 可不强调；
- 是否包含检测；
- 不包含项目。

---

### **价格区间服务**

适用于：

- 刹车；
- 空调；
- 底盘；
- 钣喷。

展示：

```
参考区间：¥300-¥1200
到店检测后确认

```

并展示影响因素：

- 车型；
- 配件品牌；
- 损伤程度；
- 是否需要更换总成；
- 工时；
- 是否涉及喷漆/拆装；
- 是否新能源车型。

---

### **事故车服务**

事故车不展示线上报价。

展示文案：

```
事故车维修无法仅凭线上图片准确报价。
平台不提供线上一口价，建议预约门店到店检测后确认维修方案。
你可以先查看类似案例，了解常见维修方案和费用影响因素。

```

按钮：

- 查看事故车案例；
- 预约到店检测。

# **13. 完工确认与评价 PRD**

## **13.1 完工确认**

商家提交完工后，用户收到提醒。

用户可操作：

1. 确认完成；
2. 申请售后；
3. 联系商家；
4. 查看维修相册。

---

## **13.2 自动完成**

建议规则：

- 商家提交完工后 72 小时；
- 用户未确认且未发起售后；
- 系统自动完成订单。

自动完成后用户仍可评价和申请售后。

---

## **13.3 评价页**

评价维度：

1. 服务态度；
2. 专业程度；
3. 价格透明；
4. 维修过程透明；
5. 完工效果；
6. 是否愿意推荐。

评分方式：

- 1-5 星；
- 可填写文字；

# **14. 评价奖励 PRD**

## **14.1 奖励入口**

1. 评价提交后结果页；
2. 我的评价奖励；
3. 订单详情页。

---

## **14.2 奖励状态**


| 状态  | 用户展示     |
| --- | -------- |
| 待计算 | 奖励计算中    |
| 待审核 | 平台审核中    |
| 待发放 | 即将发放     |
| 已发放 | 已到账      |
| 已驳回 | 不符合奖励条件  |

```
完成真实评价后，可获得平台评价奖励。
奖励金额会根据服务复杂度、车辆价值和订单金额综合计算。
平台会对异常评价、虚假交易和售后争议订单进行审核。

```

# **15. 用户授权公开 PRD**

## **15.1 入口**

1. 订单完成后弹窗；
2. 微信服务通知；
3. 订单详情页；
4. 维修档案页。

---

## **15.2 授权页内容**

必须包含：

1. 公开说明；
2. 脱敏说明；
3. 奖励说明，如有；
4. 预览脱敏效果；
5. 手动打码/裁剪；
6. 同意公开；
7. 不公开；
8. 后续下架申请说明。

---

## **15.3 授权文案示例**

```
是否愿意将本次维修记录脱敏后公开为维修案例？

公开后，其他车主可以参考你的维修过程和方案。
平台将隐藏车牌、人脸、VIN、手机号等隐私信息。
公开页面默认不展示你的实际支付金额，只展示参考价格区间和影响价格的因素。
你可以先查看脱敏预览，也可以手动打码或删除不想公开的图片。

```

# **16. 分享 PRD**

## **16.1 可分享内容**

1. 公开案例；
2. 门店页；
3. 服务页；
4. 用户自己的脱敏维修档案。

---

## **16.2 分享限制**

1. 分享必须使用脱敏图；
2. 未授权公开的订单档案，只能生成脱敏后的私密分享页或不支持分享，MVP 建议仅支持授权后分享；
3. 不在微信分享链路中明示“浏览有奖、点击有奖、点赞有奖、分享有奖”等诱导性文案。
4. 不做多级分销。
5. 后台可基于有效浏览、有效点赞、有效收藏、有效咨询、有效预约、有效订单等传播贡献计算奖励。
6. 奖励仅在小程序内部的奖励/收益中心中展示，分享落地页不展示奖励规则。
