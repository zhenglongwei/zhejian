## 2. 模块定位

搜索与筛选模块用于帮助用户快速找到目标内容。

核心目标：

1. 支持用户按关键词搜索服务；
2. 支持用户搜索门店；
3. 支持用户搜索维修案例；
4. 支持按距离、价格、评分、销量、资质等维度筛选；
5. 支持按服务类型、车辆品牌、门店能力等条件过滤；
6. 降低用户从需求到下单/预约的路径成本；
7. 为平台沉淀用户需求数据；
8. 为服务推荐和运营活动提供基础数据。

3. 未公开维修档案；
4. 未审核评价；
5. 未审核案例；
6. 用户手机号；
7. 车牌号；
8. VIN；

## **15. 排序规则**

### **15.1 通用排序**


| 排序     | 说明      |
| ------ | ------- |
| 综合排序   | 默认排序    |
| 距离最近   | 门店适用    |
| 评分最高   | 门店/服务适用 |
| 价格从低到高 | 标准服务适用  |
| 价格从高到低 | 标准服务适用  |
| 销量优先   | 服务适用    |
| 最新发布   | 案例适用    |
| 热度最高   | 案例适用    |


### **15.2 综合排序因素**

综合排序可考虑：

1. 关键词匹配度；
2. 城市匹配；
3. 距离；
4. 服务上架状态；
5. 门店营业状态；
6. 评分；
7. 订单量；
8. 案例质量；
9. 是否支持维修相册；
10. 是否可预约；
11. 运营权重。

### **15.3 排序合规**

排序如涉及广告或商业推广，应在 P1 明确标识。

P0 不建议混入广告排序。

## **16. 定位与城市**

### **16.1 定位授权**

用户可授权定位以查看附近门店。

提示：

```
开启定位后，可为你推荐附近门店和服务。

```

### **16.2 未授权处理**

未授权定位时：

1. 使用用户手动选择城市；
2. 默认平台配置城市；
3. 仍可搜索服务和案例；
4. 距离排序不可用或置灰。

### **16.3 城市切换**

支持手动切换城市。

切换后：

1. 搜索结果按新城市刷新；
2. 门店距离重新计算；
3. 热门搜索可按城市变化。

## **18. 敏感词与纠错**

### **18.1 敏感词处理**

搜索词包含敏感内容时：

1. 不展示违规内容；
2. 可提示无结果；
3. 记录风控日志；
4. 严重违规可限制搜索。

### **18.2 纠错能力**

P1 支持错别字纠错。

示例：

```
你是否想搜索：补漆

```

### **18.3 同义词**

建议配置同义词：

```
喷漆 = 补漆 = 漆面修复
保养 = 小保养 = 机油机滤
电瓶 = 蓄电池
轮胎 = 换胎

```

## **21. 合规要求**

1. 搜索结果不得泄露用户隐私；
2. 私密订单和相册不得被搜索；
3. 事故车相关结果不得展示线上最终报价；
4. 案例必须脱敏审核后展示；
5. 热门词不得包含虚假承诺；
6. 搜索结果中的评分、销量不得造假；
7. 广告结果需明确标识，P1；
8. 不得通过搜索词引导诱导分享；
9. 不得展示违规评价内容；
10. 用户定位授权必须遵守微信和隐私规则。

## **27. 验收标准**

1. 用户可从首页进入搜索页；
2. 用户可输入关键词并搜索；
3. 搜索结果可展示服务、门店、案例；
4. 用户可切换结果类型；
5. 用户可使用筛选和排序；
6. 搜索历史可展示和清空；
7. 热门搜索可点击搜索；
8. 无结果时有空状态和推荐；
9. 私密订单和私密相册不会出现在结果中；
10. 公开案例必须脱敏后展示；
11. 事故车相关服务不展示线上最终价；
12. 未授权定位时搜索仍可使用；
13. 搜索接口异常时有提示；
14. 关键词过长时有拦截；
15. 埋点可记录搜索行为。
