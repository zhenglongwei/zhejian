## **13. 公开授权入口**

### **13.1 入口定位**

订单完成后，可在服务相册或服务相册页展示：

```
授权生成公开案例

```

或：

```
申请公开为案例

```

### **13.2 展示条件**

必须满足：

1. 订单已完成；
2. 相册图片数量达到最低要求；
3. 无未处理售后争议；
4. 用户为订单本人；
5. 订单未被风控限制；
6. 门店允许案例公开，或平台规则允许；
7. 不涉及禁止公开内容。

### **13.3 授权说明**

点击授权前展示：

```
你的服务相册可供其他车主参考。公开并通过平台审核后，可按平台规则参与利益共享（以政策为准）。平台会自动脱敏车牌、手机号等信息，请你核对后再确认公开。公开版不展示完整订单信息与实际支付金额。

```

说明区下方勾选文案后附 **《利益共享政策》** 链接（跳转规则页占位，正式政策文档待补齐）。

### **13.4 用户可选择公开范围**

P0 可简化为：

```
同意平台脱敏后公开展示本次维修案例

```

P1 可支持选择：

1. 仅公开车型和服务项目；
2. 公开维修过程图；
3. 公开门店名称；
4. 公开评价内容；
5. 不公开价格；
6. 允许被分享。

### **13.5 授权勾选**

用户必须主动勾选：

```
我已核对公开内容，同意将本次服务相册公开为案例。《利益共享政策》

```

### **13.6 拒绝公开**

用户可选择不公开，页面需提供「拒绝公开」操作（与「确认公开为案例」并列，不使用警告色）。

点击后二次确认：

```
拒绝后，本次服务相册仍仅作为你的私密记录保存，不会生成公开案例。

```

确认后记录 `user_rejected` 状态；相册仍可正常查看，后续可在规则允许时重新发起公开（P1）。

### **13.7 确认公开与脱敏预览**

用户勾选并点击「确认公开为案例」后：

1. 若商家已提交完工且 **`order_pre_mask` 任务** `preMaskStatus=ready`，直接进入 `pages/desensitize/preview` 预览工作台，**无需等待** AI 脱敏；
2. 预览页展示原图 / 脱敏图对比，用户须逐张查看脱敏图；
3. 用户可重新处理失败项；手工打码 P1；
4. 勾选确认项后点击「确认并公开」，写入授权记录；
5. 预脱敏未完成时，提示稍后再试或走一键脱敏（降级）。

## **16. 与授权公示的关系（V2.0）**

服务相册本身不绑定交易评价或现金奖励。

**授权公示**（非「分享给朋友」）的条件：

1. 用户主动向 **辙见** 授权平台公开展示；
2. 平台/商家完成脱敏（平台公示 **强制脱敏**）；
3. 案例审核通过；
4. 案例达到质量与合规要求。

授权通过后，案例 **自动** 出现在案例 Tab、H5 等公示位，**无需** 用户再手工发布。

V2.0 **不做**：订单完成评价、评价奖励、分享领钱。

**口径**：`00_Phase1_服务相册产品口径.md` §4.5（授权公示 vs 用户自主分享）。

## **17. 分享规则**

> **定稿 2026-05-28**：**用户自主分享** 与 **授权公示** 分离。分享 **不要求** 授权公示；平台 **不强制** 私人分享必须脱敏（默认脱敏 + 可选原图 + 风险提示留痕）。

### **17.1 用户自主分享（私人传播）**

| 项 | 规则 |
| --- | --- |
| **谁可以分享** | 关联车主（服务相册所属用户） |
| **何时可分享** | 相册 **已完工**（维修进度完成，含 `completed` 等等价态）且有关联权限即可；**不必** 先授权公示或平台审核 |
| **分享对象** | 微信好友、朋友圈、复制链接、其他社交 App 等 |
| **脱敏** | **默认** 脱敏后分享；用户 **可勾选「原图分享」**；选原图时须展示隐私风险提示并 **留痕**，平台 **不得拦截** |
| **与平台公示** | 私人分享 **不** 自动进入案例 Tab / H5；与授权公示 **独立** |

私密服务相册 **不可** 把「仅登录可见的相册详情链接」当作公开外链发给陌生人；私人分享应走 **分享能力**（卡片/链接由产品生成，可配置脱敏/原图），而非泄露后台私密 URL。

### **17.2 授权公示**

用户 **授权公示** 后，平台在审核通过后将内容 **自动公示** 至：

1. 小程序案例 Tab；
2. H5 公开案例页；
3. 官网/GEO 页（P1）。

此路径 **强制脱敏 + 审核**；与用户是否点击「分享给好友」 **无关**。

### **17.3 分享文案限制**

分享卡片不得出现：

```
分享有奖
点击有奖
点赞有奖
浏览有奖
帮我赚钱

```

可使用内容型文案：

```
这次维修过程已整理成案例，可查看参考。

```

后台可做传播归因；**无** 用户侧奖励结算或诱导文案。

### **17.4 半屏分享面板（定稿 2026-05-28）**

> **目标**：统一分享入口，避免在页面正文中散落「复制链接 / 转发」按钮；区分 **微信内分享** 与 **站外公网 H5 链接**；与 §4.5 两条链路对齐。

#### **17.4.1 为什么需要半屏面板**

| 问题 | 说明 |
| --- | --- |
| 右上角「复制链接」 | 微信固定行为，只能复制 **小程序专用链**（`#小程序://…`），**不能**改为 HTTPS 公网地址；开发者 **无法** 通过 API 改写 |
| 页面内单独放「复制网页链接」 | 与右上角能力易混淆，视觉突兀 |
| 多种分享意图并存 | **私人相册分享**（token 落地页）与 **平台公开案例**（案例详情 / H5）需分开展示，但入口宜合并为 **一个「分享」** |

半屏分享 = 点击「分享」后，自底部弹出 **ShareSheet**（约半屏高），列出渠道与说明；**不是**新页面路由。

#### **17.4.2 入口与页面布局（相册详情）**

| 区块 | 展示 | 操作 |
| --- | --- | --- |
| **授权公示** | 已完工且仍为私密（未提交/已拒绝/已撤回公示） | 档位 + 授权公示 |
| **分享** | 已完工且有图 | 仅 **一个主按钮「分享」** → 打开半屏面板 |
| **平台公示** | 已 `public_approved` 时 | **状态文案** + **查看公示页**；**不再**在正文放转发/复制链接按钮 |

**禁止**：「须先授权才能分享」「分享即公开」等混用文案。

#### **17.4.3 半屏面板结构**

```
┌─────────────────────────────┐
│  分享                    ✕  │
│  [PrivacyBanner 场景化文案]   │
│  ── 分享我的服务相册 ──       │
│  （completed 即可；不必授权公示）│
│  □ 使用原图分享（默认不勾选）   │
│  [ 转发给微信好友 ]            │
│  [ 复制分享链接 ]              │
│  ── 平台公示案例（可选）──     │
│  （仅 public_approved 时出现） │
│  [ 转发公开案例 ]              │
│  [ 复制公示网页链接 ]          │
│  说明：「分享链接」随上方脱敏/  │
│  原图选项；「公示网页」为平台   │
│  已审核脱敏内容。右上角「复制   │
│  链接」为微信内小程序链。       │
│  [ 取消 ]                     │
└─────────────────────────────┘
```

**PrivacyBanner**：

- 默认（脱敏私人分享）：`ownerShare`
- 勾选原图：`ownerShareOriginal`
- 若展示「平台公示案例」分组：该组前追加 `publicCaseShare` 说明（平台链接 **始终脱敏**，与上方原图勾选 **无关**）

**原图分享**（仅 **私人分享** 分组）：

- 勾选前须 **Modal 风险提示**（留痕不变，见 §17.1）
- **复制分享链接** 与 **转发好友** 共用同一 `shareToken`，token 创建时写入 `mode`（`desensitized` | `original`）

#### **17.4.4 渠道规则**

| 渠道 | 可见条件 | 行为 | 链接/卡片类型 |
| --- | --- | --- | --- |
| **转发给微信好友**（私人） | 相册 `completed`；**不必** 授权公示 | `shareIntent=owner`；复制前刷新 token | 小程序 `/pages/album/share/index?token=…` |
| **复制分享链接**（私人） | **同上** | 复制前 `POST …/share` 写入当前 `mode` → `setClipboardData` | **HTTPS 公网** `…/album/share.html?token=…`（见 §17.4.4.1） |
| **转发公开案例** | `public_approved` 且有 `publicCaseId` | `shareIntent=publicCase` | 小程序 `/pages/case/detail/index?id=…` |
| **复制公示网页链接** | **同上** | 复制 `case/view.html?id=` | **HTTPS 公网**；内容 **强制脱敏**（平台公示） |
| **分享到朋友圈** | 私人或公示均可 | 面板 **文字说明** + 右上角 `onShareTimeline` | 不可从面板内一键调起；见 §17.4.5 |

##### **17.4.4.1 私人「复制分享链接」与脱敏/原图**

**能做到。** 链接不直接携带图片，而是携带 **一次性 shareToken**；服务端按 token 记录中的 `mode` 决定落地页展示 **脱敏图** 或 **原图**：

```text
用户打开面板 → 默认 desensitized
  → 可选勾选「原图分享」→ 风险 Modal → mode=original
  → 点击「复制分享链接」或「转发好友」前
  → POST /service-albums/:id/share { mode, channel }
  → 返回 token
  → 复制 URL：https://{domain}/album/share.html?token={token}
  → 好友打开 H5（或微信内打开小程序同 token）→ GET /shared-albums/:token → 按 mode 出图
```

| 环节 | 脱敏（默认） | 原图（用户勾选） |
| --- | --- | --- |
| token.mode | `desensitized` | `original` |
| 落地页图片 | 预脱敏 / 脱敏 URL | 相册原图 |
| 是否需授权公示 | **否** | **否** |
| 留痕 | 是 | 是 + 风险确认 |

**与平台公示链接的区别**：

| | **复制分享链接**（私人） | **复制公示网页链接**（平台） |
| --- | --- | --- |
| 前置条件 | `completed` | `public_approved` |
| 是否依赖授权 | **否** | 是（已授权且审核通过） |
| 脱敏/原图 | 用户可选（默认脱敏） | **强制脱敏** |
| URL | `/album/share.html?token=` | `/case/view.html?id=` |
| 进入案例 Tab/H5 索引 | **否** | 是（平台公示） |

**复制公示网页链接** 不要求脱敏封面字段就绪；有 `publicCaseId` 即可（H5/API 强制脱敏展示）。

#### **17.4.5 与右上角菜单的关系**

| 能力 | 半屏面板 | 右上角 ··· |
| --- | --- | --- |
| 转发好友 | ✅ 私人 / 公示分组内按钮 | ✅ 转发 |
| 朋友圈 | 📄 面板内说明 + `onShareTimeline` | ✅ 分享到朋友圈 |
| 复制 **分享**链接（私人·公网） | ✅ `album/share.html?token=` | ❌ 不能（仅小程序链） |
| 复制 **公示**网页链接（平台） | ✅ `case/view.html?id=`（须已公示） | ❌ 不能 |

面板底部须有一句 **对比说明**（`text-caption`），避免用户混淆两种「链接」。

#### **17.4.6 留痕与 API**

| 动作 | 留痕 |
| --- | --- |
| 私人 · 转发好友 | `POST …/share` `{ mode, channel: 'wechat' }` |
| 私人 · 复制分享链接 | `{ mode, channel: 'owner_h5_link' }` |
| 私人 · 原图勾选 | `mode: 'original'`（转发/复制链接共用 token） |
| 复制公示网页链接 | `{ channel: 'public_h5_link', caseId }`（仅已公示） |
| 转发公开案例 | 可选 `channel: 'public_case_wechat'`（P1 归因） |

#### **17.4.7 状态与禁用**

| 状态 | 面板 |
| --- | --- |
| 分享 token 准备中 | 「转发给微信好友」「复制分享链接」disabled |
| 无 `publicCaseId` | 隐藏「平台公示案例」整组；**私人组仍可用** |
| 未 `completed` | 不展示「分享」入口 |

#### **17.4.8 Phase 范围**

- **Phase 1（U-CASE-07）**：`pages/album/detail` + `ShareSheet` + **`h5/album/share.html`**（私人 token 公网落地，与小程序 `pages/album/share` 同源 API）
- **P1**：`pages/case/detail` 复用 ShareSheet

## **17.5 配件验真（`pages/album/part-verify` · U-ALB-PV · 2026-06-24）**

> **命名**：全站统一 **「配件验真」**（原「核对配件」废弃）。  
> **定位**：帮消费者对照 **维修方案零配件** 与 **相册登记零配件** 是否一致；**不鉴定** 真伪，**不保证** 已装车实物一致。

### **17.5.1 三层对照模型（页顶必展示）**

```text
  ① 维修方案配件（planParts · 阶段三锁定）
           │  车主可对照 ✓
  ② 相册登记配件（partsJson · 阶段四上传 + 编码 OCR）
           │  车主可对照 ✓（含编码自行查询）
  ③ 车上实物配件
           │  平台无法监督 → 更换时建议在场；不在场仅提醒
```

### **17.5.2 入口**

| 入口 | 文案 |
| --- | --- |
| 相册详情 · 尾页 | **配件验真** |
| 相册详情 · 底栏「配件」 | 同左 |
| 信息抽屉 · 配件清单下 | **配件验真** + 一行摘要 |
| `pages/album/engage` | 第三项 **配件验真** |

副标题 hint：**「对照维修方案与相册配件」**。

### **17.5.3 页面结构（线框见 `11_工具相册UI线框` §5）**

1. **说明区**：价值主张 + `PrivacyBanner scene=partVerify` + 三步指引  
2. **方案摘要条**：`planParts` 表格摘要或阶段三 note（旧相册降级）  
3. **验真卡（逐件）**：左 **方案** / 右 **相册** — 名称 · 类型 Tag · 品牌 · **编码** · 数量 · 凭证缩略图  
4. **差异提示**：字段不一致 → warning 样式（**非平台结论**）  
5. **结论（单选）**：方案与相册一致 / 不一致或有疑问 / 暂未验真  
6. **装车提醒折叠块**：建议更换时在场；可联系门店展示旧件/包装  
7. **提交**：勾选 consent + **保存验真记录**

**方案外增项**：单独分组「方案外增项」，仅展示相册侧，无方案对照列。

### **17.5.4 验真状态（对用户文案）**

| 内部 value | 用户文案 |
| --- | --- |
| `matched` | 方案与相册一致 |
| `question` | 不一致或有疑问 |
| `skipped` | 暂未验真 |

### **20.5 用户撤回授权**

1. 公开案例下架；

3. 奖励是否撤销按奖励规则处理；
4. 私密服务相册仍保留。

---

## **21. 合规要求**

1. 原始服务相册默认私密；
2. 未经用户授权不得公开；
3. 公开前必须脱敏；
4. 公开前必须审核；
5. 不得展示完整车牌、VIN、手机号、证件；
6. 不得展示用户实际支付金额；
7. 不得把私密相册链接分享给陌生用户；
8. 不得在分享入口展示诱导奖励文案；
9. 事故车案例公开时不得暗示线上最终报价；
10. 涉及保险、事故责任、违法信息时需谨慎审核。
