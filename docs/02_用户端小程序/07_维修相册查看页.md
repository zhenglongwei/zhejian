## 2. 模块定位

维修相册是平台“透明维修”的核心能力。

它的目标是：

1. 让用户看到车辆维修过程；
2. 降低用户对维修不透明的担忧；
3. 为订单履约提供过程证明；
4. 沉淀维修档案；
5. 在用户授权后，可生成脱敏公开案例；
6. 为评价、公开案例奖励和传播奖励提供内容基础。

维修相册默认属于用户私密订单内容，未经授权不得公开。

## **13. 公开授权入口**

### **13.1 入口定位**

订单完成后，可在维修相册或维修档案页展示：

```
授权生成公开案例

```

或：

```
申请公开为案例

```

### **13.2 展示条件**

必须满足：

1. 订单已完成；
2. 相册图片数量达到最低要求；
3. 无未处理售后争议；
4. 用户为订单本人；
5. 订单未被风控限制；
6. 门店允许案例公开，或平台规则允许；
7. 不涉及禁止公开内容。

### **13.3 授权说明**

点击授权前展示：

```
你的维修档案可供其他车主参考。公开并通过平台审核后，可按平台规则参与利益共享（以政策为准）。平台会自动脱敏车牌、手机号等信息，请你核对后再确认公开。公开版不展示完整订单信息与实际支付金额。

```

说明区下方勾选文案后附 **《利益共享政策》** 链接（跳转规则页占位，正式政策文档待补齐）。

### **13.4 用户可选择公开范围**

P0 可简化为：

```
同意平台脱敏后公开展示本次维修案例

```

P1 可支持选择：

1. 仅公开车型和服务项目；
2. 公开维修过程图；
3. 公开门店名称；
4. 公开评价内容；
5. 不公开价格；
6. 允许被分享。

### **13.5 授权勾选**

用户必须主动勾选：

```
我已核对公开内容，同意将本次维修档案公开为案例。《利益共享政策》

```

### **13.6 拒绝公开**

用户可选择不公开，页面需提供「拒绝公开」操作（与「确认公开为案例」并列，不使用警告色）。

点击后二次确认：

```
拒绝后，本次维修档案仍仅作为你的私密记录保存，不会生成公开案例。

```

确认后记录 `user_rejected` 状态；相册仍可正常查看，后续可在规则允许时重新发起公开（P1）。

### **13.7 确认公开与脱敏预览**

用户勾选并点击「确认公开为案例」后：

1. 若商家已提交完工且 **`order_pre_mask` 任务** `preMaskStatus=ready`，直接进入 `pages/desensitize/preview` 预览工作台，**无需等待** AI 脱敏；
2. 预览页展示原图 / 脱敏图对比，用户须逐张查看脱敏图；
3. 用户可重新处理失败项；手工打码 P1；
4. 勾选确认项后点击「确认并公开」，写入授权记录；
5. 预脱敏未完成时，提示稍后再试或走一键脱敏（降级）。

---

公开前必须脱敏：

1. 车牌；
2. VIN；
3. 手机号；
4. 姓名；
5. 人脸；
6. 身份证；
7. 行驶证；
8. 保险单号；
9. 详细住址；
10. 二维码；
11. 其他可识别个人身份的信息。

### **14.1 图片脱敏**

图片脱敏包括：

1. 遮挡车牌；
2. 遮挡人脸；
3. 遮挡 VIN 铭牌；
4. 遮挡证件；
5. 遮挡门禁/地址等敏感信息；
6. 必要时裁剪图片。

### **14.2 文本脱敏**

文本脱敏包括：

```
张三 → 张**
13812345678 → 138****5678
浙A12345 → 浙A****5
VIN 完整码 → 不展示或仅显示后四位

```

## **16. 与评价奖励的关系**

维修相册本身不直接等同于评价奖励。

评价奖励通常依赖：

1. 用户完成订单；
2. 用户提交真实评价；
3. 评价审核通过；
4. 符合奖励规则。

公开案例奖励可依赖：

1. 用户授权公开维修档案；
2. 平台完成脱敏；
3. 案例审核通过；
4. 案例达到质量要求。

具体奖励规则以：

```
08_评价与奖励领取.md
09_分享与收益中心.md

```

为准。

P1 可支持撤回公开授权。

撤回后：

1. 公开案例下架；
2. 已产生的传播数据保留后台记录；
3. 奖励是否撤销按奖励规则处理；
4. 私密维修档案仍保留。

## **21. 合规要求**

1. 原始维修相册默认私密；
2. 未经用户授权不得公开；
3. 公开前必须脱敏；
4. 公开前必须审核；
5. 不得展示完整车牌、VIN、手机号、证件；
6. 不得展示用户实际支付金额；
7. 不得把私密相册链接分享给陌生用户；
8. 不得在分享入口展示诱导奖励文案；
9. 事故车案例公开时不得暗示线上最终报价；
10. 涉及保险、事故责任、违法信息时需谨慎审核。

10. 私密相册不可直接对外分享；
11. 图片加载失败和空状态处理；

## **27. 验收标准**

1. 用户可从订单详情进入维修相册；
2. 非订单本人不可查看相册；
3. 相册按节点展示；
4. 图片可正常预览；
5. 图片加载失败有提示；
6. 车牌、手机号、VIN 不在公开区域展示；
7. 私密相册不可直接分享给陌生用户；
8. 用户可看到隐私提示；
9. 订单完成后可申请公开授权；
10. 授权前必须提示脱敏和审核；
11. 未授权相册不会公开；
12. 公开案例不展示实际支付金额；
13. 事故车案例不展示线上最终价；
14. 分享入口不出现诱导奖励文案；
15. 用户可对相册问题进行反馈或进入售后。
