## 2. 模块定位

服务商品详情页模块负责承接用户从首页、搜索、案例、门店等入口进入后的服务决策流程。

该模块的核心目标是：

1. 让用户看懂服务项目；
2. 让用户理解服务价格模式；
3. 让用户知道哪些门店可提供该服务；
4. 让用户查看类似维修案例；
5. 区分标准服务、价格区间服务、复杂维修预约和事故车检测；
6. 引导用户进入下单、预约或咨询流程；
7. 避免复杂维修和事故车线上误导报价。

### **6.4 价格展示规则**

服务列表页必须根据服务价格模式展示不同文案。

#### **6.4.1 一口价/套餐价**

示例：

```
套餐价 ¥399
¥299 起

```

适用：

1. 小保养；
2. 电瓶更换；
3. 轮胎安装；
4. 标准检测；
5. 固定套餐服务。

#### **6.4.2 价格区间**

示例：

```
参考区间：¥300-¥1200
到店检测后确认

```

适用：

1. 刹车系统；
2. 空调维修；
3. 底盘维修；
4. 钣喷修复；
5. 部分非标准服务。

#### **6.4.3 事故车/复杂维修**

示例：

```
到店检测后报价

```

必须避免展示：

```
线上估价
上传照片精准报价
一口价
最低价

```

### **6.5 筛选和排序**

MVP 可支持基础排序：

1. 默认推荐；
2. 价格从低到高，仅对有明确价格的服务有效；
3. 案例数量优先；
4. 支持门店数量优先。

后续 P1 可支持：

1. 按车型适配；
2. 按新能源/燃油车；
3. 按服务复杂度；
4. 按是否支持在线支付；
5. 按是否支持当天预约。

## **9. 价格模式**

### **9.1 价格模式枚举**


| 价格模式             | 说明       | 示例          |
| ---------------- | -------- | ----------- |
| fixed_price      | 一口价      | ¥399        |
| package_price    | 套餐价      | 套餐价 ¥499    |
| start_price      | 起步价      | ¥299 起      |
| range_price      | 价格区间     | ¥300-¥1200  |
| inspection_quote | 到店检测后报价  | 到店检测后确认     |
| accident_quote   | 事故车检测后报价 | 需到店检测/拆检后确认 |


### **9.2 一口价/套餐价展示**

需要明确展示：

1. 服务价格；
2. 包含项目；
3. 不包含项目；
4. 适用车型或限制；
5. 是否含配件；
6. 是否含工时；
7. 是否支持退款；
8. 是否支持在线支付。

示例：

```
套餐价 ¥399
包含：机油、机滤、基础检查、工时
不包含：额外维修、更换其他配件、特殊车型差价

```

### **9.3 起步价展示**

示例：

```
¥299 起
具体费用以车型、配件品牌和门店检测结果为准

```

### **9.4 价格区间展示**

示例：

```
参考区间：¥300-¥1200
实际费用会因车型、配件品牌、损伤程度、工时和门店检测结果不同而变化。

```

### **9.5 事故车价格展示**

事故车服务必须展示：

```
事故车维修无法仅凭线上图片准确报价。
平台不提供线上最终报价，实际维修方案和费用需门店检测或拆检后确认。
你可以先查看类似案例，了解常见维修方案和费用影响因素。

```

禁止展示：

1. 事故车线上最终价；
2. 上传照片立即报价；
3. 最低价承诺；
4. 保证报价；
5. 按图片报价；
6. 固定维修金额。

## **10. 服务包含与不包含**

### **10.1 服务包含**

展示该服务明确包含的内容。

示例：

```
包含：
- 指定配件
- 标准工时
- 基础检测
- 维修过程相册，门店支持时

```

### **10.2 服务不包含**

必须展示容易引发争议的排除项。

示例：

```
不包含：
- 额外损坏维修
- 非套餐配件
- 特殊车型差价
- 拆检后新增项目
- 用户到店后额外选择的升级服务

```

### **10.3 特殊提示**

如果服务价格可能变化，需要展示：

```
如到店检测发现车辆情况与所选服务不符，门店会与你确认新的维修方案和费用。

```

```
- 需要保险定损的维修
- 需要拆检后才能判断的复杂故障

```

不适用时应引导用户进入复杂维修预约或联系客服。

## **12. 价格影响因素**

根据服务类型展示影响因素。

通用因素：

1. 车型；
2. 配件品牌；
3. 配件规格；
4. 车辆年款；
5. 损伤程度；
6. 是否需要拆装；
7. 是否需要喷漆；
8. 工时差异；
9. 是否新能源车型；
10. 门店检测结果。

事故车额外因素：

1. 是否涉及结构件；
2. 是否涉及安全气囊；
3. 是否涉及底盘；
4. 是否涉及电池包；
5. 是否需要保险定损；
6. 是否需要拆检；
7. 是否需要更换总成；
8. 是否存在二次损伤。

## **13. 服务流程**

### **13.1 标准服务流程**

```
选择服务
→ 选择门店
→ 选择车辆
→ 选择预约时间
→ 确认套餐
→ 提交订单
→ 微信支付
→ 等待商家接单
→ 到店服务
→ 查看维修相册
→ 确认完工
→ 评价

```

### **13.2 复杂维修预约流程**

```
选择服务
→ 查看价格说明
→ 选择门店
→ 填写问题描述
→ 上传车辆照片，可选
→ 选择预约时间
→ 提交预约
→ 等待商家确认
→ 到店检测
→ 确认维修方案

```

### **13.3 事故车预约流程**

```
查看事故车说明
→ 查看类似案例
→ 选择门店
→ 填写事故/损伤描述
→ 上传车辆照片，可选
→ 勾选知晓到店检测后报价
→ 提交预约
→ 到店检测或拆检后确认方案

```

## **17. 常见问题**

```
支持透明维修的门店会上传维修过程图片，你可以在订单中查看维修相册。
```

## **20. 异常与边界场景**

### **20.1 服务下架**

提示：

```
该服务已下架，请查看其他服务。

```

### **20.2 服务城市未开放**

提示：

```
当前城市暂未开放该服务，你可以先浏览案例或联系客服。

```

### **20.3 无可预约门店**

提示：

```
当前暂无可预约门店，你可以查看类似案例或联系客服。

```

### **20.4 套餐售罄**

提示：

```
该套餐暂不可选，请选择其他套餐或联系门店。

```

### **20.5 价格信息缺失**

如果服务已上架但价格缺失：

1. 标准服务不可展示为可下单；
2. 复杂服务可展示“到店检测后确认”；
3. 后台需告警运营补齐信息。

### **20.6 门店不支持当前服务**

提示：

```
该门店暂不支持当前服务，请选择其他门店。

```

## **21. 合规要求**

### **21.1 价格合规**

1. 标准套餐必须明确包含和不包含项目；
2. 价格区间必须展示影响因素；
3. 事故车不得展示线上最终报价；
4. 不使用“最低价”“保证修好”等绝对化表达；
5. 不展示用户实际支付金额作为宣传；
6. 不诱导用户误解价格确定性。

### **21.2 案例合规**

1. 服务详情页案例必须脱敏；
2. 不展示用户隐私；
3. 不展示未经授权图片；
4. 不展示未审核案例；
5. 不展示被下架门店的关联案例。

### **21.3 分享合规**

服务详情页可分享，但分享卡片不得出现：

```
分享有奖
点击有奖
点赞有奖
浏览有奖
帮我赚钱

```

分享标题应为内容型：

```
杭州透明维修服务：刹车片更换流程与价格说明

```
