### **1.1 平台身份（V2.0）**

平台**不是**：交易平台、维修服务提供方、统一报价方、维修质量担保方、售后赔偿主体。

平台**是**：内容展示平台、案例相册工具、GEO 本地搜索引流工具、门店获客工具、咨询线索转接工具。

统一说明（服务页、门店页、案例页、咨询页、相册页均需展示）：

> 平台提供门店信息展示、维修案例展示、服务相册、咨询预约和隐私脱敏工具。实际维修服务、检测报价、收款、发票、质保和售后由门店线下提供和承担。案例内容由门店上传或经用户授权后展示，仅供参考，不构成平台报价、维修承诺或质量保证。

## **2. 产品目标**

1. **看得懂**：看懂服务项目、价格影响因素、类似案例与维修过程。
2. **敢选择**：通过门店资质、脱敏案例、服务相册建立信任，降低不透明担忧。
3. **有档案**：线下维修后通过服务相册留存过程与配件确认记录；用户可授权脱敏公开。

V2.0 **不做**：在线下单、微信支付、订单履约、完工确认、交易评价奖励、售后仲裁。

---

## **3. 核心闭环**

```text
用户进入小程序
→ 搜索服务/车型/故障关键词
→ 浏览服务说明、案例和门店
→ 查看门店案例相册
→ 电话咨询 / 在线咨询 / 预约到店
→ 线下到店检测、报价、维修、付款
→ 商家创建服务相册
→ 用户查看服务相册、完成配件待确认（如有）
→ 用户选择是否授权脱敏公开
→ 平台生成公开案例 / GEO 内容页
→ 案例继续为门店带来搜索流量和咨询线索
```

## **6. 咨询与预约（摘要）**

- 入口：服务详情、案例详情、门店详情、GEO 页。
- 按钮文案：联系门店、预约到店、咨询报价、提交咨询（禁用：立即购买、提交订单、在线支付）。

- 事故车须额外勾选：需到店检测后由门店报价；平台不提供线上最终报价。
- 详情见 `05_咨询与到店预约.md`。

## **10. 案例、授权公示与分享**

### **10.1 授权公示**

- 案例来源：**用户授权案例**、**商家历史案例**（无「平台订单案例」）。
- **授权公示**：用户授权平台将脱敏内容发布到案例 Tab、H5、GEO 等公开展示位；审核通过后 **自动公示**，用户无需再手工发布。
- 平台公开展示 **强制脱敏 + 审核**（见 `00_Phase1_服务相册产品口径.md` §4.5）。

### **10.2 用户自主分享（私人传播）**

- 用户分享 **自己的** 服务相册/案例给好友、朋友圈、社交账号等，**不需要** 先完成授权公示。
- 分享前提供脱敏选项：**默认脱敏**，用户 **可选择原图分享**；平台展示风险提示并留痕，**不强制** 脱敏（见口径 §4.5.3）。
- 私人分享 **不** 自动进入平台案例 Tab / H5。

### **10.3 合规**

- V2.0 不做评价奖励、分享领钱类诱导。
- 禁止混用文案：「分享即公开」「须授权后才能分享」。

## **13. P0 验收标准**

1. 用户可微信登录、绑定手机号；
2. 可浏览首页、服务、门店、案例并搜索；
3. 可查看服务/门店详情（含资质、案例、咨询入口）；
4. 可查看公开脱敏案例；
5. 可通过电话、在线咨询或表单提交咨询/预约，提交前确认线下服务条款；
6. 可查看咨询/预约记录；
7. 可通过链接/扫码查看私密服务相册；
8. 可在我的服务相册中查看已关联相册；
9. 可预览脱敏效果并同意/拒绝 **授权公示**；
10. 可在 **我的服务相册** 自主分享（默认脱敏、可选原图），且分享 **不依赖** 授权公示；
11. 平台可生成服务/案例/门店/车型/故障相关 GEO 页面。
