# **2. 归因定义**

转化归因是指：

```
当用户通过某个分享链接进入平台，并产生访问、咨询、下单、支付或完成订单等行为时，系统判断该行为应归属于哪个分享人、哪个分享对象、哪个分享链接。

```

归因结果用于：

1. 分享数据统计；
2. 内容传播效果分析；
3. 门店转化分析；
4. 服务商品转化分析；
5. 后续分享转化奖励依据；
6. 风控识别。

---

# **3. 归因对象**

归因对象包括：


| 对象   | 说明                  |
| ---- | ------------------- |
| 分享人  | 发起分享的用户             |
| 被分享人 | 点击或访问分享链接的用户        |
| 分享链接 | 携带 share_code 的链接   |
| 分享对象 | 案例、门店、服务商品、问答、小程序首页 |
| 转化行为 | 浏览、咨询、下单、支付、完成订单    |
| 转化订单 | 被分享人后续产生的订单         |


---

# **4. 分享链接有效期**

分享链接本身可长期有效，但转化归因有效期应有限制。

## **4.1 默认归因有效期**


| 行为     | 默认有效期 | 说明               |
| ------ | ----- | ---------------- |
| 浏览归因   | 7 天   | 用于统计访问来源         |
| 咨询归因   | 7 天   | 用户在 7 天内咨询归属于该分享 |
| 下单归因   | 30 天  | 汽修决策周期较长，可配置     |
| 支付归因   | 跟随下单  | 下单已归因则支付归因一致     |
| 完成订单归因 | 跟随订单  | 订单归因后完成状态继承      |


说明：

- 以上时间均可由运营后台配置；
- 超出归因有效期后，不再归属该分享；
- 分享链接仍可打开，但不再计入转化归因。

---

# **5. 归因模型**

MVP 推荐：

```
最后一次有效点击归因 Last Click

```

即：

```
用户在归因有效期内多次点击不同人的分享链接，最终转化归属于距离转化最近的一次有效分享点击。

```

---

## **5.1 为什么采用最后点击归因**

原因：

1. 规则简单；
2. 用户和运营容易理解；
3. 便于实现；
4. 更符合小程序分享场景；
5. 便于处理多人分享冲突；
6. 可降低长期抢占归因的问题。

---

## **5.2 首次点击记录**

虽然收益或转化归因采用最后点击，但系统仍应记录首次点击。

用途：

1. 分析最早触达来源；
2. 统计内容冷启动传播；
3. 辅助风控判断；
4. 为后续多触点归因提供数据。

---

# **6. 多人分享冲突处理**

当用户点击多个分享链接时，按以下优先级处理：

```
有效最后点击 > 有效首次点击 > 无归因

```

## **6.1 标准规则**


| 场景                     | 归因结果            |
| ---------------------- | --------------- |
| 用户只点击 A 的链接并下单         | 归因 A            |
| 用户先点击 A，后点击 B，并在有效期内下单 | 归因 B            |
| 用户点击 A，超过有效期后下单        | 不归因             |
| 用户点击 A 后又点击自己分享的链接     | 自点无效，仍归因上一个有效链接 |
| 用户点击多个异常链接             | 不归因或进入风控        |
| 多个链接时间完全相同             | 以服务端接收时间为准      |


---

## **6.2 同一分享人多次分享**

同一分享人分享同一对象多次：

1. 可记录多条分享记录；
2. 转化归因到最近一次有效 share_code；
3. 统计时可聚合到同一分享人；
4. 不允许重复计算多份收益。

---

## **6.3 同一内容被多人分享**

同一案例或门店被多人分享：

1. 访问归因到具体 share_code；
2. 转化归因到最后一次有效点击；
3. 内容本身可统计总传播效果；
4. 不允许多人同时获得同一笔转化收益。

---

# **7. 用户已注册情况下的归因**

已注册用户点击分享链接后，仍可产生内容归因或订单归因。

## **7.1 已注册用户规则**


| 场景               | 是否归因   |
| ---------------- | ------ |
| 已注册用户点击案例分享并浏览   | 记录访问归因 |
| 已注册用户点击门店分享并咨询   | 记录咨询归因 |
| 已注册用户点击服务商品分享并下单 | 记录订单归因 |
| 已注册用户点击后超过有效期下单  | 不归因    |
| 已注册用户自己点击自己分享链接  | 不归因    |
| 已注册用户与分享人存在异常关联  | 进入风控   |


说明：

- 已注册用户不属于“拉新”；
- 如果后续存在邀请奖励，应与分享转化归因分开；
- 分享归因只说明“本次转化来源”，不代表邀请关系。

---

# **8. 新用户情况下的归因**

新用户通过分享链接进入：

1. 未登录时先记录 anonymous_id；
2. 登录后绑定 user_id、openid、unionid；
3. 若在有效期内下单，则归因至对应 share_code；
4. 若多个分享链接冲突，使用最后有效点击；
5. 如后续启用邀请奖励，需另行判断是否满足邀请条件。

---

# **9. 跨小程序、H5 归因**

跨端归因需要统一使用 share_code。

## **9.1 小程序归因**

小程序分享链接应携带：

```
scene=share_code

```

或页面参数：

```
share_code=xxxx
object_type=case
object_id=123

```

进入页面后：

1. 服务端解析 share_code；
2. 记录访问事件；
3. 记录 openid/unionid；
4. 未登录用户生成 anonymous_id；
5. 登录后合并归因链路。

---

## **9.2 H5 归因**

H5 链接应携带：

```
https://example.com/case/123?share_code=xxxx

```

H5 端处理：

1. 读取 share_code；
2. 写入 cookie/localStorage；
3. 上报访问事件；
4. 若跳转小程序，透传 share_code；
5. 用户登录后绑定 user_id；
6. 下单时带上最近有效 share_code。

---

## **9.3 H5 跳小程序**

H5 跳转小程序时，应携带：

```
share_code
object_type
object_id
source=h5

```

小程序打开后：

1. 校验 share_code；
2. 记录跨端访问；
3. 继承 H5 归因；
4. 继续使用最后有效点击规则。

---

## **9.4 小程序跳 H5**

小程序跳 H5 时，应携带：

```
share_code
openid_hash
anonymous_id

```

H5 端记录后，后续再绑定用户身份。

---

# **10. 自点与强关联处理**

以下行为不计入有效归因：

1. 分享人点击自己的链接；
2. 同一设备多账号反复点击；
3. 同一 IP 高频访问；
4. 分享人和被分享人使用同一手机号；
5. 分享人和被分享人同一 openid/unionid；
6. 分享人和被分享人疑似同一自然人；
7. 商家员工自刷；
8. 脚本访问。

处理：

```
记录访问 → 标记无效 → 不计收益 → 必要时进入风控

```

---

# **11. 订单归因规则**

订单创建时，应尝试读取最近有效 share_code。

## **11.1 标准订单归因**

```
用户点击分享链接
→ 系统记录有效点击
→ 用户在有效期内下单
→ 订单绑定 share_code
→ 订单支付成功
→ 订单完成
→ 归因成立

```

---

## **11.2 订单归因字段**

订单或归因表应记录：


| 字段                    | 说明         |
| --------------------- | ---------- |
| order_id              | 订单 ID      |
| share_code            | 分享码        |
| sharer_user_id        | 分享人        |
| attributed_user_id    | 被分享用户      |
| object_type           | 分享对象类型     |
| object_id             | 分享对象 ID    |
| attribution_model     | last_click |
| attribution_time      | 归因时间       |
| attribution_expire_at | 归因过期时间     |
| risk_status           | 风控状态       |


---

# **12. 退款、售后与归因**

归因不等于收益。


| 订单状态   | 归因处理         |
| ------ | ------------ |
| 已下单未支付 | 可记录订单归因      |
| 已支付    | 可记录支付归因      |
| 已完成    | 可记录完成归因      |
| 退款申请中  | 如有收益则冻结      |
| 部分退款   | 如有收益则人工审核    |
| 全额退款   | 如有收益则撤销      |
| 售后纠纷中  | 如有收益则冻结      |
| 风控异常   | 归因可保留，但收益不发放 |

# **14. 后台管理能力**

运营后台应支持：

1. 查看分享归因链路；
2. 查看用户点击历史；
3. 查看订单归因来源；
4. 查看首次点击和最后点击；
5. 查看跨端归因；
6. 标记归因为无效；
7. 查看归因风控原因；
8. 导出归因数据；
9. 支持人工修正归因，需权限；
10. 所有修正操作留痕。

---

# **15. P0 验收标准**

P0 阶段必须满足：

1. 分享链接包含 share_code；
2. 小程序可解析 share_code；
3. H5 可解析 share_code；
4. 支持记录分享点击；
5. 支持记录访问来源；
6. 支持最后点击归因；
7. 支持多人分享冲突处理；
8. 支持已注册用户归因；
9. 支持未登录用户 anonymous_id 归因；
10. 支持登录后绑定归因；
11. 支持小程序/H5 跨端透传 share_code；
12. 支持自点识别；
13. 支持订单绑定归因来源；
14. 支持退款、售后时冻结或撤销后续收益；
15. 支持后台查看归因链路。
