# **2. 与《评价体系PRD》的关系**

`01_评价体系PRD.md` 是评价与奖励模块的总体设计文档。

本文件是对其中 **评价奖励规则** 的细化，必须遵循以下总体原则：

1. 奖励不是返佣；
2. 奖励不与好评绑定；
3. 奖励只鼓励真实有效评价和透明维修内容贡献；
4. 退款中、售后中、纠纷中的订单奖励必须冻结；
5. 全额退款订单奖励必须撤销；
6. MVP 不做浏览收益、点击收益、分享点击奖励、多级分销；
7. 所有奖励操作必须留痕，可审核、可冻结、可撤销。

---

# **3. 奖励设计目标**

评价奖励体系的目标是：

1. 鼓励真实用户在完成服务后提交评价；
2. 提升门店、服务商品、维修案例的可信度；
3. 沉淀可用于门店主页、案例库、服务页、SEO/GEO 的内容资产；
4. 鼓励用户授权公开脱敏维修案例；
5. 防止刷评、虚假评价、奖励套利；
6. 让奖励与真实订单、真实履约、真实内容绑定。

---

# **4. 基础原则**

评价奖励遵循以下原则：

1. 只有已完成订单才能评价；
2. 只有真实交易评价才可能获得奖励；
3. 奖励与好评、差评无直接绑定关系；
4. 商家不得诱导用户给好评；
5. 评价内容必须真实、合规、与订单相关；
6. 订单退款中、售后中、纠纷中时，奖励冻结；
7. 全额退款成功后，奖励撤销；
8. 命中风控的评价不发放奖励；
9. 同一订单同一奖励类型只能领取一次；
10. 奖励计算、审核、发放、冻结、撤销必须记录日志。

---

# **5. 可评价条件**

用户可以评价的条件如下：


| 条件              | 是否必须 | 说明               |
| --------------- | ---- | ---------------- |
| 订单状态为 COMPLETED | 是    | 用户确认完成或系统自动完成    |
| 支付状态为 PAID      | 是    | 未支付订单不可评价        |
| 订单未全额退款         | 是    | 全额退款订单不可评价或评价不公开 |
| 当前用户为订单本人       | 是    | 防止代评             |
| 未超过评价有效期        | 是    | 评价有效期可运营配置       |
| 用户未被风控限制        | 是    | 高风险用户不可领奖        |
| 订单无严重售后争议       | 是    | 售后中建议冻结或隐藏评价     |


---

# **6. 不可评价场景**

以下场景不可评价：

1. 订单未支付；
2. 订单未完成；
3. 订单已取消；
4. 订单已关闭；
5. 订单已全额退款；
6. 当前用户不是订单用户；
7. 订单命中作弊风控；
8. 用户被平台限制评价权限；
9. 超过评价有效期；
10. 当前订单处于严重售后或纠纷中。

说明：

- 已评价后发生售后，评价可进入 `售后冻结` 状态；
- 已评价后发生全额退款，奖励应撤销，评价可隐藏或标记争议；
- 已评价后发生部分退款，奖励需重新判断。

# **7. 评价内容要求**

评价内容与 `01_评价体系PRD.md` 保持一致。

## **7.1 六维评分**

用户评价包含六维评分：


| 维度     | 说明               |
| ------ | ---------------- |
| 服务态度   | 接待、沟通、响应是否良好     |
| 专业程度   | 检查、维修、解释是否专业     |
| 价格透明   | 报价、费用、增项是否清晰     |
| 维修过程透明 | 是否能看到维修过程、相册是否清楚 |
| 完工效果   | 修复效果、交付质量        |
| 是否愿意推荐 | 综合推荐意愿           |


评分方式：

```
1-5 星

```

MVP 综合评分：

```
综合评分 = 六维评分平均值

```

# **9. 评价奖励资格**

## **9.1 基础评价奖励资格**

用户获得评价奖励需要满足：

1. 订单已完成；
2. 支付状态为已支付；
3. 订单未全额退款；
4. 用户已提交六维评分；
5. 用户已提交真实有效文字评价；
6. 评价内容通过机器审核或人工审核；
7. 评价未命中风控；
8. 用户未超过每日、每月奖励上限；
9. 商家与用户不存在异常关联；
10. 同一订单未领取过评价奖励。

---

## **9.2 不发放评价奖励的场景**

以下场景不发放评价奖励：

1. 仅提交星级但无有效内容，运营可配置是否低奖励或不奖励；
2. 评价内容明显无意义；
3. 评价复制粘贴；
4. 评价与订单服务不相关；
5. 评价包含广告、导流、手机号、微信号；
6. 评价包含辱骂、威胁、违法违规内容；
7. 评价图片与订单无关；
8. 用户或商家命中高风险；
9. 订单进入售后、退款、纠纷；
10. 订单全额退款。

---

# **10. 公开授权奖励资格**

公开授权奖励用于鼓励用户将维修记录脱敏后授权平台公开为案例。

## **10.1 获得条件**

用户获得公开授权奖励需要满足：

1. 订单已完成；
2. 用户已提交评价；
3. 用户主动授权公开；
4. 维修相册、维修记录、车辆信息完成脱敏；
5. 案例通过平台审核；
6. 案例无售后、纠纷、高风险标记；
7. 同一订单未领取过公开授权奖励。

---

## **10.2 不发放公开授权奖励的场景**

以下场景不发放公开授权奖励：

1. 用户未授权公开；
2. 车辆信息无法脱敏；
3. 相册或维修记录缺失；
4. 相册图片疑似造假；
5. 案例内容与订单不一致；
6. 订单发生严重售后；
7. 订单全额退款；
8. 平台审核不通过；
9. 用户频繁授权后申请下架；
10. 商家存在案例造假风险。

# **11. 奖励计算规则**

## **11.1 基础公式**

评价奖励建议公式：

```
评价奖励 = 基础奖励 × 服务复杂度系数 × 订单金额系数 × 风控系数

```

公开授权奖励建议公式：

```
公开授权奖励 = 授权基础奖励 × 案例质量系数 × 风控系数

```

说明：

- 基础奖励由运营后台配置；
- 风控系数正常为 1；
- 命中高风险时风控系数为 0 或进入冻结；
- MVP 阶段可先使用固定金额配置，后续再启用复杂公式。

---

## **11.2 服务复杂度系数**


| 服务等级 | 示例           | 系数建议 |
| ---- | ------------ | ---- |
| L1   | 简单检测、雨刮      | 0.5  |
| L2   | 保养、轮胎、电瓶、刹车  | 1.0  |
| L3   | 底盘、空调、复杂故障   | 1.5  |
| L4   | 事故车、大修、新能源三电 | 2.0  |


---

## **11.3 订单金额系数**


| 金额区间      | 系数建议 |
| --------- | ---- |
| 0-299     | 0.5  |
| 300-999   | 1.0  |
| 1000-2999 | 1.5  |
| 3000 以上   | 2.0  |


---

## **11.4 案例质量系数**

公开授权奖励可根据案例质量调整。


| 案例质量 | 说明               | 系数建议  |
| ---- | ---------------- | ----- |
| 普通   | 有基础相册和维修说明       | 1.0   |
| 较好   | 有清晰过程图和完工图       | 1.2   |
| 优质   | 过程完整、说明清楚、适合公开展示 | 1.5   |
| 高风险  | 信息不完整或需复核        | 0 或冻结 |


---

## **11.5 奖励封顶**

奖励必须配置封顶。


| 奖励类型      | 单次封顶         |
| --------- | ------------ |
| 评价奖励      | 5-20 元，运营配置  |
| 公开授权奖励    | 10-50 元，运营配置 |
| 单用户每日奖励   | 运营配置         |
| 单用户每月奖励   | 运营配置         |
| 单门店每日奖励总额 | 运营配置         |

# **12. 奖励状态**

奖励状态建议如下：


| 状态   | 状态码           | 说明           |
| ---- | ------------- | ------------ |
| 待计算  | PENDING_CALC  | 用户已评价，系统待计算  |
| 待风控  | RISK_CHECKING | 等待风控检查       |
| 待审核  | WAIT_REVIEW   | 需要平台人工审核     |
| 待发放  | WAIT_GRANT    | 已通过，等待发放     |
| 已发放  | GRANTED       | 奖励到账或已发放     |
| 已冻结  | FROZEN        | 售后、退款、风控原因冻结 |
| 已驳回  | REJECTED      | 不符合奖励条件      |
| 已撤销  | REVOKED       | 已发放或待发放奖励被撤销 |
| 已失效  | EXPIRED       | 超时未领取或规则失效   |
| 发放失败 | GRANT_FAILED  | 发放通道异常       |

# **13. 奖励发放方式**

## **13.1 MVP 推荐方式**

与 `01_评价体系PRD.md` 保持一致，MVP 初期建议：

```
先以“奖励记录 + 运营人工发放/优惠券”方式验证，再接入自动转账。

```

可选方式：


| 发放方式   | 说明              | MVP建议 |
| ------ | --------------- | ----- |
| 奖励记录   | 用户可见，运营后台可管理    | 推荐    |
| 运营人工发放 | 人工通过微信、线下或后台记录  | 推荐初期  |
| 优惠券    | 合规压力较低，可促进复购    | 可选    |
| 平台余额   | 技术可控，但涉及资金账户合规  | 谨慎    |
| 微信零钱转账 | 用户感知强，但风控和实名要求高 | 后置    |


---

## **13.2 发放流程**

```
用户提交评价
→ 生成奖励记录
→ 系统计算奖励金额
→ 风控检测
→ 必要时进入人工审核
→ 审核通过
→ 奖励进入待发放
→ 运营人工发放或系统发放
→ 用户收到通知
→ 奖励状态变为已发放

```

# **14. 奖励冻结规则**

以下情况奖励应冻结：

1. 用户申请退款；
2. 用户发起售后；
3. 商家发起投诉；
4. 评价被举报；
5. 订单进入纠纷；
6. 用户命中作弊规则；
7. 商家被判定存在诱导评价；
8. 平台人工复核中；
9. 支付或退款状态异常；
10. 公开案例发生争议。

冻结后：

```
reward_status = FROZEN

```

冻结期间：

1. 不可提现；
2. 不可重复发放；
3. 用户可查看冻结原因；
4. 平台可人工解冻、驳回或撤销。

---

# **15. 奖励撤销规则**

以下情况奖励应撤销：

1. 订单全额退款成功；
2. 评价审核不通过；
3. 评价内容虚假；
4. 用户刷评；
5. 商家诱导评价；
6. 用户删除评价，视平台规则；
7. 内容涉及违法违规；
8. 同一图片或文字重复使用；
9. 风控判定为作弊；
10. 公开案例被判定造假；
11. 用户撤回公开授权且平台规则要求追回。

撤销后：

```
reward_status = REVOKED

```

如奖励已发放：

1. 生成冲正记录；
2. 可限制用户后续提现；
3. 可从用户后续奖励中抵扣；
4. 严重作弊可封禁账号。

# **16. 与退款、售后的关系**


| 订单/退款/售后状态 | 奖励处理          |
| ---------- | ------------- |
| 无退款、无售后    | 正常判断奖励        |
| 退款申请中      | 冻结奖励          |
| 退款审核中      | 冻结奖励          |
| 部分退款成功     | 视规则保留、降低或人工审核 |
| 全额退款成功     | 撤销奖励          |
| 退款失败       | 恢复奖励判断        |
| 售后申请中      | 冻结奖励          |
| 售后纠纷中      | 暂停奖励发放        |
| 售后驳回       | 可恢复奖励判断       |
| 售后成立且商家责任  | 平台人工判断奖励是否保留  |
| 公开案例争议中    | 冻结公开授权奖励      |

# **17. 评价审核规则**

## **17.1 自动审核**

自动审核检查：

1. 敏感词；
2. 广告引流；
3. 电话、微信号；
4. 辱骂攻击；
5. 涉政涉黄涉暴；
6. 隐私信息；
7. 重复文本；
8. 图片违规；
9. 疑似虚假评价；
10. 与订单服务不匹配。

---

## **17.2 人工审核**

以下情况进入人工审核：

1. 高额奖励评价；
2. 低分评价；
3. 被商家申诉；
4. 被用户举报；
5. 包含图片；
6. 命中敏感词；
7. 命中风控规则；
8. 系统无法判断真实性；
9. 涉及售后或退款；
10. 涉及公开案例授权。

# **19. 后台管理能力**

运营后台需要支持：

1. 查看评价列表；
2. 查看评价详情；
3. 审核评价；
4. 隐藏评价；
5. 恢复评价；
6. 查看奖励资格；
7. 查看奖励计算明细；
8. 审核奖励；
9. 修改奖励金额，需权限；
10. 冻结奖励；
11. 解冻奖励；
12. 驳回奖励；
13. 撤销奖励；
14. 查看风控原因；

# **21. P0 验收标准**

上线前必须满足：

1. 订单完成后才能评价；
2. 取消、关闭、全额退款订单不可评价；
3. 支持六维评分；
4. 支持文字评价；
5. 支持匿名评价；
6. 支持评价审核；
7. 支持评价公开展示；
8. 支持评价奖励；
9. 支持公开授权奖励；
10. 支持奖励状态流转；
11. 支持退款中冻结奖励；
12. 支持全额退款撤销奖励；
13. 支持售后中冻结奖励；
14. 支持风控拦截；
15. 支持后台人工审核；
16. 支持奖励发放日志；
17. 支持用户查看奖励记录；
18. 不允许奖励绑定好评；
19. 不做浏览收益、点击收益、多级分销；
20. 支持评价内容隐私保护。
