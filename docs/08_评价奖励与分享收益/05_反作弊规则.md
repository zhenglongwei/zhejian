# **2. 反作弊目标**

反作弊目标包括：

1. 防止虚假订单获取奖励；
2. 防止退款后继续领取奖励；
3. 防止批量注册账号刷奖励；
4. 防止同设备多账号刷评价；
5. 防止商家诱导或组织刷好评；
6. 防止恶意差评勒索；
7. 防止重复内容评价；
8. 防止虚假公开案例授权；
9. 防止虚假分享归因；
10. 防止未来分享转化收益作弊；
11. 支持奖励冻结、撤销和人工复核。

---

# **3. 作弊类型**


| 类型     | 说明                   |
| ------ | -------------------- |
| 刷单作弊   | 虚假下单、虚假支付、快速退款       |
| 刷评价    | 大量重复好评、模板化评价         |
| 刷奖励    | 通过评价、公开授权、分享转化获取不当奖励 |
| 羊毛党    | 多账号、多设备、多手机号薅奖励      |
| 商家自刷   | 商家组织用户或员工刷单刷评        |
| 恶意差评   | 用户以差评威胁商家或平台         |
| 虚假授权   | 授权公开的案例非真实订单或素材造假    |
| 虚假分享   | 分享链接未产生真实访问或转化       |
| 图片作弊   | 重复使用维修图片或盗用案例图片      |
| 退款套利   | 领取奖励后退款              |
| 关联账号作弊 | 同 IP、同设备、同支付账户异常聚集   |

# **4. 风险等级**


| 等级  | 说明   | 处理方式           |
| --- | ---- | -------------- |
| C0  | 正常   | 正常展示、正常发放奖励    |
| C1  | 轻微异常 | 记录观察           |
| C2  | 中度异常 | 奖励延迟发放或进入审核    |
| C3  | 高风险  | 冻结奖励、限制提现、人工复核 |
| C4  | 确认作弊 | 撤销奖励、封禁、加入黑名单  |


---

# **5. 奖励发放基础条件**

任何评价奖励或公开授权奖励发放前必须满足：

1. 用户账号状态正常；
2. 订单真实有效；
3. 订单已支付；
4. 订单已完成；
5. 订单未全额退款；
6. 评价已提交并通过审核；
7. 公开案例已授权并通过审核，如适用；
8. 用户未被判定为高风险；
9. 商家未被判定为高风险；
10. 同一订单同一奖励类型未重复领取。

---

# **6. 不得发放奖励的场景**

以下情况不得发放奖励：

1. 未支付订单；
2. 已取消订单；
3. 已关闭订单；
4. 全额退款订单；
5. 虚假订单；
6. 风控冻结订单；
7. 未完成订单；
8. 未通过审核的评价；
9. 涉及违规内容的评价；
10. 用户撤回评价或删除评价，视平台规则；
11. 用户被封禁；
12. 商家被封禁；
13. 用户与商家存在异常关联；
14. 同一订单重复提交评价领奖；
15. 同一素材重复用于多个评价或案例；
16. 后台标记作弊。

---

# **7. 退款与奖励联动规则**


| 订单/退款状态 | 奖励处理          |
| ------- | ------------- |
| 未支付     | 不生成奖励         |
| 已支付待服务  | 不生成奖励         |
| 服务中     | 不生成奖励         |
| 已完成未评价  | 不生成奖励         |
| 已完成已评价  | 可进入待审核或待发放    |
| 退款申请中   | 冻结奖励          |
| 部分退款    | 按规则降低、冻结或人工审核 |
| 全额退款    | 撤销奖励          |
| 退款失败    | 恢复原奖励状态，视风控结果 |
| 售后争议中   | 冻结奖励          |
| 风控异常    | 冻结奖励          |

# **8. 用户维度反作弊规则**

## **8.1 高频行为**

触发风险：

1. 单用户短时间大量下单；
2. 单用户短时间大量评价；
3. 单用户短时间大量分享；
4. 单用户短时间多次授权公开案例；
5. 单用户短时间多次提现；
6. 单用户多次退款后仍评价领奖。

建议动作：


| 风险        | 动作          |
| --------- | ----------- |
| 高频评价      | 延迟奖励、人工审核   |
| 高频下单取消    | 限制下单        |
| 高频退款      | 冻结奖励        |
| 高频分享无转化   | 降低分享权重或标记异常 |
| 高频公开授权后下架 | 限制公开授权奖励    |
| 高频提现      | 提现审核        |


---

## **8.2 账号异常**

风险特征：

1. 新注册账号立即下单评价；
2. 多账号绑定同一手机号；
3. 多账号使用同一设备；
4. 多账号同 IP 密集操作；
5. 账号资料高度相似；
6. 登录地频繁变化；
7. 账号长期无行为但集中参与奖励活动。

处理：

1. 奖励延迟；
2. 进入人工审核；
3. 限制提现；
4. 要求补充验证；
5. 加入黑名单。

---

# **9. 商家维度反作弊规则**

商家触发风险：

1. 同一商家短时间好评异常集中；
2. 同一商家评价文本高度相似；
3. 同一商家低价订单异常集中；
4. 同一商家退款率异常；
5. 同一商家用户来源高度集中；
6. 商家员工或关联账号下单；
7. 商家诱导用户好评返现；
8. 商家引导私下交易；
9. 商家重复上传相同案例；
10. 商家被多名用户投诉刷评。

处理动作：

1. 商家评价进入严格审核；
2. 暂停商家奖励权益；
3. 降低评价权重；
4. 冻结相关用户奖励；
5. 暂停案例公开；
6. 下线违规评价；
7. 商家警告、限制或封禁。

---

# **10. 订单维度反作弊规则**

风险订单特征：

1. 下单到支付时间极短且批量出现；
2. 支付后快速完成；
3. 完成后立即评价并领奖；
4. 低价订单大量集中；
5. 同一商家同一服务短时异常成交；
6. 订单无相册、无履约证据；
7. 相册图片重复；
8. 订单服务内容与评价内容不符；
9. 订单完成后快速退款；
10. 订单来源异常集中。

处理：

```
低风险：记录观察
中风险：奖励延迟发放
高风险：冻结奖励并人工审核
确认作弊：撤销奖励，标记订单作弊

```

---

# **11. 评价内容反作弊规则**

评价内容风险：

1. 内容过短；
2. 大量重复文案；
3. 模板化好评；
4. 与服务项目不相关；
5. 含手机号、微信号、二维码；
6. 含外部平台导流；
7. 含辱骂、威胁、违法内容；
8. 图片与订单无关；
9. 图片重复使用；
10. 评价时间异常集中。

处理规则：


| 场景    | 动作         |
| ----- | ---------- |
| 内容过短  | 不计入奖励或降低奖励 |
| 重复评价  | 进入审核       |
| 含隐私信息 | 驳回或脱敏后展示   |
| 含导流信息 | 驳回         |
| 恶意辱骂  | 隐藏并人工处理    |
| 图片重复  | 冻结奖励       |
| 与订单不符 | 审核或驳回      |


---

# **12. 公开授权奖励反作弊规则**

公开授权奖励风险：

1. 案例非真实订单；
2. 相册图片重复；
3. 相册图片来源于网络；
4. 图片包含其他平台水印；
5. 图片与维修服务不匹配；
6. 用户授权后频繁申请下架；
7. 商家批量引导用户授权；
8. 相册存在明显摆拍或造假；
9. 车辆信息未脱敏；
10. 案例与订单记录不一致。

处理动作：

1. 拒绝公开；
2. 冻结公开授权奖励；
3. 撤销公开授权奖励；
4. 下架案例；
5. 标记商家风险；
6. 标记用户风险；
7. 重复素材加入黑名单。

---

# **13. 分享归因反作弊规则**

MVP 不做浏览收益和点击收益，但仍需要识别异常分享归因。

风险特征：

1. 自己点击自己的分享链接；
2. 同一设备反复点击；
3. 同一 IP 大量点击；
4. 点击无停留、无浏览深度；
5. 虚假访问刷 PV；
6. 分享来源异常集中；
7. 分享转化用户与分享人强关联；
8. 使用脚本批量访问；
9. 多账号互点；
10. 商家组织员工刷分享。

有效分享建议条件：

1. 独立访问用户；
2. 停留时间达到最低要求；
3. 有有效页面浏览；
4. 非同一设备或账号自点；
5. 非脚本访问；
6. 如涉及未来收益，必须产生有效订单；
7. 订单未退款或未被风控标记。

处理：

```
无效点击不计入有效访问
异常点击降权
高风险分享归因标记
若启用分享收益，高风险收益冻结
确认作弊撤销收益

```

---

# **14. 未来分享转化收益反作弊**

如后续启用分享转化收益，必须满足：

1. 被分享用户产生真实订单；
2. 订单支付成功；
3. 订单已完成；
4. 订单未全额退款；
5. 订单无严重售后；
6. 分享人与被分享人非同一自然人；
7. 非同设备刷单；
8. 非商家组织自刷；
9. 通过归因规则；
10. 通过风控审核。

不得发放分享转化收益：

1. 自点转化；
2. 同设备互刷；
3. 同手机号重复注册；
4. 虚拟号或异常号段；
5. 新账号无真实行为；
6. 订单快速退款；
7. 批量账号集中领取；
8. 风控标记异常。

| 维度                | 说明        |

| 分享渠道              | 异常渠道      |

| 地理位置              | 异常集中位置    |

# **16. 奖励状态机**

奖励建议状态：

```
待计算
→ 待风控
→ 待审核
→ 待发放
→ 已发放
→ 已冻结
→ 已撤销
→ 发放失败

```

状态说明：


| 状态   | 说明                  |
| ---- | ------------------- |
| 待计算  | 评价、授权或转化行为已发生但未计算奖励 |
| 待风控  | 等待风控检查              |
| 待审核  | 进入人工审核              |
| 待发放  | 风控通过，等待发放           |
| 已发放  | 奖励已到账或已人工发放         |
| 已冻结  | 风险、退款、争议导致冻结        |
| 已撤销  | 确认不应发放              |
| 发放失败 | 发放通道异常              |


---

# **17. 人工审核规则**

进入人工审核的场景：

1. 风险等级 C2 及以上；
2. 高频评价；
3. 高频退款；
4. 商家好评异常集中；
5. 公开案例异常；
6. 分享归因异常；
7. 用户申诉；
8. 商家投诉恶意差评；
9. 系统无法判断的内容风险；
10. 奖励金额超过阈值。

审核结果：


| 结果  | 后续动作         |
| --- | ------------ |
| 通过  | 发放或恢复奖励      |
| 驳回  | 不发放奖励        |
| 冻结  | 暂停发放，等待进一步核查 |
| 撤销  | 已发放奖励追回或抵扣   |
| 封禁  | 账号或商家限制使用    |
| 误判  | 恢复状态并记录规则优化  |


---

# **18. 申诉机制**

用户或商家可对以下结果申诉：

1. 评价被驳回；
2. 奖励被冻结；
3. 奖励被撤销；
4. 公开案例被拒绝；
5. 分享归因不计入；
6. 账号被限制；
7. 商家被限制。

申诉处理要求：

1. 展示处理原因；
2. 允许提交补充证据；
3. 后台人工复核；
4. 复核结果记录；
5. 误判可恢复奖励；
6. 恶意申诉可限制申诉频率。

---

# **19. P0 验收标准**

上线前必须满足：

1. 退款订单奖励可冻结；
2. 全额退款奖励可撤销；
3. 同一订单不可重复领奖；
4. 未完成订单不可领奖；
5. 未审核评价不可领奖；
6. 违规评价不可领奖；
7. 高频评价可识别或审核；
8. 高频退款可识别或审核；
9. 同设备多账号可记录风险；
10. 公开案例重复素材可识别或审核；
11. 分享自点不计入有效访问；
12. 虚假访问不计入有效转化；
13. 商家刷评可进入风控；
14. 后台可冻结、解冻、驳回、撤销奖励；
15. 奖励操作有日志；
16. 风控误判可人工恢复；
17. 黑名单用户不可继续刷奖励；
18. MVP 不做浏览收益和点击收益。
