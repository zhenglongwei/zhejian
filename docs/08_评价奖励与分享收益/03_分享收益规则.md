# **2. 模块定位**

分享收益模块用于记录用户将平台内容分享给他人后产生的访问、咨询、下单等行为。

其核心作用包括：

1. 追踪内容传播效果；
2. 统计案例、门店、服务商品的分享转化；
3. 为后续分享转化激励提供数据基础；
4. 防止虚假点击、刷浏览、刷分享；
5. 支持运营分析和内容优化。

---

# **3. MVP 边界**

MVP 阶段明确：

```
只做分享记录、访问记录、咨询归因、下单归因，不做浏览现金收益，不做点击现金收益。

```

MVP 不做：

1. 分享点击奖励；
2. 浏览量奖励；
3. PV 奖励；
4. UV 奖励；
5. 多级分销；
6. 邀请返佣；
7. 用户拉用户返现；
8. 分享后立即发现金奖励；
9. 虚拟浏览收益；
10. 商家私设分享返现。

| 分享对象  | object_type  | 说明         |

# **5. 分享行为定义**

| 字段             | 说明                   |
| -------------- | -------------------- |
| share_id       | 分享记录 ID              |
| share_code     | 分享码                  |
| sharer_user_id | 分享人用户 ID             |
| object_type    | 分享对象类型               |
| object_id      | 分享对象 ID              |
| share_channel  | 分享渠道，如微信好友、朋友圈、群聊、H5 |
| scene          | 小程序 scene 参数         |
| created_at     | 分享时间                 |

# **6. 分享来源**

支持以下分享来源：

1. 小程序右上角分享；
2. 页面内分享按钮；
3. 案例详情页分享；
4. 门店主页分享；
5. 服务商品页分享；
6. H5 分享链接；
7. 二维码分享；
8. 海报分享，后续可选。

# **7. 分享带来的行为**

分享可能带来以下行为：


| 行为   | 是否记录 | 是否产生现金收益    |
| ---- | ---- | ----------- |
| 浏览   | 记录   | MVP 不产生     |
| 有效浏览 | 记录   | MVP 不产生     |
| 咨询   | 记录   | MVP 不产生或后置  |
| 下单   | 记录   | 后续可作为转化收益依据 |
| 支付   | 记录   | 后续可作为转化收益依据 |
| 完成订单 | 记录   | 后续可作为转化收益依据 |
| 复购   | 后期可选 | 后置          |


---

# **8. 收益来源定义**

## **8.1 MVP 阶段收益**

MVP 阶段的“收益”主要指：

1. 分享数据统计；
2. 内容传播贡献记录；
3. 分享转化归因；
4. 用户个人分享记录展示；
5. 后台运营分析。

MVP 不直接按浏览、点击发现金收益。

---

## **8.2 后续可扩展收益**

后续如启用分享转化收益，收益来源只能基于真实有效行为：


| 收益来源 | 是否建议支持 | 条件         |
| ---- | ------ | ---------- |
| 浏览   | 不建议    | 易刷，不发现金    |
| 咨询   | 谨慎     | 需有效咨询，且需风控 |
| 下单   | 可选     | 需真实下单      |
| 支付   | 可选     | 需支付成功      |
| 完成订单 | 推荐     | 需订单完成且未退款  |
| 复购   | 后期可选   | 需明确归因周期    |


推荐后续收益依据：

```
有效分享 → 被分享人下单 → 支付成功 → 订单完成 → 无退款/无售后 → 风控通过 → 计算分享转化奖励

```

# **9. 有效分享规则**

一次分享被认为有效，需满足：

1. 分享人账号正常；
2. 分享对象存在且可访问；
3. 分享链接包含合法 share_code；
4. 被分享访问不是分享人本人自点；
5. 非同设备、同 IP 高频刷访问；
6. 页面有基础停留时间；
7. 访问行为被服务端记录；
8. 未命中分享作弊规则。

---

# **10. 有效浏览规则**

有效浏览用于统计，不直接产生现金收益。

建议条件：

1. 独立访客；
2. 页面停留时间达到最低阈值，例如 5 秒；
3. 非分享人本人访问；
4. 非同设备重复刷访问；
5. 非脚本访问；
6. 页面加载成功；
7. 用户未被风控标记。

---

# **11. 有效咨询规则**

有效咨询定义：

1. 被分享用户进入咨询入口；
2. 提交有效咨询内容；
3. 非空内容、非广告、非刷屏；
4. 非分享人本人咨询；
5. 非同设备多账号模拟咨询；
6. 咨询内容与页面对象相关。

MVP 仅记录咨询归因，不直接发放现金收益。

---

# **12. 有效下单规则**

分享带来的下单需满足：

1. 被分享用户通过分享链接进入；
2. 在归因有效期内下单；
3. 订单与分享对象存在合理关联；
4. 订单支付成功；
5. 订单最终完成；
6. 未全额退款；
7. 无严重售后或纠纷；
8. 未命中风控。

---

# **13. 分享收益计算，后续可选**

MVP 不启用自动分享收益。

后续如启用，可采用：

```
分享转化收益 = 基础奖励 × 订单金额系数 × 内容贡献系数 × 风控系数

```

必须配置：

1. 单笔奖励封顶；
2. 单用户每日封顶；
3. 单用户每月封顶；
4. 单内容每日封顶；
5. 单门店每日封顶；
6. 风控冻结规则；
7. 退款撤销规则。

---

# **14. 分享收益状态，后续可选**

如启用分享转化收益，状态建议：


| 状态                  | 说明     |
| ------------------- | ------ |
| WAIT_ATTRIBUTION    | 待归因    |
| ATTRIBUTED          | 已归因    |
| WAIT_ORDER_COMPLETE | 等待订单完成 |
| WAIT_REVIEW         | 待审核    |
| WAIT_GRANT          | 待发放    |
| GRANTED             | 已发放    |
| FROZEN              | 已冻结    |
| REJECTED            | 已驳回    |
| REVOKED             | 已撤销    |

# **15. 不计入收益的场景**

以下行为不计入收益：

1. 分享人点击自己的链接；
2. 同一设备反复点击；
3. 同一 IP 批量访问；
4. 页面停留时间极短；
5. 脚本访问；
6. 虚假咨询；
7. 被分享用户快速退款；
8. 订单全额退款；
9. 订单售后纠纷成立；
10. 用户与商家存在异常关联；
11. 分享人和被分享人强关联异常；
12. 平台判定为作弊。

---

# **16. 与评价奖励的关系**

分享收益与评价奖励必须分离。


| 模块     | 触发依据          | 是否 MVP 支持 |
| ------ | ------------- | --------- |
| 评价奖励   | 用户完成真实评价      | 支持        |
| 公开授权奖励 | 用户授权公开脱敏案例    | 支持        |
| 分享统计   | 用户分享内容带来访问或转化 | 支持        |
| 分享转化收益 | 分享带来有效完成订单    | 后置        |
| 浏览收益   | 分享带来浏览        | 不支持       |
| 点击收益   | 分享链接被点击       | 不支持       |

# **18. 后台管理能力**

运营后台需要支持：

1. 查看分享记录；
2. 查看访问记录；
3. 查看咨询转化；
4. 查看订单转化；
5. 查看分享对象效果；
6. 查看分享人贡献；
7. 查看异常分享；
8. 标记无效访问；
9. 标记作弊分享；
10. 导出分享数据。

---

# **19. P0 验收标准**

P0 阶段必须满足：

1. 用户可分享维修案例；
2. 用户可分享门店主页；
3. 用户可分享服务商品；
4. 分享链接携带 share_code；
5. 系统可记录分享来源；
6. 系统可记录访问行为；
7. 系统可识别分享人本人自点；
8. 系统可记录咨询归因；
9. 系统可记录下单归因；
10. MVP 不按浏览发现金收益；
11. MVP 不按点击发现金收益；
12. 支持后台查看分享数据；
13. 支持基础反作弊标记；
14. 支持与转化归因规则联动。
