# **2. 模块定位**

H5公开内容站承担平台的外部内容入口。

主要目标：

1. **SEO 搜索流量**
  - 用户搜索“杭州刹车片更换案例”；
  - 用户搜索“杭州钣喷修复多少钱”；
  - 用户搜索“事故车维修案例”。
2. **GEO/AI 引用**
  - 让案例内容结构化；
  - 方便搜索引擎和 AI 摘要理解；
  - 提升平台内容被引用概率。
3. **微信分享落地页**
  - 商家分享门店；
  - 用户分享案例；
  - 平台推广落地页。
4. **小程序转化**
  - H5 页面引导打开微信小程序；
  - 进入服务详情、门店详情、预约流程。

# **4. H5 内容边界**

## **4.1 允许展示**

1. 已审核公开案例；
2. 脱敏图片；
3. 车型品牌/车系，必要时模糊；
4. 服务项目；
5. 价格区间；
6. 价格影响因素；
7. 门店公开信息；
8. 门店资质标签；
9. 门店透明度指标（案例数、相册完整率等，非交易评价）；
10. 服务说明；
11. 平台保障说明。

---

## **4.2 禁止展示**

1. 未授权服务档案；
2. 原图；
3. 车牌；
4. VIN；
5. 手机号；
6. 用户真实姓名；
7. 精确订单金额，默认不展示；
8. 保险定损单个人信息；
9. 身份证、行驶证、驾驶证等证件信息；
10. 用户拒绝公开的任何内容。

# **12. 小程序转化设计**

## **12.1 转化入口**

H5 页面固定展示：

1. 顶部打开小程序按钮；
2. 底部悬浮预约按钮；
3. 案例详情页“预约类似服务”；
4. 门店页“预约该门店”；
5. 服务页“查看可预约门店”；
6. 价格说明处“到小程序获取门店方案”。

---

## **12.2 打开小程序方式**

微信内：

- 使用微信开放标签 `wx-open-launch-weapp`；
- 跳转到对应小程序页面。

微信外：

- 展示二维码；
- 提示复制链接或扫码；
- 可引导下载/打开微信扫码。

---

## **12.3 跳转参数**

| service_item_id | 服务项目 ID            |

# **13. SEO/GEO 设计**

## **13.1 每页基础 SEO**

每个可收录页面必须具备：

1. title；
2. meta description；
3. canonical；
4. robots；
5. H1；
6. 面包屑；
7. 结构化数据；
8. Open Graph；
9. 图片 alt；
10. sitemap 收录。

---

## **13.2 案例页 title 规则**

```
杭州{车型}{服务项目}维修案例_辙见过程与价格参考

```

示例：

```
杭州宝马3系刹车片更换维修案例_辙见过程与价格参考

```

---

## **13.3 服务页 title 规则**

```
杭州{服务项目}价格参考与维修案例_透明汽车维修平台

```

示例：

```
杭州钣喷修复价格参考与维修案例_透明汽车维修平台

```

---

## **13.4 description 规则**

案例页：

```
查看杭州{车型}{服务项目}真实维修案例，包含故障表现、检查结果、维修方案、脱敏维修过程图片和价格影响因素。实际费用需以门店检测为准。

```

服务页：

```
了解杭州{服务项目}适用情况、参考价格、价格影响因素和真实维修案例，可预约本地辙见门店。

```

---

## **13.5 Schema 结构化数据**

建议使用：

1. `LocalBusiness`，门店页；
2. `AutoRepair`，如适配；
3. `Service`，服务页；
4. `Article` 或 `BlogPosting`，案例页；
5. `FAQPage`，常见问题；
6. `BreadcrumbList`，面包屑。

---

## **13.6 AI 可引用摘要**

每个案例页可生成一段结构化摘要：

```
本页为杭州某车型某维修项目案例。案例展示故障表现、检查结果、维修方案、脱敏过程图片和价格影响因素。页面不展示用户车牌、手机号、VIN 和实际支付金额。

```

用于提升 AI 搜索理解。

# **14. noindex 规则**

以下页面需设置 noindex：

1. 用户要求不被搜索引擎收录；
2. 案例风险中等但允许站内展示；
3. 稀有车型存在识别风险；
4. 内容质量不足；
5. 审核通过但仅门店展示；
6. 测试页面；
7. 参数组合页；
8. 搜索结果页。

# **18. 异常与边界**

## **18.1 案例下架**

案例下架后：

1. H5 返回 404 或 410；
2. sitemap 移除；
3. 小程序不再展示；
4. 若为隐私风险，应立即不可访问；
5. 搜索引擎可提交删除请求，后续人工。

---

## **18.2 门店关闭**

门店关闭后：

1. 门店页不再推荐预约；
2. 可展示“门店暂停服务”；
3. 相关案例可保留但不引导该门店；
4. 严重违规门店案例可整体下架。

---

## **18.3 服务下架**

服务下架后：

1. 服务页可显示为“暂停预约”；
2. 已公开案例可保留；
3. 不再显示购买按钮；
4. 引导用户查看相近服务。

# **20. P0 验收标准**

必须满足：

1. H5 可展示城市首页；
2. H5 可展示服务页；
3. H5 可展示案例列表；
4. H5 可展示案例详情；
5. H5 可展示门店详情；
6. 案例页只使用脱敏图；
7. 未授权内容不可访问；
8. 支持打开小程序；
9. 微信外支持二维码引导；
10. 页面具备 title 和 description；
11. 案例页具备来源标签；
12. 事故车页面明确不线上报价；
13. 案例下架后 H5 不可访问；
14. sitemap 可生成或预留；
