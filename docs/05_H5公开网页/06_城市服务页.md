## **2. 页面定位**

城市服务页用于承接本地化维修服务搜索需求。

示例：

```
北京刹车片更换
上海宝马保养
广州汽车空调维修
杭州钣喷修复
杭州事故车维修

```

页面主要承担：

1. 展示城市内某服务项目说明；
2. 展示本地价格区间；
3. 展示本地真实案例；
4. 展示本地可预约门店；
5. 展示本地透明度指标；
6. 承接 SEO 搜索流量；
7. 引导用户进入小程序预约。

---

## **3. 页面生成条件**

城市服务页不应生成大量空页面。

建议满足：

1. 城市已开通；
2. 服务项目已启用；
3. 城市内至少有 1 家可展示门店；
4. 城市内至少有服务商品或公开案例；
5. 页面内容达到 SEO 质量要求。

不满足时：

- 不生成页面；
- 或生成但 noindex；
- 或引导至城市首页/服务项目页。

## **6. 页面模块说明**

---

## **6.1 城市服务标题**

格式：

```
{城市}{服务项目}服务案例与门店推荐

```

示例：

```
杭州刹车片更换服务案例与门店推荐

```

---

## **6.2 城市服务说明**

说明该城市内可提供的服务。

示例：

```
平台收录杭州本地可提供刹车片更换服务的维修门店，并展示已审核、已脱敏的真实维修案例。用户可查看参考价格、维修流程、门店信息和透明度指标后，通过小程序预约到店服务。

```

事故车城市服务说明：

```
杭州事故车维修通常需要根据车辆损伤程度、拆检结果和配件情况确认维修方案，平台页面仅提供案例参考和门店信息，最终费用需以到店检测或拆检结果为准。

```

---

## **6.3 价格区间**

展示该城市该服务项目的参考价格。

字段：


| 字段      | 说明       |
| ------- | -------- |
| 最低参考价   | 可选       |
| 最高参考价   | 可选       |
| 起步价     | 可选       |
| 价格说明    | 必须       |
| 是否需到店检测 | 复杂项目必须展示 |


示例：

```
杭州刹车片更换参考区间：¥300-¥1200

```

复杂服务：

```
杭州钣喷修复费用需根据损伤面积、喷漆部位和修复方案确认，建议到店检测后报价。

```

事故车：

```
事故车维修不支持线上最终报价，需到店检测或拆检后确认方案和费用。

```

---

## **6.4 价格影响因素**

展示本地价格差异原因。

建议包括：

1. 车型品牌；
2. 配件品牌；
3. 损伤程度；
4. 工时复杂度；
5. 是否需要拆检；
6. 是否涉及喷漆；
7. 是否涉及编程或匹配；
8. 门店服务标准。

---

## **6.5 本地案例**

展示该城市该服务项目下的公开案例。

筛选条件：

1. 城市匹配；
2. 服务项目匹配；
3. 审核通过；
4. 已脱敏；
5. 允许公开；
6. 优先展示用户授权案例。

案例卡片字段：


| 字段   | 说明             |
| ---- | -------------- |
| 主图   | 脱敏图            |
| 标题   | 城市 + 车型 + 服务项目 |
| 门店   | 本地门店           |
| 价格区间 | 可选             |
| 来源标签 | 用户授权/商家历史      |
| 脱敏标签 | 已脱敏            |


空状态：

```
当前服务的本地案例正在补充中，可先查看相关服务案例或预约门店咨询。

```

若无案例，建议 noindex，除非页面已有足够本地门店和服务内容。

---

## **6.6 本地门店**

展示该城市可提供该服务的门店。

推荐逻辑：

1. 门店城市匹配；
2. 服务项目匹配；
3. 门店营业状态正常；
4. 服务商品已上架；
5. 有相关案例优先；
6. 透明度评分高优先；

门店卡片字段：


| 字段   | 说明       |
| ---- | -------- |
| 门店名称 | 展示       |
| 区域   | 如西湖区、朝阳区 |
| 地址   | 简要地址     |
| 营业时间 | 可选       |
| 服务价格 | 可选       |
| 案例数量 | 展示       |
| 透明度 | 可选       |
| 预约按钮 | 打开小程序    |


---

## **6.7 透明度指标**（V2.0 替代交易评价）

展示该城市该服务相关的可验证透明度数据，**不展示**订单交易评价。

数据来源：

1. 该服务项目公开案例数；
2. 相关门店服务相册完整率；
3. 平台审核通过的资质信息。

无数据时可不展示该模块或展示空状态。

---

## **6.8 服务流程**

展示该服务标准流程，可复用服务项目页内容，但加入城市语境。

示例：

```
1. 选择杭州本地门店；
2. 在小程序提交预约；
3. 到店检查车辆；
4. 门店确认维修方案和费用；
5. 维修过程中可生成服务相册；
6. 完工后用户可在小程序查看服务相册档案。

```

## **6.10 FAQ**

示例：

```
Q：杭州刹车片更换一般多少钱？
A：费用会因车型、刹车片品牌、是否前后轮更换和门店工时不同而变化，页面价格仅供参考。

Q：可以线上确认最终价格吗？
A：标准项目可查看参考价格，复杂维修或事故车维修需到店检测后确认。

Q：如何预约杭州本地门店？
A：点击页面中的预约入口，可打开小程序查看门店和服务并提交预约。

```

## **9. 结构化数据**

建议支持：

1. `Service`
2. `LocalBusiness`
3. `FAQPage`
4. `BreadcrumbList`

注意：

1. 门店列表中的透明度指标必须真实；
2. FAQ 必须页面可见；
3. 不得伪造本地案例数量；
4. 不得生成没有真实本地内容的结构化数据。

---

## **10. noindex 规则**

以下情况必须 noindex 或不生成：

1. 城市未开通；
2. 服务项目未启用；
3. 本地无门店；
4. 本地无案例且内容不足；
5. 页面只有模板文案；
6. 参数组合页；
7. 搜索结果页；
8. 内容重复度过高。

## **12. 异常处理**


| 场景    | 处理            |
| ----- | ------------- |
| 城市不存在 | 404           |
| 服务不存在 | 404           |
| 城市未开通 | 跳城市首页或 404    |
| 本地无门店 | noindex 或不生成  |
| 服务下架  | 页面显示暂停预约或 404 |
| 案例下架  | 从本地案例中移除      |
| 门店关闭  | 不再推荐预约        |


---

## **13. P0 验收标准**

1. 城市服务页有唯一 URL；
2. 页面包含城市服务说明；
3. 页面包含价格区间或价格说明；
4. 页面包含价格影响因素；
5. 页面包含本地案例模块；
6. 页面包含本地门店模块；
7. 页面包含透明度指标模块；
8. 页面包含 FAQ；
9. 页面支持打开小程序；
10. 页面具备 title、description、canonical、robots；
11. 无真实本地内容时不得开放 index；
12. 事故车相关页面明确不支持线上最终报价；
13. 页面不展示未审核或未脱敏内容；
14. 页面访问和预约点击可埋点。
