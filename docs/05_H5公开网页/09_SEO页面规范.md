## **2. SEO页面类型**

平台公开网页主要包括：


| 页面类型    | 说明                 | 是否P0 |
| ------- | ------------------ | ---- |
| 公开案例详情页 | 单个维修案例详情           | 是    |
| 门店主页    | 商家门店公开页            | 是    |
| 服务项目页   | 如刹车片更换、空调清洗        | 是    |
| 城市服务页   | 如北京汽车保养、上海钣喷维修     | 是    |
| 车型服务页   | 如宝马3系保养、奥迪A4L刹车片更换 | P1   |
| 案例列表页   | 按城市、服务、门店聚合案例      | 是    |
| 问答FAQ页  | 用户常见问题             | P1   |
| 内容文章页   | 养车知识、维修指南          | P1   |


MVP 阶段优先保证：

```
公开案例详情页
门店主页
服务项目页
城市服务页
案例列表页

```

## **3. 基础原则**

SEO页面必须遵守：

1. 只展示已审核、已脱敏、允许公开的内容；
2. 页面内容必须真实，不得伪造维修案例；
3. 不得暴露用户隐私、车牌、VIN、手机号、人脸等敏感信息；
4. 不得堆砌关键词；
5. 不得生成大量低质量空页面；
6. 页面必须有唯一 URL；
7. 页面必须有唯一 title、description、H1；
8. 图片必须有 alt；
9. 公开案例必须能追溯来源；
10. 下线页面需要正确处理索引状态。

---

## **4. URL规范**

## **4.1 URL设计原则**

URL 应满足：

- 简洁；
- 可读；
- 稳定；
- 不包含隐私信息；
- 不包含手机号、车牌号、VIN；
- 不频繁变更；
- 可被搜索引擎抓取。

---

## **4.2 推荐URL格式**

### **公开案例详情页**

```
/case/{case_id}.html

```

示例：

```
/case/cas_202501010001.html

```

如果后期需要增强可读性：

```
/case/bmw-3series-brake-pad-replacement-beijing-cas_202501010001.html

```

---

### **门店主页**

```
/store/{store_id}.html

```

示例：

```
/store/sto_202501010001.html

```

---

### **服务项目页**

```
/service/{service_slug}.html

```

示例：

```
/service/brake-pad-replacement.html
/service/car-maintenance.html
/service/ac-cleaning.html

```

---

### **城市服务页**

```
/city/{city_slug}/{service_slug}.html

```

示例：

```
/city/beijing/brake-pad-replacement.html
/city/shanghai/car-maintenance.html

```

---

### **案例列表页**

```
/cases.html
/cases/{city_slug}.html
/cases/{service_slug}.html

```

示例：

```
/cases/beijing.html
/cases/brake-pad-replacement.html

```

## **5. 页面Meta规范**

每个可收录页面必须包含：


| 字段              | 是否必填 | 说明                     |
| --------------- | ---- | ---------------------- |
| title           | 是    | 页面标题                   |
| description     | 是    | 页面描述                   |
| keywords        | 可选   | 可不重点依赖                 |
| canonical       | 是    | 标准URL                  |
| robots          | 是    | index/follow 或 noindex |
| og:title        | 建议   | 社交分享标题                 |
| og:description  | 建议   | 社交分享描述                 |
| og:image        | 建议   | 分享图                    |
| structured data | 建议   | 结构化数据                  |

## **6. Title规范**

## **6.1 公开案例页Title**

格式建议：

```
{城市}{车型}{服务项目}维修案例_{门店名称}

```

示例：

```
北京宝马3系刹车片更换维修案例_朝阳XX汽车服务中心

```

要求：

1. 包含城市；
2. 包含车型或车辆类别；
3. 包含服务项目；
4. 包含案例属性；
5. 不超过 30-35 个中文字符为佳；
6. 不得出现夸大词。

禁止：

```
全网最低价
100%修好
最好最专业
官方认证但无资质

```

---

## **6.2 门店页Title**

格式：

```
{门店名称}_{城市}{主营服务}汽车维修保养门店

```

示例：

```
朝阳XX汽车服务中心_北京汽车保养维修门店

```

---

## **6.3 城市服务页Title**

格式：

```
{城市}{服务项目}_{城市}汽车维修保养服务案例与门店推荐

```

示例：

```
北京刹车片更换_北京汽车维修保养服务案例与门店推荐

```

---

## **7. Description规范**

Description 应控制在 80-120 个中文字符左右。

## **7.1 公开案例页Description**

格式：

```
本案例展示{城市}{门店名称}为{车型}提供{服务项目}的维修过程，包括故障现象、施工过程、维修结果和透明度说明，图片已做隐私脱敏处理。

```

示例：

```
本案例展示北京朝阳XX汽车服务中心为宝马3系提供刹车片更换服务的过程，包括检查、拆装、安装和试车结果，图片已做隐私脱敏处理。

```

---

## **7.2 门店页Description**

格式：

```
{门店名称}位于{城市}{区域}，提供{主营服务列表}等汽车维修保养服务，可查看真实维修案例、透明度指标和门店信息。

```

## **8. H标签规范**

页面应有清晰的标题层级。

## **8.1 H1**

每个页面只能有一个 H1。

案例页 H1 示例：

```
北京宝马3系刹车片更换维修案例

```

门店页 H1 示例：

```
朝阳XX汽车服务中心

```

城市服务页 H1 示例：

```
北京刹车片更换维修服务案例

```

---

## **8.2 H2**

用于页面主要模块：

```
维修问题
维修过程
维修结果
案例图片
服务门店
透明度说明
常见问题
相关案例

```

---

## **8.3 H3**

用于 H2 下的细分内容。

## **9. 图片SEO规范**

所有公开图片必须满足：

1. 使用脱敏后图片；
2. 图片不可含车牌、VIN、人脸、手机号；
3. 图片大小合理；
4. 图片具备 alt；
5. 图片文件名不包含敏感信息；
6. 优先使用 webp 或压缩后的 jpg；
7. 首屏图片应优化加载速度。

---

## **9.1 图片alt格式**

公开案例图片 alt：

```
{城市}{车型}{服务项目}{维修阶段}

```

示例：

```
北京宝马3系刹车片更换施工前检查
北京宝马3系刹车片更换安装过程
北京宝马3系刹车片更换完成效果

```

禁止：

```
图片1
维修图
宝马宝马宝马刹车片刹车片
含车牌号的描述

```

## **10. 内容质量规范**

## **10.1 公开案例页内容要求**

公开案例详情页至少包含：


| 内容      | 是否必填     |
| ------- | -------- |
| 案例标题    | 是        |
| 城市      | 是        |
| 门店      | 是        |
| 服务项目    | 是        |
| 车型或车辆类别 | 是        |
| 故障/需求描述 | 是        |
| 维修过程    | 是        |
| 维修结果    | 是        |
| 脱敏图片    | 是        |
| FAQ     | 建议至少 3 条 |
| 相关案例    | 建议       |
| 门店入口    | 是        |


---

## **10.2 低质量页面判定**

以下页面不得开放收录：

1. 没有正文内容；
2. 只有图片没有文字说明；
3. 没有真实案例；
4. 只有标题堆砌关键词；
5. 页面内容重复度过高；
6. 没有审核通过；
7. 图片未脱敏；
8. 门店或服务不存在；
9. 案例已下线；
10. 用户撤回授权。

---

## **11. FAQ规范**

FAQ 用于提升用户理解和搜索覆盖。

每个案例页建议至少 3 条 FAQ。

示例：

```
Q：宝马3系刹车片更换一般需要多久？
A：常规情况下约1-2小时，具体时间取决于车辆状况和门店排队情况。

Q：刹车片更换后需要注意什么？
A：更换后建议进行试车检查，初期避免急刹，确认刹车脚感正常。

Q：刹车片是否必须前后一起更换？
A：不一定，需要根据前后刹车片磨损程度判断。

```

FAQ 要求：

1. 问题真实；
2. 回答准确；
3. 不夸大；
4. 不承诺绝对结果；
5. 不包含外部联系方式；
6. 不诱导绕过平台交易。

## **12. 结构化数据规范**

建议支持 Schema.org 结构化数据。

## **12.1 门店页**

可使用：

```
LocalBusiness
AutoRepair

```

字段包括：


| 字段              | 说明            |
| --------------- | ------------- |
| name            | 门店名称          |
| address         | 门店地址，注意隐私和准确性 |
| telephone       | 平台允许展示时才展示    |
| openingHours    | 营业时间          |
| aggregateRating | 评分，需真实        |
| image           | 门店图片          |
| url             | 门店公开页URL      |


---

## **12.2 案例页**

可使用：

```
Article
ImageObject
FAQPage
BreadcrumbList

```

要求：

1. 结构化数据与页面内容一致；
2. 不得伪造评分；
3. 不得伪造评论数量；
4. FAQPage 内容必须页面可见；
5. 图片 URL 必须是公开脱敏图片。

## **13. Robots规范**

## **13.1 可收录页面**

满足以下条件可以设置：

```
<meta name="robots" content="index,follow">

```

条件：

1. 审核通过；
2. 已发布；
3. 已脱敏；
4. 授权有效；
5. 内容质量达标；
6. 不处于纠纷、售后、风险锁定状态。

---

## **13.2 不可收录页面**

以下页面必须设置：

```
<meta name="robots" content="noindex,nofollow">

```

场景：

1. 草稿；
2. 审核中；
3. 审核驳回；
4. 已下线；
5. 用户撤回授权；
6. 隐私风险；
7. 争议订单；
8. 内容质量不达标；
9. 测试页面；
10. 内部预览页。

---

## **14. Sitemap规范**

系统应生成：

```
/sitemap.xml

```

可按类型拆分：

```
/sitemap-cases.xml
/sitemap-stores.xml
/sitemap-services.xml
/sitemap-city-services.xml

```

Sitemap 只包含：

1. 已发布；
2. 可收录；
3. canonical URL；
4. 最近更新时间准确；
5. 非 noindex 页面。

已下线页面应从 sitemap 中移除。

---

## **15. Canonical规范**

每个公开页面必须有 canonical。

示例：

```
<link rel="canonical" href="https://www.example.com/case/cas_202501010001.html" />

```

用于避免：

1. 多参数重复页面；
2. 列表筛选页面重复；
3. 分享链接参数导致重复收录；
4. 移动端和桌面端 URL 不一致。

---

## **16. 页面性能要求**

SEO页面应满足：


| 指标     | 建议        |
| ------ | --------- |
| 首屏加载   | 尽量小于 2.5s |
| 图片懒加载  | 必须        |
| JS体积   | 控制        |
| CSS阻塞  | 优化        |
| 移动端适配  | 必须        |
| 404处理  | 必须        |
| 服务器状态码 | 正确返回      |


---

## **17. 状态码规范**


| 场景      | 状态码              |
| ------- | ---------------- |
| 正常页面    | 200              |
| 页面不存在   | 404              |
| 页面永久迁移  | 301              |
| 页面临时维护  | 503              |
| 审核中/未公开 | 404 或 noindex 预览 |
| 已下线     | 404 或 410，视策略    |
| 用户无权限预览 | 403              |

## **19. P0验收标准**

上线前必须满足：

1. 公开案例页有唯一 URL；
2. 公开案例页有 title、description、H1；
3. 图片使用脱敏图；
4. 图片 alt 不为空；
5. 未审核案例不可收录；
6. 已下线案例不可收录；
7. 用户撤回授权后页面下线或 noindex；
8. 页面有 canonical；
9. sitemap 只包含可收录页面；
10. 公开案例页至少包含维修过程和维修结果；
11. 页面不得暴露车牌、VIN、手机号、人脸；
12. 低质量页面不得开放 index；
13. H5公开页只能读取已发布案例；
14. 页面移动端可正常访问；
15. 页面访问埋点可采集。
