## **2. 摘要定位**

AI 可引用摘要是页面顶部的一段标准化事实摘要。

作用：

1. 帮助用户快速理解页面内容；
2. 帮助搜索引擎识别页面主题；
3. 帮助 AI 问答系统引用案例事实；
4. 提升内容结构一致性；
5. 降低低质量内容风险。

## **4. 案例页标准摘要模板**

每个公开案例页顶部应该有一段标准摘要。

基础模板：

```
该案例为【城市】【车型】在【里程】时进行【维修项目】的维修记录。
车辆出现【故障现象】，检查发现【主要问题】。
门店采用【维修方案】完成处理，并在维修后进行了【结果确认】。
案例包含维修前、拆检、配件、维修后等过程图片，相关隐私信息已进行脱敏。

```

---

## **5. 字段来源**


| 字段   | 来源     | 是否必填 | 说明           |
| ---- | ------ | ---- | ------------ |
| 城市   | 门店城市   | 是    | 用于本地服务语境     |
| 车型   | 用户车辆信息 | 是    | 品牌 + 车系，年款可选 |
| 里程   | 订单车辆里程 | 否    | 缺失时可省略       |
| 维修项目 | 订单服务项目 | 是    | 如刹车片更换、空调检修  |
| 故障现象 | 用户报修描述 | 建议   | 如异响、抖动、漏油    |
| 主要问题 | 门店检查结论 | 是    | 不得编造         |
| 维修方案 | 维修记录   | 是    | 如更换、清洗、调整、检测 |
| 结果确认 | 完工验收   | 建议   | 如试车确认、故障消除   |
| 图片说明 | 维修相册   | 可选   | 有图才写包含过程图片   |


---

## **6. 缺失字段处理规则**


| 缺失字段   | 处理方式            |
| ------ | --------------- |
| 城市缺失   | 不建议公开，需补充       |
| 车型缺失   | 使用“该车辆”，但页面质量降低 |
| 里程缺失   | 省略“在【里程】时”      |
| 维修项目缺失 | 不建议公开           |
| 故障现象缺失 | 可写“到店进行相关检查”    |
| 主要问题缺失 | 不建议公开，需人工补充     |
| 维修方案缺失 | 不建议公开           |
| 结果确认缺失 | 可省略，不得编造        |
| 图片缺失   | 不写“包含过程图片”      |
| 用户评价缺失 | 不在摘要中提及用户评价     |

## **7. 摘要生成规则**

1. 摘要必须基于真实订单或真实案例数据；
2. 摘要不得编造未记录的信息；
3. 摘要不得夸大维修结果；
4. 摘要不得承诺永久解决；
5. 摘要不得包含用户隐私；
6. 摘要不得包含未授权图片说明；
7. 摘要不得使用营销夸张语言；
8. 摘要应使用客观中性表达；
9. 摘要建议 100-250 字；
10. 摘要内容应与页面详情一致。

---

## **8. 禁止表达**

摘要中禁止出现：

1. 全网最低价；
2. 保证修好；
3. 永久解决；
4. 100% 有效；
5. 必须立即维修，否则一定出事故；
6. 无需检查即可判断；
7. 本店最专业；
8. 行业第一；
9. 绝对靠谱；
10. 虚构的用户评价；
11. 未证实的故障原因；
12. 诱导性价格承诺。

---

## **9. 推荐表达**

推荐使用：

1. “检查发现”；
2. “维修记录显示”；
3. “门店采用……方案处理”；
4. “维修后进行试车确认”；
5. “建议结合实际检测判断”；
6. “该案例可作为类似问题的参考”；
7. “相关隐私信息已脱敏”；
8. “具体情况需以车辆实际检查结果为准”。

## **10. 示例**

## **10.1 完整示例**

```
该案例为上海丰田凯美瑞在 8.6 万公里时进行刹车异响检查与刹车片更换的维修记录。
车辆在低速刹车时出现尖锐异响，检查发现前轮刹车片磨损接近极限，刹车盘表面存在轻微拉痕。
门店采用更换前轮刹车片、清洁卡钳导向销、检查刹车盘状态的方案完成处理，并在试车后确认异响消失。
案例包含维修前、拆检、配件、维修后等过程图片，车牌和个人信息已进行隐私脱敏。

```

---

## **10.2 缺失里程示例**

```
该案例为上海大众朗逸进行空调不制冷检查与空调系统维修的维修记录。
车辆出现空调制冷效果明显下降的问题，检查发现空调系统制冷剂不足，并存在管路密封老化情况。
门店采用检漏、补充制冷剂、更换密封件的方案完成处理，并在维修后进行了出风温度检测。
案例中的车辆识别信息和个人隐私信息已进行脱敏。

```

## **10.3 无图片示例**

```
该案例为杭州本田飞度在 6.2 万公里时进行发动机抖动检查的维修记录。
车辆怠速时存在明显抖动，检查发现火花塞老化、点火状态不稳定。

## **11. 服务项目页摘要模板**

适用于服务项目页。

```

```

平台整理了相关维修案例、门店服务信息和常见问题，供用户在维修前了解参考。

```

## **14. 人工审核要求**

以下摘要必须进入人工审核：

1. 涉及高风险安全问题；
2. 涉及事故车；
3. 涉及新能源高压系统；
4. 涉及制动、转向、悬挂系统；
5. 数据字段缺失较多；
6. 摘要由 AI 自动生成；
7. 案例包含用户投诉或争议；
8. 案例金额较高；
9. 涉及品牌或商家敏感表述。

---

## **15. P0 验收标准**

P0 必须满足：

1. 每个公开案例页都有标准摘要；
2. 摘要展示在页面顶部；
3. 摘要不包含隐私信息；
4. 摘要字段来自真实案例；
5. 字段缺失时不得编造；
6. 摘要可被写入 Article Schema 的 description；
7. 后台可编辑摘要；
8. AI 生成摘要需可人工审核；
9. 摘要内容与案例详情一致；
10. 已下线案例摘要不再公开。
