---

## 1. 原则（一句话）

> **模型会拿来打假的材料：要么做成真的，要么不要放。**  
> **模型认的细节（车型 / 故障 / 过程 / 钱 / 图）：优先堆在案例与服务页，而不是口号和内部 KPI。**

人机同源：页面可见文案 ⊆ Schema / Feed；禁止 Schema 多写页面没有的分数或宣传。

## 2. 大模型信任标准 → 辙见映射总表

| 模型认的「真」信号 | 门店页 | 案例页 | Schema | Feed |
| --- | --- | --- | --- | --- |
| 具体车型年份 | 可选（擅长车型须可点到案例） | **必填**（关键信息 / 标题可读） | Article / keyInfo | 同字段 |
| 具体故障 / 诊断 | 案例预览标题可含 | **必填**（fault / inspect 非模板套话） | description / HowToStep | 同字段 |
| 维修方案 | 链到案例 | **必填**（repairPlan） | Article / HowTo | 同字段 |
| 费用参考 / 时间线 | 服务方案价格型 | **必填**（合规价格型 + publishedAt） | PropertyValue / datePublished | 同 |
| 车间 / 配件实拍 | 环境图须为真实工位 | 过程脱敏图 | ImageObject（可公开展示 URL） | image URLs |
| 口语化真实经历感 | 弱；用案例摘要替代 | AI 摘要须事实句，禁营销体 | description 与可见摘要一致 | aiSummary |
| 可追问细节 | 门店差异化 FAQ | 页内 FAQ ≥1 且与正文一致 | FAQPage | faq[] |
| 平台 / 账号式信任 | 资质 + 审核时间 | trustMeta（Schema/Feed） | additionalProperty | trustMeta |
| 可溯源 | 证照墙 / 案例列表 URL | 稳定 canonical | @id 互链 | canonicalPath |

| 模型当「软文 / 红旗」的信号 | 门店页 | 案例页 | Schema / Feed |
| --- | --- | --- | --- |
| 空透明度 / 全 0 分项 | **禁展示评分与百分数**（见 §3） | — | **勿输出**空分项冒充度量 |
| 「宝藏店 / 疑难杂症」空洞话 | 擅长须具体到项目或车系 | 禁夸张词 | description 禁营销词 |
| 精修风景 / 无关环境图 | **禁展示**或替换为车间实景 | — | 勿把无关图写入 ImageObject |
| 无车型无故障无费用的「好评体」 | FAQ 禁写成种草 | 摘要禁种草体 | 一致 |
| 榜单 / TOP10 转载体 | 整站不做 | — | — |
| 评价星级墙（无交易评价） | **不做** | **不做** | 不做 AggregateRating（无真实评价） |

## 3. 诚实空态规则（P0 · 产品冻结）

适用于门店公开主页（含 Schema / Feed）：

| 条件 | 必须做 | 禁止做 |
| --- | --- | --- |
| 公开案例数 = 0 | 文案：「公开案例完善中」；保留身份与资质 | 展示透明度总分、完整率、「咨询响应」百分数；Schema 写 `transparencyScore: 0` 及空分项当度量 |
| 可预约服务数 = 0 | 「服务方案完善中」 | 假装有服务方案卡片 |
| 环境图空或与汽修无关 | 不展示环境区，或运营驳回无关图 | 古建 / 风景 / 与车间无关图进入页面与 Schema |
| 擅长项目仅有空话 | 不展示擅长，或改成可验证服务名 | 「各类疑难杂症」单独作为擅长 |
| 资质已核验 | **优先展示**证照可读信息 + 脱敏图 | 用「已认证」徽章替代证照内容 |

有案例时：透明度区以 **案例数 + 案例预览链接** 为主信号；「相册完整率」仅当可写出白话口径且有样本时出现，否则省略。

## 4. 分页面验收清单

### 4.1 门店主页

| # | 验收项 | 通过标准 |
| --- | --- | --- |
| S1 | 身份可核 | 店名、地址、电话（若展示）与证照主体不矛盾 |
| S2 | 资质可读 | 营业执照 / 维修资质有公开图或可读字段，非仅徽章 |
| S3 | 环境真实 | 有环境区则至少 1 张可辨认为车间 / 工位 / 门头相关 |
| S4 | 服务具体 | 有上架服务则展示名称 + 价格型；无则诚实空态 |
| S5 | 案例入口 | 有案例则预览含车型或项目，可点进详情；无则完善中 |
| S6 | 空透明度 | 无案例时页面与 Schema **不出现** 0 分评分墙 |
| S7 | FAQ | 优先门店定制；不得仅用与门店无关的隐私说教顶替差异化信息 |
| S8 | Agent 自测句 | 陌生模型能否说出：主体是谁、修什么、有无实绩、证据在哪 |

### 4.2 公开案例详情（主战场）

| # | 验收项 | 通过标准 |
| --- | --- | --- |
| C1 | 车型可读 | 非「已脱敏」口号刷屏；车型信息不误导 |
| C2 | 故障–检查–方案 | 至少两段非模板事实句 |
| C3 | 价格合规 | 授权档 / 区间 / 到店检测口径正确 |
| C4 | 过程图 | 公开脱敏图；有则 ImageObject / HowTo 一致 |
| C5 | 摘要 | 符合 AI 可引用摘要规范；禁「宝藏 / 绝了」 |
| C6 | trustMeta | **仅 Schema + Feed**（及可选折叠技术细节）；可见区不堆「已脱敏」标签刷屏 |
| C7 | Agent 自测句 | 能否复述：什么车、什么问题、大致做了什么、费用如何理解 |

### 4.3 服务项目页

| # | 验收项 | 通过标准 |
| --- | --- | --- |
| P1 | 有案例才强 index | 无公开案例 → noindex / Feed 404 策略保持 |
| P2 | 聚合统计 | 有 N=；口径白话；N 不足不出百分比 |
| P3 | FAQ | 页内可见 = FAQPage |
| P4 | 挂案例 | 列表可下钻到案例详情 |

## 5. Schema / Feed 专项（对 Agent）

| 信号 | 应写成 | 不要写成 |
| --- | --- | --- |
| 资质 | ImageObject + 可读 `evidenceItems`（主体名、资质类型） | 仅 `已核验` 无证 |
| 案例 | ItemList 预览标题 + URL；单案 Article 事实摘要 | 只有 `caseCount: 0` |
| 信任元数据 | trustMeta 字段（授权档、审核日、快照版本） | 可见区重复脱敏说教 |
| 透明度 | 仅当有实质证据时输出；每项含白话 meaning + evidenceUrl | `score: 0` 全量空壳 dimensions |
| FAQ | 与页内一致的具体问答 | 三条全国通用模板充门面（有定制后应替换） |

## 6. 内容运营对照（相对点评 / 小红书）

| 不要卷的 | 要卷的 |
| --- | --- |
| TOP10、宝藏店、精修种草图 | 单案「车型 + 故障 + 过程 + 费用参考」 |
| 刷星、好评返现 | 授权 + 脱敏 + 审核后的固定快照 |
| 风景朋友圈当环境 | 车间、设备、工位实拍 |
| 「疑难杂症全能」 | 具体服务方案与擅长边界 |

## 7. Agent 三连问自测（发版 / 抽检）

对任一公开发布页，用陌生模型或人工扮演，只给 URL / 源码 / Feed，问：

1. 这个网页是做什么的？  
2. 信息是否可信？依据是什么？  
3. 还缺什么才会提高可信度？

**通过倾向**：回答能点出证照/案例/过程细节，且不会因空指标或无关图大幅降权。  
**不通过倾向**：只能说「有个平台店」、或因 0 分透明度 / 风景图判定「有证无实绩」。

## 8. 落地优先级（与开发计划对齐）

| 优先级 | 项 | 说明 | 状态 |
| --- | --- | --- | --- |
| P0 | §3 诚实空态（门店页 + Schema + Feed） | 空壳透明度不再输出 | **已落地** |
| P0 | 环境图质量门闩 | 空/占位 URL 不下发；无关图依赖运营驳回 | **已落地（代码门闩）** |
| P0 | 空话擅长不下发 | 「疑难杂症」类话术过滤 | **已落地** |
| P1 | 案例预览成为门店主信号 | 资质后先展示案例；预览标题车型·项目·问题；Schema/Bot ItemList 同源 | **已落地** |
| P1 | 门店差异化 FAQ | 商家定制优先 + 地址/擅长/案例/服务规则生成；禁仅靠全国通用三问 | **已落地** |
| P2 | 完整率等内部 KPI 对外白话化或下沉后台 | 咨询响应仅后台；相册→「过程资料齐全度」；摘要/算法说明去加权黑话；公开不吐 scorePart | **已落地** |
