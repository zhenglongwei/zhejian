## **4. URL 设计原则**

1. URL 应稳定，不频繁变更；
2. URL 应语义化；
3. 城市、车型、服务项目使用 slug；
4. 门店和案例可使用 ID；
5. 不在 URL 中暴露用户 ID、订单 ID、手机号；
6. 页面下线后应支持 301、404 或 noindex；
7. 重复页面应使用 canonical；
8. URL 中不使用中文空格和特殊符号。

## **17. 页面收录规则**


| 页面状态      | 处理方式                                  |
| --------- | ------------------------------------- |
| 内容完整、公开授权 | index, follow                         |
| 内容较少但有效   | index, follow 或 noindex, follow，由运营配置 |
| 重复内容      | canonical 指向主页面                       |
| 案例授权撤回    | noindex，下线或 404                       |
| 门店下架      | noindex 或 301 到城市门店页                  |
| 涉及隐私风险    | 立即 noindex 并下线                        |
| 售后纠纷中案例   | 暂停公开或 noindex                         |
| 空列表页      | noindex, follow                       |
| 测试页面      | noindex, nofollow                     |

## **19. P0 验收标准**

P0 阶段必须满足：

1. 首页可访问；
2. 门店页可访问；
3. 服务项目页可访问；
4. 案例详情页可访问；
5. 案例页包含 AI 摘要；
6. 案例页包含 FAQ；
7. 案例页包含面包屑；
8. 案例页可链接到门店、车型、服务项目；
9. 页面支持 TDK；
10. 页面支持 canonical；
11. 页面支持 index/noindex；
12. 页面支持 sitemap；
13. 页面支持隐私风险下线；
14. 后台可管理页面公开状态。
