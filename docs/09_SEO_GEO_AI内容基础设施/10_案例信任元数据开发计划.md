## 2. 问题与目标

### 2.1 现状缺口

| 已有 | 缺口 |
| --- | --- |
| CaseSnapshot 冻结事实（故障/方案/价格） | 公开页 **未系统展示** 授权档、审核结论、快照版本 |
| 脱敏图 + 闸门 B 审核 | 用户/AI 无法从结构化数据判断「为何可信」 |
| `authorizationTier` 在快照内 | H5/Feed/Schema **未统一输出** trustMeta 块 |
| 相册图减少（PublicView 改造） | 需用 **元数据信任链** 替代「图多=可信」 |

### 2.2 目标（一句话）

在 **不改快照正文** 前提下，为每个已发布案例输出标准化 **trustMeta**，贯穿 H5 可见区、JSON Feed、Schema，使 AI 可引用「来源可验证」而非仅「有一段描述」。

### 2.3 分层原则

```text
CaseSnapshot（不可变事实）
    ↓ 只读引用
trustMeta（提炼层 · 可变展示形态，事实来源于审核/授权记录）
    ↓ 同步
H5 信任条 + JSON Feed + Schema CreativeWork.additionalProperty
```

**禁止**：trustMeta 中的数字/结论与快照矛盾；写入运营员姓名、完整审核对话。

## 3. 数据模型

### 3.1 存储位置

挂于 `public_cases.enrichment_json.trustMeta`（与 `aiSummary`/`faq` 同层，**不写入** `snapshot_json`）。

### 3.2 字段契约

| 字段 | 类型 | 来源 | 必填 | 说明 |
| --- | --- | --- | ---: | --- |
| `snapshotVersion` | int | `snapshot.version` | 是 | 授权快照版本号 |
| `authorizedAt` | ISO8601 | 授权记录 | 是 | 用户提交授权公示时间 |
| `authorizationTier` | enum | 快照 | 是 | `user_authorized` / `merchant_history` |
| `authorizationTierLabel` | string | 规则映射 | 是 | 「用户授权案例」/「商家历史案例」 |
| `reviewStatus` | enum | `public_cases.status` | 是 | `approved` |
| `reviewedAt` | ISO8601 | 审核通过时间 | 是 | 闸门 B 通过时间 |
| `desensitized` | boolean | 脱敏管线 | 是 | 固定 `true`（已发布案例） |
| `evidenceLevel` | enum | 规则计算 | 是 | 见 §3.3 |
| `publicImageCount` | int | 快照脱敏图清单 | 是 | 公开图数量（诚实标注） |
| `nodeStageCount` | int | 快照节点 | 是 | 有内容的阶段数（1–6） |
| `auditLogSummary` | string | 审核记录摘要 | 建议 | ≤120 字；如「脱敏复核通过；无车牌/人脸残留」 |
| `complianceGateA` | enum | 相册合规 | 是 | `passed` |
| `complianceGateB` | enum | 公示审核 | 是 | `passed` |
| `trustStatement` | string | 规则模板 | 是 | 面向用户的 1 句信任说明 |
| `updatedAt` | ISO8601 | 提炼层 | 是 | trustMeta 最后生成时间 |

### 3.3 evidenceLevel 枚举

| 值 | 条件 | 展示文案 |
| --- | --- | --- |
| `text_primary` | 公开图 = 0，有文案摘要 | 「以文字记录为主」 |
| `partial_images` | 1 ≤ 公开图 ≤ 3 | 「含少量脱敏过程图」 |
| `rich_images` | 公开图 > 3 | 「含多阶段脱敏过程图」 |

> PublicView 改造后多数案例为 `text_primary` 或 `partial_images`；**禁止**为提升观感夸大图量。

### 3.4 trustStatement 模板

```
本案例为{authorizationTierLabel}；经隐私脱敏与平台审核后公开（快照版本 v{snapshotVersion}，{reviewedAt}）。{evidenceLevel对应短句}。价格与方案以快照记录为准，仅供参考。
```

## 4. 可见展示（H5）

### 4.1 信息架构

案例详情页在 **ai_summary 下方** 增加「信任与来源」折叠区（默认展示 1 行摘要，可展开）：

| 元素 | 内容 |
| --- | --- |
| 标签行 | `authorizationTierLabel` Tag + 「已脱敏」+ 「已审核」 |
| 折叠详情 | 授权时间、审核时间、快照版本、公开图数、auditLogSummary |
| 免责 | 复用 `GEO_DISCLAIMER` |

**禁止**：展示车牌/VIN/运营员 ID；用「平台订单案例」标签。

### 4.2 与 PublicView 关系

| PublicView 授权档 | trustMeta 映射 |
| --- | --- |
| 用户授权公示 | `user_authorized` → 「用户授权案例」 |
| 商家历史/冷启动 | `merchant_history` → 「商家历史案例」 |

## 6. 生成规则

### 6.1 触发时机

| 事件 | 动作 |
| --- | --- |
| 闸门 B 审核通过 | 首次生成 trustMeta |
| 用户撤回再授权（新 snapshotVersion） | 全量重算 |
| 运营修改 enrichment（不含快照） | 不重算 trustMeta，除非审核记录变更 |
| 定时任务（可选） | 仅当审核记录回填时批量 refresh |

### 6.2 生成器

新建 `backend/src/services/case-trust-meta.service.js`：

1. 只读 `snapshot_json` + `public_cases` 审核字段 + 授权记录；
2. 规则拼接 `trustStatement`；
3. 合规校验（禁词、隐私字段扫描）；
4. 写入 `enrichment_json.trustMeta`。

**禁止 LLM** 生成 trustMeta 事实字段。

## 8. 验收标准

1. 已发布案例 H5 首屏可见 `authorizationTierLabel` + 「已脱敏」「已审核」。
2. `GET /public/v1/cases/{slug}.json` 含完整 `trustMeta`，与页面一致。
3. View Source 含 trustMeta 对应 Schema 字段。
4. `evidenceLevel` 与真实公开图数一致，无夸大。
5. 快照正文未被 trustMeta 生成器修改。
6. 冒烟脚本对 ≥5 条已发布案例通过。
