---

## **2. 适用对象**

`consultation_lead`（或 `lead`）：用户通过服务/门店/案例/GEO 页提交的咨询或到店预约信息。

**不是**平台订单，不绑定支付、履约、退款状态。

---

## **3. 主状态 lead_status**

| 状态 | 说明 |
| --- | --- |
| SUBMITTED | 已提交，待商家查看 |
| VIEWED | 商家已查看 |
| CONTACTED | 商家已联系用户 |
| CANCELLED | 用户取消 |
| CLOSED | 已关闭（商家标记无效/已到店/重复等） |

---

## **4. 状态流转**

```text
SUBMITTED → VIEWED → CONTACTED → CLOSED
SUBMITTED → CANCELLED（用户）
SUBMITTED / VIEWED → CLOSED（商家关闭）
```

**不允许**：待支付、待接单、维修中、待确认、已完成、售后等订单态。

---

## **5. 关闭原因 close_reason（商家）**

| 枚举 | 说明 |
| --- | --- |
| USER_CANCELLED | 用户主动取消 |
| UNREACHABLE | 无法联系 |
| VISITED | 用户已到店（线下后续不在平台跟踪） |
| DUPLICATE | 重复提交 |
| INVALID | 无效/垃圾咨询 |
| OTHER | 其他（须填备注） |

---

## **6. 平台侧扩展（可选）**

| 字段 | 说明 |
| --- | --- |
| risk_flag | 风控标记（垃圾/骚扰） |
| platform_note | 运营内部备注 |

平台**不可**将线索改为「已支付」「已完工」等状态。

---

## **7. 状态日志**

每次状态变更写入 `lead_status_log`：`lead_id`、`from_status`、`to_status`、`operator_type`（user/merchant/platform）、`operator_id`、`reason`、`created_at`。

---

## **8. P0 验收标准**

1. 用户提交后状态为 SUBMITTED；
2. 商家可标记 VIEWED、CONTACTED、CLOSED；
3. 用户可 CANCELLED（未关闭前）；
4. 状态流转有日志；
5. 前端展示状态与枚举一致，无订单态文案。
