# **《商家工作台PRD》**

---

## **1. 文档信息**


| 项目   | 内容                                       |
| ---- | ---------------------------------------- |
| 文档名称 | 商家工作台PRD                                 |
| 当前版本 | V1.0                                     |
| 适用端  | 微信小程序内商家工作台                              |
| 用户角色 | 商家主账号、门店管理员、员工                           |
| 核心目标 | 让商家在小程序内完成入驻、接单、履约、相册上传、案例上传、服务管理和门店资产管理 |
| 重要说明 | MVP 不做 PC 商家后台，全部在微信小程序内实现               |


---

# **2. 模块定位**

商家工作台是商家在平台上的经营入口。

它承担：

1. 商家入驻；
2. 门店管理；
3. 员工管理；
4. 服务方案管理；
5. 订单接单与履约；
6. 维修相册上传；
7. 历史案例上传；
8. 评价查看；
9. 门店主页预览；
10. 数据概览。

---

# **5. 门店切换**

## **5.1 适用场景**

一个商家主体可能有多个门店。

门店切换影响：

1. 订单列表；
2. 服务方案；
3. 历史案例；
4. 门店主页；
5. 数据概览；
6. 员工权限。

---

## **5.2 切换规则**

- 主账号可查看全部门店；
- 门店管理员只能查看授权门店；
- 员工只能查看绑定门店；
- 默认进入最近使用门店；
- 可选择“全部门店”，主账号专用。

## **6.4 待接单操作**

可操作：

1. 接单；
2. 拒单；
3. 建议改期，复杂预约；
4. 联系用户。

接单后：

- 标准订单进入已接单/待到店；
- 复杂预约进入已接单/待到店；
- 用户收到通知；
- 相册可上传。

拒单后：

- 必须填写原因；
- 标准订单触发退款；
- 预约单关闭；
- 用户收到通知。

---

## **6.5 到店和维修操作**

商家操作：

```
标记到店
→ 开始维修
→ 上传相册
→ 提交完工

```

提交完工前建议填写：

1. 维修说明；
2. 配件说明；
3. 价格区间，复杂维修；
4. 质保说明；
5. 完工建议。

## **7.3 上传规则**


| 项目    | 规则       |
| ----- | -------- |
| 单相册上限 | 20 张     |
| 单次上传  | 建议最多 9 张 |
| 视频    | MVP 不支持  |
| 图片压缩  | 必须       |
| 节点    | 建议，不强制   |

2. 拆检过程；
3. 故障点；
4. 新旧件对比；
5. 安装过程；
6. 完工结果；
7. 其他。

未选择节点时可归类为“其他”。

---

## **7.5 空相册处理**

商家不上传图片也可以提交完工。

但页面提示：

1. 门店主页；
2. 案例列表；
3. 服务详情页；
4. H5 公开页面；
5. SEO/GEO 内容资产。

---

## **8.2 入口**

```
商家工作台
→ 历史案例

```

---

## **8.3 案例列表**

Tab：

1. 草稿；
2. 待审核；
3. 已通过；
4. 要求修改；
5. 已驳回；
6. 已下架。

卡片字段：

- 案例标题；
- 服务项目；

## **8.4 新建历史案例**

流程：

```
选择门店
→ 选择服务项目
→ 选择复杂度，系统带出
→ 填写车型
→ 填写故障表现
→ 填写维修方案
→ 填写价格区间
→ 上传图片
→ 确认隐私责任
→ 提交审核

```

---

## **8.5 历史案例字段**


| 字段     | 是否必填   | 说明      |
| ------ | ------ | ------- |
| 门店     | 必填     | 绑定门店    |
| 服务项目   | 必填     | 平台服务项目  |
| 车型品牌   | 建议必填   | 如宝马     |
| 车系     | 建议必填   | 如 3 系   |
| 年款     | 可选     |         |
| 里程     | 可选     |         |
| 故障表现   | 必填     | 用户问题    |
| 检查结果   | 可选     |         |
| 维修方案   | 必填     | 如何处理    |
| 配件说明   | 可选     |         |
| 价格区间   | 建议必填   | 不展示实际价格 |
| 图片     | 至少 1 张 | 最多 20 张 |
| 隐私责任确认 | 必填     | 勾选      |


---

## **8.6 隐私责任确认**

提交前必须展示：

```
请确认你上传的历史案例已取得必要授权，或已确保图片和文字内容不会暴露客户身份、车牌、VIN、手机号、人脸、支付信息等隐私信息。

平台会进行自动脱敏和审核，但商家仍需对上传内容的合法性、真实性和不侵犯客户隐私承担责任。

```

商家必须勾选后才可提交。

# **16. 异常与边界**

## **16.1 商家未审核通过**

不能使用：

1. 接单；
2. 服务上架；
3. 历史案例公开；
4. 门店主页公开展示。

可使用：

1. 查看入驻进度；
2. 修改资料；
3. 联系平台。

---

## **16.2 门店被冻结**

冻结后：

- 用户端不展示；
- 服务方案暂停预约；
- 新订单不可接；
- 已有订单继续处理或平台介入；
- 案例可隐藏或保留，按平台规则。

---

## **16.3 员工离职**

主账号应停用员工。

停用后：

- 员工无权访问工作台；
- 操作日志保留；
- 不影响其历史上传图片和案例。
