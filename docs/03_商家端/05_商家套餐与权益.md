## 4. 支付

- 渠道：微信小程序 **JSAPI 微信支付**（商户平台已配置）
- 流程：创建订单 → prepay → `wx.requestPayment` → 回调激活套餐
- 开发环境：未配置微信支付时，prepay 返回 mock 提示，可走 `mock-pay` 联调
- 样板店：运营标记 `founderFlag` + 赠送深度版；续费 `founderRenewDiscount=0.5`

### 环境变量（backend）

| 变量 | 说明 |
| --- | --- |
| `WECHAT_PAY_MCH_ID` | 商户号 |
| `WECHAT_PAY_API_V3_KEY` | APIv3 密钥 |
| `WECHAT_PAY_CERT_SERIAL` | 商户证书序列号 |
| `WECHAT_PAY_PRIVATE_KEY` 或 `WECHAT_PAY_PRIVATE_KEY_PATH` | 商户私钥 **apiclient_key.pem**（与证书同次下载） |
| `WECHAT_PAY_NOTIFY_URL` | 可选，默认 `{PUBLIC_BASE_URL}/api/v1/pay/wechat/notify` |

**验签失败（SIGN_ERROR）排查**：

```bash
cd backend && node scripts/wechat-pay-smoke.js
```

1. `WECHAT_PAY_CERT_SERIAL` = `openssl x509 -in apiclient_cert.pem -noout -serial` 输出去掉 `serial=` 与冒号，**大写**  
2. `WECHAT_PAY_PRIVATE_KEY` = 同目录 **apiclient_key.pem**（勿用 APIv3 密钥、勿用公钥）  
3. `WECHAT_PAY_MCH_ID` = 商户平台商户号，与证书一致  
4. 小程序 **AppID** 已在商户平台绑定为 JSAPI 支付 AppID  
5. 改 `.env` 后：`pm2 restart zhejian-api --update-env`

## 5. 车主评价 × 相册反馈 × 配件核对（Phase 2 · 已落地）

### 5.1 服务与记录评价（四维 · 一次性）

| 域 | 维度 | key |
| --- | --- | --- |
| 维修服务 | 服务态度 / 专业可靠 | `repairAttitude` / `repairTrust` |
| 相册记录 | 过程记录是否充分 / 图文是否清楚 | `albumSufficiency` / `albumClarity` |

- 完工后可提交，一相册一车主一条；公开案例展示 **维修分 / 记录分** 与 **脱敏配图**（勾选公开时）
- 入口：相册尾页「评价与反馈」→ `pages/album/engage`

### 5.2 配件核对（可选 · 随时可填）

- 表：`service_album_part_verifications`（按配件 upsert，可多次更新）
- 状态：`matched` 与登记一致 / `question` 有疑问 / `skipped` 暂未核对
- **浅入口（主）**：相册底栏「配件」、档案抽屉配件清单下按钮、列表卡「核对配件」、尾页「核对配件」
- **深入口（辅）**：`engage` 枢纽第三项
- API：`GET/POST /user/service-albums/:albumId/part-verifications`

### 5.3 问题反馈（保留）

- `pages/album/feedback` · 转达门店与产品，非星级

**不做**：V1 六维订单评价、好评返现。
