## 2. 套餐定义

| 套餐 | 代码 | 价格 | 相册额度 | 公域基础收录 |
| --- | --- | --- | --- | --- |
| 体验版 | `free` | 0 | **12 单/自然月**（建议默认；可配置） | **是**（须车主发布 + 审核通过；仍受内容质量规则约束） |
| 专业版 | `tool_480` | **480 元/年**（已定锚点：低门槛覆盖成本 + 付费认真感；非主收入） | 不限量（或配置为远高于体验版） | **是**（同上） |

**已停售（禁止新购）**：

| 旧代码 | 说明 |
| --- | --- |
| `index_99` | 原「标准版 99 元/年 · 付费收录」 |
| `optimize_299` | 原深度优化版 |

**存量过渡**：

- `index_99` / `optimize_299` 有效期内：权益不降级；到期续费引导至 `tool_480`。  
- 原「付费才 `seo_noindex=0`」逻辑废止为主策略；迁移见 §3。  
- 字段 `standard_trial_used`：可保留兼容；新专业版试用改用样板店 `founderFlag` / 运营赠送时长（见 §4）。

**中立性**：自然排序仅依据案例完整度、车主反馈等质量信号；**付费不干预排名、不买置顶**。

**专业版交付（目标态，与商业 P0 对齐）**：在体验版之上，强调额度与（排期上线后）确认/交车报告/证据包/工具统计等——详见 `10_商业模式产品需求调整.md` §7～§8。确认类能力**不以付费墙卡合规**：体验版也可走基础确认；专业版提供完整额度与导出次数等。

## 3. 公域策略（公示免费 · 2026-07-19）

### 3.1 原则

> 车主「发布到公开网站」+ 脱敏 + 人工审核通过后，**体验版与专业版均可基础收录**（可进 sitemap、`seo_noindex=0`），**不**再以套餐作为收录主 gate。

付费买的是**工具使用深度**，不是「能不能被搜索到」。

### 3.2 对象规则

| 对象 | 体验版 / 专业版（审核通过后） |
| --- | --- |
| 公开案例 H5 | 可生成 URL；默认允许收录（`seo_noindex=0`），仍受质量/违规下架规则约束 |
| sitemap | 可包含已发布且允许收录的案例/门店 |
| 门店 H5 | 可访问；允许收录（同上） |
| 小程序内分享 | 允许 |
| 额度用尽（仅体验版） | **阻断新建相册**；已发布案例的收录状态不因降级而批量改回 `noindex`（避免惩罚已公开内容） |

### 3.3 迁移（工程）

| 步骤 | 说明 |
| --- | --- |
| 1 | 目录下架 `index_99` / `optimize_299` 新购；下单 API 拒绝新购旧码 |
| 2 | 批量：有效商家已审核公开案例 → 按质量规则设为可收录（取消「仅付费可收录」） |
| 3 | 工作台文案：「套餐与公域收录」改为「套餐与工具权益」；去掉「H5 私域 / 付费才公域」对立 Tag，改为额度/专业版标识 |
| 4 | 开通专业版回调：激活 `tool_480` 与额度；**不再**以「打开收录」为唯一权益差分 |

## 4. 支付与样板

- 渠道：微信小程序 **JSAPI 微信支付**（商户平台已配置）
- 流程：创建订单 → prepay → `wx.requestPayment` → 回调激活套餐  
- `POST /merchant/subscription/orders` body：`{ plan: "tool_480" }`（拒绝 `index_99` 等停售码）  
- 开发环境：未配置微信支付时，prepay 返回 mock；可走 `mock-pay`  
- 样板店：运营标记 `founderFlag` + **赠送专业版时长（如 90 天）**；续费 `founderRenewDiscount=0.5`（半价）

### 环境变量（backend）

| 变量 | 说明 |
| --- | --- |
| `WECHAT_PAY_MCH_ID` | 商户号 |
| `WECHAT_PAY_API_V3_KEY` | APIv3 密钥 |
| `WECHAT_PAY_CERT_SERIAL` | 商户证书序列号 |
| `WECHAT_PAY_PRIVATE_KEY` 或 `WECHAT_PAY_PRIVATE_KEY_PATH` | 商户私钥 **apiclient_key.pem** |
| `WECHAT_PAY_NOTIFY_URL` | 可选，默认 `{PUBLIC_BASE_URL}/api/v1/pay/wechat/notify` |
| `ALBUM_FREE_MONTHLY_QUOTA` | 可选，体验版月额度，默认 `12` |

**验签失败（SIGN_ERROR）排查**：

```bash
cd backend && node scripts/wechat-pay-smoke.js
```

1. `WECHAT_PAY_CERT_SERIAL` = `openssl x509 -in apiclient_cert.pem -noout -serial` 输出去掉 `serial=` 与冒号，**大写**  
2. `WECHAT_PAY_PRIVATE_KEY` = 同目录 **apiclient_key.pem**  
3. `WECHAT_PAY_MCH_ID` 与证书一致；小程序 AppID 已绑定 JSAPI  
4. 改 `.env` 后：`pm2 restart zhejian-api --update-env`

## 5. 车主评价 × 相册反馈 × 配件核对（已落地）

### 5.1 服务与记录评价（四维 · 一次性）

| 域 | 维度 | key |
| --- | --- | --- |
| 维修服务 | 服务态度 / 专业可靠 | `repairAttitude` / `repairTrust` |
| 相册记录 | 过程记录是否充分 / 图文是否清楚 | `albumSufficiency` / `albumClarity` |

- 完工后可提交，一相册一车主一条；公开案例展示 **维修分 / 记录分** 与 **脱敏配图**（勾选公开时）
- 入口：相册尾页「评价与反馈」→ `pages/album/engage`

### 5.2 配件验真（可选 · 随时可填）

- 表：`service_album_part_verifications`（按配件 upsert，可多次更新）
- 状态：`matched` / `question` / `skipped`
- 入口：相册底栏「配件验真」、档案抽屉、详情尾页、`engage`
- API：`GET/POST /user/service-albums/:albumId/part-verifications`
- **不等于** 配件告知车主确认（确认见商业 P0 / 原 A-PH2-01）

### 5.3 问题反馈（保留）

- `pages/album/feedback` · 转达门店与产品，非星级

**不做**：V1 六维订单评价、好评返现。

### 5.4 API 摘要

| 方法 | 路径 |
| --- | --- |
| GET/POST | `/user/service-albums/:albumId/review` |
| GET/POST | `/user/service-albums/:albumId/part-verifications` |
| GET | `/merchant/reviews*` · `/merchant/part-verifications` |
