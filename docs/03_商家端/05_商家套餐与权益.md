## 2. 套餐定义

| 套餐 | 代码 | 价格 | 公域收录 |
| --- | --- | --- | --- |
| 免费自用版 | `free` | 0 / 永久 | 否（H5 可生成，`noindex`，不进 sitemap） |
| AI 全网收录版 | `index_99` | 99 元 / 年 | 是 |
| AI 深度优化版 | `optimize_299` | 299 元 / 年 | 是 + 运营交付项（Phase 3） |

**中立性**：自然排序仅依据案例完整度与车主反馈；付费不干预排名。

## 3. 公域 Gate（方案 B · 已确认）

| 对象 | 免费商家 | 收录/深度版（有效期内） |
| --- | --- | --- |
| 授权案例 H5 | 生成 URL，可分享；`seo_noindex=1` | `seo_noindex=0`（仍受内容质量规则约束） |
| sitemap | 不包含案例/门店 | 包含 |
| 门店 H5 | 可访问；`noindex` | 可收录 |
| 小程序内分享 | 允许 | 允许 |

开通/续费/到期后，后端 **批量同步** 该商家已审核案例的 `seo_noindex`。

## 4. 支付

- 渠道：微信小程序 **JSAPI 微信支付**（商户平台已配置）
- 流程：创建订单 → prepay → `wx.requestPayment` → 回调激活套餐
- 开发环境：未配置微信支付时，prepay 返回 mock 提示，可走 `mock-pay` 联调
- 样板店：运营标记 `founderFlag` + 赠送深度版；续费 `founderRenewDiscount=0.5`

### 环境变量（backend）

| 变量 | 说明 |
| --- | --- |
| `WECHAT_PAY_MCH_ID` | 商户号 |
| `WECHAT_PAY_API_V3_KEY` | APIv3 密钥 |
| `WECHAT_PAY_CERT_SERIAL` | 商户证书序列号 |
| `WECHAT_PAY_PRIVATE_KEY` 或 `WECHAT_PAY_PRIVATE_KEY_PATH` | 商户私钥 |
| `WECHAT_PAY_NOTIFY_URL` | 可选，默认 `{PUBLIC_BASE_URL}/api/v1/pay/wechat/notify` |

## 5. 车主评价 × 相册反馈（Phase 2 · 设计方向）

用户确认：**评价**针对本次维修服务/商家；**反馈**针对相册功能或 UI。二者可有机结合：

| 类型 | 对象 | 入口 | 可见性 |
| --- | --- | --- | --- |
| 服务评价 | 相册对应的一次服务 | 相册尾页 / 完工后 | 可经车主授权进入公开案例补充区；商家可回复举证 |
| 问题反馈 | 相册产品体验 | 现有 `album/feedback` | 转达门店 + 产品改进，非星级评价 |

**不做**：V1 订单绑定六维交易评价、好评返现。

**Phase 2 任务**：`service_album_reviews` 表、车主提交 UI、商家回复、公开案例挂载（脱敏）。
