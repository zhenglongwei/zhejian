# **《商家数据看板PRD》**

---

## **1. 文档信息**


| 项目   | 内容                                    |
| ---- | ------------------------------------- |
| 文档名称 | 商家数据看板PRD                             |
| 当前版本 | V1.0                                  |
| 适用端  | 微信小程序商家工作台                            |
| 用户角色 | 商家主账号、门店管理员、员工                        |
| 核心目标 | 帮助商家了解门店曝光、咨询、订单、案例和透明度表现，指导商家提升内容和转化 |
| 重要说明 | MVP 以轻量数据看板为主，不做复杂 BI                 |


---

# **2. 模块定位**

商家数据看板用于让商家知道：

1. 有多少用户看过门店；
2. 有多少用户咨询；
3. 有多少订单产生；
4. 服务和案例是否带来转化；
5. 用户评价表现如何；
6. 维修相册是否完整；
7. 门店透明度评分如何；
8. 哪些内容需要优化。

---

# **3. 入口**

```
商家工作台
→ 数据概览

```

首页也展示简版数据卡片。

---

# **4. 数据维度**

## **4.1 时间维度**

支持：

1. 今日；
2. 昨日；
3. 近7天；
4. 近30天；
5. 本月；
6. 自定义时间，MVP 可后置。

---

## **4.2 门店维度**

- 主账号：可查看全部门店和单门店；
- 门店管理员：只能查看授权门店；
- 员工：只能查看绑定门店的基础数据。

---

## **4.3 内容维度**

支持按以下对象查看：

1. 门店；
2. 服务方案；
3. 历史案例；
4. 订单相册；
5. 平台案例；
6. 用户评价。

---

# **5. 核心指标**

## **5.1 浏览量**


| 指标     | 定义          |
| ------ | ----------- |
| 门店浏览量  | 用户访问门店主页次数  |
| 服务浏览量  | 用户访问服务详情页次数 |
| 案例浏览量  | 用户访问案例详情页次数 |
| H5浏览量  | H5公开页面访问次数  |
| 小程序浏览量 | 小程序内访问次数    |


---

## **5.2 咨询数**


| 指标    | 定义                   |
| ----- | -------------------- |
| 总咨询数  | 用户点击咨询、拨打电话、发起在线咨询次数 |
| 门店咨询数 | 从门店主页发起的咨询           |
| 服务咨询数 | 从服务详情页发起的咨询          |
| 案例咨询数 | 从案例页发起的咨询            |
| 有效咨询数 | 去重后的用户咨询数，后续可做       |


---

## **5.3 订单数**


| 指标    | 定义         |
| ----- | ---------- |
| 下单数   | 用户提交订单或预约数 |
| 接单数   | 商家接单数      |
| 完成订单数 | 已完成订单数     |
| 取消订单数 | 用户取消或商家拒单数 |
| 售后订单数 | 进入售后的订单数   |


---

## **5.4 转化率**

建议 MVP 先展示基础转化：

```
咨询转化率 = 订单数 / 咨询数
浏览转化率 = 订单数 / 浏览量
接单率 = 接单数 / 待接单订单数
完单率 = 完成订单数 / 接单数

```

当分母为 0 时显示：

```
暂无数据

```

---

## **5.5 案例浏览量**

展示：

1. 总案例浏览量；
2. 单案例浏览量排行；
3. 门店案例浏览量；
4. 服务详情页带来的案例浏览；
5. H5 案例浏览。

---

## **5.6 案例带来的咨询**

定义：

用户从案例详情页点击：

1. 咨询门店；
2. 拨打电话；
3. 预约服务；
4. 查看服务方案后下单。

均可计入案例贡献。

MVP 可先统计：

```
案例咨询数 = 从案例页点击咨询/电话/预约的次数

```

---

## **5.7 用户评价**

展示：


| 指标   | 说明            |
| ---- | ------------- |
| 门店评分 | 用户综合评分        |
| 评价数  | 总评价数量         |
| 好评率  | 高分评价占比        |
| 六维评分 | 服务态度、专业度、透明度等 |
| 最新评价 | 最近用户评价        |


---

## **5.8 相册完整率**

定义：

```
相册完整率 = 有维修相册的完成订单数 / 完成订单数

```

可扩展：

```
节点完整率 = 已上传节点数 / 建议节点数

```

展示建议：

- 相册完整率；
- 有图订单数；
- 无图完成订单数；
- 平均每单图片数；
- 节点完整度。

---

## **5.9 门店透明度评分**

门店透明度评分用于表达门店在平台上的可信展示程度。

建议评分维度：


| 维度      | 权重建议 |
| ------- | ---- |
| 相册完整率   | 30%  |
| 案例数量    | 20%  |
| 案例质量    | 15%  |
| 用户评价    | 15%  |
| 服务信息完整度 | 10%  |
| 资质资料完整度 | 10%  |


MVP 可用简化模型：

```
透明度评分 = 相册完整率分 + 案例数量分 + 评价分 + 资料完整度分

```

满分 100 分。

# **7. 优化建议**

系统根据数据生成简单建议。

示例：

```
你的门店近7天有12个完成订单未上传维修相册，建议补充相册提升透明度评分。

```

```
你的刹车片更换案例浏览量较高，但咨询转化较低，建议完善服务方案价格和适用车型。

```

```
门店主页已有浏览，但咨询较少，建议补充门店照片、资质和公开案例。

```

# **11. P0 验收标准**

必须满足：

1. 商家可查看浏览量；
2. 商家可查看咨询数；
3. 商家可查看订单数；
4. 商家可查看转化率；
5. 商家可查看案例浏览量；
6. 商家可查看案例带来的咨询；
7. 商家可查看用户评价；
8. 商家可查看相册完整率；
9. 商家可查看门店透明度评分；
10. 支持按门店筛选；
11. 支持按时间筛选；
12. 主账号可看全部门店；
13. 门店管理员只能看授权门店；
14. 员工只能看基础数据；
15. 核心数据需支持按日统计。
