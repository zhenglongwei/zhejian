# **《商家数据看板 PRD》**

> **当前版本：V2.0**

---

## **1. 文档信息**

| 项目 | 内容 |
| --- | --- |
| 文档名称 | 商家数据看板 PRD |
| 当前版本 | **V2.0** |
| 适用端 | 微信小程序商家工作台 |
| 用户角色 | 商家主账号、门店管理员、员工 |
| 核心目标 | 帮助商家了解曝光、**咨询线索**、案例/相册与透明度表现，优化内容与转化 |
| 重要说明 | MVP 轻量看板，不做复杂 BI |

V2.0 **不做**：订单/接单/完单/售后指标、交易评价、佣金/结算看板。

---

## **2. 模块定位**

商家需知晓：

1. 门店/服务/案例被浏览多少；
2. 收到多少**咨询/预约线索**、处理情况如何；
3. 案例与 GEO 是否带来咨询；
4. **服务相册**是否完整、待车主确认多少；
5. 透明度评分与优化建议。

**删除**：订单数、评价表现、订单相册（改为服务相册）。

---

## **3. 入口**

```text
商家工作台 → 数据概览
```

首页展示简版卡片（与 `00_商家工作台PRD V2.0.md` §4.3 一致）。

---

## **4. 维度**

**时间**：今日、昨日、近 7 天、近 30 天、本月（自定义 P1）。

**门店**：主账号全部门店；管理员授权门店；员工绑定门店基础数据。

**内容对象**：门店、服务方案、历史案例、**服务相册**、GEO 页（P1）。

---

## **5. 核心指标**

### **5.1 浏览量**

门店主页、服务详情、案例详情、H5/GEO、小程序内访问次数。

### **5.2 咨询线索（替代订单）**

| 指标 | 定义 |
| --- | --- |
| 线索提交数 | 用户提交咨询/预约表单数 |
| 待处理 | 商家未查看/未联系 |
| 已联系 | 商家标记已联系 |
| 已关闭 | 含用户取消、无效、已到店等 |
| 电话点击 | 从门店/服务/案例发起拨号 |
| 有效线索 | 去重用户（P1） |

**删除**：下单数、接单数、完成订单、售后订单。

### **5.3 转化率（V2.0）**

```text
线索率 = 线索提交数 / 浏览量（或分页面）
联系率 = 已联系数 / 线索提交数
案例咨询率 = 从案例页发起的咨询+电话 / 案例浏览量
```

分母为 0 显示「暂无数据」。

### **5.4 案例**

总浏览、单案例排行、案例带来的咨询/电话。

### **5.5 服务相册（替代订单相册完整率）**

| 指标 | 定义 |
| --- | --- |
| 新建相册数 | 周期内创建的服务相册 |
| 已完工相册数 | 标记完工 |
| 待车主确认数 | 配件 pending |
| 相册节点完整度 | 已传节点 / 建议节点（可选） |
| 已授权公开数 | 进入案例流程 |

```text
相册完整率（V2.0）= 达到完工且核心节点齐全的相册数 / 已完工相册数
```

**删除**：「有图完成订单数 / 完成订单数」。

### **5.6 门店透明度评分（V2.0 权重）**

| 维度 | 权重建议 |
| --- | --- |
| 服务相册完整率/使用率 | 30% |
| 公开案例数量与质量 | 25% |
| 服务信息完整度 | 15% |
| 资质资料完整度 | 15% |
| 咨询响应（已联系/待处理比） | 15% |

**删除**：用户评价维度。

MVP 简式：相册分 + 案例分 + 资料分 + 响应分，满分 100。

---

## **6. 页面结构**

```text
时间 / 门店筛选
核心卡片（浏览、线索、转化率、案例浏览、相册待办、透明度分）
趋势图（浏览、线索、案例浏览）
内容排行（热门案例、热门服务）
优化建议
```

**删除区块**：订单表现、评价表现。

### **6.1 热门服务排行**

服务名称、浏览量、咨询数、**线索转化率**（线索/浏览）。**无**订单数列。

---

## **7. 优化建议示例**

```text
你有 8 条咨询线索超过 24 小时未联系，建议尽快回电。
```

```text
近 7 天有 5 个已完工服务相册未邀请车主查看，建议发送链接完成配件确认留痕。
```

```text
刹车片案例浏览较高但咨询较少，建议完善服务方案参考价与适用车型说明。
```

## **8. 权限**

| 数据 | 主账号 | 门店管理员 | 员工 |
| --- | --- | --- | --- |
| 全部门店 | 是 | 否 | 否 |
| 授权门店 | 是 | 是 | 绑定门店 |
| 线索数据 | 是 | 是 | 基础 |
| 案例/相册 | 是 | 是 | 基础 |
| 透明度分 | 是 | 是 | 是 |

**删除**：订单数据、评价数据、财务（MVP）。

---

## **9. 埋点（摘要）**

`store_view`、`service_view`、`case_view`、`consult_click`、`phone_click`、`lead_submit`、`lead_contacted`、`album_created`、`album_completed`、`case_submit`、`case_publish`。

**删除**：`order_submit`、`order_accept`、`order_complete`、`review_submit`。

---

## **10. 数据结构（摘要）**

**merchant_daily_stats** 含：`store_view_count`、`service_view_count`、`case_view_count`、`lead_submit_count`、`lead_contacted_count`、`lead_closed_count`、`case_consult_count`、`album_complete_rate`、`transparency_score`。

**不含**：`order_count`、`accepted_order_count`、`completed_order_count`、`store_rating`。

**content_stats**：`view_count`、`consult_count`、`lead_count`（可选）、`share_count`；**无** `order_count`。

---

## **11. P0 验收标准**

1. 可查看浏览量与咨询线索数；
2. 可查看线索处理状态分布；
3. 可查看案例浏览与案例带来的咨询；
4. 可查看服务相册相关指标（新建/待确认/完工）；
5. 可查看透明度评分（无评价维度）；
6. 按门店、时间筛选；
7. 角色权限正确；
8. **无**订单/评价指标入口；
9. 核心指标按日聚合（T+1 可接受）。
