## **2. 入口**

```text
商家工作台 → 咨询线索
```

首页待办：待处理线索数角标。

---

## **3. 列表 Tab**

| Tab | 对应用户端状态（摘要） |
| --- | --- |
| 待处理 | 已提交、门店未查看/未联系 |
| 已联系 | 门店已联系 |
| 已关闭 | 用户已取消、已关闭、商家关闭 |

商家侧可合并「待处理」= 新线索 + 已查看未联系（P0 简版）。

---

## **4. 线索卡片字段**

服务类型、提交时间、用户手机号（脱敏，主账号/管理员可拨号）、车辆品牌/车系、问题摘要、期望到店时间、状态。

> **曝光与渠道归因**（H5/GEO/社媒浏览、电话点击等）见 `06_商家数据看板.md` · **M-DASH** / **B-STATS**，不在线索卡片展示小程序内 `sourcePage`。

---

## **5. 线索详情**

- 用户信息、服务类型、门店（本店）
- 问题描述、上传图片（若有）
- 期望到店时间、联系电话（一键拨打）
- **操作**：
  - 标记已查看
  - 标记已联系（可选填联系备注，P1）
  - 关闭线索（必选原因）
- **平台说明**（页脚固定）：

```text
本线索仅为咨询/预约信息转接，不代表平台交易订单。实际维修、报价、收款与售后由你与用户线下确认。
```

---

## **6. 关闭原因**

用户取消、无法联系、已到店、重复提交、无效线索、其他（须填说明）。

关闭后用户端咨询记录同步为「已关闭」（与用户端状态机一致，见 `docs/11_数据结构与状态机/05_咨询线索状态机.md`）。

---

## **7. 商家操作边界**

**允许**：查看、电话/微信联系、标记状态、关闭。

**禁止**：平台内改价、收款、开订单、退款、标记「维修中/待支付/已完成」等订单态。

---

## **8. 通知**

新线索：微信订阅消息 / 站内提醒（按技术方案）。商家应在合理时间内联系用户（运营规则可另文，MVP 不惩罚）。

---

## **9. 权限**

| 操作 | 主账号 | 门店管理员 | 员工 |
| --- | --- | --- | --- |
| 查看本店线索 | 是 | 是 | 是 |
| 标记已联系/关闭 | 是 | 是 | 是 |
| 查看完整手机号 | 是 | 是 | 可配置 |

---

## **10. 与 V1.0 差异**

删除：接单、拒单、标记到店、开始维修、提交完工、支付与退款。线索状态**不**与订单状态机绑定。

---

## **11. P0 验收标准**

1. 列表 Tab 与待办角标正确；
2. 可查看详情与用户上传图；
3. 可标记已查看、已联系、关闭（含原因）；
4. 可一键拨号；
5. 页脚平台说明可见；
6. 无订单/支付/履约按钮；
7. 员工按门店权限隔离。
