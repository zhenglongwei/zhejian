# **《商家入驻PRD》**

---

## **1. 文档信息**


| 项目   | 内容                             |
| ---- | ------------------------------ |
| 文档名称 | 商家入驻PRD                        |
| 当前版本 | V1.0                           |
| 适用端  | 微信小程序商家工作台                     |
| 用户角色 | 商家主账号、平台审核人员、平台超级管理员           |
| 核心目标 | 让合规维修商家入驻平台，并沉淀门店、资质、服务能力和案例资产 |
| 重要说明 | 商家端集成在同一个小程序中，无 PC 入口          |


---

## **2. 商家入驻目标**

商家入驻模块需要解决 5 个问题：

1. **确认商家是谁**
  - 营业执照；
  - 主体名称；
  - 负责人信息。
2. **确认门店在哪里**
  - 门店名称；
  - 地址；
  - 腾讯地图坐标；
  - 门头照片；
  - 工位照片。
3. **确认商家能做什么**
  - 维修资质；
  - 擅长项目；
  - 技师能力；
  - 新能源专项能力。
4. **确认服务复杂度匹配**
  - L1-L4 服务权限；
  - 事故车、三电等高复杂项目资质约束。
5. **沉淀门店资产**
  - 门店主页；
  - 服务方案；
  - 历史案例；
  - 用户评价；
  - 透明度数据。

---

# **3. 入驻角色**

## **3.1 商家主账号**

可操作：

1. 创建商家主体；
2. 创建门店；
3. 上传资质；
4. 提交审核；
5. 修改资料；
6. 添加员工；
7. 创建服务方案；
8. 上传历史案例。

---

## **3.2 平台审核人员**

可操作：

1. 查看入驻申请；
2. 审核营业执照；
3. 审核门店信息；
4. 审核维修资质；
5. 审核门头和工位照片；
6. 审核服务能力；
7. 通过/驳回/要求修改。

---

## **3.3 平台超级管理员**

除审核人员权限外，还可：

1. 修改商家状态；
2. 冻结商家；
3. 关闭门店；
4. 修改服务权限；
5. 配置佣金；
6. 分配平台运营人员。

---

# **4. 入驻入口**

入口位于：

```
我的
→ 商家工作台
→ 申请入驻

```

如果用户未绑定手机号，先进入手机号绑定流程。

---

## **4.1 未入驻用户进入商家工作台**

展示页面：

```
成为透明维修服务商
把每一单维修变成可信案例，让专业被用户看见

```

展示平台价值：

1. 获得本地维修需求；
2. 展示门店真实案例；
3. 沉淀门店公开主页；
4. 维修过程透明化；
5. 提升用户信任；
6. 前期样板商家免佣。

按钮：

- 立即入驻；
- 查看入驻要求；
- 联系平台。

# **5. 入驻流程**

## **5.1 标准流程**

```
绑定手机号
→ 填写商家主体信息
→ 上传营业执照
→ 填写负责人信息
→ 创建门店
→ 上传门店照片
→ 上传维修资质
→ 选择擅长服务
→ 提交审核
→ 平台审核
→ 审核通过
→ 商家工作台开通

```

---

## **5.2 多门店流程**

商家主体审核通过后，可继续添加门店：

```
商家工作台
→ 门店管理
→ 添加门店
→ 填写门店资料
→ 上传门店照片和资质
→ 提交审核
→ 审核通过
→ 门店上线

```

---

# **6. 商家主体信息**

## **6.1 字段**


| 字段        | 是否必填 | 说明       |
| --------- | ---- | -------- |
| 商家主体名称    | 必填   | 与营业执照一致  |
| 统一社会信用代码  | 必填   | 营业执照识别   |
| 营业执照照片    | 必填   | 图片上传     |
| 法定代表人/负责人 | 必填   | 可与执照一致   |
| 负责人手机号    | 必填   | 接收通知     |
| 联系邮箱      | 可选   | 后续合同/结算  |
| 主体类型      | 可选   | 个体工商户/公司 |
| 经营范围      | 可选   | OCR 辅助识别 |


---

## **6.2 营业执照 OCR**

MVP 可选做，建议后续做。

如做 OCR，可自动识别：

1. 主体名称；
2. 统一社会信用代码；
3. 法人；
4. 经营范围。

但必须允许商家手动修改。

---

# **7. 门店信息**

## **7.1 字段**


| 字段     | 是否必填 | 说明         |
| ------ | ---- | ---------- |
| 门店名称   | 必填   | 用户端展示      |
| 门店简称   | 可选   | 卡片展示       |
| 门店地址   | 必填   | 详细地址       |
| 腾讯地图坐标 | 必填   | 用于距离和导航    |
| 营业时间   | 必填   | 用户预约参考     |
| 门店电话   | 必填   | 用户联系       |
| 门头照片   | 必填   | 可信展示       |
| 工位照片   | 必填   | 至少 1 张     |
| 接待区照片  | 可选   | 增强信任       |
| 门店简介   | 可选   | 用户端展示      |
| 擅长品牌   | 可选   | 如宝马、奔驰、特斯拉 |
| 擅长项目   | 必填   | 服务能力       |
| 是否品牌授权 | 可选   | 有则上传证明     |


---

## **7.2 地图选点**

使用腾讯地图。

商家填写地址后：

1. 调用腾讯地图搜索；
2. 商家选择定位点；
3. 保存经纬度；
4. 用户端用于距离排序和导航。

---

## **7.3 门店照片要求**

门头照片：

- 必须能看清门店招牌；
- 不得盗用其他门店图片；
- 不得包含明显个人隐私。

工位照片：

- 至少 1 张；
- 建议展示维修环境；
- 可上传多张；
- 有助于门店可信度。

---

# **8. 维修资质**

## **8.1 必传资质**

除品牌授权外，其余核心资质必须上传。

必传：

1. 营业执照；
2. 维修经营备案/机动车维修经营资质；
3. 门头照片；
4. 工位照片；
5. 负责人信息；
6. 门店地址；
7. 服务项目能力说明。

品牌授权：

- 非必填；
- 有则上传；
- 会提升推荐排序和门店可信度。

---

## **8.2 资质字段**


| 字段      | 是否必填        |
| ------- | ----------- |
| 维修资质类型  | 必填          |
| 资质照片/文件 | 必填          |
| 资质有效期   | 可选          |
| 资质编号    | 可选          |
| 新能源专项资质 | 如承接新能源服务则必填 |
| 品牌授权证明  | 可选          |


---

## **8.3 资质等级与服务权限**


| 资质         | 可承接服务             |
| ---------- | ----------------- |
| 三类及以上      | L1 基础服务、部分专项服务    |
| 二类或三类专项及以上 | L2 常规维保、刹车、轮胎、电瓶等 |
| 二类及以上      | L3 中复杂故障、底盘、空调等   |
| 一类维修资质     | L4 事故车、大修、整车修复    |
| 新能源专项资质    | 新能源三电相关服务         |


---

# **9. 擅长服务选择**

商家可选择擅长服务：

1. 小保养；
2. 大保养；
3. 刹车片/刹车盘；
4. 电瓶更换；
5. 轮胎服务；
6. 空调清洗/维修；
7. 钣喷修复；
8. 底盘维修；
9. 事故车维修；
10. 新能源专项。

系统根据资质判断是否允许展示或接单。

# **10. 入驻审核**

## **10.1 审核状态**


| 状态   | 说明         |
| ---- | ---------- |
| 草稿   | 商家填写中      |
| 待审核  | 已提交，等待平台审核 |
| 审核通过 | 可使用商家工作台   |
| 审核驳回 | 不符合要求      |
| 要求修改 | 需补充或修改资料   |
| 已冻结  | 平台限制经营     |
| 已关闭  | 门店不再展示     |


---

## **10.2 审核项**

平台审核人员检查：

1. 营业执照真实性；
2. 主体名称一致性；
3. 统一社会信用代码格式；
4. 门店地址是否清晰；
5. 地图定位是否正确；
6. 门头照片是否真实；
7. 工位照片是否合理；
8. 维修资质是否匹配；
9. 擅长项目是否夸大；
10. 新能源服务是否有专项资质；
11. 品牌授权证明是否真实，若上传。

---

## **10.3 审核结果**

### **通过**

商家可进入工作台，并进行：

- 服务方案创建；
- 历史案例上传；
- 接单；
- 员工管理；
- 门店主页展示。

### **要求修改**

需给出具体原因，例如：

```
门头照片不清晰，请重新上传能看到完整招牌的照片。

```

### **驳回**

适用情况：

1. 无法证明经营主体；
2. 营业执照明显异常；
3. 门店不存在；
4. 资质不符合平台要求；
5. 存在严重虚假信息。

# **11. 员工账号管理**

## **11.1 员工添加入口**

```
商家工作台
→ 员工管理
→ 添加员工

```

---

## **11.2 添加方式**

MVP 可采用手机号邀请：

1. 输入员工手机号；
2. 选择门店；
3. 选择权限；
4. 生成邀请；
5. 员工登录小程序后绑定。

---

## **11.3 员工权限**

MVP 简化权限：


| 权限   | 说明        |
| ---- | --------- |
| 查看订单 | 可查看本门店订单  |
| 接单   | 可接单/拒单    |
| 相册上传 | 可上传维修图片   |
| 提交完工 | 可提交订单完工   |
| 案例草稿 | 可创建历史案例草稿 |
| 咨询回复 | 可回复用户咨询   |
| 门店资料 | 默认不可修改    |
| 员工管理 | 默认不可操作    |


---

# **12. 商家工作台首页**

审核通过后进入商家工作台。

首页模块：

```
今日订单
待接单
维修中
待完工
待审核案例
历史案例
服务方案
门店主页
员工管理
数据概览

```

核心数据：

- 今日订单数；
- 待接单数；
- 维修中订单数；
- 已公开案例数；
- 本月案例浏览量；
- 预约咨询数。

---

# **13. 门店主页生成**

商家入驻审核通过后，系统生成门店主页。

门店主页包含：

1. 门店名称；
2. 地址；
3. 导航；
4. 营业时间；
5. 联系方式；
6. 资质标签；
7. 擅长项目；
8. 门店照片；
9. 服务方案；
10. 公开案例；
11. 用户评价；
12. 透明维修说明。

门店主页支持：

- 小程序内展示；
- H5 公开展示；
- 分享；
- SEO 基础结构。

# **14. 商家入驻合规提示**

在提交审核前，展示：

```
请确认你提交的商家主体、门店地址、维修资质和服务能力真实有效。
平台会对资料进行审核，并可能要求补充证明。
如存在虚假资料、冒用资质、夸大服务能力或侵犯客户隐私等行为，平台有权下架门店、限制接单或终止合作。

```

---

# **15. 商家入驻 P0 验收标准**

必须满足：

1. 用户可申请成为商家；
2. 商家可填写主体信息；
3. 商家可上传营业执照；
4. 商家可填写门店信息；
5. 商家可地图选点；
6. 商家可上传门头和工位照片；
7. 商家可上传维修资质；
8. 商家可选择擅长服务；
9. 平台可审核入驻申请；
10. 审核通过后商家工作台可用；
11. 支持多门店数据结构；
12. 商家主账号可添加员工；
13. 品牌授权非强制，但可上传；
14. 资质等级影响可承接服务范围。
