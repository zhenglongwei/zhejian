## **2. 模块定位**

解决新商家缺少公开案例的问题：

1. 展示维修能力；
2. 丰富门店主页与服务详情；
3. 支撑 GEO/搜索；
4. 带来咨询线索（非订单转化）。

---

## **3. 入口**

```text
商家工作台 → 历史案例
```

快捷：工作台新建、**服务相册 → 申请公开/转案例草稿**、门店主页补充案例。

**删除**：依赖平台订单生成案例的路径。

---

## **4. 上传流程**

```text
选择门店 → 选择服务项目
→ 系统带出复杂度与建议节点
→ 批量上传原图（商家侧可见）
→ 图片脱敏工具：一键脱敏 → 预览 → 确认
→ 填写车型、故障、方案、参考价格区间
→ 生成草稿 → 勾选隐私责任 → 提交审核
→ 平台审核 → 发布（门店主页 / 案例库 / H5）
```

细则见 `docs/04_维修过程相册/08_图片脱敏工具PRD.md`（或项目内脱敏文档路径）。

## **7. 建议节点（按服务）**

刹车：维修前、旧件、新旧对比、安装、完工。  
钣喷：损伤、修复、喷漆、完工。  
事故车：事故外观、拆检、损伤细节、配件、修复、完工（**仅参考区间**）。

---

## **8. 脱敏**

识别/处理：车牌、VIN、人脸、手机号、支付凭证、证件。

状态：未检测、已脱敏、疑似风险、脱敏失败、人工完成。未通过脱敏不可提交审核。

---

## **9. 隐私责任（提交前）**

```text
请确认你上传的历史案例已取得必要授权，或已确保内容不会暴露客户身份、车牌、VIN、手机号、人脸、支付信息等。
平台会进行自动脱敏和审核，但商家仍对上传内容的合法性、真实性和隐私负责。
```

---

## **10. 审核与发布**

审核：真实、脱敏、无隐私、不夸大、项目匹配、无盗用案例、价格区间合理。

| 结果 | 说明 |
| --- | --- |
| 通过 | 可发布 |
| 要求修改/驳回 | |
| 仅门店展示 | 不进全站案例库 |
| 平台下架 | 违规 |

发布后展示于：门店主页、服务详情、案例库、H5/GEO、搜索（若收录）。

**标签**：必须标明 **「商家历史案例」**；用户授权案例由相册流程生成，标 **「用户授权案例」**。

**删除**：「平台真实订单案例」标签与 `order_id` 来源。

---

## **12. P0 验收标准**

1. 可新建历史案例、选模板、批量上传；
2. 脱敏确认后可提交审核；
3. 平台审核与发布；
4. 用户端展示为「商家历史案例」且已脱敏；
5. 可从服务相册进入案例草稿流程（授权路径另验）；
6. 无订单 ID 依赖；
7. 浏览与咨询贡献可统计（P0 简版）。
