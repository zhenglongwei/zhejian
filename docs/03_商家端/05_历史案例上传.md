| 项目   | 内容                                      |

# **2. 模块定位**

历史案例用于解决新商家缺少平台订单内容的问题。

它承担：

1. 展示门店真实维修能力；
2. 丰富门店主页内容；
3. 支撑服务详情页案例展示；
4. 增加用户信任；
5. 形成 SEO/GEO 内容资产；
6. 为后续推荐和转化提供内容基础。

---

# **3. 入口**

```
商家工作台
→ 历史案例

```

快捷入口：

1. 工作台首页 → 新建历史案例；
2. 维修相册 → 转为历史案例；
3. 独立相册 → 生成案例草稿；
4. 门店主页 → 补充案例；
5. 平台运营活动入口。

---

# **4. 案例上传流程**

```
选择门店
→ 选择服务模板/服务项目
→ 系统生成节点
→ 批量上传图片（原图，仅商家可见）
→ 进入【图片脱敏工具】一键脱敏 → 预览 → 确认脱敏结果（见 08_图片脱敏工具PRD.md）
→ 填写车型信息
→ 填写维修项目
→ 填写故障表现
→ 填写维修方案
→ 填写价格区间
→ 生成案例草稿
→ 提交审核（须已完成脱敏结果确认 + 隐私责任勾选）
→ 平台审核
→ 发布到门店主页和平台案例库

```

# **6. 新建历史案例**

## **6.1 选择服务模板**

商家选择平台服务项目后，系统自动带出：

1. 服务复杂度；
2. 建议图片节点；
3. 推荐填写字段；
4. 价格模式建议；
5. 案例标题建议。

---

## **6.2 批量上传图片**

支持：

- 多图上传；
- 图片压缩；
- 图片排序；
- 图片说明；
- 节点归类；
- 上传后进入统一【图片脱敏工具】（与车主端相同工作台）；
- 一键脱敏 + 脱敏图预览 + **确认后才可提交审核**；
- 手动打码：脱敏工具 P1（见 `08_图片脱敏工具PRD.md`）。

---

## **6.3 系统自动生成节点**

根据服务项目生成节点。

例如：

### **刹车维修**

1. 维修前；
2. 旧件磨损；
3. 新旧件对比；
4. 安装过程；
5. 完工检查。

### **钣喷修复**

1. 损伤位置；
2. 修复过程；
3. 喷漆过程；
4. 完工效果。

### **事故车**

1. 事故外观；
2. 拆检过程；
3. 损伤细节；
4. 更换配件；
5. 修复过程；
6. 完工外观。

# **8. 自动脱敏**

## **8.1 脱敏对象**

系统需尽量识别：

1. 车牌；
2. VIN；
3. 人脸；
4. 手机号；
5. 支付凭证；
6. 身份证件；
7. 门店以外的第三方隐私信息。

---

## **8.2 脱敏结果**


| 状态     | 说明          |
| ------ | ----------- |
| 未检测    | 图片刚上传       |
| 已脱敏    | 可进入审核       |
| 疑似风险   | 需商家修改或平台复核  |
| 脱敏失败   | 不允许提交审核     |
| 人工处理完成 | 平台处理后可提交/发布 |


---

# **9. 案例草稿生成**

系统可根据商家填写内容生成案例草稿。

草稿内容包括：

1. 案例标题；
2. 故障摘要；
3. 维修过程说明；
4. 节点图片结构；
5. 价格区间展示文案；
6. 隐私提示；
7. 服务项目关联。

示例标题：

```
宝马3系刹车片更换案例
特斯拉Model 3底盘异响检修案例
奥迪A6L空调不制冷维修案例

```

---

# **10. 隐私责任确认**

提交前必须展示：

```
请确认你上传的历史案例已取得必要授权，或已确保图片和文字内容不会暴露客户身份、车牌、VIN、手机号、人脸、支付信息等隐私信息。

平台会进行自动脱敏和审核，但商家仍需对上传内容的合法性、真实性和不侵犯客户隐私承担责任。

```

商家必须勾选后才可提交审核。

---

# **11. 审核规则**

平台审核人员检查：

1. 图片是否真实；
2. 图片是否完成脱敏；
3. 是否涉及客户隐私；
4. 案例内容是否夸大；
5. 服务项目是否匹配；
6. 维修方案是否合理；
7. 是否存在虚假品牌授权；
8. 是否使用他人案例；
9. 价格区间是否明显异常；
10. 是否适合公开展示。

---

# **12. 审核结果**


| 结果    | 说明           |
| ----- | ------------ |
| 通过    | 可发布到门店主页和案例库 |
| 要求修改  | 商家修改后重新提交    |
| 驳回    | 不允许发布        |
| 平台下架  | 发布后因违规下架     |
| 仅门店展示 | 不进入平台案例库     |
| 仅后台留存 | 不公开          |


---

# **13. 发布规则**

审核通过后，案例可展示在：

1. 门店主页；
2. 服务详情页；
3. 平台案例库；
4. H5 公开页面；
5. 搜索结果页，后续；
6. 推荐内容流，后续。

展示必须满足：

- 图片已脱敏；
- 不展示用户真实身份；
- 不展示车牌/VIN；
- 历史案例需标明来源为“商家历史案例”；
- 平台订单案例可标明“平台真实订单案例”。

# **15. P0 验收标准**

必须满足：

1. 商家可新建历史案例；
2. 商家可批量上传图片；
3. 商家可选择服务模板；
4. 系统可根据服务模板生成节点；
5. 商家可填写车型信息；
6. 商家可填写维修项目；
7. 商家可填写故障表现和维修方案；
8. 系统可自动脱敏；
9. 系统可生成案例草稿；
10. 商家必须确认隐私责任；
11. 商家可提交审核；
12. 平台可审核案例；
13. 审核通过后可发布到门店主页；
14. 审核通过后可进入平台案例库；
15. 案例浏览量可被统计。
