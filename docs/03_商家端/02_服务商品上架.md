## **2. 模块定位**

| 端 | 职责 |
| --- | --- |
| 用户端 | 了解价格模式、内容、门店与案例；进入**咨询/预约** |
| 商家端 | 发布可承接服务、参考价、预约说明 |
| 平台端 | 审核真实性、复杂度与资质匹配、合规文案 |

---

## **3. 核心概念**

**服务类目**（平台维护）：保养、刹车、轮胎、电瓶、钣喷、事故车、底盘、新能源等。

**标准服务项目**（平台预设）：小保养、换刹车片、换电瓶、钣喷、事故车检测预约等。

**商家服务方案**：商家基于标准项目创建的、在指定门店展示的**参考方案**（非线上 SKU 支付）。

---

## **4. 价格模式**

| 模式 | 适用 | 用户端行为 |
| --- | --- | --- |
| 一口价 | 标准电瓶、轮胎、部分保养 | 展示金额；预约到店 / 提交咨询 |
| 到店检测后确定 | 刹车、空调、钣喷、事故车等 | 参考价选填；不填则只展示「到店检测后确定」 |

**删除 V1.0**：用户可在线支付、退款规则字段、佣金配置（MVP）。  
**不再作为独立价格模式**：参考区间、事故车（事故车归入「到店检测后确定」；不另挂「无法仅凭线上信息准确报价」类长文）。

### **4.1 一口价字段**

售价、包含工时与材料、不包含项、适用车型。一口价**不展示**「参考价 / 参考区间 / 以门店检测为准」类提示。

### **4.2 到店检测后确定字段**

参考价（**单个金额，选填**）。填了则用户端展示「参考价 ¥X」+「到店检测后确定」；不填则仅展示「到店检测后确定」。

---

## **5. 服务复杂度（L1–L4）**

每个标准项目绑定复杂度，用于：

1. 资质校验（商家是否可发布）；
2. 服务相册节点与配件确认强度（见 `10_复杂度分级详细标准.md`）；
3. 用户端价格展示模式建议。

| 等级 | 示例 |
| --- | --- |
| L1 | 洗车、滤芯、读码 |
| L2 | 小保养、刹车、轮胎、电瓶 |
| L3 | 底盘、空调压缩机、发动机局部 |
| L4 | 事故车、大修、三电总成 |

**删除 V1.0**：低奖励、竞价、评价权重等交易绑定。

---

## **6. 上架流程**

> **2026-06-02 定调**：无平台 **发布前审核**；商家保存后可 **自助上架**，违规依赖 **用户举报 + 运营事后抽查/处罚**。见 `02_业务模式说明.md` §2.1。

```text
商家工作台 → 服务方案 → 新建
→ 选择门店 + 平台服务项目
→ 选择价格模式 + 填写内容与参考价
→ 填写适用车型 + 咨询/预约说明
→ 可选上传服务图
→ 保存草稿 或 保存并上架（用户端立即可见，若门店已上线）
```

### **6.1 状态**

草稿、已上架、已下架（商家自行）、平台强制下架、限制预约（运营处罚，可浏览不可咨询）。

**删除 V2.0**：待审核、审核通过、审核驳回（无前置审核队列）。历史 `audit_status` 字段若存在，仅用于 **事后监管留痕**（抽查/举报处置），不阻塞上架。

## **7. 服务方案字段**

### **7.1 基础**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 所属门店 | 是 | |
| 服务类目/项目 | 是 | 平台预设 |
| 方案名称/简介/详情 | 是 | |
| 复杂度等级 | 系统带出 | 可展示 |
| 价格模式 | 是 | |
| 服务图片 | 否 | |
| 是否接受咨询/预约 | 是 | 默认是 |
| 上架状态 | 商家操作或运营处罚 | 默认草稿；上架后用户端可见 |

**删除**：`是否支持在线支付`、`退款规则`（交易）。

### **7.2 咨询/预约说明（替代纯预约规则）**

可预约日期/时段、是否需提前预约、最早/最晚预约、节假日营业、**咨询表单说明**（用户提交后门店电话联系）。

取消：用户可在用户端取消咨询记录；**无**平台订单退款流程。

---

## **8. 用户端展示（摘要）**

卡片：名称、门店、价格模式与参考价、案例数、距离、资质。**不展示**交易评分、在线支付标签。

详情结构见用户端 `03_服务商品详情页.md`；底栏「预约到店 / 提交咨询」，无「立即购买」。

事故车须突出：不支持线上最终报价。

---

## **9. 平台治理（事后，非发布前审核）**

**不做**：商家提交服务方案后的「待审核队列」、审核通过才可上架。

**做**：

1. **用户举报**：详情页举报虚假价格、夸大承诺、资质造假等（`docs/00_开发计划.md` §7.3）；
2. **运营抽查**：`admin-web` `/admin/services` 对已上架服务 spot-check、强制下架、限制预约；
3. **商家责任**：上架即视为商家对展示内容负责；详情页配合平台免责声明。

抽查/举报成立后的处置（示例）：平台下架、限制预约、累计严重则冻结门店（联动商家审核）。L 级与资质 **不作为上架拦截条件**（产品定调 2026-06-02）；可在风控策略中作事后参考。

**文案建议（非审核闸门）**：事故车避免一口价表述；禁止「保证全赔」「全网最低」等 —— 由举报与抽查处理，不在发布时硬阻断（与实现 `B-SVC-04 [-]` 一致）。

## **10. 服务与案例、相册关联**

**案例（用户端 · 未绑定）**：优先同门店同服务 → 同门店相近 → 同服务平台案例。  
**案例（用户端 · 已绑定相册）**：仅当前门店案例，不回落其他门店。  
**案例（商家预览）**：仅本店案例（同服务 / 相近），不展示其他门店案例。

**相册节点模板**：每个标准项目配置建议六阶段/节点（见 `docs/04_维修过程相册/02_相册模板与节点规则.md`）。

商家**创建服务相册**时按服务项目带出模板，**非**订单创建时生成。

---

## **11. 数据结构（摘要）**

**service_item** 含：`complexity_level`、`default_price_mode`、`need_qualification_level`；**不含** `allow_online_payment`（V2.0）。

**merchant_service_plan** 含：`price_mode`、`price`/`min_price`/`max_price`、`price_factors`、`appointment_required`、`status`、`audit_status`；**不含** `online_payment_enabled`。

**service_album_node_template**：`service_item_id`、节点名、排序、`required_level`（L1/L2/L3）。

## **12. 异常规则**

| 场景 | 处理 |
| --- | --- |
| 资质不足 | 不可提交审核 |
| 事故车一口价 | 系统拦截 |
| 区间价 max < min | 表单校验 |
| 下架后 | 用户端不展示；**无**进行中平台订单履约（V2.0） |

---

## **13. P0 验收标准**

1. 平台可配置类目与标准项目（含 L 级展示字段）；
2. 商家可创建方案并选价格模式；
3. 一口价 / 到店检测后确定展示规则正确（一口价无参考价提示；到店检测参考价选填）；
4. **自助上架**与上下架；**无**发布前审核队列；
5. 用户端可见且仅进入咨询/预约；详情含 **平台免责声明**；
6. 可关联案例与相册节点模板；
7. **无**在线支付与佣金配置（MVP）；
8. 运营可对已上架服务 **抽查/强制下架**；用户可 **举报**（§7.3 MVP）。
