# **《服务商品/服务方案上架PRD》**

---

## **1. 文档信息**


| 项目   | 内容                       |
| ---- | ------------------------ |
| 文档名称 | 服务商品/服务方案上架PRD           |
| 当前版本 | V1.0                     |
| 适用端  | 商家工作台、平台运营工作台、用户端小程序     |
| 核心对象 | 服务类目、服务项目、商家服务方案         |
| 文档目标 | 定义商家如何创建可被用户预约或购买的维修服务方案 |


---

## **2. 模块定位**

服务商品/服务方案模块连接三端：

1. **用户端**
  - 用户通过服务方案了解价格、内容、门店和案例；
  - 用户基于服务方案下单或预约。
2. **商家端**
  - 商家发布自己能做的服务；
  - 设置价格模式、服务范围、预约时间和门店。
3. **平台端**
  - 平台审核服务真实性；
  - 控制服务复杂度和商家资质匹配；
  - 配置佣金、奖励、推荐规则。

---

# **3. 核心概念**

## **3.1 服务类目**

平台级分类，例如：

- 保养服务；
- 刹车系统；
- 轮胎服务；
- 电瓶服务；
- 钣喷修复；
- 事故车维修；
- 底盘维修；
- 新能源专项。

由平台创建和维护。

---

## **3.2 标准服务项目**

平台预设的标准项目，例如：

- 小保养；
- 更换刹车片；
- 更换刹车盘；
- 更换电瓶；
- 更换轮胎；
- 空调清洗；
- 钣喷修复；
- 事故车维修预约。

由平台创建，商家选择后发布自己的方案。

---

## **3.3 商家服务方案**

商家基于平台服务项目创建的具体可售卖/可预约服务。

例如：

```
杭州 XX 汽修 - 途虎同款品牌电瓶更换套餐
杭州 XX 钣喷 - 单面喷漆修复预约
杭州 XX 维修 - 事故车到店检测预约

```

---

# **4. 服务价格模式**

MVP 支持三种价格模式。

---

## **4.1 一口价**

适用于：

- 标准电瓶更换；
- 标准轮胎服务；
- 部分小保养；
- 标准检测服务。

用户可在线支付。

字段：

- 售价；
- 原价，可选；
- 服务包含；
- 不包含项目；
- 适用车型；
- 库存/可预约名额，可选。

---

## **4.2 套餐价**

适用于：

- 小保养套餐；
- 轮胎套餐；
- 电瓶套餐；
- 部分刹车套餐。

用户可在线支付。

字段：

- 套餐名称；
- 套餐价；
- 包含配件；
- 包含工时；
- 可选规格；
- 适用车型；
- 预约规则。

---

## **4.3 价格区间 + 到店确认**

适用于：

- 刹车片/刹车盘复杂情况；
- 空调维修；
- 底盘维修；
- 钣喷修复；
- 大保养部分项目；
- 新能源部分维修。

用户提交预约，不强制在线支付最终维修价。

展示：

```
参考区间：¥300-¥1200
到店检测后确认

```

字段：

- 最低参考价；
- 最高参考价；
- 价格影响因素；
- 是否支持检测费；
- 是否支持抵扣维修费，后续。

---

## **4.4 事故车预约到店报价**

事故车不做一口价，不支持线上最终报价。

展示方式：

```
事故车维修需到店检测/拆检后报价

```

可设置：

- 是否收取检测费；
- 检测费是否可抵扣维修费；
- 可预约时间；
- 可上传事故照片；
- 类似案例展示。

---

# **5. 服务复杂度等级**

每个服务项目必须绑定复杂度等级。


| 等级  | 示例                 | 作用        |
| --- | ------------------ | --------- |
| L1  | 洗车、简单检测、雨刮         | 低奖励、低资质要求 |
| L2  | 小保养、轮胎、电瓶、刹车片      | 标准交易主力    |
| L3  | 底盘、空调压缩机、发动机局部维修   | 需更高资质     |
| L4  | 事故车、大修、泡水车、新能源三电大修 | 高资质、高审核   |

# **6. 服务上架流程**

## **6.1 商家创建服务方案流程**

```
商家工作台
→ 服务方案
→ 新建服务方案
→ 选择门店
→ 选择平台服务项目
→ 选择价格模式
→ 填写服务内容
→ 填写价格/区间
→ 填写适用车型
→ 填写预约规则
→ 上传服务图片，可选
→ 提交平台审核
→ 审核通过后上架

```

---

## **6.2 服务方案状态**


| 状态   | 说明          |
| ---- | ----------- |
| 草稿   | 商家编辑中       |
| 待审核  | 已提交平台审核     |
| 审核通过 | 可上架         |
| 审核驳回 | 不符合要求       |
| 已上架  | 用户可见        |
| 已下架  | 商家主动下架      |
| 平台下架 | 平台因违规下架     |
| 暂停预约 | 暂时不可预约但保留展示 |

# **9. 服务审核规则**

## **9.1 审核内容**

平台审核人员检查：

1. 服务项目是否真实；
2. 价格是否明显异常；
3. 服务说明是否夸大；
4. 资质是否匹配；
5. 服务复杂度是否匹配；
6. 是否涉及禁止线上报价的事故车一口价；
7. 新能源专项是否有资质；
8. 图片是否合规；
9. 文案是否存在虚假承诺。

---

## **9.2 禁止内容**

服务方案不得出现：

1. 事故车一口价包修；
2. 保证保险全赔；
3. 违规返现；
4. 虚假原厂授权；
5. 未授权使用品牌标识；
6. 夸大维修效果；
7. 明显低价诱导；
8. 隐含强制消费；
9. 不具备资质却承接高复杂服务。

---

## **9.3 审核结果**


| 结果   | 说明         |
| ---- | ---------- |
| 通过   | 可上架        |
| 驳回   | 不允许发布      |
| 要求修改 | 修改后可重新提交   |
| 降级服务 | 可承接较低复杂度服务 |
| 限制预约 | 可展示但不能接单   |
| 平台下架 | 已上架后因违规下架  |

# **10. 佣金配置**

服务方案需支持佣金配置，但 MVP 前期默认免佣。

## **10.1 配置维度**

可按以下维度配置：

1. 城市；
2. 商家；
3. 门店；
4. 服务类目；
5. 服务项目；
6. 价格模式；
7. 时间周期。

---

## **10.2 佣金字段**


| 字段               | 说明      |
| ---------------- | ------- |
| commission_rate  | 佣金比例    |
| commission_fixed | 固定佣金，可选 |
| commission_cap   | 佣金封顶，可选 |
| commission_floor | 最低佣金，可选 |
| effective_start  | 生效时间    |
| effective_end    | 结束时间    |
| status           | 启用/停用   |


MVP 默认：

```
commission_rate = 0

```

---

# **11. 服务与案例关联**

每个服务方案应关联案例。

关联方式：

1. 同门店同服务项目案例；
2. 同服务项目全平台案例；
3. 同车型案例；
4. 平台精选案例。

展示优先级：

```
同门店同服务
→ 同门店相近服务
→ 同服务全平台案例
→ 平台精选案例

```

---

# **12. 服务与相册节点关联**

每个服务项目应配置建议相册节点。

例如：

### **刹车片/刹车盘**

- 旧刹车片状态；
- 刹车盘状态；
- 新旧件对比；
- 安装完成；
- 完工检查。

### **事故车**

- 事故损伤外观；
- 拆检照片；
- 定损/维修明细；
- 核心配件；
- 修复过程；
- 完工外观。

服务方案上架后，订单创建时系统根据服务项目生成建议相册节点。

# **14. 异常与边界规则**

## **14.1 商家资质不足**

如果商家选择超出资质范围的服务：

提示：

```
当前门店资质暂不支持发布该服务。
如需开通，请补充对应维修资质。

```

可保存草稿，但不能提交审核或上架。

---

## **14.2 事故车设置一口价**

系统禁止。

提示：

```
事故车维修不支持一口价或线上最终报价，请选择“预约到店报价”。

```

---

## **14.3 价格区间异常**

如果最高价低于最低价：

提示：

```
最高参考价不能低于最低参考价。

```

如果价格明显异常，可提交但进入重点审核。

---

## **14.4 服务被下架后的订单**

已下架服务：

- 不再对新用户展示；
- 已产生订单继续履约；
- 若存在违规，平台可联系用户处理退款或改约。

---

# **15. P0 验收标准**

必须满足：

1. 平台可配置服务类目；
2. 平台可配置标准服务项目；
3. 服务项目可绑定复杂度等级；
4. 服务项目可绑定资质要求；
5. 商家可创建服务方案；
6. 商家可选择价格模式；
7. 一口价/套餐价可在线支付；
8. 区间价服务可预约；
9. 事故车只能预约到店报价；
10. 商家提交后平台可审核；
11. 审核通过后用户端可见；
12. 服务可关联门店；
13. 服务可关联案例；
14. 服务可关联相册建议节点；
15. 佣金比例支持后台配置，默认 0%。
