## **2. 模块定位**

服务相册是平台 **P0 核心能力**，Phase 1 定位 **留档为主**：

1. 记录维修六阶段过程、配件类型、参考价与凭证图（**争议证据**）；
2. 可选关联车主，邀请查看私密留档（**只读，无需签字**）；
3. 车主 **授权公开** 后，脱敏沉淀案例与 GEO 内容（Phase 1 唯一车主主动确认环节）；
4. 提升门店透明度与咨询转化。

**Phase 2** 再启用：配件/方案车主确认、L2+ 阻断完工（见 `10_配件告知确认规则.md`）。

**配件验真链路（2026-06-24 定稿）**：阶段三报价表 OCR → `planParts` → 阶段四逐件上传 + 编码 OCR → 车主 **配件验真** 页对照（见 §13–§14、`07_维修相册查看页` §17.5）。

## **5. 六阶段编辑（Phase 1）**

按 `02_相册模板与节点规则.md` 分阶段上传：

1. 接车登记  
2. 检测诊断  
3. 维修方案与 **固定参考报价**（门店填写 `planAmount`；**上传详细报价表图**，OCR 生成 `planParts` 目录，见 §13）  
4. **配件告知**（**按 `planParts` 逐项**上传凭证图 + 编码 OCR；标准类型 + 价格 — **录入留档**，见 §14）  
5. 施工记录（新旧件、包装、编码等）  
6. 完工交付  

**Phase 1 无阻断规则**：商家可在配件未「车主确认」时标记完工。  
**Phase 2**：L2+ 非原厂件未确认不可完工。

## **8. 邀请车主与公开案例**

### **8.1 邀请查看（私密）**

- 已填手机号：车主可在「我的 → 我的服务相册」查看；  
- 可分享小程序路径 / 码（P1）；  
- **仅查看留档**，不要求确认配件或方案。

### **8.2 授权公开（公开案例）**

- 须档案 `completed` 且至少 1 张有效图片；  
- **车主**在 user 端发起授权（勾选 + 留痕；Phase 2 电子签名）；  
- 脱敏工作台 → 平台展示审核 → 发布 H5 / 案例库；  
- Tag：**已授权 · 已脱敏 · 已审核**（按授权档位展示字段）。

商家 **不可** 代车主授权公开。

## **10. P0 验收标准（Phase 1）**

1. 商家可新建服务相册（门店 + 服务 + **车主扫码关联手机号**）；  
2. 可按六阶段上传图片与说明，可录入配件类型与价格；  
3. **不发起**配件车主确认，**不阻断**完工；  
4. 可邀请已关联车主查看留档；  
5. 车主可授权公开；公开前完成脱敏；  
6. 员工上传与关键操作有日志。

## **13. 阶段三 · 方案报价表与 OCR 确认台（B-PLAN · 2026-06-24）**

> **真源字段**：`02_相册模板与节点规则.md` §3.5 · 接口见 `10/04_接口规范.md`（待 B-ALB-09 补全）。

### **13.1 商家上传**

| 步骤 | UI | 规则 |
| --- | --- | --- |
| 1 | 阶段三编辑页 | 填写方案说明 + **`planAmount`** |
| 2 | **上传报价表** | 1～N 张图（门店 DMS/手写报价单/系统截图） |
| 3 | 触发识别 | 异步 `plan_quote_ocr`：先 OCR，失败时多模态 LLM 兜底（手写报价表） |
| 4 | **确认台** | 表格预览：名称 / 类型 / 品牌 / 编码 / 数量 / 单价 |
| 5 | 锁定 | 「确认方案配件目录」→ `planPartsLockedAt` |

**确认台交互**

- 每行可 **编辑/删除/新增**（OCR 草稿不可直接对外展示）。  
- 页脚 Compliance：`识别结果仅供参考，请以你确认的报价表为准；确认后将作为车主验真时的方案侧对照。`  
- 锁定后：阶段四 **清单向导** 只读引用 `planParts`；修改方案目录须 **解锁** 并记录版本（P1：`planPartsRevision++`）。

### **13.2 OCR 失败降级**

- 无 OCR 结果：允许 **纯手工** 维护 `planParts` 表格（与确认台同 UI）。  
- L3/L4：可配置 **必须有报价表图** 才允许锁定（Phase 2 与阻断规则一并启用）。

### **13.3 与 `planAmount` 校验**

- OCR 行金额合计 ≠ `planAmount` → 黄条：「请核对报价表与方案总额是否一致」。  
- **不自动修改** `planAmount`。

## **14. 阶段四 · 配件清单向导与编码 OCR（B-PART · 2026-06-24）**

### **14.1 清单式向导（非自由表单）**

```text
方案配件目录（3 项）                    进度 2/3
┌────────────────────────────────────────────┐
│ ✓ 前刹车片 · 品牌件 · ×2                  │
│   编码 341168XXXX · 已上传 2 张            │
├────────────────────────────────────────────┤
│ ○ 后刹车片 · 品牌件 · ×2                  │
│   [ 上传包装/编码/实物图 ]                 │
│   识别编码：（可编辑）                      │
└────────────────────────────────────────────┘
[ + 添加方案外增项 ]
```

| 项 | 规则 |
| --- | --- |
| 进度 | `已完成 / planParts.length` |
| 每行必做 | **标准类型** + **≥1 凭证图**（保存本项时校验） |
| 每行选填 | **品牌、编码** — 商家视配件重要程度填写；**机油/油液/冷却液与刹车片等同一表单**，不做「油液简化登记」 |
| 配件 OCR | 多图顺序识别编码候选 → 商家选择或单候选自动填入（可编辑） |
| 确认 | 单行「保存本项」→ 写入 `partsJson` |
| 增项 | `source: extra`；须名称 + 类型；原因说明选填 |
| L3+ 建议 | 主要更换件 **建议** 填写编码或编码图（非 Phase 1 阻断） |

### **14.2 与 Phase 2 配件确认卡的关系**

| 能力 | 时机 | 文档 |
| --- | --- | --- |
| **配件确认卡** | 事前：非原厂件 **同意使用** | `10_配件告知确认规则.md` §4–§6 |
| **配件验真** | 事后：方案 vs 相册 **对照留痕** | `07_维修相册查看页` §17.5 |

二者 **不合并** 为同一页面；商家完成录入后，车主在验真页 **自愿** 对照。

### **14.3 登记分级与列表分组（定稿 · 不做 · 2026-06-25）**

曾讨论：识别后为行标记「主要更换件 / 油液 / 辅料」、向导分组排序、油液类弱化编码必填与验真 diff。

**产品结论：保持现状。**

- 商家 **不看到** 上述分类文案；后端 **不** 为商家流程持久化 `partCategory`。
- 识别层仍过滤纯工时行；清单 **不** 按重要程度自动删减或分组。
- 详见 `04/02_相册模板` §3.6.3 · `00_开发计划` §7.6.13。
