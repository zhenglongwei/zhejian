# **2. 模块定位**

维修相册是平台透明维修的核心内容资产。

它承担：

1. 记录维修过程；
2. 向用户解释维修内容；
3. 支撑订单履约；
4. 沉淀门店案例；
5. 提升门店透明度评分；
6. 为历史案例和公开案例提供素材。

---

# **3. 相册创建模式**

## **3.1 从订单创建维修相册**

适用于平台订单。

入口：

1. 订单详情页；
2. 维修中订单卡片；
3. 待完工订单；
4. 工作台快捷操作。

特点：

- 自动关联订单；
- 自动带出用户车辆信息；
- 自动带出服务项目；
- 自动生成建议节点；
- 可用于用户查看；
- 完工后可转为案例草稿。

---

## **3.2 独立创建维修相册**

适用于非平台订单或门店日常记录。

入口：

```
商家工作台
→ 维修相册
→ 新建相册

```

独立相册用途：

1. 记录线下维修；
2. 上传历史案例；
3. 创建门店展示案例；
4. 为后续内容冷启动积累素材。

特点：

- 不强制关联订单；
- 需选择门店；
- 需选择服务项目或服务模板；
- 可填写车型和维修说明；
- 可转为历史案例草稿；
- 公开前必须审核。

# **8. 节点管理**

## **8.1 节点来源**

节点来自服务项目模板。

例如刹车片服务：

1. 维修前；
2. 旧件状态；
3. 新旧件对比；
4. 安装过程；
5. 完工结果。

事故车服务：

1. 事故外观；
2. 拆检过程；
3. 损伤细节；
4. 配件更换；
5. 修复过程；
6. 完工外观。

---

## **8.2 节点规则**

- 节点为建议，不强制；
- 未选择节点的图片归为“其他”；
- 商家可调整图片所属节点；
- 平台可为不同服务项目配置节点模板；
- 节点完整度计入数据看板和透明度评分。

---

# **9. 自动脱敏**

## **9.1 检测对象**

系统需尽量识别：

1. 车牌；
2. VIN；
3. 人脸；
4. 手机号；
5. 支付信息；
6. 身份证件；
7. 其他敏感文字。

---

## **9.2 脱敏规则**

- 公开展示图片必须使用脱敏版本；
- 未完成脱敏的图片不可公开；
- 商家可手动标记需要打码区域；
- 平台审核人员可要求重新上传或手动处理。

# **11. 相册转案例**

## **11.1 订单相册转案例**

订单完成后，商家可将相册转为案例草稿。

系统带出：

- 门店；
- 服务项目；
- 车型；
- 维修图片；
- 维修说明；
- 配件说明；
- 订单真实性标签。

---

## **11.2 独立相册转历史案例**

独立相册可转为历史案例草稿。

需补充：

- 故障表现；
- 维修方案；
- 价格区间；
- 隐私责任确认；
- 提交审核。

# **14. P0 验收标准**

必须满足：

1. 商家可从订单创建维修相册；
2. 商家可独立创建维修相册；
3. 独立相册可用于记录线下维修；
4. 独立相册可转为历史案例草稿；
5. 相册可上传多张图片；
6. 图片可排序；
7. 图片可选择节点；
8. 系统可根据服务项目生成建议节点；
9. 图片公开前需脱敏；
10. 相册可关联订单；
11. 相册可关联门店和服务项目；
12. 员工可上传相册；
13. 相册操作有日志。
