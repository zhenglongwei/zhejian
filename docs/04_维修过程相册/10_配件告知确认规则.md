## **2. 模块定位**

**Phase 1**：配件 **录入留档**（标准类型、价格、照片）已满足基本证据需求；**不要求**车主确认。

**Phase 2**：配件确认卡 + 车主确认 + 阻断完工，用于强化「提前、明确、显著、可追溯告知」的合规能力。

目标（Phase 2）：

- 满足「提前、明确、显著、可追溯告知并取得车主确认」的合规要求；
- 降低非原厂件、同质件、以修代换等场景的「退一赔三」风险；
- 为 PDF 证据包提供可引用的确认记录。

### **2.1 与「配件验真」的分工（定稿 2026-06-24）**

| 维度 | **配件确认卡**（本文档 · Phase 2） | **配件验真**（`07_…` §17.5 · 已落地留痕） |
| --- | --- | --- |
| 时机 | **事前**：施工前/告知时 | **事后**：相册录入完成后，车主自愿 |
| 目的 | 同意使用某类型配件（尤其非原厂） | 对照 **方案 `planParts`** 与 **相册 `partsJson`** 是否一致 |
| 是否阻断完工 | Phase 2 可配置 L2+ | **否**（仅留痕 + 引导反馈） |
| 平台承诺 | 告知 + 确认留痕 | **不鉴定真伪**；不保证装车件一致 |
| 数据 | 确认卡字段 + 勾选 | `service_album_part_verifications` + pairs |

**OCR 辅助录入**（方案报价表、配件编码图）属于 **商家留档** 能力，见 `02_相册模板` §3.5–§3.6、`03/04_维修相册` §13–§14；**不等于** 车主确认或平台鉴定。

## **3. 配件标准类型**

商家 **必须** 从下列类型中选择，**禁止** 自造模糊词：

| 类型 | 标识色建议 | 须车主确认 |
| --- | --- | --- |
| 原厂件 | `--color-success` / 蓝 | 否（仍建议记录） |
| 同质配件 | 蓝 | **是** |
| 品牌件 | 蓝 | **是** |
| 副厂件 | `--color-warning` | **是** |
| 再制造件 | warning | **是** |
| 拆车件 | warning | **是** |
| 修复件 | warning | **是** |
| 翻新件 | warning | **是** |
| 以修代换 | warning | **是** |
| 未知来源件 | `--color-danger` | **禁止使用**（系统不允许或须特批） |

## **4. 配件确认卡字段**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| part_name | 是 | 配件名称，如「左前大灯总成」 |
| part_brand | 建议 | 配件品牌 |
| part_type | 是 | 标准类型（上表） |
| is_oem | 系统 | 是否原厂（由 type 推导） |
| oem_reference_price | 建议 | 原厂件参考价 |
| actual_price | 建议 | 本次使用配件价格 |
| diff_desc | 非原厂建议 | 差异说明 |
| risk_hint | 非原厂必填 | 年检/二手车/质保等风险提示 |
| supplier | 内部 | 供应商（不公开展示） |
| photos | L2+ 建议 | 配件/包装/编码图 |
| confirm_status | 系统 | pending / confirmed / rejected |
| confirm_record_id | 系统 | 关联车主确认记录 |

## **5. 非原厂件强提醒文案（车主端）**

选择非原厂类型后，车主确认页 **必须** 展示：

```text
该配件并非主机厂原厂件。

请确认你已了解：
1. 该配件与原厂件在品牌、渠道、包装或规格上可能存在差异；
2. 后续年检、二手车检测、保险理赔或质保判断中，可能被识别为非原厂件；
3. 本次选择是基于维修方案、价格和适配情况后的确认。

如你不同意，请选择原厂件或联系门店重新报价。
```

交互要求（来自判例启示）：

- 字体足够大；风险提示 **独立卡片**（浅黄底 `--color-warning-bg`）；
- 配件类型 **高亮标签**；
- 车主 **主动勾选**，**禁止默认勾选**；
- 确认按钮文案：**「确认使用非原厂件」** 或 **「我已知晓并同意使用该配件」**；
- 禁止仅写「确认」二字。

## **6. 维修方案确认（阶段三）**

L3/L4 或金额超过阈值（如单项配件 >5000 元）时，除配件卡外须 **整单方案确认**：

| 字段 | 说明 |
| --- | --- |
| 项目清单 | 每项价格 |
| 配件类型汇总 | 列出所有非原厂件 |
| 质保/不保 | 文本 |
| 预计完工 | 可选 |

车主勾选：「我已阅读维修方案，理解费用构成与配件类型说明。」

---

## **7. 交付验收确认（阶段六，L3/L4 建议）**

车主勾选：「我已验收车辆，了解本次维修内容与质保范围。」

**不等同** 于 V1.0「平台确认完工」；仅为相册内交付留痕，不涉及平台订单状态。

---

## **8. 阻断式确认规则**

| 条件 | 阻断行为 |
| --- | --- |
| L2+ 存在 `pending` 配件确认 | 不可将相册标为 `completed`；不可进入阶段五完工（可配置为不可标记阶段五完成） |
| L3/L4 方案未确认 | 不可录入施工完成 |
| 方案变更（配件 type/项目变化） | 作废原确认，`confirm_version++`，重新确认 |
| 未知来源件 | 不允许保存或须超管特批 |

商家工作台 **合规检查清单** 示例：

```
接车照片        ✓
检测记录        ✓
报价/方案       ✓
配件类型        ✓
非原厂告知      待车主确认  ← 阻断
新旧件对比      未上传
完工验收        未完成
```

---

## **9. 高风险配件二次确认（L3/L4）**

以下项目即使为原厂件，也建议 **二次确认**（方案确认 + 配件确认 + 新旧件对比必传）：

- 发动机、变速箱、安全气囊；
- 大灯总成、轮胎、刹车系统、转向系统；
- 底盘/车身结构件、三电系统；
- 事故车大额维修、保险定损维修。

## **11. 数据留痕**

每次确认须写入 `part_confirmation`：

| 字段 | 说明 |
| --- | --- |
| album_id, part_id | 关联 |
| user_id | 确认人 |
| content_snapshot | 确认时展示的完整文案 JSON |
| content_version | 版本号 |
| checkbox_text | 勾选项原文 |
| button_text | 按钮文案 |
| confirmed_at | 时间 |
| confirm_method | checkbox / esign |
| ip, device_info | 设备留痕 |

**禁止** 商家事后篡改已确认内容；变更须新版本 + 重新确认。

---

## **12. 与公开案例的关系**

- 公开案例 **不展示** 车主签名、完整确认单、供应商信息；
- 可展示脱敏后的「本次使用品牌件/副厂件」等 **类型摘要**（若用户授权且不含识别风险）；
- 配件确认记录全文仅存在于私密相册与证据包。

---

## **13. MVP 范围**

**P0**：标准类型字段、非原厂确认页（勾选+按钮+时间戳）、待确认入口、L2+ 阻断完工、确认留痕。

**P1**：电子签名、OCR 包装识别、采购凭证上传、AI 风险提示。

---

## **14. 验收标准**

1. 非原厂件未确认时 L2 相册无法 `completed`；
2. 确认页无默认勾选；
3. 确认记录含 content_version 与 time；
4. 方案变更后旧确认失效；
5. 证据包（P0.5）含配件确认摘要。
