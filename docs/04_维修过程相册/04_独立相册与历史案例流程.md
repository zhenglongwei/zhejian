## 2. 模块定位

独立相册与历史案例用于解决平台冷启动和商家内容资产沉淀问题。

它允许商家将过往维修案例整理为公开案例，但必须满足隐私保护、内容真实性、来源标识和平台审核要求。

## 3. 相册类型

### 3.1 历史案例相册

商家上传历史维修案例，不关联平台订单。

特点：

1. 不关联平台用户订单；
2. 商家承担授权与真实性责任；
3. 必须确认隐私责任；
4. 必须脱敏；
5. 必须平台审核；
6. 公开时标注“商家历史案例”。

---

### 3.2 独立线下相册，P1

商家为线下到店用户创建维修相册。

MVP 可暂不完整支持。

后续能力：

1. 商家创建线下维修记录；
2. 用户扫码绑定；
3. 用户确认车辆与维修记录；
4. 用户授权后可转为平台可验证案例；
5. 可信等级高于普通历史案例。

## 4. 总流程

历史案例流程：

```text
商家点击创建相册
→ 选择“历史案例”

→ 选择相册模板
→ 批量上传图片（原图）
→ 系统或商家分类图片
→ 选择节点
→ 自动生成说明
→ 【图片脱敏工具】一键脱敏 → 预览 → 确认脱敏结果（商家责任）
→ 生成公开案例草稿
→ 平台审核
→ 发布（仅使用已确认脱敏图）

```

## **8. 图片上传与节点分类**

### **8.1 上传规则**

1. 单个历史案例最多 20 张图；
2. 单次建议最多上传 9 张；
3. 支持 jpg/jpeg/png/webp；
4. 上传后自动压缩；
5. 上传后自动生成缩略图；
6. 上传后进入自动脱敏队列；
7. MVP 不支持视频公开展示。

---

### **8.2 图片分类方式**

支持两种方式：

#### **方式一：商家手动分类**

商家上传图片后，逐张选择节点：

1. 维修前；
2. 故障点；
3. 拆检过程；
4. 新旧件对比；
5. 维修过程；
6. 完工效果；
7. 质检结果。

#### **方式二：系统辅助分类，P1**

系统根据图片内容识别：

1. 是否为车辆外观；
2. 是否为配件；
3. 是否为单据；
4. 是否为前后对比；
5. 是否含隐私风险；
6. 推荐节点分类。

商家可确认或修改。

平台会对图片和文字进行自动脱敏与审核，但商家仍需对上传内容的合法性、真实性以及不侵犯客户隐私承担责任。

如平台发现隐私风险、侵权投诉、虚假案例或内容不实，有权立即下架案例，并视情况限制门店案例发布权限。

## **11. 自动脱敏流程**

提交后系统自动处理：

```
图片压缩
→ 缩略图生成
→ 车牌识别
→ 人脸识别
→ OCR 识别
→ VIN/手机号/姓名/证件识别
→ 自动打码
→ 风险评级
→ 输出脱敏图

```

风险规则：


| 风险等级 | 处理      |
| ---- | ------- |
| 低风险  | 可进入审核   |
| 中风险  | 审核重点检查  |
| 高风险  | 必须人工复核  |
| 禁止公开 | 不允许提交公开 |


---

## **12. 平台审核流程**

### **12.1 审核入口**

```
平台运营工作台
→ 内容审核
→ 历史案例审核

```

---

### **12.2 审核列表字段**


| 字段   | 说明             |
| ---- | -------------- |
| 案例标题 | 自动生成或商家填写      |
| 门店   | 关联门店           |
| 服务项目 | 服务类型           |
| 案例来源 | 商家历史案例         |
| 图片数量 | 图片数量           |
| 风险等级 | 低/中/高/禁止       |
| 质量评分 | 系统评分           |
| 提交时间 | 提交审核时间         |
| 审核状态 | 待审核/通过/驳回/要求修改 |


---

### **12.3 审核检查项**

审核人员必须检查：

1. 是否存在车牌；
2. 是否存在 VIN；
3. 是否存在人脸；
4. 是否存在手机号；
5. 是否存在姓名、身份证；
6. 是否存在保险单、报案号、支付信息；
7. 是否存在虚假宣传；
8. 是否存在夸大维修效果；
9. 是否存在盗图风险；
10. 门店资质是否匹配；
11. 车型是否过于稀有导致可识别；
12. 价格是否为区间；
13. 是否适合公开传播。

---

### **12.4 审核结果**


| 结果           | 说明         |
| ------------ | ---------- |
| 通过并公开        | 生成公开案例     |
| 通过但仅门店展示     | 不进入平台案例库   |
| 通过但不允许 H5 收录 | H5 noindex |
| 要求修改         | 退回商家修改     |
| 驳回           | 不允许公开      |
| 禁止公开         | 高隐私或违规     |
| 平台隐藏         | 已发布后临时隐藏   |
| 下架           | 已发布后移除     |


---

## **13. 发布规则**

历史案例发布后必须展示来源标识：

```
商家历史案例 · 已脱敏 · 已审核 · 价格仅供参考

```

不得展示为：

```
平台订单案例
真实车主订单
用户认证案例

```

除非后续完成用户绑定与授权。

---

## **14. H5 规则**

历史案例可生成 H5 页面，但需满足：

1. 审核通过；
2. 脱敏图存在；
3. 标题和摘要完整；
4. 服务项目明确；
5. 门店信息完整；
6. 无禁止公开风险；
7. 平台允许搜索收录。

如质量一般或风险较高，应设置：

```
<meta name="robots" content="noindex,nofollow">

```

---

## **15. 修改与下架**

### **15.1 商家修改**

商家可修改：

1. 标题；
2. 摘要；
3. 维修方案；
4. 价格区间；
5. 图片说明；
6. 图片排序；
7. 是否展示。

修改后需重新审核的场景：

1. 修改图片；
2. 修改维修方案；
3. 修改价格；
4. 修改车型；
5. 修改可能影响真实性的核心字段。

---

### **15.2 商家下架**

商家可申请下架历史案例。

平台可直接下架：

1. 隐私投诉；
2. 侵权投诉；
3. 虚假案例；
4. 门店违规；
5. 商家资质失效。

---

### **15.3 用户投诉下架**

若有用户投诉历史案例侵权或泄露隐私：

1. 平台应优先临时隐藏；
2. 进入用户申请下架状态；
3. 平台审核；
4. 确认后下架或修改后恢复；
5. 通知商家。

---

## **16. 商家风控**

商家出现以下行为时限制历史案例能力：

1. 多次上传隐私违规图片；
2. 多次被投诉；
3. 疑似盗图；
4. 大量重复低质内容；
5. 虚假案例；
6. 夸大宣传；
7. 冒充平台订单案例；
8. 上传非维修相关内容。

限制措施：

1. 降低案例曝光；
2. 增加人工审核；
3. 暂停历史案例发布；
4. 下架已发布案例；
5. 限制门店内容展示；
6. 严重时清退。

2. 未确认隐私责任不可提交；

7. 历史案例不得冒充平台订单案例；

9. 修改关键内容后需重新审核；
