## 2. 流程概览

订单相册流程：

```text
订单进入“已接单”
→ 系统生成相册任务
→ 系统匹配相册模板
→ 商家按节点拍摄/上传
→ 用户实时查看进度
→ 商家提交完工
→ 系统生成交付报告
→ 用户确认完成/订单完成
→ 系统生成案例草稿
→ 用户授权公开
→ 自动脱敏
→ 平台审核
→ 公开发布

```

## **3. 核心规则**

### **3.1 自动创建**

订单进入可履约状态后，系统自动创建订单相册。

建议触发状态：

```
已接单

```

兼容复杂预约场景：

```
预约已提交
商家已确认
已到店
服务中

```

---

### **3.2 不阻断订单**

订单相册不阻断订单主流程。

即：

1. 相册为空不影响订单确认完成；
2. 节点未完成不影响订单结算；
3. 图片上传失败不影响服务履约；
4. 未公开不影响用户查看个人维修档案；
5. 商家不上传只影响透明度、案例质量和门店内容资产。

---

### **3.3 用户实时可见**

商家上传图片成功后，用户可在订单详情或维修相册页查看。

用户可查看：

1. 相册图片；
2. 节点时间线；
3. 图片说明；
4. 上传时间；
5. 门店信息；
6. 维修摘要；
7. 完工报告。

---

### **3.4 授权前不得公开**

订单相册属于用户维修档案。

未获得用户明确授权前：

1. 不得生成公开案例；
2. 不得展示到门店主页；
3. 不得展示到服务详情页；
4. 不得发布 H5；
5. 不得被搜索引擎收录；
6. 不得用于广告素材。

## **6. 订单阶段流程**

### **6.1 已接单：生成相册任务**

触发条件：

1. 订单状态变为已接单；
2. 订单关联服务项目；
3. 订单关联门店；
4. 未存在订单相册。

系统动作：

1. 创建 album 记录；
2. 匹配相册模板；
3. 生成建议节点；
4. 创建商家待办；
5. 在商家订单详情展示“维修相册”入口；
6. 在用户订单详情展示“维修进度/维修相册”入口。

---

### **6.2 商家上传：按节点拍摄**

商家操作：

1. 进入订单详情；
2. 点击维修相册；
3. 查看建议节点；
4. 选择节点；
5. 拍照或从相册选择；
6. 填写图片说明，可选；
7. 上传；
8. 调整排序；
9. 保存。

上传后系统动作：

1. 图片压缩；
2. 生成缩略图；
3. 写入图片记录；
4. 更新相册图片数量；
5. 更新节点完成状态；
6. 用户端可实时查看；
7. 可触发用户消息提醒，按策略。

---

### **6.3 用户查看：实时进度**

用户入口：

1. 订单详情页；
2. 维修进度入口；
3. 消息通知；
4. 我的维修档案，P1。

用户可见内容：

1. 订单信息；
2. 门店信息；
3. 服务项目；
4. 节点时间线；
5. 图片缩略图；
6. 图片详情；
7. 图片说明；
8. 上传时间；
9. 联系门店；
10. 联系客服。

---

### **6.4 商家提交完工**

触发条件：

1. 维修服务完成；
2. 商家点击“提交完工”；
3. 订单状态允许提交完工。

商家提交内容：


| 字段                 | 是否必填 | 说明    |
| ------------------ | ---- | ----- |
| album_id           | 是    | 相册 ID |
| repair_summary     | 否    | 维修总结  |
| parts_desc         | 否    | 配件说明  |
| warranty_desc      | 否    | 质保说明  |
| maintenance_advice | 否    | 后续建议  |
| completed_images   | 否    | 完工图   |
| completed_at       | 是    | 完工时间  |


系统动作：

1. 相册状态变为 completed；
2. 生成交付报告；
3. **触发预脱敏任务**（见 §6.4.1，后台异步，不对用户展示）；
4. 通知用户查看；
5. 订单进入待用户确认完成，若业务允许；
6. 判断是否可进入案例草稿流程。

---

### **6.4.1 预脱敏（Pre-mask）**

**目标**：用户同意公开时可直接加载脱敏预览，无需等待 AI 处理。

| 项 | 规则 |
|---|---|
| 触发时机 | 商家提交完工（§6.4）时全量预脱敏；商家上传新图时可增量预脱敏（P1） |
| 可见性 | **不对用户、商家公开展示**；写入 **`order_pre_mask` 脱敏任务**，`rawAssets[].preMaskedUrl` 为 private |
| 存储 | **统一 `desensitize_tasks`**，禁止另建平行 pre-mask storage |
| 与用户授权关系 | 预脱敏 ≠ 可公开；须再建 **`order_authorize`** 任务，用户在工作台 `maskingConfirmed` 后才可提交公开 |
| 原图变更 | `fingerprint` 变更 → `preMaskVersion++`、重跑 pre-mask、作废未确认的 `order_authorize` 任务 |
| 交付报告 | 仍用订单压缩图，**不使用** pre-mask 图 |

推荐流程：

```text
商家提交完工
→ 后台 pre-mask（异步）
→ 订单完成，用户看到「公开为案例」入口
→ 用户点确认公开
→ 直接加载 pre-mask 预览（零等待或极短等待）
→ 用户逐张核对 / 手动打码（可选）
→ 确认并公开

```

---

### **6.5 生成交付报告**

交付报告用于用户查看，不等于公开案例。

报告内容：

1. 服务项目；
2. 门店信息；
3. 维修摘要；
4. 维修过程节点；
5. 图片；
6. 配件说明；
7. 质保说明；
8. 后续建议；
9. 联系商家；
10. 评价入口。

交付报告图片可以使用订单内压缩图，不要求公开脱敏图。

但用户端仍应避免直接展示明显隐私字段，如完整手机号、VIN 等。

---

### **6.6 订单完成：生成案例草稿**

触发条件：

1. 订单已完成；
2. 相册至少有 1 张有效图片；
3. 未发生严重售后或投诉；
4. 相册未被标记禁止公开；
5. 未生成过案例草稿。

系统动作：

1. 创建 public_case 草稿；
2. 读取订单、车辆、服务、门店、相册信息；
3. 根据模板生成标题、摘要、正文草稿；
4. 生成待授权记录；
5. 向用户展示授权公开入口；
6. 可发送订阅消息提醒。

---

### **6.7 用户授权公开**

用户可选操作：


| 操作        | 结果             |
| --------- | -------------- |
| 同意公开      | 进入脱敏和审核流程      |
| 拒绝公开      | 不公开，仅保留为维修档案   |
| 删除部分图片后公开 | 仅使用用户选择图片      |
| 手动打码后公开   | 使用用户编辑后的图片进入审核 |
| 暂不处理      | 保持待授权状态        |


授权必须记录：

1. 用户 ID；
2. 订单 ID；
3. 相册 ID；
4. 授权结果；
5. 授权协议版本；
6. 脱敏预览版本；
7. 授权时间；
8. 设备信息；
9. IP。

---

### **6.8 自动脱敏**

用户同意公开并**在预览工作台确认脱敏结果**后，正式采纳脱敏图用于公开案例。

若商家已提交完工且预脱敏（§6.4.1）已完成，授权预览页**直接加载** `preMaskedUrl`，无需再次排队等待 AI。

系统处理：

1. 图片压缩；
2. 车牌识别；
3. 人脸识别；
4. OCR 识别；
5. VIN、手机号、姓名等识别；
6. 自动打码；
7. 生成脱敏图；
8. 输出风险等级；
9. 写入检测记录。

公开页面只能使用脱敏图。

---

### **6.9 平台审核**

审核人员检查：

1. 用户是否授权；
2. 图片是否脱敏；
3. 文字是否含隐私；
4. 价格是否展示为区间；
5. 是否存在虚假宣传；
6. 是否适合公开；
7. 是否允许 H5 收录；
8. 是否进入门店主页/平台案例库。

审核结果：


| 结果          | 说明           |
| ----------- | ------------ |
| 通过并公开       | 生成公开案例       |
| 通过但仅门店展示    | 不进入平台案例库     |
| 通过但 noindex | H5 不允许搜索引擎收录 |
| 要求修改        | 退回商家或平台修改    |
| 驳回          | 不公开          |
| 禁止公开        | 高风险，不允许公开    |


---

### **6.10 公开发布**

发布内容：

1. 小程序案例详情页；
2. 门店主页案例；
3. 服务详情页关联案例；
4. H5 公开案例页；
5. SEO/GEO 元信息；
6. sitemap，若允许收录。

## **10. 异常场景**

### **10.1 订单取消**

规则：

1. 相册随订单关闭；
2. 用户仍可查看已上传内容，视业务策略；
3. 不进入公开案例流程；
4. 已生成草稿需关闭。

---

### **10.2 商家拒单**

规则：

1. 相册任务关闭；
2. 不展示公开入口；
3. 已上传图片，如存在，应内部保留或清理。

---

### **10.3 售后或投诉**

如订单出现严重售后或投诉：

1. 暂停公开案例流程；
2. 已公开案例可临时隐藏；
3. 平台审核后决定恢复、修改或下架；
4. 奖励可冻结。

---

### **10.4 用户拒绝公开**

规则：

1. 相册仅作为用户维修档案；
2. 不生成公开案例；
3. 商家不可二次发起强提醒；
4. 可在合理时间后允许用户主动重新授权，P1。

---

### **10.5 脱敏失败**

规则：

1. 标记图片脱敏失败；
2. 进入人工审核或重新处理；
3. 高风险图片不得公开；
4. 用户可删除该图片后继续授权。

## **12. 接口建议**

### **12.1 获取订单相册**

```
GET /api/user/orders/{order_id}/album

```

---

### **12.1.1 触发/查询预脱敏任务（`order_pre_mask`）**

商家提交完工或系统异步补偿时：

```
POST /api/system/albums/{album_id}/pre-mask

```

查询任务状态（运营/系统）：

```
GET /api/system/albums/{album_id}/pre-mask-task

```

响应含 `taskId`、`preMaskStatus`、`preMaskVersion`、`rawAssets[].preMaskedUrl`（private）。

---

### **12.1.2 用户授权预览（`order_authorize`）**

用户点「确认公开」时，基于 `order_pre_mask` 创建或返回授权预览任务：

```
POST /api/user/orders/{order_id}/album/authorize-preview

```

响应：`taskId`（order_authorize）、`preMaskTaskId`、`preMaskVersion`、`fromPreMask`。

```
GET /api/merchant/orders/{order_id}/album

```
```
POST /api/merchant/albums/{album_id}/images

```

### **12.4 更新图片节点/说明**


## **14. P0 范围**

P0 实现：

1. 订单已接单后自动生成相册任务；
2. 自动匹配模板；
3. 商家可上传图片；
4. 图片可绑定节点；
5. 用户可实时查看；
6. 商家可提交完工；
7. 系统生成交付报告；
8. 订单完成后生成案例草稿；
9. 用户可授权或拒绝公开；
10. 同意公开后进入脱敏；
11. 平台审核后发布；
12. 公开案例只使用脱敏图。

---

## **15. 验收标准**

1. 订单进入已接单后能自动创建相册；
2. 同一订单不会重复创建多个订单相册；
3. 商家可按节点上传图片；
4. 用户可实时查看商家上传的图片；
5. 商家可提交完工并生成交付报告；
6. 未经用户授权的订单相册不会公开；
7. 用户同意公开后才进入脱敏审核流程；
8. 脱敏失败或高风险图片不得公开；
9. 平台审核通过后案例可发布；
10. 订单取消、投诉、售后等异常场景可暂停公开流程。
