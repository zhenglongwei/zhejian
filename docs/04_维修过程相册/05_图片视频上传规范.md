## 2. 总体原则

1. 服务相册 P0 以图片为主；
2. 视频能力作为 P1 扩展；
3. 上传文件必须压缩；
4. 公开展示必须使用脱敏版本；
5. 图片需记录上传人、时间、节点、**凭证类型**；
6. 高风险图片不得自动公开；
7. **关键凭证** 不可随意删除，删除须留痕；
8. 用户私密查看与公开案例使用不同资源策略；
9. **P0 证据链水印**：相册编号、脱敏车辆、门店、拍摄时间、节点名称、拍摄人员。

## 3. 图片上传规范

### 3.1 支持格式

| 类型 | 是否支持 |
|---|---|
| jpg | 支持 |
| jpeg | 支持 |
| png | 支持 |
| webp | 支持 |
| heic | P1，可前端转码 |
| gif | 不支持 |
| bmp | 不支持 |
| tiff | 不支持 |

---

### 3.2 文件大小

| 项目 | 规则 |
|---|---|
| 单张原始上传上限 | 10MB |
| 前端压缩后建议大小 | 1-2MB |
| 服务端压缩后建议大小 | 300KB-1.5MB |
| 缩略图大小 | 50KB-300KB |
| 脱敏图大小 | 300KB-1.5MB |
| 单次上传数量 | 建议最多 9 张 |
| 单个相册图片上限 | 20 张 |

---

### 3.3 图片尺寸

| 项目 | 规则 |
|---|---|
| 最低建议宽度 | 720px |
| 最低建议高度 | 540px |
| 推荐最大宽度 | 1920px |
| 推荐最大高度 | 1920px |
| 缩略图宽度 | 300-500px |
| H5 展示图宽度 | 750-1200px |

低于最低清晰度的图片可上传，但应提示：

```text
图片清晰度较低，可能影响用户查看和案例公开效果。

```

---

### **3.4 图片质量要求**

图片应满足：

1. 主体清晰；
2. 不严重模糊；
3. 不过度曝光；
4. 不过暗；
5. 维修部位可辨认；
6. 不含明显无关内容；
7. 不含违法违规内容。

系统可对图片进行基础质量检测，输出：


| 质量等级    | 说明          |
| ------- | ----------- |
| good    | 清晰可用        |
| normal  | 基本可用        |
| poor    | 模糊、过暗或主体不明确 |
| invalid | 无效或违规       |

## **4. 图片压缩规则**

### **4.1 前端压缩**

小程序端上传前建议执行：

1. 尺寸压缩；
2. 质量压缩；
3. EXIF 处理；
4. 格式兼容转换；
5. 上传前预览。

建议参数：

```
宽高限制：最长边 1920px
质量：70%-85%
目标大小：1-2MB

```

---

### **4.2 服务端压缩**

服务端接收后执行：

1. 二次压缩；
2. 生成缩略图；
3. 生成展示图；
4. 保留压缩原图；
5. 推送脱敏任务。

---

### **4.3 原图保留策略**


| 场景    | 策略             |
| ----- | -------------- |
| 订单相册  | 可不长期保留高清原图     |
| 历史案例  | 可不长期保留高清原图     |
| 公开案例  | 不使用原图，只使用脱敏图   |
| 平台审核  | **不展示原图**，仅查看脱敏图与 OCR 摘要   |
| 高风险争议 | 可按合规策略短期保留证据版本 |


建议保存：

1. 压缩原图；
2. 缩略图；
3. 脱敏图；
4. 检测结果；
5. 上传元数据。

## **5. 图片水印规则（V2.0 证据链）**

### **5.1 相册内查看 / 凭证图（P0 基础）**

服务相册上传后自动叠加 **业务水印**（非仅公开图）：

```
相册：ZJ202605210001
车辆：浙A**** 宝马3系
节点：左前大灯更换-新旧对比
时间：2026-05-21 14:32
门店：杭州某某汽车服务中心
拍摄：张师傅
```

| 图片类型 | 水印 |
|---|---|
| 普通过程 | 建议加轻量水印 |
| 关键凭证 | **必须** 加水印 |

### **5.2 是否加水印（公开案例）**


| 图片类型     | 是否加水印       |
| -------- | ----------- |
| 用户订单内查看图 | 默认不加或弱水印    |
| 商家工作台查看图 | 可加内部水印      |
| 平台审核图    | 可加内部水印      |
| 公开案例图    | 建议加平台/门店弱水印 |
| 分享图      | 建议加平台水印     |


---

### **5.2 水印内容**

公开图水印可包含：

1. 平台名称；
2. 门店简称；
3. “已脱敏”标识；
4. 案例 ID，弱显示。

不应包含：

1. 用户姓名；
2. 手机号；
3. 车牌；
4. VIN；
5. 订单号明文；
6. 详细地址。

---

### **5.3 水印位置**

建议：

```
右下角
透明度 20%-35%
不遮挡维修主体
不遮挡脱敏区域

```

## **8. 隐私与安全**

### **8.1 上传前提示**

商家上传页提示：

```
请勿上传包含车主姓名、手机号、车牌、VIN、身份证、支付信息、保险单据等隐私内容的图片。如需上传单据，请先遮挡敏感信息。

```

---

### **8.2 上传后检测**

图片上传后必须进入检测流程：

1. 车牌检测；
2. 人脸检测；
3. OCR 文字识别；
4. VIN 检测；
5. 手机号检测；
6. 单据敏感信息检测；
7. 内容安全检测。

---

### **8.3 EXIF 处理**

为避免泄露拍摄位置和设备信息，服务端应默认清除图片 EXIF。

保留必要业务元数据到数据库，不依赖图片 EXIF。

---

### **8.4 内容安全**

需拦截：

1. 色情低俗；
2. 暴力血腥；
3. 政治敏感；
4. 广告引流；
5. 非维修相关；
6. 侵权图片；
7. 二维码引流；
8. 违法违规内容。

---

## **9. 图片展示规则**

### **9.1 用户订单相册**

用户查看自己订单相册时：

1. 可展示压缩原图；
2. 可按节点展示；
3. 可查看大图；
4. 不等于公开授权；
5. 可提供保存或分享脱敏档案，视业务策略。

---

### **9.2 公开案例**

公开案例必须使用：

```
masked_path

```

不得使用：

```
original_path

```

公开页面图片必须满足：

1. 已脱敏；
2. 已审核；
3. 无禁止公开风险；
4. 与案例内容相关；
5. 质量可接受。

---

### **9.3 平台审核**

平台审核人员可查看：

1. 压缩原图；
2. 脱敏图；
3. OCR 结果；
4. 风险区域；
5. 图片元数据；
6. 操作记录。

## **10. 视频上传规范**

### **10.1 MVP 策略**

MVP 不支持视频上传和公开展示。

原因：

1. 存储成本高；
2. 转码成本高；
3. 隐私识别难度高；
4. 审核成本高；
5. 小程序加载体验不稳定。

---

### **10.2 P1 视频要求**

后续支持视频时，建议规则：


| 项目       | 规则        |
| -------- | --------- |
| 支持格式     | mp4/mov   |
| 单段最大时长   | 30 秒      |
| 最大文件大小   | 50MB      |
| 推荐压缩后大小  | 5-15MB    |
| 分辨率      | 720p 优先   |
| 是否转码     | 必须        |
| 是否生成封面   | 必须        |
| 是否支持公开展示 | 审核通过后可    |
| 是否自动脱敏   | 必须支持关键帧脱敏 |
| 是否默认公开   | 否         |


---

### **10.3 视频处理流程，P1**

```
上传
→ 格式校验
→ 转码
→ 截取封面
→ 关键帧抽取
→ 内容安全检测
→ 车牌/人脸/OCR 检测
→ 风险评级
→ 审核
→ 展示

```

---

### **10.4 视频公开限制**

以下视频不得公开：

1. 出现未脱敏车牌；
2. 出现未脱敏人脸；
3. 出现完整 VIN；
4. 出现客户对话；
5. 出现手机号或支付信息；
6. 出现事故现场敏感内容；
7. 涉及安全风险操作；
8. 无法有效脱敏。
