| 档位 | 说明 |

**废止（2026-07-12 起新写入）**：`authorizationTier = anonymous`（匿名公示档）。  
**产品文案废止（2026-07-18）**：面向车主的主按钮/主说明 **不再使用「授权公示」**；改用 **「发布到公开网站」**（见 §4.5）。实现层字段名（如 `authorizationTier`、`public-case`）可暂保留，**以产品文案与本节流程为准**。

**原则（不可妥协）**：

1. **用户点一下才上线**——未点「发布到公开网站」前，任何长文草稿、预览、私人分享 **均不上** H5 / GEO。  
2. **不骗车主**——发布前须说清将出现在公开网站；预览 = 即将上网内容（WYSIWYG = PublicView）。  
3. **降低顾虑与 GEO 供给同向**——靠透明、荣誉鼓励与控制感提高愿公开比例；**禁止**用隐瞒渠道或分享领钱换量。  
4. **平台不做物质分享奖励**——产品内仅语言/荣誉激励；物质答谢或现金奖励由运营指导门店线下处理，**不在小程序内配置、承诺或结算**，不在分享按钮上诱导现金。

**不属于「发布到公开网站」**：发给微信好友/朋友圈、复制链接私传、复制文案自行发到小红书等（见 §4.5.4）。

**PublicView**：发布预览与 H5/Feed 读侧真源为 `snapshot.publicView`（轻量公开池）；详见 §4.5.8、`14_PublicView与隐私改造计划.md`。  
**公域知识包与教练**：商家正文归属、公域藏价、配图硬拦、文字-only 完工生成、相册教练规则引擎 → §4.5.9 与 [`15_…`](./15_公域知识包与相册教练规则引擎.md)。

## 4.5 发布到公开网站 vs 自主分享（定稿 2026-07-18）

> **产品原则**：主语是 **车主自己发布**，不是「授权平台随便用」。公开网站仍须 **明确同意 + 留痕 + 闸门 B 过审**（低风险可自动过审；高风险须人审）；私人传播与站外第三方平台由车主自行操作。  
> **双系统转型（2026-06-08）**：公域发现主战场 = **H5 / 站外 GEO**；小程序 = **服务相册工具**（详见 `08_双系统转型开发计划.md`）。  
> **历史别名**：旧文档中的「授权公示」= 本节 **「发布到公开网站」**（法律实质仍为同意公开展示；交互与文案改为用户主导）。  
> **闸门 B 风险分流（2026-07-24）**：**废止**「必须人工审核才公开」的全量表述；改为低风险自动过审并立即上网、高风险拦截人审、低风险事后抽检可下架。

### 4.5.1 对照（分享面板三列）

| 维度 | **① 发给微信** | **② 复制去社交媒体** | **③ 发布到公开网站** |
| --- | --- | --- | --- |
| **谁发出** | 车主（微信原生） | 车主（自行粘贴到小红书/知乎/头条等） | 车主点击发布 → 平台闸门 B 过审后展示 |
| **典型渠道** | 好友、朋友圈 | 小红书、知乎、今日头条等（车主自选） | **H5 公开案例页、站外 GEO/SEO**（**不含** 小程序案例 Tab） |
| **是否上公开网站** | 否 | 否（第三方由车主账号负责） | **是**（须点发布 + 过审） |
| **脱敏** | 强制脱敏图 | 长文与配图建议仅基于脱敏公开池 | 强制脱敏（PublicView） |
| **平台审核** | 否 | 否 | **要**（低风险自动过审；高风险人审；自动过审可抽检） |
| **商家/平台代复制站外长文** | — | **禁止**（只给车主复制） | — |

### 4.5.2 发布到公开网站

- **含义**：车主主动将 **脱敏后、预览过** 的内容提交至公开网站展示位；闸门 B 过审后出现在 H5 / GEO（可被搜索收录）；**不含** 小程序公域案例 Tab。  
- **流程**：`completed` →（可选）生成可分享长文草稿 → 车主打开分享面板 → **预览将上网内容**（WYSIWYG = PublicView）→ 确认 **发布到公开网站** + 留痕 → 计算 `gateBRisk`：
  - **low**：自动 `public_approved` / `published`，写 `publishedAt`，**立即** H5/GEO 可见；约 **10%**（可配置）打标抽检（**不挡上线**）。
  - **high**：`pending_review`，**先不上网**，人审通过后才展示。
- **高风险默认条件**（命中任一）：公开配图 `riskLevel` 非 `low` / 需人工；公开就绪 `privacyBlocks` 非空；评价/正文扫出 PII 或绝对化承诺；存在进行中隐私/配件纠纷；模板为 `accident`（事故车弱公开）。  
- **默认不上线**：未完成上述确认的内容 **不会** 出现在 H5 / GEO；亦 **不会** 经小程序主导航进入公域发现。  
- **可撤回**：发布后车主可撤回；撤回后公开页下线或不可索引，相册可按既有规则再编辑后再发布。  
- **文案语气**：优先「由你发布」「发布前可预览」；低风险成功态说「已发布」，高风险说「审核中，通过后展示」；避免「授权给我们用于全网营销」类压迫表述。渠道说明须诚实：公开网站可能被搜索与第三方引用；**不**用恐吓清单替代主说明，也 **不** 隐瞒。

### 4.5.3 完工后长文草稿与荣誉鼓励

- 相册 **完工后** 可自动生成「可分享长文」草稿（标题/正文/配图建议），**仅车主可见**，**不上网**。  
- **荣誉/心理鼓励**（允许）：帮助同城车主、透明维修体验官（称号可随机轮换，无等级差异）、发布成功感谢等。  
- **禁止**：分享有奖、晒图返现、转发领钱、现金红包挂在分享按钮上（与设计体系 §2.4 / §9、主 PRD §10 一致）。  
- **物质答谢**：不在产品内配置；由运营指导门店线下自行处理。  
- **文案语气**：优先长句邀请（荣誉 + 可下架承诺），避免「授权公示」「发布到公开网站」等敏感短句作主按钮。

### 4.5.3a 小程序 · 公域只读访客态（2026-06-15 · 修订 2026-07-22）

与 §4.5 公开网站展示 **并行、不替代**：

| 用户态 | 小程序内可读 |
| --- | --- |
| **未绑定**任何服务相册（含未登录） | 搜一搜 / 分享深链直达的 **已审核公开** 门店/案例/服务详情（`public_guest`）；关联推荐可含同服务其他门店案例 |
| **已绑定**相册 | 仅相册 **关联门店集合** 内的门店及该店公开内容；**多店可看、互不串内容**（看 A 店只出 A 的案例/服务，不回落别店）；跨店浏览拒绝 |
| 商家分享 / 单店上下文 | 按 `storeId` 隔离展示（与 DS-A-09～11 一致） |

**小程序内 H5 公域出口**：已绑定用户不展示（工具台条件出口、关于辙见「浏览门店」等）；**H5/GEO 网页本身保持公域**，不改。

**禁止**：在小程序首页/Tab 增加跨店发现入口；本规则 **不** 改变「公开网站主渠道仍为 H5/GEO」。

### 4.5.4 用户自主分享（私人传播 + 社交媒体复制）

- **发给微信**：关联车主把 **自己的** 服务相册发给好友/朋友圈；完工即可；**不必** 先发布到公开网站。  
- **复制去社交媒体**：提供适配平台的长文（如小红书/知乎/头条风格）+ **一键复制**；由车主自行粘贴发布；平台与商家 **不代发、不代复制到商家运营账号**。  
- 入口：**我的服务相册** 详情 → **半屏 ShareSheet**（三列，见上表；`docs/02_用户端小程序/07_维修相册查看页.md` §17.4）。  
- **脱敏规则（定稿 2026-07-12）**：私人分享与社交媒体复制稿 **仅脱敏图/脱敏文案**；完整原图 **仅在小程序内**（车主/商家登录态）查看。  
- **废止**：私人分享的「原图模式」（`mode=original`）、`share_raw` 场景授权的新增留痕。  
- **复制私人分享链接**：`completed` 即可；链接带 token，**仅脱敏图**；**不** 自动进公开网站。  
- **复制公开网页链接**：仅 **已发布到公开网站且闸门 B 过审** 后出现。  
- 私人分享 / 社交媒体自发 **不** 等同于上公开网站。

### 4.5.5 文档与实现约束

- 文案禁止：「先授权才能分享」「分享即公开」「分享就有钱」。  
- **允许且推荐**：「发给微信不会自动上网」「发布到公开网站才会出现在案例站（低风险立即展示，高风险需审核）」。  
- **A-MASK**：公开网站链路与私人/社交媒体链路均 **禁止原图对外传播**。  
- **商家端**：可展示「用户已发布且过审」的案例与 H5 链接；**允许**商家可复制**同一确认稿**发自媒体（平台不代发）；**禁止**「商家一键代发到平台运营账号」作主能力。  
- **运营审核台**：人审队列以 **高风险** 为主；另设 **自动过审抽检**（不挡上线，不通过可下架）；审核员 **仅查看脱敏图 + OCR 摘要**，不展示原图（不变）。  
- 历史任务名 **U-CASE-06**（分享与公示解耦）精神保留；产品文案与 ShareSheet 三列以本节为准（落地见开发计划 §7.10 / `CASE-OPS-GATEB-RISK`）。

### 4.5.6 双闸门 · 留档质量 vs 合规 vs 脱敏（定稿 2026-07-08 · 文案修订 2026-07-18 · 风险分流 2026-07-24）

| 维度 | **商家** | **车主** | **平台** |
| --- | --- | --- | --- |
| **闸门 A**（完工后） | 合法合规；A 驳回→改相册 | 见「门店已提交，内容待您确认」（A 通过后） | 自动规则+抽检；**不审** pre-mask 实现 |
| 六阶段 / 过程图 | **义务**（不阻断完工） | **权利**（监督）；不对完整度负责 | A/B **均不** 因缺节点 alone 驳回用户发布 |
| **闸门 B**（车主点「发布到公开网站」后） | — | 脱敏预览、评价/配图、发布确认 | 脱敏完整性 + **用户侧内容**；**风险分流**（低→自动过审；高→人审；低→事后抽检） |
| B 驳回 / 抽检下架 | **不解锁**改相册（驳回）；下架保留 snapshot | **重试/手工脱敏、改评价**；须见驳回原因 | 合规类问题应在 A 已拦截；抽检不通过 → `offline` |
| pre-mask | **确认案例稿并完工后**后端自动 | 发布前预览核对 | — |

**废止主路径**：「公示合规驳回 → 用户撤回 → 商家改相册」。

**实现约束**：`assertGeoPublishable` 仅用于商家冷启动等路径；用户提交公开案例 **不得** 因 GEO block 硬拦（口径不变）。

### 4.5.7 案例快照分层与相册锁定（定稿 2026-07-09 · 卷九 · 文案修订 2026-07-18）

> 真源：`13_案例全流程快照与GEO分层开发计划.md` · 代码：`case-snapshot.schema.js` · `case-enrichment.schema.js`

| 层级 | 冻结时机 | 可变 | 读侧真源 |
| --- | --- | --- | --- |
| **CaseSnapshot** | 车主确认 **发布到公开网站** 时写入 `content_json.snapshot` | **否**（公开后任何人不得改） | H5 / Feed / 小程序公开案例 **只读 snapshot** |
| **GeoEnrichment** | 审核通过后可由运营维护 | **是**（SEO/FAQ/专题挂载；**不得**改 snapshot 事实与数字） | 页顶摘要、FAQ、TDK 等 **优先 enrichment** |
| **相册 live 数据** | 闸门 A 通过后商家不可 save | 撤回发布后可再编辑 | **仅** 商家/车主私密查看；**不** 作为已公开案例真源 |

**锁定规则**：

1. **闸门 A 通过** 或 **车主已提交发布** → 商家 `save` / `complete` → **409**（文案：须车主撤回后方可改）。  
2. **闸门 B 驳回** → **不解锁**商家；用户重试脱敏/改评价。  
3. **用户撤回** → `publicCase` → `offline` 留痕；相册解锁；可再发布 → `snapshotVersion++` + 强制 `pending_review`。  
4. **商家 299 内容优化**（可选）：仅 **完工后 + 发布前**；写 `albums.content_optimize_draft_json`，采用后更新节点说明；**发布确认时**一并打入 snapshot，**不**直接写 `public_cases`。

**隐私**：公开链仅 snapshot 内 **脱敏图 URL**（`publicView.media`）；禁止 live 相册原图回退到 H5/站外。

### 4.5.8 PublicView 上传分流（定稿 2026-07-12 · PV-REFORM）

> 真源：`14_PublicView与隐私改造计划.md`

| 原则 | 说明 |
| --- | --- |
| **单上传** | 商家每阶段仍一个「添加照片」；系统按阶段策略 + `publicGate` 写入 `album_images.visibility` |
| **stage_1 / stage_3 图** | 永远 `private`（接车、报价单仅留档） |
| **stage_2/4/5/6** | gate 通过 → `public`；含敏感信息 → `private` + 提示（不阻断 save） |
| **stage_3 文本** | `note` + `planParts` + `planAmount` → PublicView 文字；报价单图不进公开 |
| **OCR** | 仅 **商家端辅助填表**（私密）；覆盖 **报价单 / 定损单 / 结算单**；触发前须商家确认提交阿里云 OCR；生产默认关闭 VL 读原图；单据原图不上网 |
| **media 上限** | 取消固定 4 张；软上限 30 张；**发布预览** = PublicView WYSIWYG |

**文案体检**：商家 save/complete 时规则层打分 + 建议（PII / 绝对化用语等）；不自动改稿。

### 4.5.9 公域知识包 · 文案归属 · 藏价 · 相册教练（定稿 2026-07-22）

> 真源：[`15_公域知识包与相册教练规则引擎.md`](./15_公域知识包与相册教练规则引擎.md) · 任务：`00_开发计划.md` §7.11

| 原则 | 说明 |
| --- | --- |
| **私域全量 / 公域精选** | 私域：六阶段全量留档；公域：商家案例正文 + 脱敏关键帧（约 5～8）+ 可选车主评价 |
| **文案归属** | 案例正文 = **商家服务内容**（类电商商品详情）；车主**不可改文案**，仅可**发布评价**并决定是否上公开网站 |
| **公域藏价** | H5 / GEO / 社交媒体长文 / 朋友圈文案 **一律不写金额**；无须「费用以门店为准」类套话；`planAmount` **仅私域**可见 |
| **完工生成** | **一稿定稿**：点完工先进案例预览；默认按节点文字+公开配图**规则拼稿**（不调大模型）；右上角可选「AI 润色」（文字-only，图不进模型）；商家改定后确认 = 最终案例稿；确认后才真正完工并触发公开配图脱敏 |
| **车主预览** | 严格只读呈现商家**确认稿**（文案不可改）+ 脱敏配图（可删不可加）；公示的是案例文案，不是整本相册 |
| **导出** | 同一确认稿可派生：商家复制发自媒体、车主复制发朋友圈/社交媒体；**平台不代发** |
| **朋友圈** | **不**因公开硬拦而阻止分享；文案仍不写金额 |
| **相册教练** | 规则引擎按**服务类型**提示：要拍什么、别拍什么、备注怎么写；域内规则，**不**用外部大模型看原图教练 |
| **AI 出域** | 未发布前禁止原图进外部多模态；外网只吃脱敏/scrub 数据；详见 15 §3 |
| **确认语文案** | 鼓励式（体验官/避坑指南 + 一键下架承诺）；合规窄授权写在副文案/法律专页；见 15 §13 |

授权确认语须写清：精选脱敏节点照 + 商家脱敏案例说明 +（可选）评价；**不含**个人信息、金额、完整工单；**不含**将全量原图交第三方训练。主界面用荣誉与控制权话术，见 `15_…` §13。

## 9. Phase 1 验收（摘要）

1. 商家可创建 **服务相册** 并完成六阶段记录（无 orderId）。  
2. 可录入配件类型与价格，**不发起**车主配件确认、**不阻断**完工。  
3. 可邀请已关联车主查看。  
4. 车主可查看服务相册；完工后可 **发给微信 / 复制去社交媒体**（**仅脱敏**）；可选 **发布到公开网站**（预览 → 留痕 → 闸门 B 风险分流过审 → 上线；含门店名）。  
5. 公开案例 Tag 符合 Phase 1；公开网站内容须强制脱敏；H5/Feed 只读 `publicView`；未点发布则不上网。
