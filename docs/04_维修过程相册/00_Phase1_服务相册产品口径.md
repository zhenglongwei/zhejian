## 2. 核心对象：服务相册

| 维度 | Phase 1 定义 |
| --- | --- |
| **定位** | 门店为线下服务创建的 **过程记录相册** |
| **创建** | 商家主动创建；**不绑定** `orderId` |
| **主要用途** | **过程记录与争议证据**（记录了什么、何时记录） |
| **次要用途** | 车主查看；授权脱敏后成为公开案例 / GEO 内容 |

> 对内：**每一次服务，都有据可查** — Phase 1 强调可查，不要求配件/方案车主签字。

## 3. Phase 1 做什么 / 不做什么

### 3.1 Phase 1（当前 MVP）

| 能力 | 说明 |
| --- | --- |
| 六阶段记录 | 接车 → 检测 → 方案与参考价 → 配件录入 → 施工 → 交付 |
| 配件**录入** | 标准类型、价格、凭证图；**商家单方留痕** |
| 关联车主 | 手机号**必填**（车主扫码关联，商家不可代填）；未关联不可标记完工 |
| 标记完工 | **不阻断**；不要求配件/方案车主确认 |
| 公开授权 | 相册 `completed` 后，车主可 **授权公开**（MVP：勾选 + 文案快照 + 时间/IP 留痕） |
| 脱敏 + 审核 | `08_图片脱敏工具PRD.md` → 平台展示审核 → **H5 / 公众号 / 站外 GEO**（不含小程序案例 Tab） |

### 3.2 Phase 2（业务开展后再做）

- 配件 / 方案 **车主确认**（勾选或电子签名）  
- L2+ **阻断完工**  
- 合规检查清单强制、PDF 证据包、正式电子签名  
- 更完整履约工具  

详见 `10_配件告知确认规则.md`（Phase 2 启用部分）。

## 4. 公开授权三档

| 档位 | 说明 |
| --- | --- |
| **① 不公开** | 默认；须已关联车主手机号方可完工留档 |
| **② 匿名授权** | 脱敏后发布至 **H5 / 公众号 / 站外 GEO** |
| **③ 实名授权** | 脱敏基础上允许店名、车型等品牌字段 |

**唯一需要车主主动操作的「平台公示」环节**：**授权公示** + 留痕（Phase 1 勾选；Phase 2 电子签名）。  
**不属于授权公示**：车主把 **自己的** 服务相册/案例 **分享给好友、朋友圈或社交账号**（见 §4.5）。

## 4.5 授权公示 vs 用户自主分享（定稿 2026-05-28 · 渠道修订 2026-06-08）

> **产品原则**：「授权」的对象是 **辙见平台**，不是用户「分享给自己朋友」的前置条件。二者意图、渠道、脱敏规则 **分开定义**。  
> **双系统转型（2026-06-08）**：授权公示 = **站外** 公开展示（H5 / 公众号 / GEO）；小程序 = **服务相册工具**，**不作为** 公域案例发现/公示出口（详见 `docs/01_项目总览与业务架构/08_双系统转型开发计划.md`）。

### 4.5.1 对照

| 维度 | **授权公示** | **用户自主分享** |
| --- | --- | --- |
| **谁发布** | **辙见** 在 **站外** 公开展示位发布 | **用户本人** 发给指定对象 |
| **典型渠道** | **H5 公开页、公众号文章、站外 GEO/SEO**（**不含** 小程序案例 Tab / 公域列表） | 微信好友、朋友圈、复制链接、其他社交 App |
| **是否需「授权公示」** | 是（授权平台发布 + 留痕） | **否**（内容为用户本人关联的服务相册） |
| **脱敏** | **强制**（平台对公开展示的法定义务） | **默认脱敏，用户可选原图**（用户权利；平台不强制） |
| **平台审核** | 须审核通过后才进入公示渠道 | **不经过** 平台案例审核通道 |
| **与 orderId** | 无关 | 无关 |

### 4.5.2 授权公示

- 含义：用户同意辙见将 **脱敏后** 的内容发布到 **站外公开展示位**（H5 内容站、公众号分发、可被搜索收录的 GEO 页面等；**不含** 小程序内公域案例 Tab / 全站案例列表）。
- 流程：`completed` → 用户选择档位（匿名/实名/不公开）→ 预览脱敏 → **授权公示** → 平台审核 → 自动发布至 **H5 / 公众号等站外渠道**（**无需** 用户再手工「发布」或「分享」）。
- 未授权公示的内容 **不会** 出现在 **H5 / 公众号 / 站外 GEO**；亦 **不会** 通过小程序主导航进入公域案例发现。

### 4.5.3a 小程序 · 公域只读访客态（2026-06-15）

与 §4.5 站外公示 **并行、不替代**：

| 用户态 | 小程序内可读 |
| --- | --- |
| **未绑定**任何服务相册（含未登录） | 搜一搜 / 分享深链直达的 **已审核公开** 门店/案例/服务详情（`public_guest`） |
| **已绑定**相册 | 仅相册 **关联门店** 及该店公开内容；跨店浏览拒绝 |
| 商家分享 / 单店上下文 | 按 `storeId` 隔离展示（与 DS-A-09～11 一致） |

**禁止**：在小程序首页/Tab 增加跨店发现入口；本规则 **不** 改变「授权公示主渠道仍为 H5/公众号」。

### 4.5.4 用户自主分享（私人传播）

- 含义：关联车主把 **自己的** 服务相册或案例 **发给朋友、朋友圈、社交账号或网络**，与是否授权公示 **无关**。
- 入口：**我的服务相册** 详情（完工后即可，**不必** 先完成授权公示或平台审核）。
- **脱敏选项**（分享前）：
  - **默认**：脱敏后分享（推荐，平台仍提供一键脱敏预览）。
  - **可选**：原图分享 — 用户明确选择后可用原图；平台须展示 **隐私风险提示** 并 **留痕**（时间、选项），**不得强制** 仅脱敏、**不得拦截** 用户选择原图。
- **分享入口**：相册详情 **一个「分享」按钮** → **半屏 ShareSheet**（见 `07_维修相册查看页.md` §17.4）。
- **复制分享链接**（私人）：`completed` 即可，**不必授权公示**；链接带 token，**默认脱敏、可选原图**（与转发好友共用 mode）。
- **复制公示网页链接**（平台）：仅 **已授权公示且审核通过** 后出现；**强制脱敏**。
- 私人分享链接/卡片 **不** 等同于平台公示；**不** 自动进入 **H5 / 公众号等公示渠道**，除非用户 **另行** 完成授权公示且审核通过。
- 合规：禁止诱导奖励文案（见主 PRD §10）；私人分享 **不做**「分享有奖」类运营。

### 4.5.5 文档与实现约束

- 文案禁止使用「先授权才能分享」「分享即公开」等混同表述。
- **A-MASK / 脱敏工具**：平台 **公示链路**（H5 / 公众号 / 站外 GEO）仍 **强制脱敏**；**用户私人分享** 走 §4.5.3 可选脱敏，两套规则 **不可混用**。
- **U-CASE-06**（分享与授权解耦）已落地；卷六 **DS-A-07 / DS-D-05** 负责小程序案例 Tab 隐藏与下线，与本节渠道定义一致。
- **运营审核台**：审核通过后才可进入站外公示渠道；审核员 **仅查看脱敏图 + OCR 摘要**，不展示原图（不变）。

### 4.5.6 双闸门 · 留档质量 vs 合规 vs 脱敏（定稿 2026-07-08）

| 维度 | **商家** | **车主** | **平台** |
| --- | --- | --- | --- |
| **闸门 A**（完工后） | 合法合规；A 驳回→改相册 | 见「门店已提交，内容待您确认」（A 通过后） | 自动规则+抽检；**不审** pre-mask 实现 |
| 六阶段 / 过程图 | **义务**（不阻断完工） | **权利**（监督）；不对完整度负责 | A/B **均不** 因缺节点 alone 驳回用户公示 |
| **闸门 B**（授权公示后） | — | 脱敏预览、评价/配图、授权档 | 脱敏完整性 + **用户侧内容** |
| B 驳回 | **不解锁**改相册 | **重试/手工脱敏、改评价**；须见驳回原因 | 合规类问题应在 A 已拦截 |
| pre-mask | 完工后后端自动（**不变**） | 授权前预览核对 | — |

**废止主路径**：「公示合规驳回 → 用户撤回 → 商家改相册」。

**实现约束**：`assertGeoPublishable` 仅用于商家冷启动等路径；用户 `POST …/public-case` 不得因 GEO block 硬拦（口径不变）。

### 4.5.7 案例快照分层与相册锁定（定稿 2026-07-09 · 卷九）

> 真源：`13_案例全流程快照与GEO分层开发计划.md` · 代码：`case-snapshot.schema.js` · `case-enrichment.schema.js`

| 层级 | 冻结时机 | 可变 | 读侧真源 |
| --- | --- | --- | --- |
| **CaseSnapshot** | 车主提交授权公示时写入 `content_json.snapshot` | **否**（公示后任何人不得改） | H5 / Feed / 小程序公开案例 **只读 snapshot** |
| **GeoEnrichment** | 审核通过后可由运营维护 | **是**（SEO/FAQ/专题挂载；**不得**改 snapshot 事实与数字） | 页顶摘要、FAQ、TDK 等 **优先 enrichment** |
| **相册 live 数据** | 闸门 A 通过后商家不可 save | 撤回授权后可再编辑 | **仅** 商家/车主私密查看；**不** 作为已公示案例真源 |

**锁定规则**：

1. **闸门 A 通过** 或 **车主已授权** → 商家 `save` / `complete` → **409**（文案：须车主撤回公示后方可改）。  
2. **闸门 B 驳回** → **不解锁**商家；用户重试脱敏/改评价。  
3. **用户撤回** → `publicCase` → `offline` 留痕；相册解锁；可再授权 → `snapshotVersion++` + 强制 `pending_review`。  
4. **商家 299 内容优化**（可选）：仅 **完工后 + 授权前**；写 `albums.content_optimize_draft_json`，采用后更新节点说明；**授权时**一并打入 snapshot，**不**直接写 `public_cases`。

**隐私**：公开链仅 snapshot 内 **脱敏图 URL**；禁止 live 相册原图回退到 H5/站外。

## 9. Phase 1 验收（摘要）

1. 商家可创建 **服务相册** 并完成六阶段记录（无 orderId）。  
2. 可录入配件类型与价格，**不发起**车主配件确认、**不阻断**完工。  
3. 可邀请已关联车主查看。  
4. 车主可查看服务相册；完工后可 **自主分享**（默认脱敏、可选原图）；可选 **授权公示**（与分享无关）。  
5. 公开案例 Tag 符合 Phase 1；平台公示内容须强制脱敏。

## 10. 方案报价与价格展示（Phase 1）

| 场景 | 展示方式 | 录入/计算 |
| --- | --- | --- |
| 商家编辑 · **方案与报价** | **固定报价**（单行金额） | 商家填写 `planAmount`，以线下确认为准 |
| 商家查看 / 车主查看 **私密服务相册** | **固定报价** | 读取 `planAmount` |
| **公开案例 · 已获得车主授权** | **固定报价**（真实方案价） | 读取 `planAmount`；展示于 **H5 / 公众号 / 站外 GEO**（**不含** 小程序案例 Tab） |
| **公开案例 · 未获车主授权** | ~~参考区间~~ | **已取消**：无关联车主不可提交平台公开；公开须车主授权公示 |

说明：

- 服务**商品**仍沿用一口价 / 参考区间 / 仅预约 / 事故车四型；**服务相册内方案报价**为门店本次留档价，Phase 1 **仅支持固定值录入**，不在商家端展示区间输入。
- 公开页系统区间仅用于隐私保护，不代表平台成交价；区间由服务端/前端共用算法按案例 ID 确定性生成，避免固定比例被批量反推。
- **站外公示页**须附带 `displayDisclaimer`（§9.4.1）及区间价「实际费用以门店检测结果为准」类说明；事故车场景禁止线上报价暗示（与 `price-compliance-check` 一致）。
