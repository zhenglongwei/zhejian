## 2. 模块定位

案例生成规则用于将服务相册结构化转化为公开案例内容。

生成对象包括：

1. 标题；
2. 摘要；
3. 正文；
4. 节点说明；
5. 图片说明；
6. FAQ；
7. SEO 标题；
8. SEO 描述；
9. AI 可引用摘要；
10. Schema 结构化数据，P1。

---

## 3. 生成来源

案例内容可来源于：

### 3.1 用户授权案例（服务相册）

来源字段：

1. 服务相册六阶段与配件摘要（脱敏）；
2. 服务项目、门店、城市；
3. 车型信息；
4. 商家填写的维修摘要；
5. 价格区间规则（非成交价）；
6. **不含**平台订单号、支付金额。

来源字段：

1. 商家填写的基础信息；

3. 门店信息；
4. 车型信息；
5. 图片节点；

## 4. 生成原则

### 4.1 不泄露隐私

生成内容不得包含：

1. 用户真实姓名；
2. 手机号；
3. 车牌号；
4. VIN；
5. 身份证号；
6. 微信号；
7. 精确地址；
8. 保险单号；
9. 报案号；
10. 实际支付金额，除非后续有明确授权策略。

---

### 4.2 不夸大宣传

不得生成：

1. 保证修好；
2. 永不复发；
3. 全网最低价；
4. 事故车保证全赔；
5. 原厂认证，除非有资质；
6. 绝对化用语；
7. 虚假承诺。

---

### 4.3 价格展示规则（V2.0 · Phase 1 修订 · **2026-07-22 公域藏价**）

| 场景 | 展示 | 数据来源 |
| --- | --- | --- |
| 商家编辑/查看服务相册 | **固定报价** | 商家录入 `planAmount` |
| 车主查看私密服务相册 | **固定报价** | `planAmount` |
| 公开案例 · **已发布且过审** | **不展示金额** | 公域知识包藏价；见 `15_…` §5 |
| 社交媒体长文 / 朋友圈文案 | **不写金额** | 无须费用免责套话 |
| 公开案例 · **未发布** | — | 不上公开网站 |

不允许公开展示：

1. 成交价 / `planAmount` / 价税数字；
2. 保险赔付金额；
3. 定损单完整金额；
4. 可关联个人身份的报价单。

**废止（2026-07-22）**：已获车主授权的公开案例「直接展示 `planAmount`」。

### **8.2 通用节点说明**


| 节点    | 默认说明                    |
| ----- | ----------------------- |
| 维修前状态 | 展示车辆维修前的状态，便于对比维修前后的变化。 |

## **10. FAQ 生成规则**

### **10.1 FAQ 数量**

每个公开案例建议生成：

```
至少 3 个 FAQ
最多 6 个 FAQ

```

---

### **10.2 FAQ 来源**

FAQ 来源优先级：

1. 服务项目 FAQ 模板；
2. 相册模板 FAQ；
3. 平台通用 FAQ；
4. AI 生成补充 FAQ，P1。

---

### **10.3 FAQ 结构**

```
Q：问题
A：回答

```

回答要求：

1. 客观；
2. 不绝对化；
3. 不承诺效果；
4. 不暴露隐私；
5. 不诱导虚假低价；
6. 可说明“需以实际检测为准”。

---

### **10.4 FAQ 示例**

#### **刹车片案例**

```
Q：刹车片多久需要更换一次？
A：刹车片更换周期与车型、驾驶习惯和路况有关，建议结合实际磨损厚度和门店检查结果判断。

Q：刹车异响一定是刹车片问题吗？
A：不一定。刹车异响可能与刹车片材质、刹车盘状态、异物或安装情况有关，需要检查后确认。

Q：刹车盘需要和刹车片一起更换吗？
A：不一定。若刹车盘磨损严重、变形或有明显沟槽，可能需要同时更换。

```

---

#### **钣喷案例**

```
Q：钣喷修复后会有色差吗？
A：色差与车辆颜色、漆面老化程度、调色和施工工艺有关，门店会尽量进行颜色匹配和过渡处理。

Q：小划痕需要整面喷漆吗？
A：不一定，需要根据划痕深度、位置、颜色和修复效果要求判断。

Q：钣喷修复一般需要多久？
A：维修时间会受到损伤面积、是否拆件、喷漆工序和天气等因素影响，具体以门店确认为准。

```

### **19.3 快照层 vs 提炼层（定稿 2026-07-09 · 卷九）**

> 计划：`13_案例全流程快照与GEO分层开发计划.md` §3 · 字段：`docs/11_数据结构与状态机/07_案例快照与提炼层字段.md`

| 内容类型 | 写入域 | 生成时机 | 运营可改（公示后） |
| --- | --- | --- | --- |
| 标题/摘要/正文/节点/过程图（脱敏 URL）/方案价 | **`content_json.snapshot`** | 用户授权提交时冻结 | **否** |
| 页顶 AI 摘要、SEO TDK、FAQ、专题挂载、聚合 N= | **`enrichment_json`** | 审核通过 / 挂载 / 聚合任务 | **是**（不改 snapshot 数字与事实） |
| 商家 299 润色草稿 | **`albums.content_optimize_draft_json`** | 授权前商家发起 | 采用后写相册；授权时 merge 进 snapshot |

**生成器规则**：

1. `buildCaseDraft` / 模板生成 → 授权瞬间写入 **snapshot**（非 live merge）。  
2. `approveAdminCase`：**有 snapshot 时不得**从 live album 重建正文/节点。  
3. LLM / 规则聚合 FAQ、IGAIN 统计句 → **仅 enrichment**；须含样本量 N=（有数据时）。  
4. 提炼层措辞可变，**价格、车型、节点事实**须与 snapshot 一致。

## **24. 验收标准**

1. 用户授权案例可自动生成标题和摘要；
2. 历史案例可自动生成标题、摘要和正文；
3. 每个公开案例至少有 3 个 FAQ；
4. SEO 标题和描述可生成；
5. AI 可引用摘要不包含隐私；
6. 生成内容不得包含车牌、手机号、VIN；
7. 价格展示必须为区间或参考说明；
8. 商家编辑内容后需重新风控检测；
9. 平台可修改生成内容；
10. 内容修改有版本记录；
11. 审核通过后才可公开发布。
