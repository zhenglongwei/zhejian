**说明**

- 「底盘异响 / 胶套」目前是**教练扩展包**，不是独立相册模板；后续可挂在 `default` 关键词或扩展模板，不纳入本次「模板全覆盖」硬清单，可另列扩展包。
- 平台服务目录（可上架项）少于模板数：保养、刹车、钣喷、电瓶、事故、自定义；轮胎/空调/大保养主要靠**创建相册时关键词 → 模板**。

**教练匹配原则（改规则时遵守）**

1. 优先 `templateId` 精确匹配规则包；  
2. 其次服务名关键词；  
3. 未命中 → `default` 中性包；  
4. **禁止**用某一类维修的专属示例充当「通用」备注样例。

## 2. 映射方法（各类模板通用）

每类服务、每个阶段填四列：

| 列 | 含义 |
| --- | --- |
| 车主关心 | 到店前/验收时最怕什么、最想确认什么 |
| AI / GEO 可引用点 | 公开案例、专题、FAQ 能沉淀的事实（无金额、无绝对化承诺） |
| 建议拍 | `shoot_prefer`；标注公域价值 / 仅留档 |

| 编码 | 取向 |
| --- | --- |
| `avoid_pitfall` | 现象 + 判断依据 + 方案对比（反对过度维修叙事） |
| `standard_5s` | 防护、工序、环境规范（打消「糊弄施工」顾虑） |
| `liability` | 拆件确认、材料说明、质保边界 |

## 3. 钣喷样板（`body_paint`）

### 3.1 双关注点总表（跨阶段）

#### 车主关心（调研 + 站内 FAQ 对齐）

| 主题 | 典型问法 | 主要落点阶段 |
| --- | --- | --- |
| 色差 | 修完会不会花？金属漆/珍珠漆更怕什么？ | 检测、方案、施工、交付 |
| 该不该整面喷 | 小划痕一定要整面吗？补漆 vs 钣金喷漆？ | 检测、方案 |
| 工艺是否缩水 | 有没有打磨/中涂/烤漆？是不是快干催熟？ | 施工、交付 |
| 拆不拆件 | 门把手饰条拆喷还是遮蔽喷？ | 方案、施工 |
| 材料 | 用什么漆？水性/油性？品牌？ | 配件、方案 |
| 工期与养护 | 多久能取？多久能洗车？ | 方案、交付 |
| 质保 | 起泡开裂管不管？书面说明？ | 交付 |
| 保值/痕迹 | 能否尽量保留原厂漆？ | 检测、方案 |
| 隐私 | 整车照会不会被人认出车牌？ | 全阶段避忌 |

站内已有口径（须继续用，勿自造对立说法）：

- `02` §11.5 / `geo-faq-templates` BODY_PAINT：补漆 vs 钣金、色差、到店确认方案  
- H5 `body-paint-repair`：费用到店确认、色差与车龄工艺相关  

#### AI / GEO 关心（可引用、可检索）

| 主题 | 希望案例里出现的「证据句」 | 主要落点 |
| --- | --- | --- |
| 部位 + 损伤类型 | 「右前门中度划痕，未见钣金变形」 | 接车/检测 |
| 工艺路径选择 | 「建议局部补漆，不整面喷」及判断依据 | 检测/方案 |
| 工序完整度 | 打磨 → 腻子/中涂 → 调色比对 → 喷涂 → 烤漆/抛光 | 施工 |
| 5S / 环境 | 遮蔽防护、无尘/烤房相关可见点（勿夸大） | 施工 |
| 验收动作 | 自然光色差复查、橘皮/光泽目视 | 交付 |
| 边界话术 | 色差受车龄与原漆老化影响；不写「100% 无色差」 | 交付/FAQ |

公域配图取向：损伤近景、工序细节、完工近景、色差比对过程；**少拍整车全景带车牌**。

### 3.2 六阶段映射（商家端教练文案草案）

> 节点标题以模板为准：接车记录 / 检测记录 / 修复方案 / 配件·材料凭证 / 施工过程 / 完工结果。  
> 「公开」列：适合进公域关键帧的倾向；最终仍受闸门与车主删图约束。

#### stage_1 接车记录

| 维度 | 内容 |
| --- | --- |
| 车主关心 | 伤在哪、多严重、到店时外观有没有被误伤 |
| AI 点 | 部位定位（哪一面/哪块板件） |
| 建议拍 | **损伤部位远景，一处一图**（与模板 meta 一致）；知悉多为留档，含车牌整车慎公开 |
| 别拍（叠加通用） | 清晰车牌特写、人脸、钥匙、他车可辨识 |
| 备注骨架 | 到店诉求一句话 + 损伤部位清单 |
| **示例（钣喷）** | 「右前门、右后视镜壳划痕到店；外观未见新增磕碰。」 |
| geo_angle | —（本阶段多 `always_private`，提示偏留档完整） |

#### stage_2 检测记录

| 维度 | 内容 |
| --- | --- |
| 车主关心 | 要不要钣金？能不能只补漆？伤到结构没有？ |
| AI 点 | 损伤深度/类型判断 +「排除了什么」 |
| 建议拍 | **各损伤点近景/特写**；多角度（侧/斜）；顺序与完工对比组一一对应 |
| 备注骨架 | 现象 + 检查手段 + 结论（钣金与否 / 补漆范围） |
| **示例（钣喷）** | 「右前门中度划痕，漆下未见钣金变形；建议局部补漆，不必整面喷。」 |
| geo_angle | `avoid_pitfall` |
| 强烈建议 | 有检测结论备注（完工清单 strong） |

#### stage_3 修复方案

| 维度 | 内容 |
| --- | --- |
| 车主关心 | 拆不拆件、大概多久好、工艺路径（钱在私域字段，公开不写金额） |
| AI 点 | 修复方式（钣金/喷漆/补漆/更换）+ 是否拆件 + 颜色类型（可选） |
| 建议拍 | 报价单/方案图**仅留档**；公开靠文字方案要点 |
| 备注骨架 | 方案要点 + 为何不采用过度方案 +「金额只填字段不进公开稿」 |
| **示例（钣喷）** | 「局部补漆不拆件；珍珠漆需调色过渡；工期视天气与烤漆排程，以门店沟通为准。」 |
| geo_angle | `avoid_pitfall` |

#### stage_4 配件/材料凭证

| 维度 | 内容 |
| --- | --- |
| 车主关心 | 漆与辅料是否说得清；有无批次/品牌可核对 |
| AI 点 | 材料品类说明（色漆/清漆/腻子等），**不冒充平台鉴定真伪** |
| 建议拍 | 漆罐/辅料包装与标签特写；含个人信息的进货单整页仅留档 |
| 备注骨架 | 材料名称 + 用途一句；验真说明用中性句，**勿写死单一品牌（如海拉）** |
| **示例（钣喷）** | 「本次使用色漆+清漆；包装批号已拍照留档，可按品牌官网渠道核对。」 |
| geo_angle | `liability`（材料告知边界） |

#### stage_5 施工过程

| 维度 | 内容 |
| --- | --- |
| 车主关心 | 工序有没有缩水；环境是否规范 |
| AI 点 | 可见工序链（对齐证据清单：钣金点 → 腻子/中涂/遮蔽 → 面漆/抛光） |
| 建议拍 | 防护/遮蔽、打磨或钣金点、调色比对过程、喷涂环节（**避开车牌**）；烤房/无尘相关可见点可拍但勿绝对化宣传 |
| 备注骨架 | 关键工序 + 质检关注点（避免「100% 无色差」） |
| **示例（钣喷）** | 「已打磨并完成调色比对；按工序喷涂，交车前自然光复查色差。」 |
| geo_angle | `standard_5s` +（可选）`avoid_pitfall` |
| 过程清单对齐 | `bp_metal` / `bp_coating` / `bp_paint` |

#### stage_6 完工结果

| 维度 | 内容 |
| --- | --- |
| 车主关心 | 色差、平整度、养护注意、质保 |
| AI 点 | 修复局部完工近景 + 验收动作描述 |
| 建议拍 | **与检测同序同角的完工近景**；色差检查（自然光对比邻板）；少拍整车招摇全景 |
| 备注骨架 | 验收项 + 养护/质保一句（无绝对承诺） |
| **示例（钣喷）** | 「自然光下过渡区复查；漆面养护期内避免高压水枪；质保范围以门店说明为准。」 |
| geo_angle | `standard_5s` + `liability` |
| 强烈建议 | 至少 1 张交付/复查图（公开价值高） |

### 3.3 钣喷 · 教练规则包字段草案（供后续灌入）

```text
RulePack.id = body_paint
label = 钣喷修复
match: templateId=body_paint；关键词 钣金/喷漆/划痕/凹陷/补漆/钣喷
geoPyramidHint = standard_5s   # 塔身信任为主；避坑（过度整喷）为辅

stages:
  stage_1: note_hints + shoot_prefer(远景定位)     # 覆盖通用，禁止底盘示例
  stage_2: note_hints + shoot_prefer(近景/多角) + geo_angle[avoid_pitfall]
  stage_3: note_hints(工艺路径) + geo_angle[avoid_pitfall]
  stage_4: note_hints(材料中性) + shoot_prefer(包装标签)
  stage_5: note_hints + shoot_prefer(遮蔽/工序/调色) + geo_angle[standard_5s]
  stage_6: note_hints + shoot_prefer(完工近景/色差查) + geo_angle[standard_5s, liability]

completeChecklist（钣喷侧重）:
  - stage_2 检测结论备注（strong）
  - stage_3 方案要点备注（strong，无金额）
  - stage_6 完工近景图（strong）
  - stage_5 至少一类工序图（tip）
```

### 3.4 合规红线（钣喷文案自检）

- 不写公域金额；不暗示「线上估价即成交价」  
- 不写「100% 无色差 / 与原厂一模一样」  
- 不羞辱式对比同行；用「建议局部补漆、避免过度整喷」叙事  
- 公开路径继续避车牌、人脸、钥匙
