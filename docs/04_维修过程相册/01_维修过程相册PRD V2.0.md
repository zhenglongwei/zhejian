## **2. 产品定位**

服务相册 **不是** 普通图片上传，而是：

> **Phase 1：服务相册 = 六阶段过程留档 + 配件/价格录入 + 照片证据 + 时间线**  
> **公开案例 = 车主授权 + 脱敏 + 平台展示审核**（与私密留档严格分离）

**争议证据**：若车主发现配件或价格与事先约定不一致，档案中的录入内容与时间戳可作为凭证；Phase 1 **不要求**配件/方案签字，Phase 2 可加强车主确认留痕。

同时承担五个角色：

| 角色 | Phase 1 说明 |
| --- | --- |
| **商家风控工具** | 过程与配件录入留痕、凭证图；Phase 2 + PDF 证据包 |
| **车主透明档案** | 查看六阶段留档；**仅授权公开**需主动操作 |
| **门店营销素材源** | 脱敏授权后 → GEO / 咨询线索 |
| **平台内容生产** | 公开案例、门店页、GEO 页 |
| **合规凭证库** | 争议时可导出时间线与录入记录（Phase 2 增强） |

**平台不提供**维修质量担保；相册内容由门店上传，平台提供记录、展示、脱敏与审核工具。

## **3. 设计原则**

### **3.1 商家：极简、有效、能获益**

维修商是相册的直接用户；工具须 **极简化、高效、能直接带来收益**，否则难以持续使用。

**轻量化助手能力（P0/P1）**

- **分阶段上传**：接车 → 检测 → 方案 → 配件 → 施工 → 交付，按阶段引导而非一次性填表；
- **扫码/手机号创建**：通过维修单号、二维码或车主手机号快速创建并关联相册；
- **模板化标注**：按服务模板自动生成节点说明（如「更换刹车片完成」）；
- **待办清单（Phase 1）**：待上传节点、待公开授权；**Phase 2**：待配件确认、待车主签字
- **检索**：按相册编号、日期、脱敏车牌检索历史记录（P1）；
- **对外 API**：与门店现有工单系统对接（P1）。

**直接收益**

- **减少纠纷**：配件确认卡 + 新旧件/包装/编码照片 + 车主确认留痕；
- **专业形象**：生成车主可查看链接/交付摘要，可脱敏分享；
- **GEO 激励**：案例完整度、相册使用率影响门店与案例推荐权重（非交易排序）。

### **3.2 车主：看得懂、信得过（Phase 1）**

- 六阶段时间线 + **翻译层**；
- **授权公开**前档案为私密；授权须主动勾选 + 留痕（Phase 2 电子签名）；
- 相册不等于公开。

**Phase 2 增加**：待确认事项置顶、非原厂件确认页、方案确认。

### **3.3 风险分级（Phase 1 建议 / Phase 2 阻断）**

| 等级 | 典型场景 | Phase 1 | Phase 2 |
| --- | --- | --- | --- |
| **L1** | 洗车、滤芯 | 可选拍照 | 同左 |
| **L2** | 刹车、轮胎、钣喷 | **建议**录入配件类型 | 非原厂须车主确认，未确认不可完工 |
| **L3/L4** | 发动机、事故车 | **建议**完整留档 | 方案+配件确认、证据包 |

**Phase 1**：均为 **建议不强制**，以降低商家负担；录入仍作争议证据。

### **3.4 公开必须授权 + 脱敏 + 展示审核**

公开案例须：**车主授权**（档位见 `00_Phase1_服务相册产品口径.md` §4：不公开 / 授权公示）+ **统一脱敏** + 平台展示审核。  
**Phase 2**：无车主时的商家责任确认路径另行定义。

展示审核 **不代表** 对维修质量、配件真伪或价格合理性背书。

### **3.5 私密相册 vs 公开案例严格分离**

| 私密服务相册 | 公开案例 |
| --- | --- |
| 完整过程、配件确认、价格、签名 | 脱敏摘要、部分授权图 |
| 车主、门店、争议处理 | 搜索、分享、GEO |
| 禁止未授权对外展示 | 禁止完整车牌/VIN/结算单/车主签名 |

## **4. 六阶段证据链**

所有 **服务相册**（非纯历史案例）建议按六阶段组织：

```
阶段一 接车记录     → 证明进店状态
阶段二 检测诊断     → 证明为什么要修、为什么要换
阶段三 方案与报价   → 可确认的维修方案（非口头报价）
阶段四 配件告知确认 → 核心合规节点（配件确认卡）
阶段五 施工记录     → 证明实际施工（新旧件、包装、编码）
阶段六 完工交付     → 验收、结算说明、交付确认
```

各阶段字段与节点细则见 `02_相册模板与节点规则.md`、`10_配件告知确认规则.md`。

## **7. 配件告知（Phase 1 录入 / Phase 2 确认）**

**Phase 1**：每个涉及更换的配件，商家 **录入** 标准类型、价格、凭证图，作为留档与争议证据；**不发起**车主确认。

**Phase 2**（见 `10_配件告知确认规则.md`）：非原厂件须车主确认；L2+ 阻断完工。

标准类型：`原厂件` | `同质配件` | `品牌件` | `副厂件` | …（禁止模糊词）。

## **9. 照片分类**

### **9.1 普通过程照片**

进店、环境、施工过程、完工展示——主要服务体验与透明度。

### **9.2 关键凭证照片**

包装、编码、新旧件对比、损伤部位、报价确认截图、车主确认、验收——用于争议与合规。

要求：**不可随意删除**；删除须管理员权限且 **留痕**；须可纳入证据包。

水印（P0 基础）：订单编号、脱敏车辆、门店、拍摄时间、节点、拍摄人。详见 `05_图片视频上传规范.md`。

## **10. 用户角色与场景**

### **10.1 车主**

1. 通过链接/「我的服务相册」查看；
2. 处理 **待确认**（方案、非原厂件）；
3. 查看六阶段时间线；
4. 授权或拒绝脱敏公开；
5. 分享 **已授权脱敏** 内容。

**不做**：平台订单评价、确认完工（交易履约）。

### **10.2 商家**

1. 创建服务相册、按阶段上传；
2. 录入配件类型、发起车主确认；
3. 查看合规检查清单；
4. 导出证据包（授权后）；
5. 上传历史案例；
6. 邀请用户授权公开。

### **10.3 平台审核**

脱敏与展示规范审核；隐私风险复核；案例下架。不做维修质量鉴定。

## **11. 授权公开流程（摘要）**

```
服务相册 completed
→ 系统生成案例草稿 + 预脱敏（后台，见 08_图片脱敏工具PRD）
→ 用户进入授权页 / 脱敏工作台
→ 用户同意 / 拒绝 / 手动打码后同意
→ 平台展示审核
→ 发布 H5 + GEO 索引
```

授权页须说明：公开内容为门店上传；平台展示审核不等于质量背书；默认不展示实际支付金额。

## **12. 前端页面（概要）**

### **12.1 车主端 · 服务相册页**

1. **待确认条**（固定顶部）：「你有 N 项待确认」+ 立即处理；
2. 六阶段进度条；
3. 配件确认卡列表；
4. 时间线 + 图片；
5. 费用/质保说明（门店填写）；
6. 平台说明（非质量担保）；
7. 授权公开入口；
8. 联系门店。

### **12.2 商家端 · 相册编辑页**

1. 门店/服务/用户关联；
2. 六阶段 Tab + 待办；
3. **合规检查清单**（接车、检测、方案、配件、新旧件、验收）；
4. 配件录入与「发起车主确认」；
5. 图片上传（节点 + 水印）；
6. 保存 / 邀请车主查看 / 申请公开（须用户授权）。

### **12.3 商家端 · 工作台待办**

```
待车主确认 N | 待上传凭证 N | 高风险订单 N | 即将交付未签字 N
```

## **15. 验收标准（Phase 1）**

1. 商家可 **独立创建** 服务相册（无平台订单）；  
2. 六阶段留档与配件录入可走通；  
3. **无** Phase 2 配件阻断；  
4. 车主可查看留档；  
5. 未授权档案不会公开；  
6. 公开页仅脱敏图 + 平台说明；  
7. 公开 Tag 为 **已授权 · 已脱敏 · 已审核**，不含「平台订单案例」。

**Phase 2 验收**见 `10_配件告知确认规则.md` §14。
