## **2. 流程总览（Phase 1）**

```text
商家创建服务相册（门店 + 服务 + 可选手机号）
→ 阶段一～六：接车、检测、方案与参考价、配件录入、施工、交付（商家留档）
→ 标记已完工
→ [可选] 邀请车主查看私密留档（只读，无需签字）
→ [可选] 车主授权公开（勾选 + 留痕；Phase 2 电子签名）
→ 脱敏工作台确认 → 平台展示审核
→ 发布 H5 公开案例 / GEO
```

**Phase 1 不含**：配件/方案车主待确认、L2+ 阻断完工。

## **3. 创建服务相册**

### **3.1 入口**

```
商家工作台 → 新建服务相册
```

快捷方式（P1）：扫码创建、从咨询线索一键创建。

### **3.2 字段**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| store_id | 是 | 门店 |
| service_id | 是 | 服务项目 |
| user_phone | **否** | 车主手机号；无则仅门店留档 |
| complexity_level | 建议 | L1–L4 |

可选：车辆品牌/车系/年款、里程、内部工单号。

### **3.3 系统动作**

1. 创建 `album`（status=`draft`）；  
2. 匹配相册模板，生成六阶段空节点；  
3. 生成分享路径（有手机号时可用于通知）；  
4. 可选：短信/订阅消息通知车主。

**不再**：监听平台订单、支付回调。

---

## **4. 六阶段操作（商家 · Phase 1）**

### **4.1～4.2 接车 / 检测**

同 V2.0：外观、里程、故障点、检测图、诊断结论。

### **4.3 方案与报价**

商家填写本次服务的 **固定参考报价**（`planAmount`，以线下确认为准）；**不在此阶段录入价格区间**。  
商家端、车主端查看私密服务相册时，均展示该固定报价。  
公开案例价格在 §7 与 `07_案例生成规则.md`：已获车主授权展示固定价；未获授权由系统计算参考区间。

**Phase 2**：L3/L4 维修方案车主确认。

### **4.4 配件告知（录入留档）**

对每个更换件 **录入**：

- 配件名称、品牌、**标准类型**、价格、凭证图；  
- **不**推送车主确认、**不**阻断完工。

### **4.5～4.6 施工 / 交付**

新旧件对比、包装编码、完工图、试车说明、质保说明。

## **5. 车主查看（Phase 1）**

### **5.1 入口**

- 门店分享链接 / 小程序路径；  
- 我的 → **我的服务相册**。

### **5.2 车主能力**

| 能力 | Phase 1 |
| --- | --- |
| 查看六阶段时间线 | ✅ |
| 查看配件录入与参考价 | ✅ |
| 授权公开 | ✅（完工后） |
| 确认配件/方案 | ❌ Phase 2 |

---

## **6. 状态流转（Phase 1 主路径）**

```text
draft
  ↓ 商家编辑/upload
in_progress
  ↓ 商家标记完工
completed
  ↓ 车主授权公开（可选）
pending_review（脱敏 + 展示审核）
  ↓ 审核通过
published
```

异常：`user_rejected`（拒绝公开）、`taken_down`。

**Phase 2 启用**：`pending_part_confirm` 及方案变更回退。

---

## **7. 授权公开与案例发布**

### **7.1 触发条件**

1. 档案 `completed`；  
2. 至少 1 张有效图片；  
3. 无进行中的隐私投诉。

**Phase 2 增加**：L2+ 配件车主确认已完成。

### **7.2 授权档位**

见 `00_Phase1_服务相册产品口径.md` §4：① 不公开 / ② 匿名 / ③ 实名。

### **7.3 脱敏与审核**

`service_pre_mask` → 用户 `service_authorize` 预览 → 平台展示审核 → 发布。

### **7.4 发布后 Tag**

**已授权 · 已脱敏 · 已审核**（不得使用「平台订单案例」）。

## **9. 消息通知（Phase 1）**

| 场景 | 接收方 | 文案方向 |
| --- | --- | --- |
| 档案创建/分享 | 车主 | 门店为你创建了服务相册，点击查看 |
| 档案已完工 | 车主 | 服务相册已更新，可查看 |
| 可授权公开 | 车主 | 是否愿意将本次记录脱敏公开为案例 |
| 案例已公开 | 车主/商家 | 脱敏案例已发布 |

**Phase 2 增加**：待确认配件、车主已确认等。

## **10. 接口建议（Phase 1）**

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/merchant/service-albums` | 创建档案 |
| GET | `/api/merchant/service-albums` | 商家列表 |
| GET | `/api/merchant/service-albums/{id}` | 商家详情 |
| POST | `/api/merchant/service-albums/{id}` | 保存留档 |
| POST | `/api/merchant/service-albums/{id}/complete` | 标记完工 |

**移除**：订单绑相册相关路径。

---

## **11. P0 验收（Phase 1）**

1. 商家可创建档案并完成六阶段留档；  
2. 可录入配件类型，**不发起**车主确认；  
3. 可标记完工，**无** L2+ 阻断；  
4. 车主可查看时间线；  
5. 授权 + 脱敏 + 审核后可发布；  
6. 公开 Tag 符合 Phase 1。
