## **2. 流程总览（Phase 1）**

```text
商家创建服务相册（门店 + 服务 + **车主扫码关联手机号**）
→ 阶段一～六：接车、检测、方案与参考价、配件录入、施工、交付（商家留档）
→ 标记已完工
→ [可选] 邀请车主查看私密留档（只读，无需签字）
→ [可选] 车主授权公开（勾选 + 留痕；Phase 2 电子签名）
→ 脱敏工作台确认 → 平台展示审核
→ 发布 H5 公开案例 / GEO
```

**Phase 1 不含**：配件/方案车主待确认、L2+ 阻断完工。

## **3. 创建服务相册**

### **3.1 入口**

```
商家工作台 → 新建服务相册
```

快捷方式（P1）：扫码创建、从咨询线索一键创建。

### **3.2 字段**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| store_id | 是 | 门店 |
| service_id | 是 | 服务项目 |
| user_phone | **是** | 车主手机号；须车主扫码关联，商家不可代填；未关联不可标记完工 |
| complexity_level | 建议 | L1–L4 |

可选：车辆品牌/车系/年款、里程、内部工单号。

### **3.3 系统动作**

1. 创建 `album`（status=`draft`）；  
2. 匹配相册模板，生成六阶段空节点；  
3. 生成分享路径（有手机号时可用于通知）；  
4. 可选：短信/订阅消息通知车主。

**不再**：监听平台订单、支付回调。

## **4. 六阶段操作（商家 · Phase 1）**

### **4.1～4.2 接车 / 检测**

同 V2.0：外观、里程、故障点、检测图、诊断结论。

### **4.3 方案与报价**

商家填写本次服务的 **固定参考报价**（`planAmount`，以线下确认为准）；**不在此阶段录入价格区间**。  
**2026-06-24 增强**：须上传 **详细报价表/说明图** → OCR 生成 **`planParts[]`** → 商家确认锁定（见 `02_相册模板` §3.5、`03/04_维修相册` §13）。

**Phase 2**：L3/L4 维修方案车主确认。

### **4.4 配件告知（录入留档 + 按目录上传）**

对每个更换件 **录入**（**按 `planParts` 逐项**，见 `02_相册模板` §3.6）：

- 配件名称、品牌、**标准类型**、**编码**（凭证图 OCR 或手工）、价格、凭证图；  
- **不**推送 Phase 2 车主确认、**不**阻断 Phase 1 完工；  
- 允许 **方案外增项**（`source: extra`），验真页单独展示。

```text
阶段三 报价表 OCR → planParts（锁定）
        ↓
阶段四 按 planParts 逐件上传 + 配件编码 OCR → partsJson
        ↓
车主   配件验真：planParts ↔ partsJson 对照留痕
```

### **4.5～4.6 施工 / 交付**

新旧件对比、包装编码、完工图、试车说明、质保说明。

## **5. 车主查看（Phase 1）**

### **5.1 入口**

- 门店分享链接 / 小程序路径；  
- 我的 → **我的服务相册**。

### **5.2 车主能力**

| 能力 | Phase 1 |
| --- | --- |
| 查看六阶段时间线 | ✅ |
| 查看配件录入与参考价 | ✅ |
| **配件验真**（对照方案与相册，自愿留痕） | ✅（`pages/album/part-verify`；满血需 `planParts`，见 §17.5） |
| 授权公开 | ✅（完工后） |
| 确认配件/方案 | ❌ Phase 2 |

## **6. 状态流转（Phase 1 主路径）**

```text
draft
  ↓ 商家编辑/upload
in_progress
  ↓ 商家标记完工
completed
  ↓ 车主授权公开（可选）
pending_review（脱敏 + 展示审核）
  ↓ 审核通过
published
```

异常：`user_rejected`（拒绝公开）、`taken_down`。

**Phase 2 启用**：`pending_part_confirm` 及方案变更回退。

## **7. 授权公开与案例发布**

### **7.1 触发条件**

1. 档案 `completed`；  
2. 至少 1 张有效图片；  
3. 无进行中的隐私投诉。

**Phase 2 增加**：L2+ 配件车主确认已完成。

### **7.2 授权档位**

见 `00_Phase1_服务相册产品口径.md` §4：① 不公开 / ② 匿名 / ③ 实名。

### **7.3 脱敏与审核**

`service_pre_mask` → 用户 `service_authorize` 预览 → 平台展示审核 → 发布。

### **7.4 发布后 Tag**

**已授权 · 已脱敏 · 已审核**（不得使用「平台订单案例」）。

### **7.5 用户撤回授权**

用户在 **任何时候** 均可撤回已发起的公开案例授权，**不受审核状态限制**：

| 当前状态 | 撤回后 |
| --- | --- |
| `pending_review` | 移出审核队列，不发布 |
| `public_approved` / 已公示 | 案例立即下线（H5 noindex 或 404，移出 sitemap，停止推荐） |

撤回后：`album.status` 保持 **已完工**（`completed`），`publicCaseStatus` 回退 **private**；用户可再次发起授权公示。脱敏授权任务（`task_auth_*`）重置 `maskingConfirmed`，预览接口 **upsert** 避免重复创建冲突。

入口见「我的 → 我的公开授权」（`12_我的页面与账户体系.md` §8）；确认交互使用 **BottomSheet**（非原生 `showModal`，避免遮罩残留）。处理细则见 `10_隐私协议与授权规则.md` §10.2。

### **7.6 用户端列表展示（Phase 1）**

| 维度 | 用户可见文案 | 说明 |
| --- | --- | --- |
| 列表 Tab | **私密相册** / **公开相册** | 仅两 Tab；审核中案例仍在私密 Tab |
| 私密 Tab 卡片 | **维修中** / **已完工**；可选 **审核中** | 审核中为 `pending_review` 时的附加 Tag |
| 公开 Tab 卡片 | （无状态 Tag） | 已 `public_approved` |
| 详情头部 | 维修进度 + 可见性 Tag | 见 `07_维修相册查看页.md` §14.1 |

列表 **禁止**「可授权 / 已授权 / 已公开 / 待授权公开」等内部态文案。

## **11. P0 验收（Phase 1）**

1. 商家可创建档案并完成六阶段留档；  
2. 可录入配件类型，**不发起**车主确认；  
3. 可标记完工，**无** L2+ 阻断；  
4. 车主可查看时间线；  
5. 授权 + 脱敏 + 审核后可发布；  
6. 公开 Tag 符合 Phase 1。
