# **2. 模块定位**

案例公开与脱敏审核模块是平台内容资产形成的关键模块。

它解决三个问题：

1. **能不能公开**
  - 用户是否授权；
  - 商家是否有权上传；
  - 是否存在隐私风险；
  - 是否通过平台审核。
2. **公开什么**
  - 脱敏图片；
  - 维修方案；
  - 车型信息；
  - 价格区间；
  - 影响价格因素；
  - 门店信息。
3. **公开后如何使用**
  - 小程序案例页；
  - H5 案例详情页；
  - 门店主页；
  - 服务详情页；
  - SEO/GEO 内容页；
  - 分享页。

# **3. 案例来源类型**

## **3.1 用户授权案例（服务相册）**

来源：

```
商家创建服务相册（线下维修）
→ 车主查看并完成配件等待确认
→ 相册 completed
→ 用户明确授权脱敏公开
→ 平台展示审核
→ 生成公开案例
```

特点：

- 有用户授权记录；
- 可有配件确认留痕（不公开展示全文）；
- 标签：**用户授权案例 · 已脱敏 · 平台展示审核**；
- **不得**称「平台订单案例」。

---

## **3.2 商家历史案例**

来源：

```
商家上传过往案例
→ 商家确认隐私和合法性责任
→ 系统脱敏
→ 平台审核
→ 生成公开案例

```

特点：

- 冷启动重要内容；
- 不关联平台订单；
- 商家承担内容真实性和隐私责任；
- 必须标注「商家上传 · 历史案例」；
- 平台展示审核不等于质量背书。

---

## **3.3 ~~线下绑定案例~~（P1）**

后续：服务相册用户认领增强；MVP 不单独设类型。

# **4. 公开案例状态**

## **4.1 案例状态**

| 状态 | 说明 |
| --- | --- |
| 草稿 | 未提交 |
| 待用户授权 | **用户授权案例** 等待用户同意 |
| 用户拒绝公开 | 不公开 |
| 待脱敏    | 已授权，等待自动脱敏   |
| 脱敏中    | 系统处理图片和文字    |
| 脱敏失败   | 自动处理失败       |
| 待审核    | 等待平台审核       |
| 要求修改   | 需补充资料或重新脱敏   |
| 审核通过   | 可公开          |
| 已公开    | 已生成公开页面      |
| 已驳回    | 不允许公开        |
| 已下架    | 公开后被下架       |
| 用户申请下架 | 等待处理         |
| 平台隐藏   | 因风险临时隐藏      |

| 状态     | 说明     |
| ------ | ------ |

| OCR 完成 | 已识别文字  |
| 车牌识别完成 | 已识别车牌  |
| 人脸识别完成 | 已识别人脸  |
| 脱敏完成   | 已生成打码图 |
| 脱敏失败   | 系统无法处理 |
| 高风险    | 存在隐私风险 |
| 禁止公开   | 无法安全公开 |

# **5. 用户授权案例公开流程**

## **5.1 触发时机**

当 **服务相册** 满足：

1. 相册 `completed`；
2. 至少有 1 张有效图片；
3. L2+ 配件确认已完成（若涉及更换件）；
4. 无严重隐私投诉。

---

## **5.2 流程**

```
服务相册 completed
→ 后台 service_pre_mask（预脱敏，可选 P0.5）
→ 用户进入授权页 / 脱敏工作台
→ 用户同意 / 拒绝 / 手动打码
→ 平台展示审核
→ 发布 H5 / GEO
```

---

## **5.3 用户授权前展示**

必须展示：

1. 公开后展示脱敏维修案例，**非平台质量背书**；
2. 不展示车牌、VIN、人脸、手机号、完整结算、车主签名；
3. 脱敏预览；
4. 默认不展示实际支付金额；
5. 可拒绝公开、可申请下架。

---

## **5.4 用户可选操作**


| 操作        | 结果            |
| --------- | ------------- |
| 同意公开      | 进入脱敏和审核流程     |
| 拒绝公开      | 案例不公开，仅作为服务相册 |
| 删除部分图片后公开 | 剔除指定图片后进入流程   |
| 手动打码后公开   | 使用用户编辑后的图片    |
| 暂不处理      | 保持待授权状态       |


---

# **6. 商家历史案例公开流程**

## **6.1 触发时机**

商家在工作台创建历史案例并提交审核。

---

## **6.2 流程**

```
商家新建历史案例
→ 填写车型/服务/故障/方案/价格区间
→ 上传图片
→ 勾选隐私责任确认
→ 提交
→ 系统压缩图片
→ 系统自动脱敏
→ 输出风险等级
→ 平台审核
→ 审核通过
→ 发布公开案例

```

---

## **6.3 历史案例必须标识**

所有历史案例展示时必须标明：

```
商家历史案例，非平台订单案例

```

同时建议展示：

- 已脱敏；
- 已平台审核；
- 价格仅供参考；
- 由商家提供。

# **7. 脱敏规则**

## **7.1 必须脱敏的信息**

图片中必须识别并遮挡：

1. 车牌号；
2. VIN 码；
3. 发动机号；
4. 手机号；
5. 姓名；
6. 身份证号；
7. 人脸；
8. 微信号；
9. 地址；
10. 支付二维码；
11. 维修单据个人信息；
12. 保险定损单个人信息；
13. 车辆登记证、行驶证、驾驶证等证件信息；
14. 能明显识别车主身份的特殊内容。

---

## **7.2 文字内容脱敏**

案例文字不得包含：

1. 用户真实姓名；
2. 手机号；
3. 精确住址；
4. 车牌；
5. VIN；
6. 身份证；
7. 微信号；
8. 保险单号；
9. 报案号；
10. 其他个人敏感信息。

如商家填写：

```
浙A12345 宝马 3 系车主张先生……

```

系统应提示或自动替换为：

```
一位宝马 3 系车主……

```

---

## **7.3 图片脱敏方式**

支持：

1. 车牌区域马赛克；
2. 人脸区域马赛克；
3. VIN/手机号 OCR 区域遮挡；
4. 单据敏感区域整块遮挡；
5. 用户手动打码；
6. 用户裁剪；
7. 删除高风险图片。

---

## **7.4 脱敏图保存规则**

每张图片至少保存：

1. 压缩原图，内部查看；
2. 脱敏图，公开展示；
3. 缩略图，列表展示；
4. 脱敏版本号；
5. 风险检测结果。

公开页面只能使用脱敏图。

---

# **8. 隐私风险等级**


| 风险等级 | 判断标准              | 处理方式        |
| ---- | ----------------- | ----------- |
| 低风险  | 未发现明显隐私           | 可进入审核       |
| 中风险  | 疑似车牌/VIN/手机号/人脸   | 人工审核重点检查    |
| 高风险  | 明显隐私且未完全遮挡        | 必须重新脱敏或删除图片 |
| 禁止公开 | 无法有效脱敏、稀有可识别、含证件等 | 不允许公开       |


---

# **9. 特殊高风险场景**

## **9.1 稀有车型风险**

某些稀有车型即使脱敏车牌，也可能被识别车主。

处理建议：

1. 不展示完整车身；
2. 裁剪局部维修部位；
3. 隐藏车型年款；
4. 不展示门店精确时间；
5. 必要时禁止公开。

---

## **9.2 事故车定损单**

定损单可能包含大量个人信息。

处理建议：

1. 默认不公开定损单原图；
2. 如需展示，只展示维修项目局部；
3. 必须遮挡车牌、VIN、姓名、手机号、保险信息；
4. 高风险人工审核。

---

## **9.3 新能源三电维修**

新能源三电信息可能涉及安全和资产识别风险。

处理建议：

1. 隐藏 VIN；
2. 隐藏电池编号；
3. 隐藏诊断设备中的车辆唯一标识；
4. 高复杂度案例人工审核；
5. 避免展示敏感诊断数据细节。

---

## **9.4 人脸和店内人员**

如图片出现车主、路人、员工人脸：

- 车主和路人必须打码；
- 员工如未授权也应打码；
- 商家可选择删除图片。

---

# **10. 平台审核 PRD**

## **10.1 审核入口**

```
我的
→ 平台运营工作台
→ 案例审核

```

---

## **10.2 审核列表**

Tab：

1. 待审核；
2. 高风险；
3. 要求修改；
4. 已通过；
5. 已驳回；
6. 已下架；
7. 用户申请下架。

列表字段：


| 字段   | 说明        |
| ---- | --------- |
| 案例标题 | 系统生成/商家填写 |
| 案例来源 | 用户授权/商家历史 |
| 门店   | 关联门店      |
| 服务项目 | 如钣喷修复     |
| 复杂度  | L1-L4     |
| 图片数量 | 总数        |
| 风险等级 | 低/中/高/禁止  |
| 授权状态 | 用户授权案例可公开展示  |
| 提交时间 | 进入审核时间    |


---

## **10.3 审核详情页**

页面模块：

```
案例基础信息
服务相册信息，用户授权案例
用户授权记录
商家责任确认，历史案例
车辆信息
维修内容
价格区间
脱敏图查看（审核依据，不展示原图）
OCR 识别结果
风险提示
案例质量评分
审核操作
审核日志

```

---

## **10.4 审核人员检查项**

必须检查（依据 **脱敏图 + OCR 摘要**，**不查看原图**）：

1. 是否有用户授权（用户授权案例）；
2. 历史案例是否有商家责任确认；
3. 脱敏图/OCR 是否仍含可见车牌；
4. 脱敏图/OCR 是否仍含可见人脸；
5. 脱敏图/OCR 是否仍含 VIN/手机号/姓名；
6. 文字是否包含隐私；
7. 是否存在夸大宣传；
8. 价格展示是否为区间；
9. 是否存在虚假承诺；
10. 资质是否匹配服务类型；
11. 案例质量是否足以公开；
12. 是否适合进入平台案例库和 H5 搜索页。

---

## **10.5 审核结果**


| 审核结果         | 说明                |
| ------------ | ----------------- |
| 通过并公开        | 生成公开案例            |
| 通过但仅门店展示     | 不进入平台案例库          |
| 通过但不允许搜索引擎收录 | 小程序内展示，H5 noindex |
| 要求修改         | 退回商家/用户重新处理       |
| 驳回           | 不允许公开             |
| 禁止公开         | 高隐私风险或违规          |
| 平台下架         | 已公开后移除            |

# **11. 公开案例内容规则**

## **11.1 标题生成规则**

标题建议格式：

```
【城市】【车型】【服务项目】维修案例

```

示例：

```
杭州宝马3系刹车片更换维修案例
杭州奥迪A6L前保险杠钣喷修复案例
杭州事故车前部损伤维修案例

```

事故车标题应避免过度暴露：

```
杭州某车型事故车前部损伤维修案例

```

---

## **11.2 摘要生成规则**

标准摘要：

```
该案例为【车型】在【里程】时进行【维修项目】的维修记录。
检查发现【主要问题】，门店采用【维修方案】完成处理。
案例包含维修前、拆检、配件、维修后等过程图片，图片已进行隐私脱敏。
价格展示为参考区间，实际费用会因车型、配件品牌、损伤程度和维修方案不同而变化。

```

---

## **11.3 价格展示规则**

禁止展示：

1. 用户实际支付金额，除非后续有明确授权策略；
2. 保险理赔具体金额；
3. 个人定损金额原图；
4. 可关联个人身份的费用单据。

允许展示：

1. 价格区间；
2. 影响价格因素；
3. 配件品牌差异；
4. 工时差异；
5. 损伤程度差异。

---

## **11.4 来源标识**

用户授权案例：

```
用户授权案例 · 已脱敏 · 平台展示审核
```

商家历史案例：

```
商家上传 · 历史案例 · 已脱敏 · 平台展示审核 · 价格仅供参考
```

# **12. H5 发布规则**

## **12.1 可发布 H5 的条件**

必须满足：

1. 审核通过；
2. 脱敏图存在；
3. 文字脱敏完成；
4. 有标题；
5. 有摘要；
6. 有服务项目；
7. 有门店信息；
8. 无禁止公开风险。

---

## **12.2 URL 规则**

建议格式：

```
/cases/{city}/{service-slug}/{case-id}

```

示例：

```
/cases/hangzhou/brake-pad/10001
/cases/hangzhou/body-paint/10002

```

---

## **12.3 SEO/GEO 元信息**

每个 H5 案例页应生成：

1. title；
2. description；
3. canonical；
4. robots；
5. Open Graph；
6. Schema；
7. sitemap 记录；
8. AI 可引用摘要。

---

## **12.4 noindex 规则**

以下案例不允许搜索引擎收录：

1. 风险较高但允许小程序展示；
2. 用户要求不被搜索；
3. 稀有车型可识别风险；
4. 内容质量不足；
5. 平台认为不适合公开传播。

页面应加：

```
<meta name="robots" content="noindex,nofollow">

```

# **13. 用户申请下架**

## **13.1 下架入口**

用户可在：

1. 订单详情；
2. 服务相册页；
3. 公开案例页；
4. 客服入口；

发起下架申请。

---

## **13.2 下架原因**

用户可选择：

1. 不希望继续公开；
2. 发现隐私未处理；
3. 图片不想展示；
4. 内容描述不准确；
5. 其他原因。

---

## **13.3 下架处理流程**

```
用户提交下架申请
→ 案例进入用户申请下架状态
→ 平台临时隐藏，建议
→ 平台审核原因
→ 确认下架/修改后恢复/驳回
→ 通知用户和商家

```

建议对隐私相关下架申请：

- 先隐藏；
- 后处理；
- 不等待审核完成再隐藏。

---

# **14. 商家申请修改或下架**

商家可对历史案例申请修改或下架。

用户授权案例：

- 商家不能单方面公开；
- 商家不能绕过用户授权；
- 商家可申请修改门店说明；
- 涉及用户内容需平台审核。

历史案例：

- 商家可申请下架；
- 修改后需重新审核；
- 若有投诉，平台可直接下架。

# **15. 案例质量评分**

## **15.1 评分维度**


| 维度         | 权重建议 |
| ---------- | ---- |
| 图片清晰度      | 20%  |

---

## **15.2 质量影响**

质量评分影响：

1. 案例列表排序；
2. 门店主页排序；
3. 服务详情页关联案例排序；
4. 是否进入 H5 公开页；
5. 是否进入 sitemap；
6. 是否被推荐为优质案例。

# **16. 风控规则**

## **16.1 内容风控**

拦截或重点审核：

1. 夸大宣传；
2. 虚假低价；
3. 保证维修效果；
4. 事故车保证全赔；
5. 未授权品牌标识；
6. 敏感个人信息；
7. 侵权图片；
8. 非维修相关内容。

---

## **16.2 商家风控**

商家存在以下情况时限制案例发布：

1. 多次上传隐私违规内容；
2. 多次被用户投诉；
3. 历史案例疑似盗图；
4. 资质不匹配；
5. 虚假案例；
6. 刷案例行为。

限制措施：

- 案例需人工审核；
- 降低曝光；
- 暂停历史案例上传；
- 冻结门店展示；
- 严重者清退。

# **18. P0 验收标准**

必须满足：

1. **用户授权案例** 须用户授权后才能公开；
2. 用户可查看脱敏预览；
3. 用户可拒绝公开；
4. 用户可删除部分图片后公开，若实现；
5. 商家历史案例必须确认隐私责任；
6. 图片必须生成脱敏版本；
7. 公开页面只能使用脱敏图；
8. 平台审核**仅查看脱敏图**（不展示原图；Admin API 不返回 `rawUrl`）；
9. 平台可审核通过、驳回、要求修改；
10. 高风险图片不得自动公开；
11. 历史案例必须标注来源；
12. 公开案例可生成小程序详情页；
13. 公开案例可生成 H5 页面；
14. 用户可申请下架；
15. 隐私投诉可先隐藏后处理；
16. 审核和授权操作必须留痕。
