## **2. MVP 核心目标**

验证三个核心假设：

### **2.1 商家愿意使用维修证据链工具**

- 创建 **服务相册**；
- 填写配件类型（Phase 1 商家单方录入）；
- ~~配合非原厂件车主确认~~ → **Phase 2**；
- 认为案例与 GEO 能获客；
- 认可证据包对风控的价值（PDF **Phase 2 / P0.5**）。

### **2.2 用户愿意通过案例建立信任**

- 查看公开案例与 GEO 页；
- 因案例产生咨询/预约；
- 查看门店创建的服务相册；
- ~~完成配件待确认（若适用）~~ → **Phase 2**；
- 可选 **授权脱敏公开**（Phase 1 唯一需车主主动操作的环节）。

### **2.3 公开内容能形成长期资产**

- H5/GEO 页可访问与被收录；
- 案例页带来咨询线索；
- 结构化摘要具备 GEO 建设价值。

## **3. MVP 范围总览**

MVP 必须覆盖 **6 个模块**（原「历史案例上传」已并入模块 4，不再单列）：

1. 用户端浏览、搜索、咨询预约；
2. 商家入驻与内容管理；
3. 服务方案展示（非在线成交）；
4. **服务相册证据链**（六阶段 + 配件录入 + 公开授权；含冷启动公开案例，**无**独立历史案例入口）；
5. H5 公开案例、门店页与 GEO 页；
6. 平台审核与运营工作台。

**已移除模块**：订单与支付、接单履约、交易评价与奖励、**独立历史案例上传流程**。

与 `01_项目背景与目标.md` 一致：P0 含刹车、轮胎、电瓶、钣喷、事故车（内容维度）；价格均为参考价/区间/不展示，**无在线成交**。

## **6. MVP 不做范围**

- 在线下单、微信支付、退款、平台订单状态机；
- 商家接单、平台完工确认、交易评价、评价奖励；
- 交易佣金、结算、售后仲裁；
- 平台订单案例；
- **独立「历史案例上传」流程与「商家历史案例」主 Tag**（考古文档保留，新功能勿引）；
- 复杂 AI 诊断、多级分销、分享现金收益；
- 事故车线上报价；
- 强制上传全部节点（改为风险分级）；
- 视频上传（可后续）；
- 全国多城市；
- 保险理赔深度对接。

**说明**：Phase 1 **需要**图片持久化存储（ECS 或 OSS），否则服务相册无法在生产环境使用；与早期「OSS 可后续」表述相比，**上线前为 P0 基础设施**（见 `00_开发计划.md` **B-MEDIA**）。

## **7. 核心规则定义（V2.0 · Phase 1）**

### **7.1 咨询/预约规则**

| 项目 | 规则 |
| --- | --- |
| 必填 | 门店、服务类型、手机号、平台说明勾选 |
| 状态 | 已提交 / 门店已查看 / 门店已联系 / 已取消 / 已关闭 |
| 平台角色 | 信息转接，不参与报价与维修 |

### **7.2 服务相册规则**

| 项目 | 规则 |
| --- | --- |
| 创建 | 商家主动创建，不依赖订单；**唯一 create 流程** |
| 配件类型 | 更换件必填标准类型（Phase 1 **录入留痕**，不发起车主确认） |
| 非原厂确认 | **Phase 2** |
| 图片上限 | 单相册 20 张（初版） |
| 公开 · 有车主 | 须 **车主授权档位** + 脱敏 + 平台审核 |
| ~~公开 · 冷启动~~ | ~~无关联车主时商家单方提交~~ **已取消**（2026-06-24）；公开须车主授权 |

### **7.3 公开授权规则**

入口为 **服务相册** `completed` 后车主操作；非订单完成触发。

### **7.4 奖励规则**

**MVP 不做**交易评价奖励。P1 可选：案例授权贡献激励（非 P0）。

## **12. MVP 上线验收标准**

### **功能验收（16 项摘要 · Phase 1 口径）**

1. 用户可登录并绑定手机号；
2. 可浏览首页、服务、门店、案例、GEO；
3. 可搜索并查看服务详情与参考价说明；
4. 可提交咨询/预约并查看记录；
5. 提交前须勾选平台说明；
6. 可查看门店详情并电话/导航/留言咨询；
7. 可查看公开脱敏案例（Tag：**已授权 · 已脱敏 · 已审核**）；
8. 可通过链接/我的相册查看服务相册；
9. ~~可完成配件告知确认~~ → **Phase 2**；
10. 可授权/拒绝公开并预览脱敏；
11. 商家可入驻、建 **服务相册**、填配件类型、管理线索；
12. 商家可通过 **服务相册** 录入冷启动内容并提交 **公开案例审核**（**无**独立历史案例入口）；
13. 平台可审核 **服务相册与公开案例**；
14. H5 案例/门店/GEO 可访问；
15. 公开内容必须脱敏；
16. 关键操作有日志。

### **合规验收**

公开案例脱敏；有车主场景须用户授权；冷启动公开须平台审核与合规文案；平台说明在关键页展示；资质审核；无在线支付与平台质量承诺文案；**禁止**新页面使用「商家历史案例」作主 Tag。
