## **2. 项目起点分析**

### **2.1 创始人行业背景**

创始人长期从事车险业务，与修理厂、4S 店、事故车维修链条有较深接触。

在长期观察中发现：

- 修理厂、4S 店并不缺低价流量，但低价活动很难转化为高价值订单；
- 事故车维修曾是重要利润来源，但近年受费改、赔付压价、车主出险意愿下降等影响持续承压；
- 黄牛推修、外拓返现等灰色渠道存在质量和法律风险。

最初曾考虑做事故车推修交易小程序，用「评价返现」替代灰色返现。推进中发现：一手信息被保险公司垄断、老司机有固定维修场所、纯交易撮合缺乏壁垒。

### **2.2 法律风险与战略转向**

2022–2026 年多起「非原厂件未充分告知」被判退一赔三的案例表明：

- 法院不要求禁止非原厂件，但要求**提前、明确、显著、可追溯地告知并取得车主确认**；
- 口头告知、结算单小字备注均不足以免责；
- 平台若作为交易撮合方，极易卷入纠纷甚至承担连带责任。

结合平台启动阶段资金与人力，**无法承担交易平台所需的投诉调解、售后仲裁、连带责任等运营责任**。

因此项目做出战略调整：

> **放弃交易平台，重点完善「服务相册」——升级为「维修过程证据链 + 配件告知确认 + 风险自保系统」。**

产品细则见 `docs/04_维修过程相册/01_维修过程相册PRD V2.0.md`。

### **2.3 前期尝试中的问题（仍成立）**

- 纯交易撮合没有足够壁垒，易陷入低价竞争；
- 优质商家对低价平台天然警惕；
- 小程序内容封闭，难以被搜索引擎和 AI 直接引用；
- 「评价返现」机制易引发刷评、合规与风控成本。

这些判断支持从「交易平台」转向「内容工具 + 证据链工具 + GEO 引流」。

## **3. 项目本质判断**

本项目的本质不是「汽修团购平台」，也不是「修车小程序」，而是：

> **以维修过程证据链为核心，以脱敏案例和 GEO 内容为资产，以咨询线索和到店转化为结果的维修案例相册与本地门店引流平台。**

长期由四层构成（**交易层已移除**）：

### **3.1 第一层：维修证据链层（核心）**

通过六阶段服务相册、配件告知确认单、车主签字留痕、照片水印与元数据、证据包导出，把维修过程变成可查看、可确认、可追溯、可导出的合规凭证。

主要解决：

- 非原厂件、同质件、副厂件等未充分告知的法律风险（面向商家自保）；
- 用户看不懂维修过程、不知道换了什么件；
- 争议发生时修理厂缺乏可追溯证据；
- 平台不参与交易，但可提供记录与展示工具。

### **3.2 第二层：案例与知识数据层**

将服务相册和用户授权脱敏后沉淀为结构化公开案例。

包括：车型、故障表现、维修方案、配件类型（脱敏）、价格区间参考、过程摘要等。

### **3.3 第三层：GEO 内容与引流层**

通过 H5 公开案例页、门店页、城市服务页、车型/故障专题页等，承接搜索与 AI 引用流量，转化为咨询/预约线索。

### **3.4 第四层：咨询线索层**

用户通过电话、在线咨询、表单预约等方式联系门店；**实际维修、报价、收款、发票、质保和售后由门店线下完成**。

## **4. 项目核心定位**

### **4.1 平台身份（必须明确）**

平台**不是**：

- 交易平台；
- 维修服务提供方；
- 统一报价方；
- 维修质量担保方；
- 售后赔偿主体。

平台**是**：

- 内容展示平台；
- 维修案例相册工具；
- GEO 本地搜索引流工具；
- 门店获客工具；
- 咨询线索转接工具；
- 维修过程记录与配件确认工具（面向商家风控）。

统一对外表述：

> 平台为车主提供维修案例浏览、门店信息展示、服务项目查询、到店咨询预约和脱敏服务相册查看服务。实际维修服务、报价、收款、发票、质保及售后由线下门店提供和承担。

### **4.2 对用户的定位**

**看维修案例，找本地门店；维修过程有相册，服务选择更清楚。**

用户核心价值：

1. **看得懂**：看懂常见服务、案例、价格影响因素、门店擅长方向；
2. **找得到**：通过服务、车型、故障、城市、门店等维度找到本地维修门店；
3. **可参考**：查看公开脱敏案例；查看门店为自己创建的服务相册；可选择是否授权脱敏公开。

说明：服务相册由门店上传，**不代表平台验收维修质量**，仅用于记录和展示商家上传的维修过程信息。

### **4.3 对商家的定位**

**降低争议风险，让每一次告知都有凭证；让专业被看见，让案例成为获客资产。**

商家核心价值：

- 维修过程证据链与配件告知确认，降低「未告知非原厂件」等法律风险；
- 合规检查清单与阻断式确认，减少员工遗漏；
- 一键导出 PDF 证据包，便于争议处理；
- 历史案例与服务相册脱敏公开，带来 GEO 搜索流量和咨询线索；
- 在 AI 搜索时代提前建立可被引用的数字资产。

### **4.4 对平台自身的定位**

平台长期价值来自：

- 标准化维修证据链（六阶段 + 配件确认卡）；
- 脱敏公开案例与 GEO 内容页；
- 咨询线索与内容转化数据；
- 可被搜索引擎和 AI 读取的公开内容。

## **5. 首发城市与冷启动策略**

### **5.1 首发城市**

MVP 首发城市：**杭州**。

### **5.2 冷启动总体策略**

原则：**先供给，后需求；先样板商家，后规模扩张；先案例资产，后平台故事；先工具价值，后流量故事。**

第一阶段重点：

1. 签约杭州首批样板维修商；
2. 商家上传历史案例、创建服务相册（含配件类型与告知确认 MVP 能力）；
3. 平台生成 GEO 页面与公开案例；
4. 用案例页、门店页建立用户信任；
5. 引导用户电话咨询/在线咨询/预约到店——**不在线成交**。

---

## **6. 首批服务范围**

MVP 覆盖常见服务作为**内容分类与 GEO 维度**，不作为在线商品成交。

### **6.1 P0 优先服务**

| 服务 | 原因 |
| --- | --- |
| 刹车片/刹车盘 | 高痛点，适合新旧件对比与配件确认 |
| 轮胎服务 | 高频，易标准化展示 |
| 电瓶更换 | 易理解，适合案例展示 |
| 钣喷修复 | 前后对比明显，案例价值高 |
| 事故车维修/专修 | 商家高价值项目，需强合规记录（到店检测后报价） |

### **6.2 P1 扩展服务**

小保养、大保养、空调清洗/维修、底盘维修、新能源专项等。

### **6.3 价格展示模式（非成交价）**

| 类型 | 说明 |
| --- | --- |
| 参考价 | 「参考价 ¥299 起，具体费用以门店检测和用户线下确认为准」 |
| 参考区间 | 「参考区间 ¥300–¥1200，费用因车型、配件、损伤程度而异」 |
| 不展示价格 | 事故车、发动机、变速箱、三电、结构件等复杂项目 |

**禁止**在线一口价成交、在线支付、平台统一报价暗示。

## **7. 维修复杂度分级**

平台保留 L1–L4 分级，用于：

- 相册节点建议与风险等级；
- 配件告知确认强度（非原厂件是否需车主签字）；
- 证据包导出要求；
- 案例审核优先级。

**不再用于**评价奖励核算、订单金额系数、交易佣金。

| 等级 | 典型场景 | 证据链要求（概要） |
| --- | --- | --- |
| L1 | 洗车、雨刮、滤芯等 | 普通记录，可选拍照 |
| L2 | 小保养、刹车、电瓶、轮胎、小钣喷 | 必须记录配件类型；非原厂件需车主确认 |
| L3 | 发动机/变速箱局部、底盘、中大面积钣喷 | 强制拍照；非原厂件确认；关键凭证不可随意删除 |
| L4 | 总成大修、事故车整车、三电大修 | 强制方案确认、配件确认、电子签名、新旧件对比、证据包导出 |

## **8. 项目阶段目标**

### **8.1 MVP 阶段目标**

验证三个核心假设：

1. **商家愿意使用维修证据链工具**（上传相册、填写配件类型、配合车主确认）；
2. **用户愿意通过案例建立信任并产生咨询**；
3. **公开内容能形成 GEO 资产并带来线索**。

MVP 必须跑通：

```
用户浏览服务/案例/门店
→ 电话咨询/在线咨询/预约到店
→ 线下检测、报价、维修、付款（门店线下完成）
→ 商家创建服务相册（含配件类型与告知确认 MVP）
→ 用户查看服务相册
→ 用户授权脱敏公开
→ 平台生成公开案例与 GEO 内容页
→ 案例带来搜索流量和咨询线索
```

### **8.2 3 个月目标**

| 目标 | 指标 |
| --- | --- |
| 样板商家 | 杭州 10–20 家 |
| 标准案例 | 100 个以上 |
| 服务相册 | 有真实相册创建与配件类型记录 |
| 公开页面 | 案例页、门店页、基础 GEO 页上线 |
| 咨询线索 | 案例页带来可统计的咨询/预约 |

### **8.3 6 个月目标**

| 目标 | 指标 |
| --- | --- |
| 标准案例 | 300–500 个 |
| GEO 页面 | 城市服务页、车型/故障专题页初步形成 |
| SEO 收录 | 公开页开始被收录 |
| 证据链能力 | 非原厂件确认、证据包导出 MVP 稳定可用 |

### **8.4 12 个月目标**

- 评估第 2 城市扩展；
- 形成车型/故障/价格参考基础内容；
- 维修证据链成为商家付费工具候选；
- 公开案例数据资产形成壁垒。

## **9. MVP 不做什么**

- 在线下单、微信支付、订单履约、退款、售后仲裁；
- 平台订单案例、确认完工、交易评价、评价奖励；
- 交易佣金与结算；
- 复杂 AI 诊断、多级分销、按浏览量分享收益；
- 事故车线上报价；
- 全国多城市同时扩张；
- 强制商家上传全部节点（改为风险分级 + 关键节点阻断）。

---

## **10. 项目成功的核心判断**

不取决于第一阶段交易额，而取决于：

1. 商家是否愿意创建服务相册并填写配件类型？
2. 非原厂件场景下，车主是否完成告知确认？
3. 商家是否认为证据链工具有助于降低争议风险？
4. 用户是否愿意查看服务相册并授权公开？
5. 公开案例是否带来咨询与到店线索？
6. GEO 页面是否开始被收录并产生自然流量？

若以上成立，平台可在内容、工具、GEO 方向持续深化；**是否恢复交易能力留待后续独立评估**，不在 V2.0 MVP 范围内。
