## **2. 核心业务总览**

平台整体业务由 **8 条核心流程**构成：

1. 用户浏览与咨询预约流程；
2. 商家咨询线索处理流程；
3. 服务相册（维修证据链）创建流程；
4. 配件告知与车主确认流程；
5. 独立历史案例上传流程；
6. 用户授权公开与案例发布流程；
7. 分享与 GEO 引流流程；
8. SEO/GEO 页面生成流程。

整体闭环：

```
用户浏览服务/案例/门店/GEO 页
→ 电话咨询 / 在线咨询 / 预约到店
→ 线下检测、报价、维修、付款（门店完成）
→ 商家创建服务相册
→ 填写配件类型，非原厂件发起车主确认
→ 用户查看服务相册并完成待确认事项
→ 用户选择是否授权脱敏公开
→ 平台脱敏与展示审核
→ 生成 H5 公开案例与 GEO 页面
→ 带来新的浏览、咨询与预约
```

# **3. 用户浏览与咨询预约流程**

## **3.1 浏览路径**

```
用户进入小程序或 H5
→ 搜索服务/门店/车型/故障关键词
→ 浏览服务说明、价格影响因素
→ 查看公开脱敏案例
→ 查看门店主页与案例相册
→ 产生咨询意向
```

## **3.2 咨询预约流程**

```
用户点击联系门店 / 预约到店 / 咨询报价
→ 填写：服务类型、门店、车型、问题描述、期望到店时间、手机号
→ 阅读并勾选平台说明
→ 提交咨询/预约
→ 门店收到线索通知
→ 门店线下联系用户
→ 用户自行决定是否到店及是否维修
```

## **3.3 咨询/预约状态**

| 状态 | 说明 |
| --- | --- |
| 已提交 | 用户已提交，待门店查看 |
| 门店已查看 | 门店已读线索 |
| 门店已联系 | 门店已联系用户 |
| 用户已取消 | 用户主动取消 |
| 已关闭 | 门店标记关闭或过期关闭 |

**不做**：待支付、待接单、维修中、待确认完工、已完成、售后、已退款。

# **4. 商家咨询线索处理流程**

## **4.1 入口**

```
我的 → 商家工作台 → 线索管理
```

## **4.2 处理流程**

```
商家收到新线索
→ 查看用户信息、问题描述、上传图片
→ 电话联系用户
→ 标记已联系
→ 用户到店后标记已到店
→ 线下检测、报价、维修（平台不参与）
→ 可选：创建服务相册关联该用户
→ 线索关闭
```

---

# **5. 服务相册（维修证据链）创建流程**

## **5.1 创建入口**

- 商家工作台 → 创建服务相册；
- 关联门店（必填）、服务项目（必填）；
- 关联用户：手机号 / 分享链接 / 扫码认领。

**不再**依赖「订单创建后默认生成相册」。

## **5.2 六阶段流程**

```
阶段一：接车记录（外观、里程、故障描述等）
→ 阶段二：检测诊断（现象、检测图、建议方案）
→ 阶段三：维修方案与报价确认（项目、价格、配件类型、质保说明）
→ 阶段四：配件告知与确认（非原厂件 → 车主确认页）
→ 阶段五：施工过程（新旧件对比、包装、编码、安装过程）
→ 阶段六：完工交付（完工图、验收确认、结算说明）
```

## **5.3 配件告知确认流程**

```
商家录入更换配件
→ 选择标准配件类型
→ 若为非原厂件：
   → 系统生成车主确认页（显著风险提示）
   → 车主勾选「我已阅读并理解…」
   → 车主点击「确认使用非原厂件」或等价文案
   → 记录确认时间、确认内容版本
→ 未完成确认时，系统阻断进入施工/完工（按风险等级配置）
```

## **5.4 用户查看规则**

| 场景 | 可见内容 |
| --- | --- |
| 用户查看自己的服务相册 | 门店上传内容 + 待确认事项 |
| 用户分享 | 仅脱敏且已授权内容 |
| H5 公开案例 | 脱敏图 + 摘要，无完整结算与签名 |
| 商家查看本店相册 | 完整记录（含合规视图） |
| 平台审核 | **脱敏图 + OCR/风险摘要**（**不展示原图**） |

## **5.5 证据包导出（MVP P0 或 P0.5）**

```
相册达到可导出条件（完工 + 关键节点完整）
→ 商家或授权角色发起导出
→ 生成 PDF（含时间线、配件确认、照片摘要、操作日志、防篡改校验码）
→ 供争议/行政调解/诉讼准备使用
```

# **6. 独立历史案例上传流程**

## **6.1 使用场景**

冷启动、门店专业展示、GEO 内容供给。

## **6.2 流程**

```
商家工作台 → 历史案例 → 新建
→ 填写城市、服务、车型、故障、方案、GEO 标签
→ 上传图片（最多 20 张）
→ 确认隐私责任
→ 自动压缩与脱敏
→ 平台展示审核
→ 发布至门店页 / 案例库 / GEO 聚合
```

## **6.3 标识要求**

必须展示：**已脱敏 / 商家上传 / 历史案例 / 平台展示审核**（不得使用「平台订单案例」「平台认证真实」等表述）。

---

# **7. 用户授权公开与案例发布流程**

## **7.1 入口**

- 服务相册查看页；
- 门店发送的相册链接；
- 微信服务通知；
- 我的服务相册；
- 商家邀请授权页。

## **7.2 发布流程**

```
用户查看脱敏预览（无图时可直接勾选确认）
→ 可手动打码/裁剪（有图时）
→ 同意或拒绝公开
→ 若同意：提交公开案例（不因 GEO 缺项拦截）→ 平台合规审核（非留档质量质检）
→ 通过 → 发布 H5 案例页 + 更新 GEO 索引
```

## **7.3 用户权利**

- 可拒绝公开；
- 可申请下架已公开案例；
- 分享仅可使用脱敏公开版本。

---

# **8. 分享流程**

```
用户/商家进入公开案例、门店页、GEO 页
→ 点击分享
→ 系统确认使用脱敏素材
→ 生成分享卡片或链接
→ 记录基础访问数据（不含现金奖励）
```

---

# **9. SEO/GEO 页面生成流程**

## **9.1 MVP 页面类型**

1. 公开案例详情页；
2. 门店公开主页；
3. 城市服务页（如「杭州刹车片更换」）；
4. 城区服务页；
5. 车型/故障专题页；
6. 案例聚合页；
7. sitemap + 基础 Schema。

## **9.2 生成流程**

```
案例/门店/服务审核通过或运营配置
→ 生成页面标题、摘要、FAQ
→ 绑定相关案例与门店
→ 生成 URL slug
→ 输出 Schema（LocalBusiness、Service、FAQPage、Article 等）
→ 加入 sitemap
→ 发布 H5
```

## **9.3 GEO 页底说明**

> 页面内容用于展示维修服务信息、门店信息和脱敏案例，不构成平台报价或维修承诺。实际维修方案、费用、配件、质保和售后由用户与门店线下确认。

# **10. 异常流程**

## **10.1 用户取消咨询**

已提交未联系：用户可取消；已联系：建议联系门店后关闭。

## **10.2 用户不同意公开案例**

相册仅作私密服务记录，不生成公开 H5 页。

## **10.3 配件确认超时**

非原厂件未获车主确认：系统保持阻断，商家不可标记施工完成/完工（L2+）。

## **10.4 方案变更**

配件类型或项目变更：作废原确认，重新发起确认流程。

## **10.5 门店被冻结**

门店主页与公开案例隐藏；不可提交新公开内容；已有私密相册只读。

## **10.6 隐私投诉**

用户或第三方投诉 → 运营审核 → 下架案例/相册公开部分 → 留痕。

# **11. 流程优先级**

## **P0 必须实现**

1. 用户浏览服务、门店、案例、GEO 页；
2. 用户提交咨询/预约（含平台说明勾选）；
3. 商家查看与处理线索；
4. 商家创建服务相册（六阶段骨架 + 配件类型字段）；
5. 非原厂件车主确认（MVP 最低：勾选 + 确认按钮 + 时间戳）；
6. 用户查看服务相册与待确认集中入口；
7. 用户授权公开 + 脱敏 + 展示审核；
8. H5 案例页、门店页、基础 GEO 页发布；
9. 历史案例上传与审核。

## **P0.5 / P1**

1. PDF 证据包导出；
2. 电子签名；
3. 照片强制水印与删除留痕；
4. 合规检查清单阻断完工；
5. 更完整 GEO 页面矩阵；
6. 门店评价（非订单绑定，谨慎上线）。

## **明确不做（MVP）**

- 微信支付、订单状态机、接单拒单、确认完工、交易评价、评价奖励、佣金结算、售后仲裁。
