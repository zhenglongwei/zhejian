## **2. 平台业务总览**

平台连接三类核心对象：

1. 车主用户；
2. 维修商家；
3. 平台运营方。

核心业务闭环：

**站外（公域 · 主战场）**

```
用户在 H5 / 公众号 / 搜索发现脱敏案例与 GEO 内容
→ 站外浏览（或跳转小程序深链详情）
→ 电话咨询 / 在线咨询 / 预约到店
→ 线下检测、报价、维修、付款（门店完成）
→ 商家创建服务相册
→ 用户查看服务相册 → 授权公示 → 审核 → 内容发布回 H5/公众号
→ 内容带来新的搜索流量和咨询线索
```

**小程序（私域 · 工具）**

```
微信搜索 / 商家分享 → 工具首页或深链相册
→ 扫码 / 相册码 / 我的服务相册
→ 查看六阶段记录 → 可选授权公示 / 私人分享
→ 咨询锁定来源门店（深链上下文内）
```

> **2026-06-08 定调**：找店、逛公开案例、GEO 专题 **不在** 小程序冷启动内完成；见 §3.1、`08_双系统转型开发计划.md`。

平台的核心不是一次性撮合交易，而是让每次线下维修可沉淀为：

1. 用户可查看的服务相册（含配件确认留痕）；
2. 商家风控证据与可导出凭证；
3. 平台脱敏内容资产；
4. 搜索和 AI 可读取的数据源。

**平台不提供**：在线交易、在线支付、统一报价、订单履约、维修售后处理、质量担保。

### **2.1 内容发布与可信信源定位（2026-06-02 定调）**

辙见 **不是交易撮合平台**，而是 **维修领域的内容发布平台**，长期目标是成为 **修车相关 AI 搜索最可靠的结构化信源**（门店、服务说明、脱敏公开案例、GEO 专题与证据链相册元数据）。

| 维度 | 平台做什么 | 平台不做什么 |
| --- | --- | --- |
| 商家服务方案 | 商家 **自助上架** 展示参考价与说明；平台提供展示位 | **发布前审核**、价格/质量担保、交易撮合 |
| 公开案例 | **重点监管**：脱敏、授权、公开展示合规（公开展示闸门） | 维修质量鉴定、相册六阶段逐本质检 |
| 门店/GEO 内容 | 展示商家填写或用户授权的内容，服务搜索与 GEO 收录 | 保证宣传绝对真实、对线下维修结果负责 |
| 用户转化 | 咨询/预约线索转交门店 | 订单、支付、退款、售后仲裁 |
| AI / 搜索 | 沉淀可索引、可引用的脱敏内容与结构化字段 | 替用户选店、承诺最低价或效果 |

**合规与治理（轻量、事后）**：

1. **免责声明**：服务/门店/案例等详情页常驻——内容为商家发布或用户授权展示，仅供参考；实际方案与费用请与门店线下确认；
2. **用户举报**：「举报虚假信息」入口（夸大宣传、虚假价格、伪造案例等），见 `09_客服与售后入口.md` §10；
3. **事后治理**：运营 **抽查** + 举报核实 → 下架服务、隐藏案例、限制预约、严重则冻结门店；**不做**商家服务方案发布前拦截。

**仍保留前置审核的少数场景**：商家 **入驻资质**、**公开案例** 公开展示（卷三 OPS-MASK）。与服务上架无关。

详见：`docs/00_开发计划.md` §7.2.3、§7.3；`docs/06_平台运营后台/01_运营后台PRD V2.0.md` §6（事后监管）。

| 场景 | 入口 | 用户得到什么 |

| **公域发现** | **H5 内容站** `geo.simplewin.cn`、**公众号** 文章 | 公开案例、门店列表、服务说明、GEO 专题；搜索收录与 AI 引用主战场 |
| **私域承接** | 商家分享 **相册 / 门店** 小程序卡片或链接 | 深链直达，单店上下文；全程不见跨店推荐 |

| **历史兼容** | 旧书签、搜索页、GEO 小程序页深链 | **仅深链兼容 / 待下线**（DS-DOC-03）；不新增此类入口 |

**不再作为产品入口（阶段 A 起）**：小程序五 Tab 发现（首页推荐、服务/案例/门店 Tab、全站搜索框）。

**原 V2.0 枚举（对照）**：微信搜索、分享案例/GEO、H5 跳小程序、地推二维码、活动页 — 其中 **「搜索进小程序逛案例」** 已改为 **搜索进工具首页 + 发现走 H5**。

### **3.2 咨询与预约（替代原「下单」）**

用户通过以下方式产生线索，**不产生平台订单**：

| 方式 | 说明 |
| --- | --- |
| 电话咨询 | 门店详情页、案例页一键拨号 |
| 在线咨询 | 表单或 IM 留资（若接入） |
| 预约到店 | 填写服务类型、门店、车型、问题描述、期望到店时间、手机号 |

流程：

```
用户查看服务/案例/门店
→ 点击联系门店 / 预约到店
→ 填写咨询信息并勾选平台说明
→ 提交给门店
→ 门店线下联系用户
→ 用户自行决定是否到店、是否维修、如何付款
```

提交前须展示并勾选：

> 平台将把你的咨询信息发送给所选门店。平台不提供线上最终报价，不参与线下维修服务、收款、发票、质保和售后。实际维修方案和费用需你与门店线下确认。

### **3.3 事故车特殊说明**

- 不做线上最终报价；
- 仅展示类似案例、维修项目和费用影响因素；
- 须勾选：「我已知晓事故车需到店检测后由门店报价。」

## **4. 商家如何管理内容与线索**

### **4.1 商家工作台入口**

商家通过同一小程序「商家工作台」管理：

- 咨询线索；
- 服务相册（证据链）；
- 历史案例；
- 服务项目展示；
- 门店资料；
- 数据概览。

**已移除**：新订单、待接单、维修中订单、待完工、售后订单、结算。

### **4.2 咨询线索管理**

Tab：待联系 / 已联系 / 已到店 / 已关闭。

商家可：查看线索、联系用户、标记到店、关闭线索、填写备注。

### **4.3 服务相册创建（核心）**

相册**不再绑定平台订单**，由商家主动创建：

```
创建服务相册（关联门店、服务项目、用户手机号/认领方式）
→ 按六阶段记录（接车、检测、方案报价、配件确认、施工、交付）
→ 填写配件类型（原厂/同质/品牌/副厂/再制造/拆车件等）
→ 非原厂件触发车主确认页（勾选 + 确认，MVP 可先电子确认）
→ 上传关键凭证照片（新旧件对比、包装、编码等）
→ 完成脱敏预览
→ 保存 / 提交
→ 邀请用户查看
→ 用户可选授权公开为案例
```

详见 `docs/04_维修过程相册/01_维修过程相册PRD V2.0.md`。

## **5. 维修过程如何记录**

### **5.1 相册定位升级**

> **服务相册 = 维修过程记录 + 配件告知确认 + 车主确认留痕 + 照片证据 + 时间线 + 可导出证据包**

六阶段：

1. 接车记录；
2. 检测诊断；
3. 维修方案与报价确认；
4. 配件告知与确认（**核心合规节点**）；
5. 施工过程记录；
6. 完工交付确认。

### **5.2 配件类型标准化**

必须使用标准类型，禁止模糊词（如「原装品质」）：

- 原厂件；
- 同质配件；
- 品牌件；
- 副厂件；
- 再制造件；
- 拆车件；
- 修复件 / 翻新件；
- 以修代换；
- 未知来源件（高风险，不建议允许）。

非原厂件须：显著风险提示、车主主动勾选、明确确认按钮文案（如「确认使用非原厂件」）。

### **5.3 阻断式确认（按风险等级）**

| 风险 | 处理方式 |
| --- | --- |
| 低（L1） | 普通记录，可选拍照 |
| 中（L2） | 必须拍照；必须记录配件类型；非原厂件需确认 |
| 高（L3/L4） | 强制方案确认、配件确认、新旧件对比；L4 强制电子签名与证据包 |

关键规则：

- 未完成配件告知确认，不可进入施工（含非原厂件时）；
- 方案变更须重新确认；
- 关键凭证照片删除须留痕。

### **5.4 三套视图**

| 视图 | 面向 | 用途 |
| --- | --- | --- |
| 车主视图 | 车主 | 进度、待确认事项、过程照片、费用说明 |
| 门店内部视图 | 员工 | 待办清单、合规检查、风险提醒 |
| 争议/合规视图 | 门店/平台 | 一键导出 PDF 证据包 |

### **5.5 照片规范**

- 自动水印：订单编号、脱敏车辆、门店、拍摄时间、节点、拍摄人；
- 保存上传人、原图 hash、修改/删除记录；
- 关键凭证类照片不可随意删除。

### **5.6 图片数量限制（MVP）**

| 项目 | 规则 |
| --- | --- |
| 单个相册图片上限 | 20 张（后续版本可调整） |
| 视频 | 暂不支持 |
| 压缩 | 必须 |
| 公开展示 | 仅脱敏图 |

## **6. 相册如何生成公开案例**

### **6.1 案例来源（V2.0）**

1. **用户授权服务相册案例**（门店创建相册 + 用户同意脱敏公开）；
2. **商家上传历史案例**；
3. **门店公开案例**（经平台展示审核）。

**已删除**：「平台订单案例」。

### **6.2 服务相册转公开案例流程**

```
服务相册完成
→ 用户收到授权提醒
→ 用户查看脱敏预览（可手动打码/裁剪）
→ 用户同意 / 拒绝公开
→ 系统自动脱敏与风险识别
→ 平台展示审核（隐私脱敏、展示规范、违规内容）
→ 审核通过 → 发布 H5 公开案例页 + 纳入 GEO 索引
```

审核说明：

> 平台展示审核仅针对隐私脱敏、图片展示规范和违规内容，**不代表**平台对维修质量、配件真伪或价格合理性作出保证。

### **6.3 历史案例**

流程与 V1.0 类似，须标记「商家历史案例 / 商家上传」，不得与「用户授权案例」混称。

## **7. 服务相册与公开案例的边界**

| 维度 | 服务相册（私密） | 公开案例 |
| --- | --- | --- |
| 面向 | 车主、门店、争议处理 | 潜在车主、搜索、分享 |
| 内容 | 完整过程、配件确认、价格、签名 | 脱敏摘要、部分授权图片 |
| 禁止公开 | — | 完整车牌、VIN、手机号、完整结算单、供应商敏感信息、车主签名 |

## **8. 分享规则**

MVP 保留：

- 公开案例分享；
- 门店页分享；
- 服务说明页 / GEO 专题页分享；
- 用户已授权公开的脱敏相册分享。

MVP 不做：

- 按浏览/点击/互动发现金；
- 多级分销；
- 与订单绑定的评价奖励；
- 「有效订单」归因奖励。

## **12. MVP 商业策略结论**

1. 对样板商家免费提供证据链相册与 GEO 展示，换取案例与内容供给；
2. 用高质量脱敏案例提升用户信任，驱动咨询而非在线成交；
3. 用 H5/GEO 获取自然流量；
4. 暂不做法定意义上的平台交易与评价奖励；
5. 验证「工具价值 + 内容资产 + 线索转化」后再考虑商业化深化。
