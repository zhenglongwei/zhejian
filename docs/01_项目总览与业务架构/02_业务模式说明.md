## **2. 平台业务总览**

平台连接三类核心对象：

1. 车主用户；
2. 维修商家；
3. 平台运营方。

通过以下业务闭环产生价值：

```
车主发现服务
→ 查看门店和案例
→ 下单/预约
→ 商家接单
→ 到店维修
→ 创建维修相册
→ 用户查看过程
→ 用户评价
→ 用户授权公开
→ 平台脱敏审核
→ 生成公开案例
→ 案例带来新的咨询和订单

```

平台的核心不是一次性撮合交易，而是让每次维修沉淀为：

1. 用户维修档案；
2. 商家可信案例；
3. 平台内容资产；
4. 搜索和 AI 可读取的数据源。

## **3. 用户如何下单**

### **3.1 用户入口**

用户可通过以下入口进入小程序：

1. 微信搜索；
2. 商家分享；
3. 用户分享的案例；
4. H5 公开案例页跳转；
5. H5 门店页跳转；
6. 地推二维码；
7. 平台活动页。

---

### **3.2 下单类型**

平台支持两类下单方式。

---

#### **3.2.1 标准服务在线支付**

适用于：

- 小保养；
- 电瓶更换；
- 轮胎服务；
- 部分明确型号的刹车片更换；
- 部分标准化服务套餐。

流程：

1. 用户选择服务；
2. 选择门店；
3. 选择车辆；
4. 选择预约时间；
5. 确认价格；
6. 微信支付；
7. 商家接单；
8. 用户到店服务。

---

#### **3.2.2 复杂维修预约到店报价**

适用于：

- 刹车系统复杂维修；
- 空调维修；
- 底盘维修；
- 钣喷修复；
- 事故车维修；
- 新能源专项；
- 疑难故障维修。

流程：

1. 用户选择服务类型；
2. 查看类似案例；
3. 选择门店；
4. 提交预约；
5. 可填写问题描述和上传车辆照片；
6. 商家接单；
7. 用户到店检测；
8. 商家线下报价；
9. 用户确认后维修；
10. 维修过程可通过平台相册记录。

说明：

- 事故车不支持线上报价；
- 平台只展示类似案例的价格区间；
- 实际价格由门店检测后确定。

## **4. 商家如何接单**

### **4.1 商家接单入口**

商家通过同一个微信小程序中的“商家工作台”处理订单。

商家登录后，根据账号权限可访问：

- 新订单；
- 待接单；
- 已接单；
- 待到店；
- 维修中；
- 待完工确认；
- 已完成；
- 售后中；
- 历史案例；
- 门店数据。

---

### **4.2 商家接单动作**

商家收到订单后可执行：

1. 查看订单详情；
2. 查看用户预约时间；
3. 查看车辆信息；
4. 查看服务项目；
5. 接单；
6. 拒单；
7. 修改预约时间；
8. 联系用户；
9. 标记用户到店；
10. 开始维修；
11. 上传维修相册；
12. 发起完工确认；
13. 处理售后。

---

### **4.3 复杂维修订单处理**

对于事故车、底盘、空调、钣喷、新能源专项等复杂服务：

1. 用户提交的是预约/咨询单；
2. 商家接单后与用户确认到店时间；
3. 到店后检测；
4. 商家可在线下形成维修报价；
5. 平台允许商家在订单内记录最终维修项目和价格区间；
6. 若用户同意，可生成维修相册和案例。

---

## **5. 维修过程如何记录**

### **5.1 相册默认创建原则**

平台订单创建后，系统默认生成一个维修相册。

但相册上传不强制。

原则是：

> 平台不强制商家上传节点照片，而是通过案例曝光、用户信任、门店转化和推荐排序，引导商家主动上传。

如果商家不上传：

- 用户订单中相册为空；
- 不影响订单基础履约；
- 但门店透明度、案例展示和后续曝光可能受到影响。

---

### **5.2 相册节点建议**

系统根据服务类型推荐拍摄节点，但商家可以跳过任意节点。

示例：刹车片/刹车盘服务建议节点：

1. 到店车辆外观；
2. 里程表；
3. 旧刹车片状态；
4. 拆卸过程；
5. 新旧配件对比；
6. 安装完成；
7. 完工检查；
8. 用户确认。

说明：

- 节点为建议，不做强制；
- 但节点完整度会影响案例质量；
- 完整案例在门店页和平台页展示权重更高。

---

### **5.3 图片数量限制**

MVP 阶段考虑服务器成本：


| 项目       | 规则            |
| -------- | ------------- |
| 单个相册图片上限 | 20 张          |
| 视频       | 暂不支持          |
| 图片压缩     | 必须压缩          |
| 原图保留     | 可在不影响展示的前提下删除 |
| 脱敏图      | 单独保存          |
| 用户订单内查看  | 原始图/未公开图      |
| 公开展示和分享  | 仅使用脱敏后图片      |

## **6. 相册如何生成案例**

### **6.1 案例来源**

平台案例分为两类：

1. 平台订单案例；
2. 商家历史案例。

---

### **6.2 平台订单案例生成流程**

```
订单完成
→ 商家提交完工
→ 系统生成案例草稿
→ 用户收到公开授权提醒
→ 用户查看脱敏预览
→ 用户可手动打码/裁剪
→ 用户确认是否公开
→ 系统自动脱敏
→ 输出风险指标
→ 高风险进入人工审核
→ 审核通过
→ 发布 H5 公开案例页

```

---

### **6.3 历史案例生成流程**

商家可上传历史案例，用于冷启动和门店展示。

流程：

```
商家创建历史案例
→ 填写车型/服务/维修说明
→ 上传图片
→ 系统提示隐私责任
→ 自动脱敏
→ AI 风险筛选
→ 平台人工审核
→ 发布到门店页/案例库

```

历史案例必须标记为：

> 商家历史案例，非平台订单案例。

历史案例可信等级低于平台订单案例。

---

### **6.4 历史案例隐私限制**

商家上传历史案例时，必须遵守：

1. 不得上传可能暴露车主身份的信息；
2. 不得上传未脱敏车牌；
3. 不得上传 VIN、手机号、姓名、身份证件；
4. 不得上传人脸；
5. 不得上传客户聊天记录；
6. 不建议上传稀有车型案例；
7. 不得上传能通过车辆外观、车牌残留、特殊改装等识别车主身份的图片；
8. 如平台发现侵权风险，可立即下架。

上传前必须提示商家：

> 请确认你已取得必要授权，或已确保案例内容不会侵犯客户隐私。因上传未授权内容导致的纠纷，商家应承担相应责任。平台将进行脱敏和审核，但不代表商家可免除隐私保护义务。

## **7. 用户如何评价并获得奖励**

### **7.1 评价触发条件**

用户订单完成后，可收到评价提醒。

评价入口包括：

1. 订单详情页；
2. 微信服务通知；
3. 维修相册页；
4. 个人中心待评价。

---

### **7.2 基础 6 维评价**

用户完成以下 6 个维度评价后，可获得基础奖励资格：

1. 服务态度；
2. 专业程度；
3. 价格透明；
4. 维修过程透明；
5. 完工效果；
6. 是否愿意推荐。

每项可采用 1-5 分评分，并允许填写文字说明。

---

### **7.3 奖励计算原则**

用户奖励综合考虑三个维度：

1. 服务复杂度；
2. 车价；
3. 订单金额。

基础公式可设计为：

```
用户评价奖励 = 复杂度基础奖励 × 车价系数 × 订单金额系数

```

同时受对应复杂度等级的封顶限制。

---

### **7.4 复杂度基础奖励**


| 复杂度      | 基础奖励  | 单笔封顶   |
| -------- | ----- | ------ |
| L1       | 10 元  | 50 元   |
| L2       | 30 元  | 200 元  |
| L3       | 100 元 | 800 元  |
| L4       | 300 元 | 2000 元 |
| L4 保险事故车 | 300 元 | 3000 元 |


---

### **7.5 车价系数建议**

车价系数用于体现高价值车辆维修决策风险更高。


| 车辆估值      | 系数  |
| --------- | --- |
| 10 万元以下   | 1.0 |
| 10-30 万元  | 1.2 |
| 30-50 万元  | 1.5 |
| 50-100 万元 | 1.8 |
| 100 万元以上  | 2.0 |


说明：

- MVP 可先由用户选择车辆品牌车型后系统估算；
- 无法判断车价时默认系数为 1.0；
- 后续可接入车型估值数据。

---

### **7.6 订单金额系数建议**


| 订单金额         | 系数  |
| ------------ | --- |
| 500 元以下      | 0.5 |
| 500-3000 元   | 1.0 |
| 3000-10000 元 | 1.5 |
| 10000 元以上    | 2.0 |


说明：

- 标准线上支付订单以支付金额为准；
- 复杂维修可由商家填写最终金额区间；
- 平台可要求上传维修明细单、定损单或收据作为凭证；
- 无法确认金额时，按最低系数计算。

---

### **7.7 奖励发放条件**

用户获得奖励需满足：

1. 订单已完成；
2. 用户完成基础 6 维评价；
3. 评价内容通过平台风控；
4. 无明显刷评、恶意评价、虚假交易；
5. 订单无退款或重大售后争议；
6. 复杂维修订单需有必要凭证。

---

### **7.8 奖励发放方式**

MVP 可采用：

1. 微信零钱红包，若合规和接口可行；
2. 平台余额；
3. 优惠券；
4. 人工审核后线下打款。

建议 MVP 初期采用：

> 平台余额 + 满额提现/人工审核提现。

这样便于控制风险。

## **8. 分享收益如何产生**

### **8.1 MVP 阶段结论**

MVP 阶段不做按浏览、点击、互动产生的分享收益。

原因：

1. 微信生态对诱导分享、现金裂变、返利拉新较敏感；
2. 按浏览或互动发钱容易引发刷量；
3. 汽修交易低频高客单，按流量结算不容易判断真实贡献；
4. 多层级收益容易触碰分销合规风险；
5. 会增加风控、财务、提现和客服复杂度。

---

### **8.2 MVP 保留普通分享**

MVP 保留：

1. 案例分享；
2. 门店分享；
3. 服务方案分享；
4. 用户维修档案脱敏分享；
5. 分享浏览数据展示。

但不直接根据浏览量发现金。

---

### **8.3 推荐成交奖励预留**

后续可考虑：

> 单层推荐成交奖励。

规则建议：

1. 只能奖励直接推荐人；
2. 不做多级分销；
3. 必须产生真实订单；
4. 必须完成支付或确认到店维修；
5. 必须通过退款期和售后观察期；
6. 奖励金额后台配置；
7. 高金额订单需人工审核；
8. 不以“拉人头”为奖励条件。

## **9. 平台如何变现**

平台商业模式包括 6 类。

---

### **9.1 交易佣金**

平台从订单金额中抽取佣金。

规则：

1. 前期样板商家可免佣；
2. 佣金比例后台可配置；
3. 可按服务类型配置；
4. 可按商家等级配置；
5. 可按城市配置；
6. 可设置起征金额和封顶金额。

示例配置：


| 服务类型   | MVP 佣金     |
| ------ | ---------- |
| 小保养    | 0%         |
| 电瓶     | 0%         |
| 轮胎     | 0%         |
| 刹车系统   | 0%         |
| 钣喷     | 0%         |
| 事故车    | 0%         |
| 后续标准佣金 | 3%-10% 可配置 |


MVP 阶段建议：

> 所有样板商家前 6-12 个月免佣，以换取案例上传和透明维修合作。

---

### **9.2 商家工具费**

后续可向商家收取工具费。

工具包括：

1. 维修相册；
2. 交付报告；
3. 门店案例库；
4. 员工账号；
5. 门店数据看板；
6. 自动生成案例；
7. H5 门店主页；
8. SEO 内容资产。

MVP 阶段不强制收费。

---

### **9.3 增值曝光**

后续可为商家提供增值曝光服务。

包括：

1. 城市页推荐；
2. 服务页推荐；
3. 车型页推荐；
4. 优质案例推荐；
5. 附近门店优先展示；
6. 品牌授权商家标识；
7. 透明度榜单推荐。

注意：

- 增值曝光不能完全替代真实服务质量；
- 推荐排序应综合案例质量、用户评价、资质、距离、响应速度等因素；
- 避免纯广告导致平台公信力下降。

---

### **9.4 线索服务费**

对于事故车、钣喷、复杂维修等非标准报价项目，平台可按有效线索收费。

有效线索标准可包括：

1. 用户留下联系方式；
2. 用户预约到店；
3. 商家确认有效；
4. 用户完成检测；
5. 用户最终成交。

MVP 阶段建议暂不收费，只做记录。

---

### **9.5 内容流量转化**

通过 H5 公开内容获得自然搜索和外部平台流量，再转化为：

1. 咨询；
2. 预约；
3. 门店访问；
4. 小程序下单；
5. 品牌合作。

核心内容资产包括：

- 公开案例页；
- 门店主页；
- 服务项目页；
- 城市服务页；
- 车型案例页；
- FAQ 问答页。

---

### **9.6 数据资产服务，后期**

长期可探索：

1. 车型常见故障数据；
2. 配件价格区间；
3. 维修方案参考；
4. 门店透明度指数；
5. 区域维修价格指数；
6. 新能源维修案例库。

注意：

- 必须建立在合法授权、脱敏、聚合统计基础上；
- 不出售个人隐私数据；
- 不出售可识别用户或车辆身份的数据。

## **10. 商家资质与服务复杂度匹配**

平台必须根据服务复杂度过滤商家承接权限。


| 复杂度   | 商家资质要求            | 是否可承接竞价/推荐订单 |
| ----- | ----------------- | ------------ |
| L1    | 三类及以上             | 可            |
| L2    | 二类或三类专项及以上        | 可            |
| L3    | 二类及以上，需对应技师       | 仅符合资质可       |
| L4    | 一类维修资质，需高级技师和专项设备 | 仅一类可         |
| 新能源三电 | 新能源专项维修资质         | 无资质不可承接      |


品牌授权不是强制要求，但会影响推荐排序。

## **12. MVP 商业策略结论**

MVP 阶段不以短期变现为第一目标。

第一阶段核心策略是：

1. 对样板商家免佣；
2. 重点换取真实案例和相册使用；
3. 用高质量案例提升用户信任；
4. 用 H5 页面获得搜索和外部流量；
5. 用评价奖励获取真实用户反馈；
6. 暂缓复杂分享收益；
7. 逐步验证交易和内容的相互促进关系。
