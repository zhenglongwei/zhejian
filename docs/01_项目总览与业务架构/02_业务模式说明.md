## **2. 平台业务总览**

平台连接三类核心对象：

1. 车主用户；
2. 维修商家；
3. 平台运营方。

核心业务闭环：

```
车主发现服务/案例/门店
→ 查看脱敏案例与服务说明
→ 电话咨询 / 在线咨询 / 预约到店
→ 线下检测、报价、维修、付款（门店完成）
→ 商家创建服务相册（证据链 + 配件告知确认）
→ 用户查看服务相册
→ 用户授权脱敏公开
→ 平台审核并生成公开案例 / GEO 内容页
→ 内容带来新的搜索流量和咨询线索
```

平台的核心不是一次性撮合交易，而是让每次线下维修可沉淀为：

1. 用户可查看的服务相册（含配件确认留痕）；
2. 商家风控证据与可导出凭证；
3. 平台脱敏内容资产；
4. 搜索和 AI 可读取的数据源。

**平台不提供**：在线交易、在线支付、统一报价、订单履约、维修售后处理、质量担保。

### **2.1 展示平台定位（2026-06-02 定调）**

辙见在 V2.0 阶段的本质角色是 **信息展示与引流平台**（类「打广告」），而非交易方或质量背书方：

| 维度 | 平台做什么 | 平台不做什么 |
| --- | --- | --- |
| 商家服务方案 | 商家自助发布、展示参考价与说明 | 前置审核、价格/质量担保、交易撮合 |
| 公开案例 | **重点监管**：脱敏、授权、公开展示合规 | 维修质量鉴定、相册六阶段质检 |
| 门店/GEO 内容 | 展示商家自行填写或授权的内容 | 保证宣传真实、对线下结果负责 |
| 用户转化 | 咨询/预约线索转交门店 | 订单、支付、退款、售后仲裁 |

**合规手段（轻量）**：

1. **免责声明**：服务/门店/案例等详情页常驻说明——内容为商家自行发布或授权展示，平台不对价格、质量、效果负责；
2. **用户举报**：提供「举报虚假信息」入口，供用户反馈夸大宣传、虚假价格、伪造案例等；
3. **事后治理**：运营通过抽查、举报核实进行处罚（强制下架服务、隐藏案例、限制预约、严重则冻结门店/商家），**不做发布前拦截**。

详见：`docs/10_技术架构与接口/09_客服与售后入口.md` §10；开发任务见 `docs/00_开发计划.md` §7.3。

## **3. 用户如何联系门店**

### **3.1 用户入口**

1. 微信搜索；
2. 商家/用户分享的案例或 GEO 页；
3. H5 公开案例页、门店页跳转小程序；
4. 地推二维码；
5. 平台活动页。

### **3.2 咨询与预约（替代原「下单」）**

用户通过以下方式产生线索，**不产生平台订单**：

| 方式 | 说明 |
| --- | --- |
| 电话咨询 | 门店详情页、案例页一键拨号 |
| 在线咨询 | 表单或 IM 留资（若接入） |
| 预约到店 | 填写服务类型、门店、车型、问题描述、期望到店时间、手机号 |

流程：

```
用户查看服务/案例/门店
→ 点击联系门店 / 预约到店
→ 填写咨询信息并勾选平台说明
→ 提交给门店
→ 门店线下联系用户
→ 用户自行决定是否到店、是否维修、如何付款
```

提交前须展示并勾选：

> 平台将把你的咨询信息发送给所选门店。平台不提供线上最终报价，不参与线下维修服务、收款、发票、质保和售后。实际维修方案和费用需你与门店线下确认。

### **3.3 事故车特殊说明**

- 不做线上最终报价；
- 仅展示类似案例、维修项目和费用影响因素；
- 须勾选：「我已知晓事故车需到店检测后由门店报价。」

## **4. 商家如何管理内容与线索**

### **4.1 商家工作台入口**

商家通过同一小程序「商家工作台」管理：

- 咨询线索；
- 服务相册（证据链）；
- 历史案例；
- 服务项目展示；
- 门店资料；
- 数据概览。

**已移除**：新订单、待接单、维修中订单、待完工、售后订单、结算。

### **4.2 咨询线索管理**

Tab：待联系 / 已联系 / 已到店 / 已关闭。

商家可：查看线索、联系用户、标记到店、关闭线索、填写备注。

### **4.3 服务相册创建（核心）**

相册**不再绑定平台订单**，由商家主动创建：

```
创建服务相册（关联门店、服务项目、用户手机号/认领方式）
→ 按六阶段记录（接车、检测、方案报价、配件确认、施工、交付）
→ 填写配件类型（原厂/同质/品牌/副厂/再制造/拆车件等）
→ 非原厂件触发车主确认页（勾选 + 确认，MVP 可先电子确认）
→ 上传关键凭证照片（新旧件对比、包装、编码等）
→ 完成脱敏预览
→ 保存 / 提交
→ 邀请用户查看
→ 用户可选授权公开为案例
```

详见 `docs/04_维修过程相册/01_维修过程相册PRD V2.0.md`。

## **5. 维修过程如何记录**

### **5.1 相册定位升级**

> **服务相册 = 维修过程记录 + 配件告知确认 + 车主确认留痕 + 照片证据 + 时间线 + 可导出证据包**

六阶段：

1. 接车记录；
2. 检测诊断；
3. 维修方案与报价确认；
4. 配件告知与确认（**核心合规节点**）；
5. 施工过程记录；
6. 完工交付确认。

### **5.2 配件类型标准化**

必须使用标准类型，禁止模糊词（如「原装品质」）：

- 原厂件；
- 同质配件；
- 品牌件；
- 副厂件；
- 再制造件；
- 拆车件；
- 修复件 / 翻新件；
- 以修代换；
- 未知来源件（高风险，不建议允许）。

非原厂件须：显著风险提示、车主主动勾选、明确确认按钮文案（如「确认使用非原厂件」）。

### **5.3 阻断式确认（按风险等级）**

| 风险 | 处理方式 |
| --- | --- |
| 低（L1） | 普通记录，可选拍照 |
| 中（L2） | 必须拍照；必须记录配件类型；非原厂件需确认 |
| 高（L3/L4） | 强制方案确认、配件确认、新旧件对比；L4 强制电子签名与证据包 |

关键规则：

- 未完成配件告知确认，不可进入施工（含非原厂件时）；
- 方案变更须重新确认；
- 关键凭证照片删除须留痕。

### **5.4 三套视图**

| 视图 | 面向 | 用途 |
| --- | --- | --- |
| 车主视图 | 车主 | 进度、待确认事项、过程照片、费用说明 |
| 门店内部视图 | 员工 | 待办清单、合规检查、风险提醒 |
| 争议/合规视图 | 门店/平台 | 一键导出 PDF 证据包 |

### **5.5 照片规范**

- 自动水印：订单编号、脱敏车辆、门店、拍摄时间、节点、拍摄人；
- 保存上传人、原图 hash、修改/删除记录；
- 关键凭证类照片不可随意删除。

### **5.6 图片数量限制（MVP）**

| 项目 | 规则 |
| --- | --- |
| 单个相册图片上限 | 20 张（后续版本可调整） |
| 视频 | 暂不支持 |
| 压缩 | 必须 |
| 公开展示 | 仅脱敏图 |

## **6. 相册如何生成公开案例**

### **6.1 案例来源（V2.0）**

1. **用户授权服务相册案例**（门店创建相册 + 用户同意脱敏公开）；
2. **商家上传历史案例**；
3. **门店公开案例**（经平台展示审核）。

**已删除**：「平台订单案例」。

### **6.2 服务相册转公开案例流程**

```
服务相册完成
→ 用户收到授权提醒
→ 用户查看脱敏预览（可手动打码/裁剪）
→ 用户同意 / 拒绝公开
→ 系统自动脱敏与风险识别
→ 平台展示审核（隐私脱敏、展示规范、违规内容）
→ 审核通过 → 发布 H5 公开案例页 + 纳入 GEO 索引
```

审核说明：

> 平台展示审核仅针对隐私脱敏、图片展示规范和违规内容，**不代表**平台对维修质量、配件真伪或价格合理性作出保证。

### **6.3 历史案例**

流程与 V1.0 类似，须标记「商家历史案例 / 商家上传」，不得与「用户授权案例」混称。

## **7. 服务相册与公开案例的边界**

| 维度 | 服务相册（私密） | 公开案例 |
| --- | --- | --- |
| 面向 | 车主、门店、争议处理 | 潜在车主、搜索、分享 |
| 内容 | 完整过程、配件确认、价格、签名 | 脱敏摘要、部分授权图片 |
| 禁止公开 | — | 完整车牌、VIN、手机号、完整结算单、供应商敏感信息、车主签名 |

## **8. 分享规则**

MVP 保留：

- 公开案例分享；
- 门店页分享；
- 服务说明页 / GEO 专题页分享；
- 用户已授权公开的脱敏相册分享。

MVP 不做：

- 按浏览/点击/互动发现金；
- 多级分销；
- 与订单绑定的评价奖励；
- 「有效订单」归因奖励。

## **9. 平台如何变现**

MVP 不以交易佣金为第一目标。

| 模式 | MVP | 说明 |
| --- | --- | --- |
| 交易佣金 | **不做** | 无平台订单 |
| 商家工具费 | 预留 | 证据链相册、证据包导出、员工账号、数据看板 |
| 增值曝光 | 预留 | GEO 页推荐、案例推荐、门店优先展示 |
| 线索服务费 | 记录不收费 | 有效咨询/到店线索，成熟后可按线索计费 |
| 内容流量转化 | **核心** | GEO/H5 带来咨询，商家为内容工具付费 |
| 数据资产服务 | 后期 | 聚合价格指数、故障洞察等，须脱敏合规 |

资质仍用于**展示与内容分类**，不用于「平台派单接单」。

| 复杂度 | 商家资质建议 | 说明 |
| --- | --- | --- |

| L3 | 二类及以上 | 高完整度案例优先展示 |

## **12. MVP 商业策略结论**

1. 对样板商家免费提供证据链相册与 GEO 展示，换取案例与内容供给；
2. 用高质量脱敏案例提升用户信任，驱动咨询而非在线成交；
3. 用 H5/GEO 获取自然流量；
4. 暂不做法定意义上的平台交易与评价奖励；
5. 验证「工具价值 + 内容资产 + 线索转化」后再考虑商业化深化。
