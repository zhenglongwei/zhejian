# 术语表

> 版本：V2.0。与各端 V2.0 主 PRD 对齐，持续维护。

| 术语 | 含义 |
| --- | --- |
| **服务相册** | 门店为一次线下维修创建的结构化记录，含六阶段时间线、配件信息与确认留痕；V2.0 核心对象，替代「订单相册」为主口径 |
| **订单相册** | 旧版绑定平台订单的相册概念；V2.0 MVP **不实现**平台订单，文档中仅作历史对照 |
| **维修证据链** | 接车→检测→方案→配件确认→施工→交付的完整可追溯记录，含照片、确认、日志 |
| **配件告知确认单** | 独立结构化模块：配件名称、标准类型、价格对比、风险提示、车主勾选与确认时间 |
| **配件标准类型** | 原厂件、同质配件、品牌件、副厂件、再制造件、拆车件、修复件、翻新件、以修代换、未知来源件 |
| **非原厂件** | 除原厂件外的上述类型；须触发强提醒与车主确认 |
| **阻断式确认** | 未完成配件告知/关键照片等节点时，系统不允许进入施工或完工 |
| **关键凭证照片** | 包装、编码、新旧件对比、报价确认、车主确认、验收等；删除须留痕 |
| **证据包** | 一键导出的 PDF 维修记录，含时间线、确认记录、照片摘要、操作日志、校验码 |
| **公开案例** | 脱敏 + 用户授权或商家历史案例 + 平台展示审核后发布的 H5 内容 |
| **平台展示审核** | 针对隐私脱敏、展示规范、违规内容的审核；**不代表**对维修质量或配件真伪背书 |
| **商家历史案例** | 商家上传的过往案例，须标记来源，不得称为平台订单案例 |
| **用户授权案例** | 由服务相册经用户同意脱敏公开形成的案例 |
| **咨询线索 Lead** | 用户通过电话、表单、预约等留下的联系意向；**非平台订单** |
| **GEO** | 生成式引擎优化；城市+服务/车型/故障等组合页，承接搜索与 AI 引用 |
| **参考价/参考区间** | 仅作信息展示，不构成平台报价或成交承诺 |
| **平台说明** | 关键页声明：实际维修、报价、收付款、质保、售后由门店线下负责 |
| **风险等级** | 低/中/高，决定拍照、配件确认、签名、证据包等强度（对应 L1–L4） |
| **六阶段** | 接车记录、检测诊断、方案报价、配件确认、施工记录、完工交付 |
| **三套视图** | 车主视图、门店内部视图、争议/合规视图 |
| **翻译层** | 车主查看时将专业术语转为易懂文案 |
| **lifecycle_status** | 内容/案例 GEO 生命周期状态（实现字段，非订单状态） |

## 已废弃或 MVP 不使用的术语

| 术语 | 说明 |
| --- | --- |
| 平台订单案例 | V2.0 无平台交易，改用用户授权案例 |
| 确认完工 | 交易履约概念，已删除 |
| 评价奖励 | 绑定订单的评价返现，MVP 不做 |
| 交易佣金 | 无平台订单，MVP 不做 |
| 草稿订单 | 旧版相册补订单概念，V2.0 以服务相册独立创建 |
| 六维评价（订单） | 交易完成后评价，MVP 不做；若未来做门店评价须非订单绑定 |
| 快速相册 | 可合并理解为「独立创建的服务相册」，不再强调补订单 |
