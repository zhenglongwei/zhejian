## 2. 战略定调（一句话）

| 系统 | 定位 | 用户感知 |
| --- | --- | --- |
| **小程序** | 商家私域的**服务相册 / 案例获取工具** | 「这是我的维修记录工具」 |
| **H5 / 公众号** | 授权案例的**公域内容发布与 GEO 获客阵地** | 「这是可以搜索、分享、咨询的维修案例内容」 |

**核心闭环**（优先于平台首页）：

```
商家创建相册 → 客户查看 → 客户授权公开 → 脱敏审核 → 生成案例文章
  → 发布 H5 / 公众号 → 站外浏览与搜索 → 咨询线索回流该门店
```

---

## 3. 设计原则（开发红线）

1. **小程序内不出现跨店发现**：无附近门店、无全站案例 Tab、无门店列表 Tab、无平台推荐、无「查看更多门店」。
2. **分享链路单店隔离**：商家分享出去的页面，全程只展示该店内容。
3. **冷启动首页是工具页**：微信搜索首次进入 = 工具入口，不是平台首页。
4. **公域发现走站外**：用户想「找店 / 看公开案例」→ H5 或公众号，不在小程序内满足。
5. **案例文章强绑门店**：H5 / 公众号底部转化只指向案例所属门店。
6. **旧页面先隐藏、后下线**：Tab 与导航先收口，路由保留兼容一段时间，避免分享链断裂。
7. **文档先于大改代码**：阶段 A 先更新 PRD 口径，再动 `app.json` 与首页。

## 4. 冷启动首页规格（阶段 A 必达）

用户**首次通过微信搜索进入**、无商家分享上下文时，展示**工具型首页**：

### 4.1 页面模块

| 模块 | 规则 |
| --- | --- |
| 主标题 | 维修相册 / 服务相册（与 Phase1 口径一致） |
| 副标题 | 查看门店为你创建的维修服务记录 |
| 主操作 | 扫描门店二维码、输入相册码 |
| 次操作 | 我的服务相册（引导登录） |
| 最近查看 | 仅 `localStorage` 有记录时展示**一条**，如「最近查看：XX汽修 · 继续查看」 |
| 我的服务记录 | 已登录且手机号关联到相册时，展示**用户自己的**相册列表（非推荐） |
| 站外出口 | 「想了解公开维修案例？前往辙见内容站」→ 打开 H5 |
| 商家入口 | 我是商家，进入工作台 |
| 帮助 | 联系客服、使用说明 |

### 4.2 禁止模块

- 附近门店、精选案例、推荐服务、GEO 专题、城市切换（平台发现）
- 跨店搜索入口
- 任何「浏览全部门店 / 全部案例」的小程序内链

### 4.3 底部 Tab（目标态）

```
首页（工具入口） | 我的
```

可选第二 Tab「相册」= 我的服务相册快捷入口；**不保留** 服务 / 案例 / 门店 Tab。

## 5. 总体规划

### 5.1 阶段划分

| 阶段 | 名称 | 目标 | 建议周期 |
| --- | --- | --- | --- |
| **A** | 小程序工具化收口 | 商家敢用：分享链路单店、冷启动工具首页、Tab 收缩 | 1～2 周 |
| **B** | 内容中台 + H5 文章化 | 相册能变成可发布文章，H5 案例页可获客 | 3～4 周 |
| **C** | H5 内容站扩展 | 城市 / 专题 / 内链 / sitemap，承接 GEO | 4～6 周 |
| **D** | 公众号 + 彻底收口 | 公众号分发、小程序公域页下线、招商话术统一 | 2～3 周 |

### 5.2 推荐执行顺序

```
阶段 A（文档 + 小程序壳）
  → 阶段 B（案例文章生成 + H5 案例页改版）  ← 与 A 可部分并行
  → 阶段 C（H5 聚合页 + SEO 基建）
  → 阶段 D（公众号 + 小程序公域下线）
```

**阻塞关系**：

- B 依赖 A 的口径定稿（授权公示渠道定义）
- C 依赖 B 的案例文章数据模型
- D 依赖 B 至少 1 条生产案例文章链路跑通
