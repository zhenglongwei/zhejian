## 1. 产品定位（修订）

### 1.1 一句话

> **现有系统负责开单收款；辙见是透明成交与合规留证的专用工具——让车主确认得清楚、出事举证得出来、愿意公开时晒得干净。**

### 1.2 不是什么

- 不是全套汽修 ERP（进销存 / 收银 / 排班）  
- 不是摄像头「透明车间」直播方案  
- 不是短期靠 GEO/AI 推荐变现的流量平台  

### 1.3 防 ERP 抄袭原则

ERP 最易抄「拍几张图 + 勾选」。护城河不在定位文案，而在：

1. **车主侧真正确认**（非店员代勾）  
2. **纠纷可举证的证据链**（确认留痕 + 证据包）  
3. **授权可公开的合规中台**（脱敏、审核、快照）——短期可免费养，但是能力必须在  

若产品停在「轻量工单插件」，中期必被 ERP 功能补丁吞掉。

## 2. 调整背景

1. 原「工具永久免费 + 99/980 卖公域收录」：**收费点难验证**，续费弱。  
2. GEO/AI 在汽修领域是**长期趋势**，短期难直接看到到店效果；用户更常经科普/AI 后去**高德/美团**成交。  
3. 市场：门店有管理系统付费习惯（轻量约千元级/年），但「过程确认 + 留证」仍是细分能力，宜与 ERP **并存补位**，勿正面替代。

## 3. 已拍板决策（第二～三轮）

| 编号 | 决策 | 说明 |
| --- | --- | --- |
| D1 | **主收费 = 工具专业版** | 已定 **480 元/年**（低门槛盖成本 + 付费认真感；工具非未来主收入） |

## 3.1 两段节奏：验证期 / 过线后（第三轮定稿）

> **原则**：先通过现有服务相册快速验证需求、积累公开案例；市场接受度高再扩充团队、上确认/证据包与正式套餐工程。

### 验证期（当前 · 建议 60～90 天）

| 做 | 不做 |
| --- | --- |
| 样板店运营：周活相册、推动「发布到公开网站」、记发布率与续费意愿 | 不为确认/证据包/额度工程扩人 |
| 用**已上线**能力跑闭环（相册 → 车主查看 → 发布 → 审核 → H5） | 不以 GEO/留资作为过线条件 |
| 话术按「工具定位 + 公示免费」；确认/证据包对外说「共建中/即将加强」，**禁止超卖已上线** | 不大改支付目录（除非阻塞试用的致命文案，可极小修补） |
| 过线指标见下表 | 不启动 A-PH2 / BIZ-SUB 大排期 |

**验证期过线参考（可微调，须书面确认才算过线）**：

| 指标 | 参考阈值 |
| --- | --- |
| 稳定合作店 | ≥8 家 |
| 供给节奏 | 多数店能维持每周 ≥2 个完工相册 |
| 公开案例 | 合计 ≥100，或样板店均 ≥15 |
| 付费/离不开信号 | ≥3 家明确愿续费或表示工具离不开 |

未过线：收缩功能幻想，只保相册体验与案例供给；不扩团队。

### 过线后（触发后再排期）

按顺序启动（详见 §7、§9 与开发计划 §4.5）：

1. **BIZ-SUB-***：专业版目录、体验额度、取消收录 gate、工作台文案（正式变现工程）  
2. **A-PH2-01 / 02 / 05**：确认、证据包、交车报告（防 ERP、撑工具定价）  
3. A-PH2-03/04、GEO/线索增值：按需后置  

套餐目标态（§4）在过线后落地；验证期可用运营赠送/宽松试用，不强制先上线额度阻断。

## 4. 套餐与权益（目标态）

### 4.1 设计原则

| 原则 | 含义 |
| --- | --- |
| 付费 = 立刻可感知的工具深度 | 额度、确认、证据包、交车报告、账号与统计 |
| 免费公示 = 养长期内容资产 | 降低公开阻力，服务未来 GEO |
| 不与 ERP 抢主系统位 | 可并存；录入路径保持轻量 |

### 4.2 套餐定义

| 套餐 | 代码建议 | 价格 | 一句话 |
| --- | --- | --- | --- |
| 体验版 | `free` | 0 | 试用透明留证；额度有限 |
| 专业版（主推） | `tool_480`（或替换原付费码） | **480 元/年** | 不限量（或高额度）透明成交与合规留证；低价门票养案例供给 |
| 增值包（以后） | 另议 | 另议 | 线索 / 专题分发等——**有数据后再立项** |

### 4.3 权益矩阵

| 能力 | 体验版 | 专业版 |
| --- | --- | --- |
| 服务相册创建与车主查看 | ✅ 月额度（建议 8～15 单/月，最终数另定） | ✅ 不限或高额度 |
| 标准配件录入、过程节点 | ✅ | ✅ |
| 车主确认 / 阻断（P0 缺口补齐后） | ✅ 基础或同专业版（确认能力不应用付费墙卡合规） | ✅ |
| 交车报告、证据包导出（P0 缺口补齐后） | 体验版可限次 | ✅ |
| 私域分享（发给微信等） | ✅ | ✅ |
| **发布到公开网站 + 基础收录** | ✅ **免费**（仍须车主发布 + 脱敏 + 审核） | ✅ **免费** |
| 门店短链 / 基础数字档案 | 基础 | ✅ |
| 工具使用统计（相册数、确认完成率等） | 极简 | ✅ |
| 获客看板 / 线索计费 | 观察期可记帐 | 正式增值另议 |

### 4.4 下线与存量

| 项 | 要求 |
| --- | --- |
| `index_99`（99 元收录） | **停止新购**；文案下线 |
| 原「获客版 980 / 付费才 sitemap」 | **废止为主结构**；gate 改为全档可基础收录（质量与审核规则仍适用） |
| 存量付费商户 | 有效期内权益不降；续费引导至 **专业版 480**（是否折算另定） |

## 5. 成功标准与线索（产品口径）

### 5.1 验证期主成功标准（当前）

> **周活相册、公开案例积累、门店「愿继续用 / 愿续费」信号。**  
> 确认完成率、证据包次数：能力未上线前**不作为**验证期考核。  
> 获客/GEO 为观察指标，不作过线必要条件与对外承诺。

### 5.1b 过线后主成功标准（工具变现期）

> 在验证期指标之上，增加：**确认完成率、证据包使用、专业版付费转化与续费**。

### 5.2 获客观察（非主收费交付）

| 层级 | 定义 | 用途 |
| --- | --- | --- |
| A | 站内预约/咨询/拨号 | 可记帐；以后增值用 |
| B | 门店回传「看过案例/内容才到店」 | 模式验证 |
| C | 高德/美团体感 | 仅复盘，不作文凭 |

### 5.3 禁止对外承诺

大模型必荐、搜索排名、保证留资/成交、付费买综合排序、平台参与质保售后。

## 6. 公开页转化（产品需求）

现实路径：科普/AI → 高德/美团 → 电话或到店。

公开案例/门店页 CTA 优先级：

1. 一键拨打门店电话  
2. 导航 / 复制地址  
3. 复制标准店名（便于去高德/美团搜）  
4. 站内留资（保留，非唯一主按钮）

## 7. 防 ERP · P0 能力清单

### 7.1 车主侧（A）

| ID | 能力 | 验收（白话） |
| --- | --- | --- |
| A1 | 交车前待办确认清单 | 车主能看到未完成确认项与状态 |
| A2 | 非原厂件阻断式确认 | 须车主主动确认；有时间与内容留痕 |
| A3 | 方案/增项变更再确认 | 关键变更后须重新确认 |
| A4 | 交车可读报告 | 一页人话说明过程与配件；可私域转发 |
| A5 | 发布到公开网站由车主决策 | 预览后同意；商家不能冒充授权 |

### 7.2 证据链（B）

| ID | 能力 | 验收（白话） |
| --- | --- | --- |
| B1 | 标准配件类型（禁模糊词） | 必须落在标准类型枚举 |
| B2 | 关键凭证照片规则 | 高风险缺图不可假完成；删图留痕 |
| B3 | 一键证据包导出 | 含时间线、确认、关键图；门店认可出事能用 |
| B4 | 操作审计 | 上传/改/删/确认可追溯 |
| B5 | 基础水印/元数据 | 图含门店/时间/节点等基础信息 |

### 7.3 与 ERP 并存（C）

| ID | 能力 | 验收（白话） |
| --- | --- | --- |
| C1 | 不替代开单收银 | 招商与产品说明明确并存 |
| C2 | 轻量录入 | 核心留证约 2～3 分钟可完成 |
| C3 | 按风险分级 | L1/L2 可简；L3/L4 强阻断——避免一律过重 |

### 7.4 公开层（D）——能力要在，短期不靠它收费

| ID | 能力 | 验收（白话） |
| --- | --- | --- |
| D1 | 脱敏预览 + 审核闸门 | 公开前脱敏与合规审核 |
| D2 | 授权/发布后快照不可乱改 | 公示内容不被商家悄悄改成广告版 |
| D3 | 基础公开展示免费 | 无「付费才收录」主 gate |

### 7.5 P0 明确不做

进销存/收银/完整 CRM、摄像头直播车间、以 GEO 为 P0 收费承诺、交易撮合与好评返现。

### 7.6 实施顺序（分两段）

**验证期（现在）**

1. **A5 + D1 + D2**（多数已有）→ 全力跑发布与案例供给  
2. **C1 + C2** → 话术与轻量体验，不堆功能  
3. 现网缺陷修复（阻塞样板店使用的 Bug）  

**过线后**

1. **BIZ-SUB-*** → 正式套餐与公示免费工程落地  
2. **A2 + A1 + B3** → 支撑工具收费与防 ERP  
3. **A4** → 交车报告，支撑续费感知  
4. **B1/B2/B4、C3、B5、A3** → 并行加深  
5. A-PH2-03/04、GEO/线索 → 更后

## 10. 验收标准

### 10.1 验证期结束时

1. 能汇报 §3.1 过线表四项是否达标（或书面决定延长/收缩）。  
2. 对外主卖点已是工具定位，不是「买收录/买 AI」。  
3. 未对样板店超卖「已上线确认/证据包」。  
4. 未因未过线而扩开发团队做 A-PH2/BIZ-SUB。

### 10.2 过线后产品验收（触发开发后）

1. 新购无 99 元收录档；基础公示不设付费 gate（BIZ-SUB）。  
2. 专业版价格与权益按 §4 展示。  
3. A2/B3 等收费关键缺口按排期交付。  
4. 招商明确与 ERP 并存（C1）。
