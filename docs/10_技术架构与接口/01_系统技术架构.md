## **2. 总体架构目标**

系统需要同时支持：

1. 用户端浏览、咨询预约、服务相册查看与授权公开；
2. 商家端入驻、服务相册、案例、线索管理；
3. 平台运营审核、GEO 内容管理；
4. H5/官网公开页用于 SEO/GEO；
5. 图片/视频上传与脱敏；
6. 案例、FAQ、结构化数据生成；
7. 消息通知；
8. 内容/隐私/合规审核。

**V2.0 不包含**：在线下单、微信支付、订单履约、退款、交易评价与奖励。

## **5. 架构演进策略**

## **5.1 MVP 阶段**

MVP 阶段不建议一开始拆复杂微服务，推荐：

```
一个主业务后端
一个媒体处理服务
一个 H5 内容生成服务
一个运营后台系统

```

### **MVP 推荐架构**

```
backend-main
├── 用户模块
├── 商家模块
├── 商品/服务模块
├── 咨询线索模块（lead）
├── 相册模块（service_album）
├── 案例内容模块
├── 审核模块
├── 消息模块
├── 搜索/SEO 模块
└── 后台管理模块

media-worker
├── 图片压缩
├── 缩略图生成
├── 视频转码
├── 隐私脱敏
└── 媒体审核预处理

content-render
├── H5公开页生成
├── SEO页面生成
├── sitemap生成
├── Schema生成
└── 静态页缓存刷新

```

---

## **5.2 成长期阶段**

当业务增长后，可逐步拆分：媒体服务、搜索/SEO 服务、消息通知、审核服务、案例内容服务等。

**V2.0 不规划**：支付服务、订单服务、奖励收益服务独立拆分。

---

## **5.3 微服务阶段**

当满足以下条件时再拆微服务：

1. 团队规模扩大；
2. 单体发布互相影响；
3. 某模块流量明显独立；
4. 支付、媒体、搜索等模块需要独立扩容；
5. 数据权限和安全要求更高；
6. 多端业务复杂度明显上升。

---

## **6. 核心业务域划分**


| 业务域 | 主要职责 |
| ----- | --------------------- |
| 用户域 | 登录、车辆（可选）、授权、隐私协议 |
| 商家域 | 门店、资质、服务范围 |
| 商品域 | 服务项目展示、价格影响因素、复杂度 |
| **线索域** | 咨询/预约提交、状态、商家跟进 |
| 相册域 | 服务相册、六阶段、配件确认 |
| 媒体域 | 上传、压缩、脱敏 |
| 案例内容域 | 案例生成、FAQ、SEO、公开状态 |
| 审核域 | 商家/服务/相册/案例审核 |
| SEO/GEO 域 | 公开页、sitemap、结构化数据 |

~~V1.0 已移除~~：订单域、支付域、评价域、奖励域、售后域。
| 消息域   | 订阅消息、短信、站内信、系统通知      |
| 收益域   | 奖励、佣金、商家结算、提现预留       |
| 搜索域   | 门店搜索、案例搜索、服务搜索、SEO收录  |

## **7. 关键业务链路**

## **7.1 用户下单链路**

```
用户选择服务
→ 填写车辆信息
→ 选择门店/预约时间
→ 创建订单
→ 创建微信支付单
→ 用户支付
→ 支付回调
→ 订单变为待接单/待服务
→ 通知商家

```

---

## **7.2 商家履约链路**

```
商家接单
→ 用户到店
→ 商家上传维修前图片
→ 检查并填写故障结论
→ 上传拆检/配件/维修后图片
→ 完工确认
→ 用户评价
→ 平台生成可公开案例草稿
→ 审核通过后公开

```

---

## **7.3 案例公开链路**

```
订单完工
→ 检查是否用户授权
→ 生成案例草稿
→ 生成AI摘要
→ 生成FAQ
→ 图片脱敏
→ 内容审核
→ 生成Schema
→ 发布公开案例页
→ 写入sitemap
→ 搜索引擎抓取

```

---

## **7.4 媒体上传链路**

```
前端申请上传凭证
→ 前端直传对象存储
→ 对象存储回调后端
→ 创建媒体记录
→ 创建压缩/缩略图任务
→ 创建脱敏任务
→ 审核通过
→ 业务绑定图片

```

---

## **7.5 支付链路**

```
创建业务订单
→ 创建支付单
→ 调用微信支付
→ 返回支付参数
→ 用户完成支付
→ 微信支付回调
→ 验签
→ 幂等处理
→ 更新支付单
→ 更新业务订单
→ 发送通知
→ 对账任务校验

```

## **8. 数据存储建议**


| 数据类型         | 推荐存储                        |
| ------------ | --------------------------- |
| 用户、商家、订单、支付  | MySQL/PostgreSQL            |
| 会话、验证码、限流、缓存 | Redis                       |
| 图片、视频、附件     | 对象存储，例如 COS/OSS/S3          |
| 搜索索引         | Elasticsearch/OpenSearch    |
| 日志           | ELK/Loki                    |
| 消息队列         | RabbitMQ/Kafka/RocketMQ/云队列 |
| 静态公开页        | CDN + 对象存储/SSR 服务           |
| 定时任务         | XXL-JOB/Quartz/云定时任务        |

5. 管理后台权限校验；
6. 限流；
7. 黑名单；
8. 请求日志；
9. API 路由；
10. 灰度发布；
11. 统一错误码；
12. 接口版本管理。

---

## **11. 鉴权体系**


| 端      | 鉴权方式                 |
| ------ | -------------------- |
| 用户小程序  | 微信登录 + 平台 Token      |
| 商家端    | 手机号/账号登录 + Token     |
| 运营后台   | 账号密码 + MFA 可选 + RBAC |
| H5公开页  | 大部分无需登录              |
| 支付回调   | 微信平台证书验签             |
| 对象存储回调 | 签名校验 + IP 白名单可选      |
| 内部服务   | 内部 Token / mTLS 可选   |


---

## **12. 权限模型**

运营后台采用 RBAC。

```
用户
→ 角色
→ 权限
→ 菜单/按钮/API

```

基础角色：

1. 超级管理员；
2. 平台运营；
3. 审核人员；
4. 财务人员；
5. 客服人员；
6. 内容运营；
7. 商家管理员；
8. 门店员工。

---

## **13. 消息通知体系**

支持：

1. 微信订阅消息；
2. 短信；
3. 站内信；
4. 后台通知；
5. 商家端通知；
6. 邮件，后期可选。

通知场景：

1. 用户支付成功；
2. 商家接单提醒；
3. 订单状态变更；
4. 退款结果；
5. 审核结果；
6. 案例发布；
7. 用户评价；
8. 售后处理。

---

## **14. 审核体系**

审核对象：

1. 门店资料；
2. 商家资质；
3. 图片；
4. 视频；
5. 维修案例；
6. FAQ；
7. 用户评价；
8. 商家回复；
9. 公开页面内容。

审核方式：

```
机器审核
→ 风险标记
→ 人工审核
→ 通过/驳回/修改后通过

```

## **15. SEO/GEO 技术要求**

H5/官网公开页需要支持：

1. SSR 或 SSG；
2. 页面 TDK；
3. canonical；
4. sitemap；
5. robots；
6. Breadcrumb；
7. JSON-LD Schema；
8. 静态页缓存；
9. 低质量页面 noindex；
10. 案例撤回后下线；
11. 页面发布后自动刷新 sitemap。

## **16. 日志与监控**

| 类型   | 内容            |
| ---- | ------------- |
| 访问日志 | 请求路径、IP、UA、耗时 |

1. API 成功率；
2. API 平均响应时间；
3. 支付回调成功率；
4. 上传成功率；
5. 媒体处理失败率；

7. 数据库慢查询；

---

## **17. 安全要求**

必须支持：

1. HTTPS；
2. 敏感数据脱敏；
3. 密码加密存储；
4. Token 过期机制；
5. 请求限流；
6. 防重复提交；
7. 支付回调验签；
8. 上传文件类型校验；
9. 文件大小限制；
10. 后台操作日志；
11. SQL 注入防护；
12. XSS 防护；
13. CSRF 防护，后台 Web 需要；
14. 对象存储私有读写，公开资源走 CDN；
15. 用户隐私授权管理。

## **19. P0 验收标准**

P0 阶段系统架构必须满足：

1. 用户端可以登录、下单、支付；
2. 商家端可以接单、上传图片、完工；
3. 后台可以管理用户、商家、订单、案例、审核；
4. 支持图片上传、压缩、缩略图、脱敏任务；
5. 支持微信支付和支付回调；
6. 支持退款申请和退款回调；
7. 支持案例公开页；
8. 支持基础 SEO；
9. 支持操作日志；
10. 支持基础监控；
11. 支持权限管理；
12. 支持用户授权与隐私脱敏。
