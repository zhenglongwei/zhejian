## **9. 统一请求规范**

前端应封装统一 request。

## **9.1 请求 Header**

```
Authorization: Bearer {token}
X-Client-Type: user-miniapp / merchant-miniapp / admin-web / official-web
X-App-Version: 1.0.0
X-Request-Id: uuid
Content-Type: application/json

```

---

## **9.2 响应格式**

```
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "req_xxx"
}

```

---

## **9.3 错误处理**


| code | 说明    | 前端处理     |
| ---- | ----- | -------- |
| 0    | 成功    | 正常展示     |
| 400  | 参数错误  | Toast 提示 |
| 401  | 未登录   | 跳转登录     |
| 403  | 无权限   | 提示无权限    |
| 404  | 数据不存在 | 展示空状态    |
| 409  | 状态冲突  | 刷新数据     |
| 429  | 请求过快  | 提示稍后重试   |
| 500  | 系统错误  | 提示并上报    |

## **10. 图片上传前端流程**

```
选择图片/视频
→ 校验大小和格式
→ 请求上传凭证
→ 直传对象存储
→ 通知后端或等待回调
→ 轮询媒体审核/处理状态
→ 绑定业务对象

```

---

## **11. 支付前端流程**

```
提交订单
→ 创建业务订单
→ 创建微信支付单
→ 获取支付参数
→ 调用 wx.requestPayment
→ 支付成功后查询订单状态
→ 展示支付结果

```

注意：

1. 前端支付成功不作为最终依据；
2. 订单状态以服务端支付回调为准；
3. 支付后需要主动查询订单状态。

| 事件              | 场景     |
| --------------- | ------ |
| page_view       | 页面访问   |
| service_view    | 服务详情查看 |

| payment_success | 支付成功   |
| upload_start    | 开始上传   |
| upload_success  | 上传成功   |
| case_view       | 查看案例   |
| consult_click   | 点击咨询   |


---

## **13. 前端安全要求**

1. 不在前端保存敏感密钥；
2. Token 使用安全存储；
3. 后台 Web 防 XSS；
4. 后台 Web 防 CSRF；
5. 上传文件限制类型和大小；
6. 不在 URL 中暴露手机号、订单号、用户 ID；
7. 用户隐私信息展示需脱敏；
8. 管理后台敏感操作需二次确认。

---

## **14. P0 验收标准**

P0 前端需要满足：

1. 用户小程序可登录、下单、支付、查看订单；
2. 商家端可接单、上传图片、完工；
3. 后台可管理用户、商家、订单、案例、审核；
4. 官网/H5 可展示公开案例页和门店页；
5. 前端统一请求封装；
6. 支持图片上传；
7. 支持支付结果查询；
8. 支持错误提示；
9. 支持基础埋点；
10. 支持权限控制。
