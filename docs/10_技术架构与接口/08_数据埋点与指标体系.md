# **2. 模块定位**

数据埋点与指标模块用于回答以下问题：

1. 用户从哪里来；
2. 用户看了什么服务；
3. 用户为什么没有下单；
4. 哪些门店转化好；
5. 哪些服务最受欢迎；
6. 哪些案例带来预约；
7. H5 是否带来小程序转化；
8. 商家是否及时接单和履约；
9. 奖励是否存在刷量风险；
10. 平台运营待办是否积压。

# **4. 核心业务漏斗**

## **4.1 用户下单漏斗**

```
进入小程序
→ 浏览服务/门店/案例
→ 点击预约/下单
→ 填写车辆信息
→ 提交订单
→ 支付/预约成功
→ 商家接单
→ 到店履约
→ 订单完成
→ 用户评价
→ 案例授权公开

```

---

## **4.2 H5 转化漏斗**

```
H5 页面访问
→ 浏览案例/服务/门店
→ 点击打开小程序
→ 进入小程序
→ 浏览对应内容
→ 提交订单/预约
→ 完成订单

```

---

## **4.3 商家履约漏斗**

```
收到订单
→ 查看订单
→ 接单
→ 用户到店
→ 开始维修
→ 上传过程相册
→ 提交完工
→ 用户确认
→ 获得评价

```

---

## **4.4 案例内容漏斗**

```
订单完成/商家提交历史案例
→ 生成案例草稿
→ 脱敏处理
→ 平台审核
→ 公开展示
→ H5 收录
→ 用户浏览
→ 转化预约

```

# **8. 核心指标定义**

## **8.1 用户增长指标**


| 指标    | 口径          |
| ----- | ----------- |
| UV    | 去重访问用户数     |
| PV    | 页面访问次数      |
| 新用户数  | 首次登录/访问用户数  |
| 登录转化率 | 登录用户 / 访问用户 |
| 城市用户数 | 按城市定位或选择统计  |


---

## **8.2 订单指标**


| 指标    | 口径            |
| ----- | ------------- |
| 订单创建数 | 创建成功的订单数      |
| 支付订单数 | 支付成功订单数       |
| 预约订单数 | 复杂服务预约成功订单    |
| 订单完成数 | 状态为已完成        |
| 支付转化率 | 支付订单数 / 创建订单数 |
| 完成率   | 完成订单数 / 接单订单数 |
| 取消率   | 取消订单数 / 创建订单数 |
| 退款率   | 退款订单数 / 支付订单数 |


---

## **8.3 商家履约指标**


| 指标     | 口径                 |
| ------ | ------------------ |
| 接单率    | 接单订单数 / 新订单数       |
| 平均接单时长 | 接单时间 - 新订单时间       |
| 拒单率    | 拒单订单数 / 新订单数       |
| 相册上传率  | 上传相册订单数 / 完成订单数    |
| 按时履约率  | 按预约时间履约订单数 / 接单订单数 |
| 平均评分   | 评价综合评分平均值          |


---

## **8.4 案例内容指标**


| 指标       | 口径                    |
| -------- | --------------------- |

---

## **8.5 H5 转化指标**


| 指标       | 口径                       |

# **10. 数据质量要求**

## **10.1 去重**

以下事件需要防重复：

1. 支付成功；
2. 订单创建；
3. 退款成功；
4. 奖励发放；
5. 审核通过；
6. 案例公开。

---

## **10.2 数据一致性**

业务结果类指标以业务表为准。

例如：

```
订单数以 order 表为准；
支付金额以 payment 表为准；
退款金额以 refund 表为准；
奖励金额以 reward 表为准。

```

行为埋点只用于过程分析。

---

## **10.3 隐私要求**

埋点中不得明文上传：

1. 手机号；
2. 车牌号；
3. VIN；
4. 身份证号；
5. 真实姓名；
6. 详细地址门牌号；
7. 原始图片 URL，如涉及隐私。

# **12. P0 验收标准**

必须满足：

1. 用户端核心页面访问可统计；
2. 服务、门店、案例详情浏览可统计；
3. 订单创建、支付、完成可统计；
4. H5 页面访问可统计；
5. H5 打开小程序点击可统计；
6. 小程序可接收 H5 来源参数；
7. 商家接单、拒单、完工可统计；
8. 平台审核操作可记录；
9. 奖励审核和发放可统计；
10. 平台运营工作台可查看基础数据；
11. 埋点不得上传敏感隐私明文；
12. 支付、退款、奖励等财务类指标必须以业务表为准。
