# 域名与部署架构

> **盈简科技** · 产品 **辙见**（辙见服务平台）  
> 仓库内部代号「辙见 / zhejian」，与产品名「辙见」指同一小程序项目。

---

## 1. 域名分工

| 域名 | 归属 | 用途 | 与本仓库关系 |
|---|---|---|---|
| **simplewin.cn** | 盈简科技官网 | 公司介绍、品牌、商务联系 | **独立站点**，不在本 monorepo 部署 |
| **geo.simplewin.cn** | 辙见专用 · 生产 | 小程序 API、SEO/GEO 公开内容、运营后台 | **本仓库生产部署目标** |
| **staging.geo.simplewin.cn** | 辙见专用 · 预发 | 发版前联调（API / H5 / `/admin/`） | **同机另一目录**，库与端口与生产隔离 |

**策略说明：** 微信生态内使用小程序；可索引、可分享、可 GEO 的内容放在 **geo.simplewin.cn**，规避微信封闭环境，并与公司主站域名隔离。预发仅用于验收，勿把正式用户流量指到 staging。

预发搭建步骤：`docs/12_测试验收部署与安全合规/B-INF_预发环境搭建.md`。

## 2. geo.simplewin.cn 路径规划

| 路径 | 服务 | 说明 |
|---|---|---|
| `/api/v1/*` | Node 主业务 API（`backend/`） | 小程序、商家端、后台均调用 |
| `/admin/*` | 运营后台 Web（`admin-web/`） | 审核、内容管理；SPA + 子路径部署 |
| `/` `/case/*` 等 | H5 公开页 | 仓库内 `h5/`，Nginx 直接映射，**不另建发布目录** |

```text
/var/www/zhejian/                 ← 生产 git clone
├── backend/                      ← :3100 → /api/v1/
├── h5/                           ← https://geo.simplewin.cn/...
└── admin-web/dist/               ← https://geo.simplewin.cn/admin/

/var/www/zhejian-staging/         ← 预发（结构相同）
├── backend/                      ← :3101；库 zhejian_staging
├── h5/                           ← https://staging.geo.simplewin.cn/...
└── admin-web/dist/               ← https://staging.geo.simplewin.cn/admin/
```

## 3. 与小程序的关系

| 端 | 访问方式 | 域名 |
|---|---|---|
| 用户/商家小程序（正式） | 微信内 | `https://geo.simplewin.cn/api/v1/...`（`ACTIVE_ENV=prod`） |
| 用户/商家小程序（预发联调） | 体验版/真机 | `https://staging.geo.simplewin.cn/api/v1/...`（`ACTIVE_ENV=staging`） |
| H5 公开案例 | 浏览器 / 搜索收录 | `https://geo.simplewin.cn/...`（生产）；预发同路径挂在 staging 子域 |
| 运营后台 | 浏览器 | 生产 `/admin/`；预发 `https://staging.geo.simplewin.cn/admin/` |

微信公众平台 **request 合法域名**须同时配置（不含路径）：

- `https://geo.simplewin.cn`
- `https://staging.geo.simplewin.cn`
