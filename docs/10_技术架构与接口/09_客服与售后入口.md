## 2. 模块定位

客服与售后入口是用户在遇到订单问题、服务争议、退款问题、维修疑问、奖励异常等情况时的统一求助入口。

核心目标：

1. 让用户快速找到问题解决路径；
2. 将常见问题通过 FAQ 自助解决；
3. 将订单相关问题关联到具体订单；
4. 将退款、投诉、售后等问题结构化提交；
5. 降低客服沟通成本；
6. 提高售后处理效率；
7. 为平台沉淀投诉、退款、商家履约质量数据；
8. 与消息通知、订单、退款、评价奖励模块联动。

## **5. 客服类型**

### **5.1 客服类型分类**


| 类型       | 说明           | MVP 是否支持 |
| -------- | ------------ | -------- |
| 微信客服     | 通过微信客服能力联系平台 | 支持       |

### **5.2 MVP 推荐方案**

MVP 阶段建议采用：

```
FAQ 自助帮助 + 微信客服 + 订单关联售后申请 + 平台待办处理

```

不建议 MVP 自研完整 IM。

### **7.4 FAQ 示例**

#### **7.4.1 如何取消订单？**

```
如订单尚未开始服务，可在订单详情页申请取消。若订单已支付，取消后退款规则以订单页面展示为准。

```

#### **7.4.2 维修进度为什么没有更新？**

```
维修进度由门店在服务过程中上传。若长时间未更新，可在订单详情页联系门店或平台客服。

```

#### **7.4.3 为什么事故车不能线上直接报价？**

```
事故车维修通常需要检测或拆检后确认损伤范围，因此平台仅展示检测、预约和方案确认信息，不提供线上最终报价承诺。

```

#### **7.4.4 奖励什么时候到账？**

```
评价或案例授权奖励需经过平台审核。审核通过后，奖励会按规则发放，具体状态可在奖励中心查看。

```

## **11. 退款入口**

### **11.1 退款入口展示条件**

展示退款入口需满足：

1. 用户已登录；
2. 订单属于当前用户；
3. 订单存在可退款金额；
4. 订单状态允许退款；
5. 不存在进行中的同类型退款申请；
6. 未命中平台限制规则。

### **11.2 退款原因**

用户申请退款时需选择原因：

```
不想要了
预约时间不合适
门店无法服务
商家未及时接单
价格/方案未达成一致
重复下单
服务未开始
其他原因

```

复杂维修/事故车场景可增加：

```
检测后不维修
方案不符合预期
拆检后费用超预期

```

### **11.3 退款申请字段**


| 字段            | 是否必填 | 说明           |
| ------------- | ---- | ------------ |
| order_id      | 是    | 订单 ID        |
| refund_reason | 是    | 退款原因         |
| refund_amount | 是    | 退款金额         |
| description   | 否    | 补充说明         |
| images        | 否    | 凭证图片         |
| contact_phone | 否    | 联系电话，默认账号手机号 |


### **11.4 退款提示**

提交前提示：

```
退款申请提交后，平台或门店将根据订单状态和服务规则进行处理。

```

### **11.5 退款结果**

退款结果以：

```
订单详情页
退款详情页
消息通知

```

展示。

```
平台需要你补充资料，以便继续处理售后申请。

```
```
补充资料
```

状态可从：

```
waiting_user
```

## **16. 取消售后申请**

### **16.1 取消条件**

用户可取消售后申请，需满足：

1. 售后未完成；
2. 未进入退款执行中；
3. 未进入平台最终处理阶段；
4. 平台规则允许取消。

### **16.2 取消提示**

```
取消后，该售后申请将关闭。如问题仍未解决，可重新提交。

```

按钮：

```
继续处理
确认取消

```

### **16.3 取消后状态**

状态变为：

```
canceled

```

## **18. 商家侧联动**

### **18.1 商家待办**

用户提交以下问题后，商家工作台可生成待办：

1. 用户申请退款；
2. 服务质量问题；
3. 维修进度问题；
4. 补充服务诉求；
5. 门店投诉；
6. 收费争议。

### **18.2 商家处理动作**

商家可执行：

1. 查看售后详情；
2. 查看订单；
3. 查看用户描述和凭证；
4. 提交处理说明；
5. 同意退款，视权限；
6. 拒绝退款并说明原因；
7. 申请平台介入；
8. 标记已联系用户。

### **18.3 商家响应时效**

平台可配置：

```
商家需在 X 小时内响应售后申请。

```

超时后：

1. 提醒商家；
2. 进入平台待办；
3. 影响商家履约指标；
4. 可能影响门店评分或排序，P1。

## **19. 平台侧联动**

### **19.1 平台待办**

平台工作台应展示：

1. 待处理售后；
2. 待处理退款；
3. 待处理投诉；
4. 商家超时未处理；
5. 高风险隐私投诉；
6. 案例下架申请；
7. 奖励争议。

### **19.2 平台处理动作**

平台可执行：

1. 查看售后详情；
2. 查看订单记录；
3. 查看支付和退款记录；
4. 查看维修相册；
5. 查看评价与奖励记录；
6. 查看商家反馈；
7. 要求用户补充资料；
8. 要求商家补充说明；
9. 审核通过；
10. 审核驳回；
11. 发起退款；
12. 关闭售后；
13. 标记风险。

### **19.3 高风险问题**

以下问题需优先进入平台待办：

1. 用户隐私泄露；
2. 车牌/VIN/手机号外泄；
3. 事故车维修重大争议；
4. 大额退款；
5. 多次投诉同一门店；
6. 评价奖励刷量；
7. 案例未经授权公开；
8. 涉及人身财产安全的问题。

## **20. 售后规则**

### **20.1 退款规则**

退款规则应根据：

1. 订单状态；
2. 服务是否开始；
3. 服务是否完成；
4. 是否产生检测/拆检成本；
5. 是否使用优惠券；
6. 是否涉及奖励冻结；
7. 门店处理结果；
8. 平台仲裁结果。

### **20.2 复杂维修/事故车规则**

复杂维修、事故车相关售后需特别说明：

1. 线上展示价格不等于最终维修价；
2. 最终费用需检测/拆检后确认；
3. 检测费、拆检费、材料费等需按订单规则处理；
4. 用户拒绝维修时，已产生费用是否退还以规则为准；
5. 不得承诺无条件全额退款，除非业务规则明确支持。

### **20.3 奖励冻结**

订单发生售后、退款、投诉时，可冻结相关奖励。

关联：

```
08_评价与奖励领取.md
09_分享与收益中心.md

```

冻结提示：

```
因订单存在售后处理，相关奖励暂时冻结。

```

## **21. 隐私与合规**

### **21.1 用户隐私**

售后与客服页面不得公开展示：

1. 完整手机号；
2. 完整车牌号；
3. VIN；
4. 身份证号；
5. 详细家庭地址；
6. 未脱敏维修相册；
7. 未授权公开的车辆图片。

### **21.2 通知隐私**

消息通知标题不得包含敏感信息。

不得出现：

```
浙A12345车辆事故维修退款失败
138xxxx用户投诉门店

```

可使用：

```
你的售后申请有新进展
你的退款申请已处理

```

### **21.3 凭证审核**

用户上传凭证需支持：

1. 图片格式校验；
2. 内容安全检测；
3. 隐私风险提示；
4. 违规内容拦截；
5. 平台审核查看。

### **21.4 案例公开争议**

用户投诉案例公开问题时，应优先处理。

场景：

1. 用户认为未授权公开；
2. 用户要求下架；
3. 图片脱敏不充分；
4. 车牌或个人信息泄露；
5. 维修内容不希望公开。

## **29. 验收标准**

1. 用户可从我的页面进入客服与帮助；
2. 用户可查看 FAQ 分类和详情；
3. 用户可点击微信客服；
4. 用户可从订单详情页联系门店；
5. 用户可从订单详情页联系平台客服；
6. 未登录用户提交售后时会触发登录；
7. 用户可对符合条件的订单申请退款；
8. 用户可对符合条件的订单申请售后；
9. 用户可提交投诉；
10. 用户提交售后后可查看售后详情；
11. 售后详情页展示当前状态和处理进度；
12. 重复售后申请需被拦截；
13. 不支持退款的订单不展示或置灰退款入口；
14. 售后处理结果可通过消息通知触达用户；
15. 商家工作台可收到相关售后待办；
16. 平台工作台可收到售后/投诉待办；
17. 通知标题不得展示敏感隐私；
18. 车牌、手机号、VIN 等不得明文展示；
19. 售后和客服关键行为可埋点统计；
20. 售后提交失败、图片上传失败等异常场景有提示。
