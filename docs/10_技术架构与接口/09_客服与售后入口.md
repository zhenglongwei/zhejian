## **2. 模块定位**

V2.0 平台**不是**交易方，**不提供**订单退款、售后仲裁、维修质量担保。

本模块用于：

1. FAQ 自助（咨询方式、相册查看、授权公开、隐私说明）；
2. 联系平台（内容/隐私/合规类问题）；
3. 投诉举报（虚假案例、隐私泄露、违规宣传）；
4. 咨询线索问题反馈（商家未联系等，平台仅协助转达，不介入线下维修）。

**不做**：订单客服、退款申请、售后进度、评价奖励争议。

## **5. 投诉类型**

| 类型 | 处理 |
| --- | --- |
| 隐私泄露 | 先隐藏内容，运营审核 |
| 未授权公开 | 下架案例，核查授权记录 |
| 虚假/违规案例 | 审核/下架，标记商家风险 |
| 咨询线索问题 | 记录并通知商家，平台不判定维修结果 |
| 配件/维修质量 | **引导联系门店或行政/司法途径**，平台不提供赔偿 |

## **10. 举报虚假信息（2026-06-02 新增）**

> **背景**：平台定位为 **展示/引流**（见 `02_业务模式说明.md` §2.1），商家服务/门店宣传 **不做发布前审核**；通过 **免责声明 + 用户举报 + 运营事后处置** 控制虚假信息风险。**公开案例**仍走卷三审核（监管重点）。

### **10.1 入口**

| 页面 | 入口文案 | 优先级 |
| --- | --- | --- |
| 服务详情 `pages/service/detail` | 举报虚假信息 | P1 |
| 门店详情 `pages/store/detail` | 举报虚假信息 | P1 |
| 案例详情 `pages/case/detail` | 举报虚假信息 | P1 |
| GEO 专题页 | 举报 | P1 |
| 我的 → 帮助与反馈 | 通用投诉（含举报） | 已有规划 |

页脚或更多菜单中展示，**不**与主 CTA（咨询/预约）竞争视觉权重。

### **10.2 举报类型**

| 类型 | 说明 | 典型处置 |
| --- | --- | --- |
| `false_price` | 虚假或误导性价格 | 要求商家下架服务 / 强制下架 |
| `false_promise` | 夸大宣传（保证修好、全网最低等） | 强制下架 + 警告 |
| `false_qualification` | 资质、授权造假 | 联动商家审核 / 冻结 |
| `fake_case` | 伪造、盗用案例 | 隐藏案例 + 案例审核复核 |
| `misleading_media` | 图片/文案与实际情况严重不符 | 下架相关内容 |
| `other` | 其他虚假信息 | 运营判定 |

**不做**：维修质量仲裁、价格重新核定、要求平台赔偿。

### **10.3 举报表单**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 举报类型 | 是 | 上表枚举 |
| 问题说明 | 是 | 10–500 字 |
| 截图/凭证 | 否 | 最多 3 张，走 `B-MEDIA` |
| 联系手机 | 否 | 便于运营回访 |
| 知晓声明勾选 | 是 | 「平台仅作信息展示，举报不代表平台已核实；恶意举报可能承担法律责任」 |

提交后展示：「已收到，我们将在 X 个工作日内处理」；**不承诺**线下维修结果。

### **10.4 运营处置（admin-web）**

入口：`平台运营工作台 → 举报管理`（`/admin/reports`）

| 操作 | 说明 |
| --- | --- |
| 受理 / 驳回 | 驳回须填原因；恶意举报可标记用户 |
| 要求商家整改 | 通知商家，限期修改 |
| 强制下架 | 服务 → `OPS-SVC` suspend；案例 → 临时隐藏 |
| 标记商家风险 | 累计举报达阈值（P2 配置） |
| 升级冻结 | 严重虚假 → 联动商家/门店冻结（人工决策） |

所有操作写入 `report_handle_log`；与 `service_review_log` / `case_review_log` 可交叉引用。

### **10.5 数据对象（摘要）**

**content_report**

| 字段 | 说明 |
| --- | --- |
| id | 举报 ID |
| user_id | 举报人 |
| target_type | `service` / `store` / `case` / `geo` |
| target_id | 对象 ID |
| report_type | 枚举 |
| description | 说明 |
| images_json | 凭证图 |
| status | `pending` / `processing` / `resolved` / `rejected` |
| resolution | 处置摘要 |
| created_at / updated_at | |

### **10.6 P1 验收标准**

1. 服务/门店/案例详情可发起举报并落库；
2. 运营台可列表查看、详情处理、留痕；
3. 举报成立可触发 **至少一种** 下架/隐藏动作（手动确认后执行）；
4. 详情页有 **平台免责声明**（U-DISCL-01）；
5. **无**「平台已核实该商家」类暗示文案。
