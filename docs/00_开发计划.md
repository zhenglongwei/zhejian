## 1. 如何使用

| 标记 | 含义 |
| --- | --- |
| `[x]` | 已完成（模拟器/dev 联调验收通过） |
| `[~]` | 部分完成（UI 或 mock/dev 桩，未达上线标准） |
| `[ ]` | 待开发 |
| `[延]` | **延期开发**（有价值，暂缓；核心链路不阻塞） |
| `[搁]` | **搁置开发**（当前不做；已有替代方案或产品口径变更） |
| `[-]` | Phase 2 / 本阶段不做 |

**优先级**：`P0` 阻塞上线 · `P1` V0.5 后增强 · `P2` 可延后 · `Ph2` = Phase 2 产品口径

**任务 ID**：`U-*` 用户端 · `A-*` 服务相册（含商家端相册页） · `M-*` 商家工作台 · `B-*` 后端 · `OPS-*` 运营后台

**规则**：所有新开发只在本表勾选进度；完成改 `[ ]`→`[x]`；暂缓改 `[延]`/`[搁]` 并注明原因。PRD 冲突以 Phase 1 口径为准。

**每日怎么开发**：只看根目录 [`每日固定动作.md`](../每日固定动作.md)（skill、红线、Cursor 模板）；新卷在彼文档 §7 追加备忘。
