## 重要：两种「目录」不要搞混

| 概念 | 路径 | 说明 |
|---|---|---|
| **Git 仓库（源码）** | `/var/www/zhejian/` | `git clone` 后与本地一样：`backend/`、`h5/`、`pages/`… |
| **浏览器 URL 路径** | `geo.simplewin.cn/...` | Nginx 把 URL **映射到** 仓库里的子目录，**不是**再复制一套文件夹 |

以前文档里的 `/var/www/geo.simplewin.cn/public` 是「复制发布」写法，**已取消**。  
现在：**只保留 `/var/www/zhejian` 一个源码目录**，Nginx 直接读其中的 `h5/`、`admin-web/`。

## 二、推荐方式：GitHub 上传 + 服务器拉取

**比手动 FTP 上传省事**，以后改代码只需 `git push`，服务器 `git pull`。

### 第 1 步：在 GitHub 网站建仓库

1. 登录 <https://github.com>  
2. 右上角 **+** → **New repository**  
3. 名称填：`zhejian`（或 `zhejian-simplewin`）  
4. **不要**勾选 “Add a README”  
5. 创建后复制仓库地址，例如：  
   `https://github.com/你的用户名/zhejian.git`

### 第 2 步：本地连接并推送（PowerShell）

把下面 `你的用户名` 换成你的 GitHub 用户名：

```powershell
cd c:\Users\longwei\WeChatProjects\zhejian
git branch -M main
git remote add origin https://github.com/你的用户名/zhejian.git
git add .
git commit -m "Prepare deployment for geo.simplewin.cn"
git push -u origin main
```

第一次 push 会弹出 GitHub 登录（浏览器或 Personal Access Token）。

> 本机未安装 `gh` CLI，所以要在网页建仓库 + 上面命令 push，**效果一样**。

### 第 3 步：服务器拉代码到 /var/www

SSH 登录服务器（8.155.0.128）后执行：

```bash
sudo mkdir -p /var/www
sudo git clone https://github.com/你的用户名/zhejian.git /var/www/zhejian
sudo chown -R $USER:$USER /var/www/zhejian
```

## 三、手动上传（不用 GitHub 时）

1. 把整个 `zhejian` 文件夹打成 zip（**不要**包含 `node_modules`、`backend/.env`）

## 七、站外埋点与看板统计（运维）

### 7.1 三件事分别是什么

| 建议 | 含义（现实世界） | 不做会怎样 |
| --- | --- | --- |
| **H5 随发版更新** | 用户打开的 `geo.simplewin.cn/case/...` 读的是仓库 `h5/`，不是 Node | 只部署 API 时，案例页没有 `track.js`，库里只有冒烟数据，**没有真实浏览** |
| **监控** | 每天自动检查：昨天有没有新埋点、凌晨 job 是否成功 | job 挂了几天才发现，商家看板一直为 0 |
| **PM2/K8s CronJob 等** | 和 Linux `crontab` **同一件事**：定时跑 `stats:aggregate` | 你们单机 ECS 用 **crontab 即可**，不必上 K8s |

数据流：

```text
用户打开 H5 → POST /api/v1/track/events → event_tracking_log（实时）
每天北京时间 0:00 → stats:aggregate → merchant_daily_stats（昨日汇总）
商家小程序看板 → GET /merchant/stats（读日表 + 部分实时排行）
```

### 7.2 H5 静态部署（你们已具备，只需养成习惯）

Nginx 已配置（见 `backend/deploy/simplewin.conf`，geo 块摘要见 `nginx-geo.simplewin.cn.conf`）：

- `/case/` → `/var/www/zhejian/h5/case/`（案例列表）
- `/store/` → `/var/www/zhejian/h5/store/`（**门店列表**，普通人入口）
- `/store/{id}.html` → 门店详情（PRD · GEO/分享深链）
- `/shared/` → `/var/www/zhejian/h5/shared/`

因此 **没有单独「发布 H5」步骤**：在服务器 `git pull` 后，新 `track.js`、案例 HTML 立刻对外生效。  
发版清单：**backend 重启 + git pull 整仓**（不要只 rsync `backend/`）。

### 7.3 日聚合定时（北京时间每天 0:00）

```bash
chmod +x /var/www/zhejian/backend/scripts/stats-aggregate-cron.sh
mkdir -p /var/www/zhejian/backend/logs
crontab -e
```

写入（路径按实际修改）：

```cron
TZ=Asia/Shanghai
0 0 * * * /var/www/zhejian/backend/scripts/stats-aggregate-cron.sh
```

日志（默认）：`/var/www/zhejian/backend/logs/stats-aggregate.log`（应看到 `[aggregate] { statDate: '...', processed: N }`）。

### 7.3.1 GEO Prompt 探测（北京时间每周一 03:00）

与 [`07_GEO引用观测开发计划.md`](./09_SEO_GEO_AI内容基础设施/07_GEO引用观测开发计划.md) **GEO-OBS-B06** 对应；**非每日**，默认每批 **20 条** Prompt（非 30 条全量）。

**ECS 生产**：2026-06-19 已部署 ✅（`git pull` + backend 重启 · crontab `0 3 * * 1` · Key 已轮换）。

**`.env` 必配**（`backend/.env`）：

```bash
GEO_PROBE_ENABLED=true
GEO_PROBE_DRY_RUN=false
DASHSCOPE_API_KEY=sk-xxx          # 或 GEO_PROBE_API_KEY
GEO_PROBE_MODEL=qwen-plus
GEO_PROBE_ENABLE_THINKING=false
```

```bash
chmod +x /var/www/zhejian/backend/scripts/geo-prompt-probe-cron.sh
mkdir -p /var/www/zhejian/backend/logs
crontab -e
```

写入：

```cron
TZ=Asia/Shanghai
0 3 * * 1 /var/www/zhejian/backend/scripts/geo-prompt-probe-cron.sh
```

日志：`/var/www/zhejian/backend/logs/geo-probe.log`。手动试跑：

```bash
bash scripts/geo-prompt-probe-cron.sh
# 或快速验收 2 条：node scripts/geo-prompt-probe.js --live --limit=2
```

**说明**：live 探测串行调千问，20 条约 **5～30 分钟**；旧版日志在全部跑完前可能只有 seed-sync 一行，属正常。`git pull` 后脚本会输出 `(1/20) probing...` 进度。

### 7.4 监控（最小可行，不依赖第三方）

**① 看日志（每天或告警脚本读一次）**

```bash
tail -5 /var/www/zhejian/backend/logs/stats-aggregate.log
# 期望：有当天时间戳，且 processed >= 1（有 ACTIVE 商家门店时）
```

**② SQL：昨天是否有埋点、是否已汇总**

```sql
-- 昨日（按上海日历，示例日期请改成昨天）
SELECT COUNT(*) AS events_yesterday
FROM event_tracking_log
WHERE created_at >= '2026-06-04 00:00:00' AND created_at < '2026-06-05 00:00:00';

SELECT store_id, case_view_count, phone_click_count
FROM merchant_daily_stats
WHERE stat_date = '2026-06-04';
```

**③ 自动化监控（OPS-DATA-03，推荐）**

```bash
chmod +x /var/www/zhejian/backend/scripts/stats-monitor-cron.sh
# 手动跑一次（需 DATABASE_URL）
cd /var/www/zhejian/backend && npm run stats:monitor

# crontab 在聚合 job 之后 30 分钟（示例路径按实际修改）
30 0 * * * /var/www/zhejian/backend/scripts/stats-monitor-cron.sh
```

脚本 `backend/scripts/ops-data-monitor.js` 会检查：昨日 `event_tracking_log` 增量、`merchant_daily_stats` 是否有行、
`logs/stats-aggregate.log` 是否含对应 `statDate` 的 `processed`。失败退出码 1；可选环境变量 `MONITOR_WEBHOOK_URL` 推送告警。

手工 SQL 见 `backend/scripts/sql/ops-data-monitor.sql`。

| 监控项 | 正常 | 异常时 |
| --- | --- | --- |
| `event_tracking_log` 日增量 | H5 有流量时 > 0 | 检查 H5 是否含 `track.js`、Nginx `/api/` |
| `stats-aggregate.log` | 有 `processed: N` 且脚本退出 0 | 查 `journalctl`、数据库连接、商家是否 ACTIVE |
| job 退出码 | cron 邮件为空 / 日志无 ERROR | 手动 `scripts/stats-aggregate-cron.sh` 复现 |

### 7.5 PM2 cron / K8s 是什么？要不要用

- **PM2 cron**：用 PM2 插件在进程管理器里设定时任务；适合「API 已用 PM2 跑、想配置写在一处」——**与 crontab 二选一**。
- **K8s CronJob**：多容器集群里跑定时 Pod；**单机 ECS 不需要**。
- **云函数定时**：阿里云「定时触发器」调一条命令；适合无常驻机，你们已有 ECS，**crontab 最简单**。

## 常见问题

| 现象 | 处理 |
|---|---|
| health 返回 502 | `sudo systemctl status zhejian-api` 看是否启动；`journalctl -u zhejian-api -n 50` 看日志 |
| db 连接失败 | 检查 `.env` 的 `DATABASE_URL`、MySQL 用户权限 |
| /admin/ 404 | 确认 `/var/www/zhejian/admin-web/placeholder/index.html` 存在且 Nginx 已 reload |
| 看板无浏览、只有冒烟有数 | `git pull` 是否更新 `h5/`；案例页是否加载成功；是否配置 §7.3 crontab |
| F12 无 `track/events` 请求 | 多为广告拦截器屏蔽 URL 含 `track`；H5 已改走 `POST /api/v1/analytics/events`，拉代码后硬刷新 |
| `merchant_daily_stats` 空 | 先查 `event_tracking_log`；再 `npm run stats:aggregate -- --date=昨天` |

更细的说明见 `backend/deploy/README.md`、`docs/10_技术架构与接口/10_域名与部署架构.md`。
