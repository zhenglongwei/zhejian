# 辙见 · 部署上手指南（照着做即可）

> **盈简科技** · 产品 **辙见**  
> 服务器代码目录：`/var/www/zhejian`（与 GitHub / 本地 **完全一致**，`git clone` 即可）

---

## 重要：两种「目录」不要搞混

| 概念 | 路径 | 说明 |
|---|---|---|
| **Git 仓库（源码）** | `/var/www/zhejian/` | `git clone` 后与本地一样：`backend/`、`h5/`、`pages/`… |
| **浏览器 URL 路径** | `geo.simplewin.cn/...` | Nginx 把 URL **映射到** 仓库里的子目录，**不是**再复制一套文件夹 |

以前文档里的 `/var/www/geo.simplewin.cn/public` 是「复制发布」写法，**已取消**。  
现在：**只保留 `/var/www/zhejian` 一个源码目录**，Nginx 直接读其中的 `h5/`、`admin-web/`。

## 一、本地环境（在你 Windows 电脑上）

### 已帮你准备好的文件

| 文件 | 作用 |
|---|---|
| `backend/.env` | 本地数据库连接（已写好，勿提交 Git） |
| `scripts/local-setup.ps1` | 一键安装依赖、建表、演示数据 |

### 你需要安装一次

1. **Node.js 20+** — <https://nodejs.org>  
2. **Docker Desktop**（可选）— 用来跑本地 MySQL；若没有 Docker，需本机自己装 MySQL 并改 `backend/.env` 里的 `DATABASE_URL`

### 本地执行（2 条命令）

在项目根目录 `c:\Users\longwei\WeChatProjects\zhejian` 打开 PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\local-setup.ps1
cd backend
npm run dev
```

浏览器打开：<http://127.0.0.1:3000/api/v1/health>  
看到 `"ok":true` 即本地 API 正常。

小程序仍默认 mock；要连本地 API 时改 `services/config.js` 的 `mode: 'dev'`、`baseUrl: 'http://127.0.0.1:3000'`（仅开发者工具可调本地，真机需 HTTPS 域名）。

## 二、推荐方式：GitHub 上传 + 服务器拉取

**比手动 FTP 上传省事**，以后改代码只需 `git push`，服务器 `git pull`。

### 第 1 步：在 GitHub 网站建仓库

1. 登录 <https://github.com>  
2. 右上角 **+** → **New repository**  
3. 名称填：`zhejian`（或 `zhejian-simplewin`）  
4. **不要**勾选 “Add a README”  
5. 创建后复制仓库地址，例如：  
   `https://github.com/你的用户名/zhejian.git`

### 第 2 步：本地连接并推送（PowerShell）

把下面 `你的用户名` 换成你的 GitHub 用户名：

```powershell
cd c:\Users\longwei\WeChatProjects\zhejian
git branch -M main
git remote add origin https://github.com/你的用户名/zhejian.git
git add .
git commit -m "Prepare deployment for geo.simplewin.cn"
git push -u origin main
```

第一次 push 会弹出 GitHub 登录（浏览器或 Personal Access Token）。

> 本机未安装 `gh` CLI，所以要在网页建仓库 + 上面命令 push，**效果一样**。

### 第 3 步：服务器拉代码到 /var/www

SSH 登录服务器（8.155.0.128）后执行：

```bash
sudo mkdir -p /var/www
sudo git clone https://github.com/你的用户名/zhejian.git /var/www/zhejian
sudo chown -R $USER:$USER /var/www/zhejian
```

## 三、手动上传（不用 GitHub 时）

1. 把整个 `zhejian` 文件夹打成 zip（**不要**包含 `node_modules`、`backend/.env`）  
2. 用 WinSCP / FileZilla 上传到服务器 `/var/www/zhejian`  
3. 在服务器解压：

```bash
sudo mkdir -p /var/www
cd /var/www
sudo unzip zhejian.zip -d zhejian
# 或上传后：sudo mv 上传的文件夹名 /var/www/zhejian
```

## 五、云端配置与部署命令（复制执行）

SSH 登录服务器后，**推荐一键脚本**（B-INF）：

```bash
cd /var/www/zhejian
sudo bash scripts/server-install.sh --init
nano backend/.env
sudo bash scripts/server-install.sh
curl -s https://geo.simplewin.cn/api/v1/health
```

或手动按顺序执行：

```bash
# 1. 配置 API 环境变量（在仓库内的 backend/）
cd /var/www/zhejian/backend
cp .env.production.example .env
nano .env
# 修改 DATABASE_URL 和 DEV_USER_TOKEN 后 Ctrl+O 保存

# 2. 安装 API 依赖 + 迁移（生产默认不 seed）
npm install
npm run db:setup:prod
# 可选首次演示：npm run db:seed

# 3. Nginx（/api/ 须反代到 Node :3000，见 B-INF 文档）
sudo cp /var/www/zhejian/backend/deploy/nginx-geo.simplewin.cn.conf /etc/nginx/sites-available/geo.simplewin.cn
sudo ln -sf /etc/nginx/sites-available/geo.simplewin.cn /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 4. API 守护进程（WorkingDirectory=/var/www/zhejian/backend）
sudo cp /var/www/zhejian/backend/deploy/zhejian-api.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable zhejian-api
sudo systemctl start zhejian-api
sudo systemctl status zhejian-api
```

### 验证

```bash
curl -s https://geo.simplewin.cn/api/v1/health
curl -sI https://geo.simplewin.cn/admin/
```

### 目录结构（clone 后与本地 / GitHub 一致）

```text
/var/www/zhejian/                 ← git clone 到这里，结构与本地完全相同
├── backend/                      ← API 运行目录（systemd 启动这里）
├── h5/                           ← Nginx 映射到 https://geo.simplewin.cn/case/...
├── admin-web/placeholder/        ← Nginx 映射到 https://geo.simplewin.cn/admin/
├── pages/                        ← 小程序（只在本地/开发者工具用，服务器不跑）
├── packageMerchant/
├── services/
└── ...
```

**URL 与仓库文件夹对应关系：**

| 浏览器地址 | 实际读的文件 |
|---|---|
| `https://geo.simplewin.cn/api/v1/...` | Node 进程（`backend/`） |
| `https://geo.simplewin.cn/admin/` | `admin-web/placeholder/index.html` |
| `https://geo.simplewin.cn/case/case_001.html` | `h5/case/case_001.html` |

## 常见问题

| 现象 | 处理 |
|---|---|
| health 返回 502 | `sudo systemctl status zhejian-api` 看是否启动；`journalctl -u zhejian-api -n 50` 看日志 |
| db 连接失败 | 检查 `.env` 的 `DATABASE_URL`、MySQL 用户权限 |
| /admin/ 404 | 确认 `/var/www/zhejian/admin-web/placeholder/index.html` 存在且 Nginx 已 reload |

更细的说明见 `backend/deploy/README.md`、`docs/10_技术架构与接口/10_域名与部署架构.md`。
