## 1. 本地 ↔ 生产怎么切换

辙见有 **两套独立运行环境**，不是一个全局开关：

| 层 | 谁决定连哪里 | 配置文件 | 说明 |
| --- | --- | --- | --- |
| **小程序**（用户端 + 商家端分包） | 你改一行代码 | `services/config.js` → `ACTIVE_ENV` | 决定请求 `127.0.0.1:3000` 还是 `geo.simplewin.cn` |
| **Backend API** | 在哪台机器跑、读哪份 `.env` | 本地 `backend/.env`；服务器 `/var/www/zhejian/backend/.env` | **两份文件互不影响**，不要提交 Git |

### 1.1 三种小程序环境（`ACTIVE_ENV`）

打开 `services/config.js`，改第 11 行：

```javascript
const ACTIVE_ENV = 'local'   // 或 'prod' / 'mock'
```

| `ACTIVE_ENV` | API 地址 | 何时用 |
| --- | --- | --- |
| **`local`** | `http://127.0.0.1:3000` | 开发者工具 + **本机** `npm run dev` |
| **`prod`** | `https://geo.simplewin.cn` | 真机预览、体验版、上线 |
| **`mock`** | 不发 HTTP | 纯 UI 演示，无需 backend |

用户端与商家端（`packageMerchant`）**共用**这一份配置。

### 1.2 切到「本地联调」（开发者工具 → 本机 API）

**① 本机 backend（Windows）**

```powershell
cd C:\Users\longwei\WeChatProjects\zhejian\backend
npm run dev
```

确认：<http://127.0.0.1:3000/api/v1/health> 返回 `"ok":true`。  
首次搭建见 [§2](#2-本地开发-windows--无-docker推荐)。

**② 本机 `backend/.env`（与生产那份无关）**

模板：`backend/.env.example`。本地关键项：

```env
NODE_ENV=development
PORT=3000
PUBLIC_BASE_URL=http://127.0.0.1:3000
DATABASE_URL=mysql://zhejian:你的密码@127.0.0.1:3306/zhejian
DEV_AUTH_ENABLED=true
DEV_USER_TOKEN=dev_user_token_change_me
DESENSITIZE_ENGINE=dev
```

**③ 小程序**

```javascript
// services/config.js
const ACTIVE_ENV = 'local'
```

微信开发者工具 → **详情 → 本地设置 → 勾选「不校验合法域名、web-view…」**（仅本地）。

**④ 登录**

- 本地 `DEV_AUTH_ENABLED=true` 时，点登录会走 `POST /user/auth/wechat-login`，backend 返回 dev token（默认 `dev_user_token_change_me`）。
- seed 演示用户：`user_demo_1` / 商家 `merchant_demo_1`（见 `npm run db:seed` 输出）。

**⑤ 编译**

改完 `ACTIVE_ENV` 后，开发者工具点 **编译** 或 **重新载入**。

### 1.3 切到「生产 / 真机」（小程序 → 线上 API）

**① 小程序**

```javascript
// services/config.js
const ACTIVE_ENV = 'prod'
```

**② 微信开发者工具**

- **关闭**「不校验合法域名」（真机/体验版必须关）。
- 公众平台 → 开发 → 开发设置 → **request 合法域名**：`https://geo.simplewin.cn`。

**③ 本机 backend**

- **不必**跑 `npm run dev`；请求直接打到服务器。
- 本地 `backend/.env` **不用改**；生产配置只在服务器 `/var/www/zhejian/backend/.env`。

**④ 登录**

- 生产推荐：`JWT_SECRET` + `WECHAT_APP_ID` / `WECHAT_APP_SECRET`，`DEV_AUTH_ENABLED=false`（真微信 JWT）。
- 联调阶段可暂留 `DEV_AUTH_ENABLED=true`（见 B-INF 文档 §3.3）。

**⑤ 提交代码前**

若团队约定默认连生产，**push 前把 `ACTIVE_ENV` 改回 `'prod'`**，避免他人拉代码后误连 localhost。

### 1.4 对照表（一眼看清）

| 项目 | 本地联调 | 生产 / 真机 |
| --- | --- | --- |
| `services/config.js` | `ACTIVE_ENV = 'local'` | `ACTIVE_ENV = 'prod'` |
| 本机 `npm run dev` | **要开** | 不需要 |
| `backend/.env` 位置 | 本机 `backend/.env` | 服务器 `/var/www/zhejian/backend/.env` |
| `NODE_ENV` | `development` | `production` |
| `PUBLIC_BASE_URL` | `http://127.0.0.1:3000` | `https://geo.simplewin.cn` |
| `PORT` | `3000` | `3100`（与官网 3000 共存时） |
| `DATABASE_URL` | 本机 MySQL | 服务器 MySQL |
| `DESENSITIZE_ENGINE` | `dev`（stub） | `aliyun` |
| 开发者工具「不校验合法域名」 | **勾选** | **关闭** |
| 运营后台 `admin-web` | 浏览器仍访问 `https://geo.simplewin.cn/admin/` | 同左 |

### 1.5 常见误区

| 误区 | 正确理解 |
| --- | --- |
| 改了 `ACTIVE_ENV` 就能切换 backend 数据库 | ❌ 只切换**小程序请求的 API 地址**；数据库由**正在跑 API 的那台机器**的 `.env` 决定 |
| 把生产 `.env` 复制到本地就能联调 | ❌ 生产 `DATABASE_URL` 指向服务器 MySQL，本机连不上；应各用各的 `.env` |
| 真机预览设 `local` | ❌ 手机访问不到 `127.0.0.1`；真机必须 `prod` + 合法域名 |
| `mock` 能测接口 | ❌ `mock` 不发起 HTTP，只走 `mock/` 假数据 |

更细的登录与真机冒烟：[§9 小程序联调](#9-小程序联调生产--体验版)、`B-INF_生产与真机联调.md`。

## 2. 本地开发（Windows · 无 Docker，推荐）

适用于：**没有 Docker**，在本机安装 MySQL 8，跑 `npm run dev`。

### 2.1 安装前置

1. **Node.js 20+** — <https://nodejs.org>  
   验证：`node -v`（应 ≥ v20）
2. **MySQL 8** — 任选：
   - [MySQL Installer for Windows](https://dev.mysql.com/downloads/installer/)（选 Server only 或 Developer Default）
   - 或 `winget install Oracle.MySQL`
3. 安装后确认 **MySQL 服务在运行**：
   - Windows「服务」→ 找 `MySQL80` → 状态「正在运行」
   - 或 PowerShell：`Get-Service *mysql*`

默认端口 **3306**。若被占用，要么改 MySQL 端口，要么改 `.env` 里 `DATABASE_URL` 的端口。

### 2.2 创建数据库与用户

用 **MySQL Workbench**（安装 MySQL 时一般会带）或命令行登录 root：

```powershell
# mysql 常见路径（若未加入 PATH）：
# & "C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p
mysql -u root -p
```

执行 SQL（**密码请改成你自己的**，下面 `zhejian_dev_pass` 仅为本地开发示例）：

```sql
CREATE DATABASE IF NOT EXISTS zhejian
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- Prisma 连 127.0.0.1 时，MySQL 会匹配 'user'@'127.0.0.1'，两个 host 都建更稳
CREATE USER IF NOT EXISTS 'zhejian'@'localhost' IDENTIFIED BY 'zhejian_dev_pass';
CREATE USER IF NOT EXISTS 'zhejian'@'127.0.0.1' IDENTIFIED BY 'zhejian_dev_pass';

GRANT ALL PRIVILEGES ON zhejian.* TO 'zhejian'@'localhost';
GRANT ALL PRIVILEGES ON zhejian.* TO 'zhejian'@'127.0.0.1';
FLUSH PRIVILEGES;
```

**不想单独建用户**（仅本机临时调试）可只用 root，跳过建用户，`.env` 里写 root 密码即可。

### 2.3 配置 `backend/.env`（必做，不会自动生成）

> **常见误区**：仓库里**没有**提交 `backend/.env`（已在 `.gitignore`）。  
> 必须从 example 复制并改密码。

在项目根目录 PowerShell：

```powershell
cd C:\Users\longwei\WeChatProjects\zhejian\backend
copy .env.example .env
notepad .env
```

**本地最少要改这几项**（其余可先保持 example 默认）：

```env
NODE_ENV=development
PORT=3000
HOST=127.0.0.1

# 【必改】用户名、密码、库名必须与 §2.2 里 MySQL 里的一致
DATABASE_URL=mysql://zhejian:zhejian_dev_pass@127.0.0.1:3306/zhejian

# 与小程序 local 环境一致（上传返回 URL、相册读图）
PUBLIC_BASE_URL=http://127.0.0.1:3000

DEV_AUTH_ENABLED=true
DEV_USER_TOKEN=dev_user_token_change_me
DEV_MERCHANT_TOKEN=dev_merchant_token_change_me
DEV_SYSTEM_TOKEN=dev_system_token_change_me
```

**密码含特殊字符**（`@` `#` `%` `!` 等）须做 [URL 编码](https://developer.mozilla.org/en-US/docs/Glossary/Percent-encoding) 再写入 `DATABASE_URL`，例如 `@` → `%40`。

本地脱敏可先 stub，不必配阿里云 Key：

```env
DESENSITIZE_ENGINE=dev
```

### 2.4 安装依赖 + 建表 + 演示数据

```powershell
cd C:\Users\longwei\WeChatProjects\zhejian\backend
npm install
npm run db:setup
```

等价于依次执行：`db:generate` → `db:migrate` → `db:seed`。

### 2.5 启动 API 并验证

```powershell
npm run dev
```

浏览器或 PowerShell：

```powershell
curl http://127.0.0.1:3000/api/v1/health
```

期望 JSON 含 `"ok":true`。

### 2.6 小程序连本地 API

按 [§1.2 切到本地联调](#12-切到本地联调开发者工具--本机-api)：`ACTIVE_ENV = 'local'` + 勾选不校验合法域名。

### 2.7 本地验收清单

- [ ] `node -v` ≥ 20  
- [ ] MySQL 服务运行中，3306 可连  
- [ ] `backend/.env` 已从 `.env.example` 复制，`DATABASE_URL` **不是** `YOUR_PASSWORD`  
- [ ] `npm run db:setup` 无报错  
- [ ] `npm run dev` 后 health 返回 ok  
- [ ] 小程序 `ACTIVE_ENV = 'local'` 能调通一个接口

## 4. 故障排查：P1000 / 数据库连不上

```text
Error: P1000: Authentication failed against database server,
the provided database credentials for `zhejian` are not valid.
```

表示：**MySQL 能连上（127.0.0.1:3306），但 `.env` 里的用户名或密码与 MySQL 里不一致**。

按顺序排查：

| 步骤 | 操作 |
| --- | --- |
| 1 | 打开 `backend/.env`，确认 `DATABASE_URL` 不是 `YOUR_PASSWORD` 或 example 占位符 |
| 2 | 用 Workbench / `mysql -u zhejian -p -h 127.0.0.1 zhejian` 用**同一组**账号密码登录 |
| 3 | 若 root 能登、zhejian 不能 → 重跑 [§2.2 SQL](#22-创建数据库与用户) 或 `ALTER USER` 改密码 |
| 4 | 确认库 `zhejian` 已创建：`SHOW DATABASES LIKE 'zhejian';` |
| 5 | 密码有特殊字符 → URL 编码后再写入 `DATABASE_URL` |
| 6 | 改完 `.env` 后**重新**在 `backend/` 目录执行 `npm run db:migrate` |

**验证连接（不经过 Prisma）：**

```powershell
cd backend
node -e "require('dotenv').config(); console.log(process.env.DATABASE_URL?.replace(/:([^:@/]+)@/, ':***@'))"
```

**MySQL 未启动时** 常见报错是 `ECONNREFUSED` 或 `Can't reach database server`，与 P1000 不同。

**迁移报「列不存在」**：说明库是旧的或迁移未跑全 → `npm run db:migrate`；仍失败可备份后 `DROP DATABASE zhejian` 重建（仅本地开发）。

### P3009：存在 failed migration

```text
Error: P3009
The `20260703140000_part_verify_guide` migration ... failed
```

表示某次迁移**半途失败**，Prisma 在 `_prisma_migrations` 里留了失败记录，后续迁移会被阻塞。

**本地开发（无重要数据）— 最快：删库重建**

```sql
DROP DATABASE zhejian;
CREATE DATABASE zhejian CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

```powershell
cd backend
npm run db:setup
```

**保留已有数据 — 按失败原因修复后重跑**

```powershell
cd backend
# 1. 看失败原因（logs 列）
node -e "require('@prisma/client').PrismaClient().$queryRawUnsafe('SELECT migration_name, logs FROM _prisma_migrations WHERE finished_at IS NULL').then(console.log)"

# 2. 确认 schema 已修复后，标记该迁移为已回滚
npx prisma migrate resolve --rolled-back 迁移名

# 3. 重新应用
npm run db:migrate
```

常见根因：`TEXT` 列带 `DEFAULT ''`（MySQL 报错 1101）— 仓库内 `20260703140000_part_verify_guide` 已改为分步 `ADD NULL → UPDATE → MODIFY NOT NULL`。

## 5. 生产环境首次部署

目标域名：**geo.simplewin.cn**（API + H5 + 运营后台）。  
服务器代码目录：**`/var/www/zhejian`**（与 GitHub / 本地结构一致）。

### 5.1 把代码弄到服务器

**方式 A · GitHub（推荐）**

```bash
sudo mkdir -p /var/www
sudo git clone https://github.com/你的用户名/zhejian.git /var/www/zhejian
sudo chown -R $USER:$USER /var/www/zhejian
```

**方式 B · 手动上传**  
zip 整个仓库（**不要**含 `node_modules`、`backend/.env`）→ 解压到 `/var/www/zhejian`。

### 5.2 服务器 MySQL（一次性）

SSH 登录服务器，MySQL 里执行（**生产请用强密码**）：

```sql
CREATE DATABASE IF NOT EXISTS zhejian
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'zhejian'@'127.0.0.1' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON zhejian.* TO 'zhejian'@'127.0.0.1';
FLUSH PRIVILEGES;
```

MySQL 应只监听 `127.0.0.1`，不对公网开放 3306。

### 5.3 配置 `backend/.env`

```bash
cd /var/www/zhejian
sudo bash scripts/server-install.sh --init
nano backend/.env
```

脚本会从 `backend/.env.production.example` 生成 `.env`。  
**必须修改**（仍含 `YOUR_PASSWORD` / `请改成` 时，部署脚本会拒绝继续）：

| 变量 | 说明 |
| --- | --- |
| `DATABASE_URL` | `mysql://zhejian:强密码@127.0.0.1:3306/zhejian` |
| `DEV_USER_TOKEN` / `DEV_MERCHANT_TOKEN` / `DEV_SYSTEM_TOKEN` | 随机长字符串 |
| `JWT_SECRET` | 随机长字符串（微信登录） |
| `ADMIN_PASSWORD` | 运营后台登录密码 |
| `PORT` | 与 Nginx 反代一致；与 simplewin.cn 共存时常用 **3100** |
| `PUBLIC_BASE_URL` | `https://geo.simplewin.cn` |
| `MERCHANT_AUTO_APPROVE` | 生产 **`false`** |
| `DEV_AUTH_ENABLED` | 真微信登录上线后 **`false`** |

脱敏生产需 `DESENSITIZE_ENGINE=aliyun` 及 ECS RAM 角色（见 `backend/.env.example` 注释）。

### 5.4 部署 API（二选一）

#### 路径 A · 一键脚本（systemd + Nginx）

适合首次裸机或希望脚本包办 Nginx/systemd：

```bash
cd /var/www/zhejian
sudo bash scripts/server-install.sh --bootstrap   # 可选：首次安装 node/nginx
sudo bash scripts/server-install.sh               # npm install + migrate + nginx + systemd
curl -s https://geo.simplewin.cn/api/v1/health
```

若 ECS 上已有 **simplewin.cn** 整站 Nginx 配置：

```bash
sudo bash scripts/server-install.sh --simplewin-nginx
```

#### 路径 B · PM2（与现有运维习惯一致，推荐）

与 **simplewin.cn :3000** 共存时，`backend/.env` 设 **`PORT=3100`**：

```bash
cd /var/www/zhejian/backend
npm install
npm run sync:shared-utils          # 同步 shared utils 到 vendor（必跑）
npm run db:setup:prod              # 生产默认不 seed

# 若曾用 systemd，先停掉避免抢端口
sudo systemctl stop zhejian-api 2>/dev/null || true
sudo systemctl disable zhejian-api 2>/dev/null || true

pm2 start ecosystem.config.cjs
pm2 save
pm2 startup                        # 按提示执行一条 sudo 命令

curl -s http://127.0.0.1:3100/api/v1/health
```

Nginx 真源：`backend/deploy/simplewin.conf` → `/etc/nginx/conf.d/simplewin.conf`  
独立 geo 模板：`backend/deploy/nginx-geo.simplewin.cn.conf`

### 5.5 运营后台静态资源

```bash
cd /var/www/zhejian/admin-web
npm install
npm run build
```

Nginx 将 `/admin/` 指到 `admin-web/dist/`（详见 `backend/deploy/simplewin.conf`）。

### 5.6 生产首次验收

```bash
cd /var/www/zhejian/backend
npm run deploy:verify -- https://geo.simplewin.cn

curl -s https://geo.simplewin.cn/api/v1/health
curl -sI https://geo.simplewin.cn/admin/
```

小程序：`services/config.js` → **`ACTIVE_ENV = 'prod'`**（见 [§1.3](#13-切到生产--真机小程序--线上-api)）  
微信公众平台 request 合法域名：`https://geo.simplewin.cn`

## 10. 常见问题

| 现象 | 处理 |
| --- | --- |
| 不知道切本地还是生产 | [§1 环境切换](#1-本地--生产怎么切换) |
| **P1000 Authentication failed** | [§4](#4-故障排查p1000--数据库连不上)：改 `DATABASE_URL` 与 MySQL 用户密码一致 |
| **P3009 failed migration** | [§4 P3009](#p3009存在-failed-migration)：`migrate resolve --rolled-back` 后重跑，或本地删库 `db:setup` |
| **ECONNREFUSED / Can't reach database** | MySQL 未启动；检查 Windows 服务或 `docker compose ps` |
| **请先修改 backend/.env 中的 YOUR_PASSWORD** | 生产脚本检测到占位符；改 `DATABASE_URL` 和 `DEV_*_TOKEN` |
| health **502** | `systemctl status zhejian-api` 或 `pm2 logs zhejian-api`；查端口是否与 Nginx 一致 |
| **端口被占用** | `ss -tlnp \| grep :3100`；改 `.env` 的 `PORT` 并同步 Nginx |
| `/admin/` **404** | 是否 `admin-web` 已 `npm run build`；Nginx root 是否指向 `dist` |
| 看板无浏览数据 | 是否 `git pull` 更新了 `h5/`；是否配置 [§7.1 crontab](#71-商家看板日聚合每天-000-上海时间) |
| Nginx SSL 报错 | `grep -r ssl_certificate /etc/nginx/` 找证书路径，写入 `.env` 的 `SSL_CERTIFICATE*` 或见 `server-install.sh` 提示 |
| 迁移列不存在 | `npm run db:migrate`；本地可重建库后重跑 `db:setup` |
