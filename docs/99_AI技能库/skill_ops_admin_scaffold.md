# Skill: Ops Admin Scaffold（运营审核台页面骨架）

## 目标

按项目规范生成 **`admin-web/`** 运营后台页面骨架（列表 / 详情 / 审核操作 / **脱敏图 + OCR 审核**），支撑卷三 **OPS-MASK-01**、**B-MERCH-04**、**A-PUB-06** 等审核能力。

## 使用场景

- 新建 `admin-web/` 子模块或页面（案例审核、商家审核、相册审核、脱敏风控）
- 扩展运营台：待办列表、筛选、审核通过/驳回/要求修改、操作留痕
- 对接 `https://geo.simplewin.cn/api/v1/` 运营类接口（待建时先 mock + 标注）

## 与 `page-scaffold` 的分工

| Skill | 端 | 目录 |

## 页面类型模板

### A. 审核列表页

- 筛选：状态、时间、门店、风险等级、来源（用户授权 / 冷启动）
- 表格：ID、标题、门店、状态 Tag、提交时间、审核员
- 分页 + loading / empty / error
- 行点击进入详情

### B. 审核详情页

必备模块（按 PRD 裁剪）：

```text
基础信息（案例/商家/相册）
关联对象（albumId、storeId、authorization）
脱敏图区 + OCR/风险摘要（OPS-MASK-01；不展示原图）
授权/责任确认记录
价格/合规摘要（只读，事故车无确定价）
审核操作区：通过 / 驳回 / 要求修改 / 下线
审核日志时间线
```

### C. 脱敏审核区（OPS-MASK-01）

- **仅展示脱敏图**（审核依据）+ OCR 命中项、风险等级（低/中/高/禁止公开）
- **禁止**在运营审核台展示原图；Admin API **不返回** `rawUrl`
- 操作：重试脱敏、要求修改（联动 B-MASK-06）

> 商家/用户端 `DesensitizeWorkbench` 仍可原图|脱敏对比；**运营审核台与之不同**，见 `04_公开案例审核.md` §8.2。

## 生成原则

1. **鉴权优先**：路由守卫 + `admin_token`；未登录跳转登录页；按角色隐藏菜单（见运营 PRD §3.3）。
2. **原图隔离**：**运营审核 API/页面不返回、不展示原图**；禁止写入公开 H5、禁止 console 持久打印。
3. **状态完整**：列表/详情均需 loading、empty、error；审核提交防重复。
4. **操作留痕**：通过/驳回/下线须调 API 写 audit log（接口未就绪时 stub + TODO 任务 ID）。
5. **合规文案**：详情页展示平台免责声明（非质量背书、价格仅供参考、审核员仅查看脱敏图等）；禁止交易/奖励导向文案。
6. **部署约定**：`base: '/admin/'`；API 同域 `/api/v1/`；见 `admin-web/README.md`。
7. **mock 标注**：接口未实现时使用 mock，文件头注释 `# MOCK` + 对应 `00_开发计划` 任务 ID。

## 实现后自查

1. 非 admin 角色无法访问页面与 API。
2. 详情页可完成「看脱敏图 + OCR → 填原因 → 提交审核」闭环（或 mock 闭环）。
3. 通过/驳回后列表状态刷新正确。
4. 文案符合 V2.0（无订单/评价/奖励审核入口）。
5. 构建产物 `base` 为 `/admin/`，本地 preview 路径正确。
6. **Admin 详情响应与页面均无 `rawUrl`/原图预览。**

## 约束

- 不要复制用户端 `components/` 到 admin-web；运营 UI 用后台组件库，**语义与 Tag 文案** 与设计体系一致。
- 不要在新运营页引入 V1 订单/评价/奖励管理。
- 不要在没有 audit API 时假装「已留痕」——须标注 mock 或先补 backend 任务。
- **不要在运营审核台复用商家端「原图|脱敏对比」布局。**
