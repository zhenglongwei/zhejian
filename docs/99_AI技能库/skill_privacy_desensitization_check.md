# Skill: Privacy Desensitization Check

## 目标

检查服务相册、公开案例、分享内容是否符合隐私和脱敏规范。

## 使用场景

- 服务相册页
- 案例详情页
- H5 公开案例页
- 分享页面
- 授权公开页面
- 商家上传相册页面

## 必读文档

- `docs/04_维修过程相册/06_自动脱敏规则.md`
- `docs/00_设计规范/00_辙见平台设计体系.md`（§7、隐私相关）
- 相关业务 PRD

## 检查规则

1. **平台公示**（H5 公开页、公众号、站外 GEO）：必须使用脱敏图；未授权公示不得出现。**不含** 小程序案例 Tab（卷六 DS-DOC-01）。
2. **用户自主分享**（私人传播）：默认脱敏；若支持原图分享须有 **风险提示 + 留痕**；**不得** 强制仅脱敏、**不得** 要求先授权公示才能分享。
3. **授权公示** 须说明公开范围（与「分享」分开展示；界面文案仅用「授权公示」）。
4. 图片/文案中 **平台公示** 不得展示：车牌、手机号、人脸、身份证、详细住址、付款码等敏感信息。
5. **用户授权案例**必须与**商家历史案例**区分展示（标签、文案、排序）。
6. 公开案例应展示：已脱敏、已审核；历史案例加价格仅供参考。
7. 相册空节点不得使用强警告样式（中性引导）。
8. **运营审核台**：审核员 **仅查看脱敏图 + OCR 摘要**；Admin API **不返回** `rawUrl`；**禁止**在审核页展示原图（见 `04_公开案例审核.md` §8.2）。

**口径**：`docs/04_维修过程相册/00_Phase1_服务相册产品口径.md` §4.5。

## 约束

- 不直接展示原图 URL 到公开页面。
- 不允许把商家端上传原图用于用户分享页。
- **运营审核台**不得展示原图；Admin 案例详情 API 不得返回 `rawUrl`。
