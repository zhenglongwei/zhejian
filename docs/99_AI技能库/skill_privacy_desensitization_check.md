# Skill: Privacy Desensitization Check

## 目标

检查维修相册、公开案例、分享内容是否符合隐私和脱敏规范。

## 使用场景

- 维修相册页
- 案例详情页
- H5 公开案例页
- 分享页面
- 授权公开页面
- 商家上传相册页面

## 必读文档

- `docs/04_维修过程相册/06_自动脱敏规则.md`
- `docs/00_设计规范/00_透明维修平台设计体系.md`（§7、隐私相关）
- 相关业务 PRD

## 检查规则

1. 用户端公开展示必须使用脱敏图。
2. 分享内容必须说明只分享脱敏内容。
3. 授权公开必须说明公开范围。
4. 图片/文案中不得展示：车牌、手机号、人脸、身份证、详细住址、付款码等敏感信息。
5. 平台订单案例必须区分于商家历史案例。
6. 公开案例应展示：已脱敏、已审核；历史案例加价格仅供参考。
7. 相册空节点不得使用强警告样式（中性引导）。

## 推荐组件

- PrivacyBanner
- ComplianceNotice
- AlbumNode
- Tag
- CaseCard

```md

## 问题清单

| 文件 | 问题 | 风险 | 建议 |
|---|---|---|---|

## 必须补充的提示

1.
2.
3.
```

## 约束

- 不直接展示原图 URL 到公开页面。
- 不允许把商家端上传原图用于用户分享页。
