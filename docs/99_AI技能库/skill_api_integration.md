# Skill: API Integration

## 目标

规范页面接口接入方式，保证 loading、empty、error、unauthorized 状态完整。

## 使用场景

- 页面接真实接口
- 替换 mock 数据
- 添加分页
- 添加提交表单

## 必读文档

- `docs/10_技术架构与接口/04_接口规范.md`（若存在）
- `docs/10_技术架构与接口/02_前端架构.md`（请求封装、错误码）
- 对应页面业务 PRD

## 接入要求

1. 请求开始设置 loading。
2. 请求成功但列表为空显示 Empty。
3. 请求失败显示错误状态和重试按钮。
4. 登录失效显示 LoginSheet 或授权提示。
5. 分页列表要处理：loadingMore、noMore、refresh。
6. 提交类接口必须防重复提交。
7. 错误信息不要直接展示后端堆栈。
8. 涉及隐私、价格、**咨询线索/相册状态**时要同步更新对应组件状态。

## 输出格式

```md
# API 接入计划

## 页面

## 接口列表

| 接口 | 用途 | 时机 |
|---|---|---|

## 状态设计

| 状态 | 变量 | 展示 |
|---|---|---|

## 错误处理

## 文件变更计划
```

## 约束

- 不要在 WXML 写过于复杂的业务判断（逻辑放 ts/js）。
- 接口字段需做兼容处理。
- 不要把 mock 数据遗留为正式逻辑（须标注或删除）。
