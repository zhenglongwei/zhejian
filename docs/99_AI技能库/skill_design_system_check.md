# Skill: Design System Check

## 目标

检查页面、组件、WXSS 是否符合透明维修平台设计体系。

## 使用场景

- 新页面开发完成后
- 老页面重构前
- Cursor 自动生成代码后
- 上线前 UI 检查

## 必读文档

执行前必须阅读：

- `docs/00_设计规范/00_透明维修平台设计体系.md`
- `docs/00_设计规范/01_设计令牌_tokens.md`
- `docs/00_设计规范/02_组件API规范.md`

## 检查范围

1. 是否使用设计令牌
2. 是否存在硬编码颜色
3. 是否存在硬编码圆角、阴影、字号、间距
4. 是否复用基础组件和业务组件
5. 是否符合透明维修平台的视觉定位
6. 是否存在促销化、低价导向、过度营销视觉
7. 是否处理 loading / empty / error / disabled 状态
8. 固定底部是否适配 safe-area
9. 标签、价格、案例来源、隐私提示是否符合规范

请按以下格式输出：

```md

## 总结

- 检查页面：
- 风险等级：低 / 中 / 高
- 是否建议立即修改：是 / 否

## 问题清单

| 序号 | 文件 | 问题 | 风险 | 建议 |
|---|---|---|---|---|

## 需要替换的硬编码样式

| 文件 | 当前写法 | 建议替换 |
|---|---|---|

## 组件复用建议

| 当前实现 | 建议组件 |
|---|---|

## 修改优先级

1. 高优先级：
2. 中优先级：
3. 低优先级：
```

## 约束

- 默认只检查，不直接改代码。
- 如需修改，必须先输出修改计划。
- 不得引入新的视觉体系。
- 不得新增未经 token 管理的颜色。
