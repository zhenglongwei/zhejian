## 2. 产品印象与设计 north star

### 2.1 我们不是什么

- 不是低价团购、秒杀式汽修 App（避免刺眼促销红、满屏「限时抢」）。
- 不是保险/事故黄牛导流工具（避免灰色诱导、夸大承诺）。
- 不是纯比价货架（排序与视觉不以「最低价」为唯一卖点）。
- **不是 B/C 撮合中介或交易担保方**（避免「平台担保」「平台订单」「平台推荐最低价」等中介平台话术；见 §2.4、§9.0）。

### 2.2 我们是什么

**辙见**面向车主展示可验证的维修过程与本地门店信息；面向商家提供**营销展示与案例沉淀工具**；面向搜索与 AI 提供**可索引、可引用的 GEO 信源**。

| 角色 | 产品本质 | 设计应传达 |
|---|---|---|
| **商家** | 营销工具：自助发布服务说明、门店信息与案例 | 「你的内容被看见」「过程透明有助于获客」 |
| **车主** | 案例库 + 咨询入口：看过程、找门店、留线索 | 「内容来自门店/授权车主」「线下与门店确认方案与费用」 |
| **GEO / AI** | 可靠信源：结构化字段 + 脱敏证据链 | 摘要块、关键信息表、可引用节点元数据 |

设计应持续传达四件事：

| 维度 | 用户感知 | 设计表达 |
|---|---|---|
| **可信** | 案例真实、门店有资质、过程有图 | 证据优先布局、来源标签、审核态 |
| **透明** | 价格有说明、过程可查看 | 价格模式分区、时间线相册、影响因素列表 |
| **易懂** | 不懂车也能看懂 | 翻译层文案、步骤化节点、少术语多结论 |
| **克制** | 不替门店打包票、不忽悠 | 合规提示常驻、事故车/区间价免责声明清晰 |

### 2.3 设计 north star（一句话）

> **像一份「可翻阅的服务相册」，而不是一张「促销传单」。**

### 2.4 小程序定位 · 文案基准（必守）

> **定调 2026-06-02** · 业务口径见 `docs/01_项目总览与业务架构/02_业务模式说明.md` §2.1

#### 本质差别：营销工具 + GEO 信源，而非中介平台

| | 辙见小程序（商家端 + 用户端） | 传统 B/C 中介平台 |
|---|---|---|
| 关系 | 商家 **自助发布** 内容与案例；用户 **自行联系** 门店 | 平台撮合交易、定价、履约、售后 |
| 责任 | 内容为商家发布或车主授权展示，**仅供参考** | 常暗示平台背书、平台价、平台保障 |
| 监管 | 公开案例脱敏与授权；服务方案 **事后** 抽查/举报 | 上架前审核、平台定价规则、交易拦截 |
| 目标 | 门店获客、案例沉淀、**GEO/AI 可引用信源** | GMV、抽佣、订单闭环 |

**设计 implication**：界面与文案应让用户/商家感知「**我在用一套展示与案例工具**」，而非「**我在和一个中介平台做交易**」。

#### 小程序可见文案总则

以下约束适用于 **用户端小程序、商家工作台（同小程序）** 中一切 **用户可见** 文案（按钮、提示、Empty、合规条、Toast、Modal、设置页协议摘要等）：

1. **默认禁用「平台」作主语**（如「平台审核」「平台担保」「平台提供订单」）。易引发中介/担保联想，与定位冲突。
2. **优先用词**：
   - 指产品：**辙见**（品牌名）
   - 指能力：**此处 / 本页 / 线上**（如「此处不提供最终报价」）
   - 指责任主体：**门店 / 商家 / 车主**（如「内容由商家发布」「经车主授权展示」）
   - 指流程：**审核 / 授权公示 / 下架**（不说「平台处罚」；商家端用「下架说明」「审核意见」）
3. **商家端额外注意**：商家正在 **自行编辑要发布的内容** 时，**不要**再写「本页由商家自行发布」类免责声明——该条仅用于 **用户端预览/详情**。
4. **允许出现「平台」的场景**：
   - **运营工作台**（`admin-web`）、内部 PRD/设计文档、法务完整协议正文（经法务确认）
   - 对外品牌全称 **「辙见服务平台」** 仅在 **关于辙见 / 协议标题** 等必要处 **少量** 使用
5. **禁止暗示**（无论是否出现「平台」二字）：统一报价、平台推荐、平台保障、平台订单、好评返现、分享赚钱。

#### 文案自检问句（PR 前必答）

- 这句话是在描述 **工具能力**，还是在扮演 **交易中介**？
- 商家看到会觉得自己 **被审核上架**，还是 **在用营销工具发布**？
- 车主看到会以为 **辙见担保维修结果**，还是 **门店内容仅供参考**？
- 若去掉「平台」二字，语义是否仍然正确？（若否，说明文案仍带中介色彩，应重写）

**关联**：合规条组件 `ComplianceNotice` 类型 `displayDisclaimer`；禁用词表 §9.1；AI 自检 skill `skill_design_system_check.md`。

## 3. 设计原则（必须遵守）

### P1 · 证据先于说服

- 案例卡、门店卡、服务详情：**先展示脱敏图 / 节点图 / 关键信息表，再展示营销文案**。
- 列表缩略图优先维修前后对比、新旧件对比，而非门头照 alone。

### P2 · 价格诚实可见

- **一口价 / 套餐价 / 参考区间 / 到店检测** 四种模式必须在视觉上可区分（标签 + 排版，见 §6.4）。
- 事故车、复杂维修：**禁止**出现类似最终成交价的暗示；主按钮为「预约检测」「查看案例」。

### P3 · 来源可识别

- 公开案例 Tag：**已授权 · 已脱敏 · 已审核**（Phase 1 统一口径；按授权档位控制展示字段）。
- ~~用户授权案例 vs 商家历史案例~~：**Phase 1 不再按来源二分 Tag**；新页面勿新增「商家历史案例」主 Tag（遗留 H5 fixture 改造见 R7）。

### P4 · 隐私默认安全

- 公开展示仅脱敏图；私密相册有锁态/提示条。
- 分享、授权公开、打码裁剪流程使用同一套「隐私安全」视觉语言（盾牌/锁图标 + 短文案）。

### P5 · 商家友好、不羞辱

- 相册为空、节点未完成：**引导上传**，不用红色警告态羞辱商家；用中性说明 + 收益提示（透明度、案例曝光）。

### P6 · 合规文案内置

- 评价奖励、分享、订阅等场景：**禁止**「好评返现」「分享赚钱」「转发领现金」等表述（见 §9）。
- 奖励在「收益中心」内说明规则，不在外部分享按钮上诱导。

### P7 · 多端一致、场景可分

- Token（色、字、间距）全局一致；**用户端偏温暖可信、商家/运营端偏高效信息密度**（见 §11）。

### P8 · 为内容资产而设计

- 案例页、门店页、服务页结构需利于 H5/SEO/GEO：标题区 → AI 摘要 → 关键信息表 → 过程图 → FAQ（与 `docs/09_*`、`docs/05_*` 对齐）。

## 4. 品牌与语气

### 4.1 品牌主文案（对外统一）

| 场景 | 文案 |
|---|---|
| 主标语 | 辙见服务平台 |
| 副标语 | 修车过程看得见，每次维修有档案 |
| 补充说明 | 查看真实维修案例，预约杭州本地可信维修门店 |
| 对用户 | 不懂车也能看明白 |
| 对商家 | 让专业被看见，把每一单变成可信案例 |

### 4.2 语气

- **用户端**：平和、解释型、少命令式；用「建议」「可以参考」「实际费用以门店检测为准」。
- **商家端**：简洁、任务导向；用「待办」「建议上传」「有助于提升透明度」。
- **运营端**：中性、可审计；状态、审核结论、风险指标一目了然。
- **禁止**：绝对化承诺（「100% 修好」「全网最低」）、恐吓式（「再不修就报废」）。

### 4.3 名称与术语

与 `docs/01_项目总览与业务架构/06_术语表.md` 保持一致：

- 服务相册、订单相册、公开案例、翻译层、六维评价、透明度评分等——**界面用语不得自造同义词**（如把「服务相册」写成「施工日记」除非产品统一改名）。

**小程序可见文案 · 禁用/替换**（详见 §2.4）：

| 避免（用户/商家可见） | 推荐替换 |
|---|---|
| 平台审核 / 平台处罚 / 平台下架 | 审核 / 下架说明 / 已下架 |
| 平台公示 | **授权公示**（与「分享」区分） |
| 平台订单 / 平台交易 | 咨询记录 / 预约信息 / 线下确认 |
| 平台担保 / 平台推荐 | （删除或改为）内容仅供参考；实际以门店为准 |
| 平台客服 | **客服** / 联系客服 |
| 平台规则 / 关于平台 | **使用规则** / **关于辙见** |
| 平台提供…（能力描述） | **辙见提供…** 或 **此处提供…** |

### 5.5 图标与插图

- 图标：**线性图标**为主，线宽 2rpx，圆角端点；与 Ant Design / Remix Icon 风格接近。

- 车辆/维修相关：优先实拍图（案例、相册），插画仅用于空状态和教育说明。

## 6. 核心业务模式 · UI 映射

以下模式在 PRD 中反复出现，**必须组件化**，禁止每页重写样式。

### 6.1 价格展示四型

| 类型 | 标签文案 | 展示格式 | 主按钮 |
|---|---|---|---|
| 一口价/套餐 | `一口价` / `套餐` | `¥299` 起 + 包含项折叠 | 咨询预约 |
| 参考区间 | `到店确认` | `参考区间 ¥300-¥1200` + 影响因素 | 预约到店 |
| 仅预约 | `预约咨询` | 不展示具体金额，展示「到店检测后报价」 | 预约检测 |
| 事故车 | `事故车` | 固定说明文案块（见 PRD） | 查看案例 / 预约检测 |

**影响因素**：使用可折叠「为什么价格不同」列表（车型、配件品牌、损伤程度…）。

### 6.2 咨询线索与服务相册状态色

| 状态组 | 颜色 | 示例状态 |
|---|---|---|
| 待处理 | `--color-warning` | 待联系、待到店、配件待确认 |
| 进行中 | `--color-primary` | 维修中、相册进行中 |
| 完成 | `--color-success` | 已到店、相册已完工、案例已公开 |
| 异常 | `--color-danger` | 已关闭、审核驳回 |

咨询记录与相册详情 **状态文案 + 颜色一致**。（V2.0 无订单/支付状态 UI）

### 6.3 维修进度时间线

用于：服务相册首页、商家履约、咨询记录关联相册。

```
● 接车登记 ──● 检测诊断 ──● 配件确认 ──● 施工维修 ──● 质检交车 ──● 完工总结
```

- 已完成节点：实心圆 + 主色；当前节点：主色脉冲或加粗；未到：灰空心。
- 每个节点可挂载「查看相册」次级链接。

### 6.4 服务相册 · 节点视图

- **纵向时间线 + 节点标题 + 图片网格（3 列）** 为默认布局。
- 节点名使用模板标准名（如「旧件状态」「新旧对比」），商家可改说明不可改节点 ID 样式。
- 空节点：虚线框 + 「商家暂未上传」；不显示破损图标。
- 用户端顶部：**隐私提示条**「仅你可见，分享将使用脱敏版本」。

### 6.5 案例卡片（列表通用）

```
[脱敏缩略图 16:9]  标签行：用户授权案例 · 已脱敏
车型 · 服务项目
维修摘要（一行）
参考价格区间 | 门店名
```

- 历史案例卡片：标签行首项为 `商家历史案例`，整体略降饱和度或加「仅供参考」脚注。

### 6.6 门店卡片

```
[门头图]  门店名  距离
资质标签 | 擅长标签
案例数 · 评分 · 透明度
```

- **透明度评分** 使用 `--color-info` 系列，与星级评分分区展示，不混在一个数字里。

### 6.7 关键信息表（案例/订单摘要）

用于案例详情、H5、授权预览：

| 字段 | 值 |
|---|---|
| 城市 | 杭州 |
| 服务项目 | 刹车片更换 |
| … | … |

表格样式：左列 `--color-text-secondary` 固定宽，右列正文；移动端可转为 label-value 行。

## 10. 页面模板与信息优先级

### 10.1 用户端 · 首页

优先级（从上到下）：城市/搜索 → 主标语 → P0 服务入口 → 事故车入口 → 附近门店 → 精选案例 → 辙见说明 → 保障说明。

**视觉分层**（与 `02_首页与服务发现.md`、进度 **R9.3 / R10** 对齐）：

- **P0 结构**（R9.3）：模块顺序、卡片容器、token 间距；宫格可用汉字占位
- **P1 体验**（R10）：统一宫格图标、Hero 层次、门店/案例缩略图、Tab 品牌稿；禁止促销传单式 Banner

### 10.2 用户端 · 服务详情

标题 → 价格模式组件 → 服务说明 → 适用/不适用 → 影响因素 → 类似案例 → 门店列表 → 流程 → FAQ → 底栏下单。

### 10.3 用户端 · 订单详情

状态区 → 进度时间线 → 门店/服务/车辆 → 费用 → **服务相册入口（显眼）** → 操作按钮 → 售后/评价入口。

### 10.4 用户端 · 服务相册

摘要 → 隐私条 → 节点时间线 → 门店说明 → 档案/授权入口。

### 10.5 商家 · 工作台首页

门店切换 → 待办数字卡片 → 快捷操作（新建案例、上传相册）→ 今日数据 → 公告。

### 10.6 商家 · 相册上传

建议节点 checklist（完成度进度条）→ 按节点上传 → 一键生成说明（占位）→ 提交。

### 10.7 H5 公开案例页

SEO 标题 → 标签行 → AI 摘要 → 关键信息表 → 维修说明 → 过程图（脱敏）→ 价格区间与因素 → 门店卡片 → FAQ → 跳转小程序 CTA。

### 12.1 加载

- 首屏：Skeleton；下拉刷新用微信默认或品牌色三点。
- 图片：渐进加载 + 缩略图模糊；失败显示占位图可点击重试。

| 场景 | 文案方向 | 操作 |

- 时长：200–300ms；缓动 `ease-out`。
- **禁止**：金币动画、全屏闪动、自动轮播大图（干扰阅读证据）。
