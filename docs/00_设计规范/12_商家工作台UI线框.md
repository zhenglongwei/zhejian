## 0. 与用户端工具台对照

| 维度 | 用户端 `pages/mine/index` | 商家端 `workbench/index` |
| --- | --- | --- |
| 定位 | 车主翻阅服务相册 | 门店生产/管理服务相册 |
| 壳层 | `showBand=false` | 同左 |
| 扉页 | `ui-mine-user-header`（用户） | **`ui-merchant-store-header`**（门店） |
| P0 Hero | `ui-album-card` ×1～2 `audience=user` | **`ui-album-card` ×1～2 `audience=merchant`** |
| 待办 | 仅有待办时一块；**仅咨询待处理** | 同语法；仅 `pendingLeads` |
| 次要 | 公示激励弱条、H5 弱链 | 案例动态 P1、GEO 弱条（条件） |
| Dock | **3 格** | 新建相册 / 咨询线索 / 服务方案 |
| 管理 | 主账号 · 页内文字链 | 门店主页 · 编辑资料 · 员工管理（与 Dock 同卡） |

**north star**：与用户端同一套「单页工具台」容器语法；差异在 **内容**（门店 vs 车主），不在卡片样式分叉。

### 1.1 Hero · 服务相册

| 项 | 规则 |
| --- | --- |
| 组件 | `ui-album-card`，`audience="merchant"`，`showProgress="{{true}}"` |
| 数量 | **默认 1 条**（最近更新）；有第二本「进行中」可叠 **2 条** |
| 数据 | `enrichMerchantAlbumListItem`；须含 `coverUrl`（与列表页一致） |
| 区头 | 「服务相册」+「全部 ›」→ `album/list` |
| 角标 | 无（相册进度见 Hero 卡） |
| 空态 | `ui-empty` +「新建服务相册」 |
| 禁止 | sticky 大待办卡内嵌主按钮；无封面的纯文字摘要作 Hero |

### 1.2 待办聚合

| 项 | 规则 |
| --- | --- |
| 展示 | **无待办时整块不渲染** |
| 样式 | 对标 `mine__todo`：headline + 圆点 + `text-caption` + › |
| 项 | 仅 **咨询待处理**（`pendingLeads > 0`）；车主授权由车主端待办，商家工作台不提醒 |
| 禁止 | 彩色左边框告警行；`0 条` 占位行 |

### 1.3 快捷入口（Dock + 文字链 · 单卡）

**Dock 三格**

| key | 标签 | 跳转 | 角标 |
| --- | --- | --- | --- |
| `createAlbum` | 新建相册 | `album/create` | — |
| `leads` | 咨询线索 | `lead/list?tab=pending` | `pendingLeads` |
| `services` | 服务方案 | `service/list` | — |

视觉：`mine__dock` 同款；**一张** `section--panel` 包裹 Dock + 下方文字链，禁止第二块宫格卡。

**数据概览**：不占 Dock；扉页 `overviewLine`（近7天咨询 · 透明度）点击 → `dashboard`。

**主账号文字链**（Dock 下方分隔线 + `text-link`，非 icon 宫格）

| key | 标签 | 跳转 |
| --- | --- | --- |
| `storeHome` | 门店主页 | 用户端门店详情（预览/分享） |
| `editStore` | 编辑资料 | `store/edit` |
| `staff` | 员工管理 | `staff/list` |

非主账号：仅 Dock 三格，无文字链。

### 1.5 移除/降级

| 原实现 | 改造后 |
| --- | --- |
| 页内 `bg-band`「商家工作台」 | 删除 |
| sticky `action-cabin` 待办大卡 | 拆为轻待办 + Hero 区头 CTA |
| 横滑 `quick-strip` | 并入 Dock + Cell |
| 顶栏通知/消息双 icon | 消息 → 扉页角标或 Dock；订阅 → **场景弹窗** |
| 底部 KPI 独立大卡 | 弱文案并入扉页或仅 Dock「数据」入口 |
| GEO 大面板 | 条件弱条 |

## 4. 组件与样式

| 复用 | 说明 |
| --- | --- |
| `ui-tool-page-shell` | `showBand=false`，gate 与 normal 统一 |
| `ui-album-card` | merchant Hero，与 `album/list` 一致 |
| `ui-empty` / `ui-skeleton` | loading 在 shell 内，勿跳出独立 page |
| `ui-cell` | 门店管理组 |
| `constants/merchant-hub.js` | 待办文案与聚合（对标 `mine-hub.js`） |
| 布局类 | 优先共享 `mine__hub` / `mine__section--panel` 语义，少 `merchant-wb__*` |

---

## 5. 禁止清单

- 页内重复「服务相册 · 汽车维修档案」band  
- 首屏 5+ 同级大阴影卡片平铺  
- Hero 无封面纯文字行（除非 API 降级且文档注明）  
- 待办 `0` 条仍展示  
- 顶栏「开启微信通知」主入口  

---

## 6. 验收要点

1. 与用户端首页并看，容器节奏一致（扉页 → Hero → 待办 → Dock）。  
2. Hero 卡片与商家相册列表页视觉一致。  
3. 无待办时无待办块；无相册时 Hero 空态可新建。  
4. Dock 三格可点通；主账号可见同卡文字链；扉页概览可进数据看板。  
5. `design-system-check` 无散落硬编码色值。
