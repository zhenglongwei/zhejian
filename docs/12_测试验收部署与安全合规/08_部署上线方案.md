## **2. 上线目标**

本方案用于保障平台从测试环境发布到生产环境时：

1. 发布范围明确；
2. 发布步骤可执行；
3. 数据库变更可控；
4. 配置切换安全；
5. 生产问题可监控；
6. 发布失败可回滚；
7. 核心链路上线后可验证。

## **4. 上线前准备**

## **4.1 代码准备**

必须确认：

1. 主分支代码已合并；
2. 发布分支已冻结；
3. 版本号已确定；
4. Tag 已创建；
5. 代码 Review 已完成；
6. 无未合并的临时修改；
7. 无测试账号、测试密钥写死；
8. 无 console/debug 级别敏感日志；
9. 构建产物已生成；
10. 依赖版本已锁定。

---

## **4.2 配置准备**

生产配置包括：


| 配置项         | 要求              |
| ----------- | --------------- |
| 数据库连接       | 使用生产库           |
| Redis/缓存    | 使用生产实例          |
| 对象存储        | 使用生产 bucket     |
| 微信小程序 AppID | 生产配置            |
| 微信支付商户号     | 生产商户            |
| 支付回调地址      | HTTPS生产域名       |
| 退款回调地址      | HTTPS生产域名       |
| 消息模板ID      | 生产模板            |
| H5域名        | 已备案、证书有效        |
| 后台域名        | 权限受控            |
| 日志级别        | INFO/WARN/ERROR |
| 内容安全服务      | 生产密钥            |
| 地图/OCR服务    | 生产密钥            |


---

## **4.3 数据库准备**

必须完成：

1. 数据库初始化；
2. 表结构迁移；
3. 索引创建；
4. 枚举字典初始化；
5. 角色权限初始化；
6. 后台管理员账号初始化；
7. 城市数据初始化；
8. 服务类目初始化；
9. 审核原因字典初始化；
10. 消息模板配置初始化；
11. 任务调度配置初始化；
12. 数据备份策略启用。

数据库变更要求：

```
所有DDL必须提前评审
所有变更脚本必须可重复检查
高风险变更需准备回滚脚本
生产执行前必须备份

```

---

## **4.4 域名与证书准备**

需要确认：

1. 小程序服务器域名已配置；
2. H5域名可访问；
3. 后台域名可访问；
4. API域名可访问；
5. HTTPS证书有效；
6. 支付回调域名可被微信访问；
7. 上传回调域名可被对象存储访问；
8. CDN配置正确，若使用；
9. robots.txt 配置正确；
10. sitemap.xml 可访问。

---

## **4.5 第三方服务准备**

必须确认：

1. 微信登录可用；
2. 微信支付可用；
3. 微信退款可用；
4. 微信订阅消息模板可用；
5. 对象存储上传可用；
6. 内容安全服务可用；
7. OCR/脱敏服务可用；
8. 地图定位服务可用；
9. 短信服务，如使用，可用；
10. 日志/监控服务可用。

---

## **5. 发布流程**

推荐发布顺序：

```
1. 数据库备份
2. 执行数据库迁移
3. 发布后端服务
4. 发布运营后台
5. 发布商家端
6. 发布H5公开页
7. 提交/发布用户端小程序
8. 配置支付、上传、消息回调
9. 执行生产冒烟测试
10. 开启灰度或全量

```

---

## **6. 后端服务部署**

后端部署步骤：

1. 拉取发布镜像或构建产物；
2. 校验版本号；
3. 更新环境变量；
4. 执行数据库迁移；
5. 启动服务；
6. 健康检查；
7. 查看启动日志；
8. 验证核心接口；
9. 验证定时任务；
10. 验证回调地址。

健康检查接口建议：

```
GET /health
GET /actuator/health

```

返回：

```
{
  "status": "UP",
  "version": "v1.0.0",
  "time": "2025-01-01T10:00:00Z"
}

```

## **7. 前端部署**

## **7.1 用户端小程序**

发布步骤：

1. 构建生产包；
2. 检查 AppID；
3. 检查 API 域名；
4. 检查隐私协议；
5. 上传微信小程序版本；
6. 提交审核；
7. 审核通过后灰度发布；
8. 观察核心指标；
9. 全量发布。

---

## **7.2 商家端**

如果商家端为小程序或H5，需要确认：

1. 生产API地址；
2. 登录授权；
3. 文件上传域名；
4. 消息通知配置；
5. 版本号；
6. 灰度策略；
7. 回滚包。

---

## **7.3 运营后台**

部署步骤：

1. 构建后台前端；
2. 发布到静态资源服务；
3. 配置后台API地址；
4. 配置登录和权限；
5. 创建管理员账号；
6. 验证审核功能；
7. 验证敏感信息脱敏；
8. 验证操作日志。

---

## **7.4 H5公开页**

部署步骤：

1. 构建H5产物；
2. 配置H5域名；
3. 配置服务端渲染或静态生成策略；
4. 配置 sitemap；
5. 配置 robots；
6. 配置 canonical 域名；
7. 验证公开案例页；
8. 验证未公开页面不可访问；
9. 验证页面不泄露隐私。

---

## **8. 生产冒烟测试**

## **8.2 商家链路**

## **8.3 后台链路**

## **8.5 H5公开页链路**

## **9. 灰度发布策略**

建议采用灰度：


| 阶段    | 流量      | 观察时间  |
| ----- | ------- | ----- |
| 内部验证  | 内部账号    | 2-4小时 |
| 小范围灰度 | 5%-10%  | 1天    |
| 扩大灰度  | 30%-50% | 1-2天  |
| 全量发布  | 100%    | 持续观察  |


灰度期间重点观察：

1. 登录成功率；
2. 下单成功率；
3. 支付成功率；
4. 支付回调成功率；
5. 商家接单率；
6. 上传成功率；
7. 退款成功率；
8. 接口错误率；
9. 页面访问错误率；
10. 投诉与客服反馈。

---

## **10. 监控与告警**

上线必须配置：


| 监控项        | 告警条件     |
| ---------- | -------- |
| 服务可用性      | 健康检查失败   |
| 接口错误率      | 5xx异常升高  |
| 支付回调失败     | 连续失败或超阈值 |
| 退款失败       | 连续失败或超阈值 |
| 订单状态异常     | 状态长时间停留  |
| 上传失败率      | 高于阈值     |
| 脱敏失败率      | 高于阈值     |
| 消息发送失败率    | 高于阈值     |
| 数据库连接      | 连接异常     |
| CPU/内存     | 超过阈值     |
| 磁盘空间       | 低于安全值    |
| H5 404/500 | 异常升高     |

## **11. 回滚方案**

## **11.1 可回滚内容**


| 内容   | 回滚方式           |
| ---- | -------------- |
| 后端服务 | 回滚到上一镜像/上一版本   |
| 后台前端 | 回滚静态资源版本       |
| H5页面 | 回滚前端构建产物       |
| 小程序  | 微信后台回退版本，视平台能力 |
| 配置   | 恢复上一配置         |
| 数据库  | 通过回滚脚本或备份恢复    |
| 定时任务 | 暂停任务或恢复旧任务     |
| 支付回调 | 恢复旧回调处理逻辑      |


---

## **11.2 回滚触发条件**

出现以下情况必须考虑回滚：

1. 支付无法完成；
2. 支付回调大面积失败；
3. 订单状态异常大面积出现；
4. 用户或商家严重越权；
5. 隐私信息公开泄露；
6. H5公开页大量泄露原图；
7. 退款金额错误；
8. 数据库写入异常；
9. 服务不可用超过可接受时长；
10. 无法通过降级修复。

---

## **11.3 回滚流程**

```
发现严重问题
→ 技术负责人确认影响范围
→ 产品/业务确认是否暂停入口
→ 暂停高风险功能或切流
→ 执行版本回滚
→ 校验核心服务
→ 修复异常数据
→ 发布事故说明
→ 复盘问题

```

---

## **12. 降级方案**

优先支持以下降级：


| 模块      | 降级方式                     |
| ------- | ------------------------ |
| 支付异常    | 暂停下单支付入口                 |
| 退款异常    | 暂停自动退款，转人工处理             |
| 上传异常    | 暂停视频上传，仅保留图片             |
| 脱敏异常    | 暂停公开案例发布                 |
| H5异常    | 暂停 sitemap 更新或临时 noindex |
| 消息异常    | 站内消息保底，微信消息重试            |
| 埋点异常    | 不影响主链路，异步降级              |
| 推荐/排序异常 | 使用默认排序                   |
| 案例审核异常  | 暂停发布，保留草稿                |

## **16. 上线后观察**

上线后至少观察：


| 时间       | 观察内容               |
| -------- | ------------------ |
| 0-30分钟   | 服务启动、错误日志、登录、首页    |
| 30分钟-2小时 | 下单、支付、商家接单、上传      |
| 2-24小时   | 退款、评价、案例审核、H5访问    |
| 1-3天     | 转化率、支付成功率、退款异常、投诉  |

---

## **17. 上线成功标准**

满足以下条件可认为上线成功：

1. 核心服务稳定；
2. 无P0事故；
3. 登录、下单、支付可用；
4. 商家接单履约可用；
5. 支付回调正常；
6. 退款流程正常；
7. 图片上传正常；
8. 后台审核正常；
9. H5公开页正常；
10. 无隐私泄露；
11. 监控无持续异常；
12. 客服反馈在可控范围内。
