## **2. 基础原则**

平台处理用户信息必须遵循：

1. 合法、正当、必要；
2. 明示收集目的；
3. 用户知情同意；
4. 最小必要采集；
5. 敏感信息单独授权；
6. 可查询、可更正、可删除；
7. 可撤回授权；
8. 公开展示必须脱敏；
9. 不得超范围使用；
10. 不得将原始隐私信息用于公开页面。

## **5. 授权类型**

平台至少需要以下授权类型：


| 授权类型   | 说明            |
| ------ | ------------- |
| 登录授权   | 微信登录及基础身份识别   |
| 手机号授权  | 咨询联系、通知      |
| 车辆信息授权 | 服务匹配、咨询预约     |
| 服务相册授权 | 商家上传服务相册供用户查看 |
| 相册影像处理授权 | 车主关联相册时，同意 OCR/脱敏/AI 辅助阅读等处理 |
| 公开案例授权 | 脱敏后用于公开展示     |
| ~~评价展示授权~~ | V2.0 不做 |
| 消息通知授权 | 微信订阅消息或站内消息   |
| 埋点分析授权 | 用于产品分析和体验优化   |
| 风控授权说明 | 用于反作弊和安全控制    |

## **6. 授权记录要求**

每次关键授权必须记录。

建议字段：


| 字段                | 说明          |
| ----------------- | ----------- |
| authorization_id  | 授权记录ID      |
| user_id           | 用户ID        |
| auth_type         | 授权类型        |
| business_id       | 关联咨询线索/服务相册/案例  |
| auth_status       | 已授权/已拒绝/已撤回 |
| auth_text_version | 协议版本        |
| auth_time         | 授权时间        |
| revoke_time       | 撤回时间        |
| client_type       | 小程序/H5/后台   |
| ip                | 授权IP，注意合规使用 |
| device_info       | 设备信息        |
| remark            | 备注          |

## **7. 隐私协议展示规则**

用户应在以下节点看到隐私说明或授权提示：

1. 首次登录；
2. 获取手机号；
3. 提交咨询/预约；
4. 填写车辆信息；
5. 上传图片；
6. 查看或生成服务相册；
7. ~~提交评价~~（V2.0 不做）；
8. 案例公开授权；
9. 分享脱敏公开案例；
10. 隐私协议更新时。

---

## **8. 服务相册授权规则**

服务相册包含车辆、环境、维修过程，可能涉及隐私。

## **8.1 用户可见规则**

用户本人可查看自己关联的服务相册。

商家可查看本门店创建的服务相册及关联用户的授权范围。

平台可在审核、客服、风控场景下查看。

其他用户不可查看未公开相册。

---

## **8.2 商家上传规则**

商家上传服务相册时必须遵守：

1. 只能上传本门店**服务相册**或历史案例素材；
2. 不得上传无关个人隐私；
3. 不得上传用户证件、聊天记录等非必要内容；
4. 不得上传包含明显人脸的照片，除非后续脱敏；
5. 不得上传外部联系方式引导交易；
6. 上传内容进入平台审核和脱敏流程。

---

## **8.3 原图存储规则**

原始图片和视频：

1. 必须私有存储；
2. 不得直接用于H5公开页；
3. 不得生成公开可访问URL；
4. 访问必须鉴权；
5. 运营后台访问应记录日志；
6. 超出保存期限后可归档或删除。

---

## **9. 公开案例授权规则**

公开案例授权是独立授权，不能默认等同于下单授权。

## **9.1 必须明确授权**

用户必须明确知道：

1. 维修图片可能脱敏后公开展示；
2. 案例可能用于平台H5页面；
3. 案例可能被搜索引擎收录；
4. 评价内容可能公开展示；
5. 用户可以撤回公开授权；
6. 撤回后平台将停止继续公开展示，但搜索引擎缓存清除存在延迟。

---

## **9.2 可公开内容**

满足以下条件才可公开：

1. 用户已授权；
2. 图片已脱敏；
3. 案例审核通过；
4. 不存在售后纠纷；
5. 不存在隐私风险；
6. 不包含车牌、VIN、人脸、手机号；
7. 不包含保险、理赔、证件等敏感材料；
8. 商家状态正常。

---

## **9.3 不可公开内容**

以下内容不得公开：

1. 未脱敏原图；
2. 用户手机号；
3. 完整车牌号；
4. 完整 VIN；
5. 用户身份证、行驶证；
6. 人脸；
7. 精确家庭地址；
8. 支付凭证；
9. 聊天记录；
10. 投诉纠纷材料；
11. 保险理赔资料；
12. 用户明确拒绝公开的内容。

---

## **10. 用户撤回授权**

用户应支持撤回以下授权。

**公开案例授权不受审核/公示状态限制**：用户可在授权后的 **任何时间** 发起撤回，包括案例 **待审核**、**已审核通过** 或 **已在 H5/小程序公开展示** 之后。平台不得以「已审核通过」或「已公示」为由拒绝或隐藏撤回入口。


| 授权     | 撤回后处理          |
| ------ | -------------- |
| 公开案例授权 | 下线案例或移除用户相关素材  |
| 评价公开授权 | 隐藏评价或匿名化处理     |
| 消息通知授权 | 停止对应通知         |
| 车辆信息授权 | 删除或匿名化非咨询履约必要信息  |
| 手机号授权  | 视咨询履约和法律留存要求处理 |
| 埋点分析授权 | 停止非必要追踪，保留匿名统计 |


---

## **10.1 撤回授权流程**

```
用户发起撤回
→ 系统确认授权类型
→ 判断是否存在未完成咨询履约或法定留存义务
→ 执行下线、匿名化或删除
→ 记录撤回日志
→ 通知用户处理结果

```

---

## **10.2 撤回公开案例授权后的处理**

应执行：

1. 案例状态改为下线或隐藏；
2. H5页面 noindex 或返回 404/410；
3. 从 sitemap 移除；
4. 不再用于推荐和聚合页；
5. 保留必要操作日志；
6. 通知搜索引擎更新，后期可选；
7. 通知商家案例已下线。

---

## **11. 脱敏规则**

公开展示前必须脱敏：


| 信息   | 脱敏方式         |
| ---- | ------------ |
| 车牌号  | 打码或模糊        |
| VIN  | 打码或裁剪        |
| 人脸   | 模糊或遮挡        |
| 手机号  | 只展示前三后四或完全隐藏 |
| 地址   | 只展示到城市/区县    |
| 证件   | 不公开展示        |
| 支付凭证 | 不公开展示        |
| 二维码  | 打码           |
| 微信号  | 删除或打码        |
| 门牌号  | 视场景隐藏        |

## **13. 商家信息展示规则**

商家公开展示的信息包括：


| 信息     | 是否可公开      |
| ------ | ---------- |
| 门店名称   | 可公开        |
| 门店地址   | 可公开，但需商家授权 |
| 营业时间   | 可公开        |
| 服务项目   | 可公开        |
| 门店电话   | 视业务策略      |
| 营业执照   | 不建议直接公开原图  |
| 负责人手机号 | 不公开        |
| 员工个人信息 | 不公开        |
| 收款信息   | 不公开        |

## **14. 后台权限与日志**

运营后台查看隐私信息时必须：

1. 按角色授权；
2. 最小权限展示；
3. 手机号、车牌默认脱敏；
4. 需要查看明文时二次确认；
5. 查看明文应记录操作日志；
6. 导出数据需审批；
7. 禁止普通运营批量导出敏感信息；
8. 风控、客服、财务权限分离。

---

## **15. 数据保存期限**

建议规则：


| 数据     | 保存建议         |
| ------ | ------------ |
| 订单数据   | 按财务和业务要求保存   |
| 支付退款数据 | 按财务和监管要求保存   |
| 操作日志   | 至少保存 6-12 个月 |
| 原始维修图片 | 按业务期限保存，可归档  |
| 脱敏公开图片 | 案例有效期内保存     |
| 授权记录   | 长期保存或按合规要求   |
| 埋点明细   | 可定期归档或匿名化    |
| 风控记录   | 按安全要求保存      |


---

## **16. 隐私协议更新规则**

隐私协议更新时：

1. 记录协议版本；
2. 标记变更内容；
3. 重大变更需重新征得用户同意；
4. 用户拒绝后，不得继续使用新增授权范围；
5. 保留历史版本；
6. 授权记录需关联协议版本。

重大变更包括：

1. 新增敏感信息采集；
2. 新增公开展示用途；
3. 新增第三方共享；
4. 新增商业化用途；
5. 用户权利处理方式变化。

---

## **17. 第三方共享说明**

如涉及第三方服务，需在隐私政策中说明：


| 第三方      | 用途         |
| -------- | ---------- |
| 微信       | 登录、支付、订阅消息 |
| 对象存储服务   | 图片视频存储     |
| 内容安全服务   | 图片、文本审核    |
| OCR/识别服务（阿里云） | 脱敏打码；商家端报价单/定损单/结算单 OCR 辅助录入 |
| 地图服务     | 门店定位、距离计算  |
| 数据统计服务   | 访问分析，需合规   |


**单据 OCR 合规要求**：

1. 仅商家主动触发；触发前须明示「将单据图提交阿里云 OCR，仅用于辅助填表」；
2. 单据原图永不进入 PublicView / H5 / Feed；
3. 公开仅允许已审核结构化摘要（方案项目、参考金额等）；
4. 生产环境默认关闭将单据原图送入通用多模态大模型通读；
5. 不得将未脱敏识别全文用于站外营销或未授权外发。

要求：

1. 不向第三方提供超出必要范围的数据；
2. 敏感信息传输需加密；
3. 第三方用途需披露；
4. 用户可查看相关说明。

## **19. P0验收标准**

上线前必须满足：

1. 有隐私协议入口；
2. 首次登录有隐私协议确认；
3. 手机号授权明确；
4. 车辆信息采集有说明；
5. 服务相册上传有授权说明；
6. 公开案例必须单独授权；
7. 用户拒绝公开时不得发布案例；
8. 用户撤回授权后案例可下线；
9. 公开图片必须脱敏；
10. 原图不得公开访问；
11. H5不得展示手机号、车牌、VIN、人脸；
12. 后台查看敏感信息有权限控制；
13. 后台敏感操作有日志；
14. 协议版本可追溯；
15. 授权记录可查询；
16. 第三方服务用途在隐私政策中说明；
17. 埋点不得采集敏感明文；
18. 支付退款数据按权限访问。
