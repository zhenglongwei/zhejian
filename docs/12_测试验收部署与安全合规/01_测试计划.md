## **2. 测试目标**

测试目标包括：

1. 验证 V2.0 核心业务链路可用（浏览、咨询、相册、案例、GEO）；
2. 验证咨询线索状态流转正确；
3. 验证服务相册上传、配件确认、脱敏、案例公开链路；
4. 验证用户端、商家端、运营权限隔离；
5. 验证 H5 公开内容不泄露隐私；
6. 验证接口鉴权、幂等、错误码；
7. 验证消息通知与埋点；
8. 验证上线前无 P0/P1 阻塞缺陷；
9. 验证部署和回滚流程可执行。

**不测试**：下单、微信支付、退款、交易评价与奖励。

## **7. 核心业务链路测试**

## **7.1 V2.0 标准业务闭环**（必须通过）

```
用户登录
→ 浏览服务/门店/案例
→ 提交咨询/预约
→ 商家处理线索
→ 商家创建服务相册
→ 六阶段证据上传 + 配件确认（L2+）
→ 用户查看相册 / 确认配件
→ 用户授权脱敏公开
→ 脱敏 + 案例审核
→ H5 公开案例展示
→ sitemap 收录

```

## **8. 接口测试要求**

接口测试必须覆盖：

1. 正常请求；
2. 参数缺失；
3. 参数类型错误；
4. 非法枚举值；
5. 未登录；
6. token过期；
7. 无权限；
8. 越权访问；
9. 资源不存在；
10. 状态不允许；
11. 重复提交；
12. 幂等校验；
13. 分页参数；
14. 错误码；
15. request_id 追踪。

---

## **9. 权限测试要求**

必须验证：

1. 用户只能查看自己的**咨询记录与服务相册**；
2. 用户不能查看其他用户相册；
3. 商家只能查看本门店**线索与服务相册**；
4. 商家不能审核自己的公开案例；
5. 商家不能修改平台审核结果；
6. 后台不同角色权限隔离；
7. 未授权H5不能访问未发布案例；
8. 已下线案例不能继续公开展示；
9. ~~支付回调不能被伪造~~（V2.0 无支付）；
10. 上传素材必须校验归属。

## **14. 缺陷等级**


| 等级  | 定义   | 示例                  |
| --- | ---- | ------------------- |
| P0  | 阻塞上线 | 支付失败、越权、隐私泄露、订单无法完成 |
| P1  | 严重问题 | 核心链路异常、退款错误、审核失效    |
| P2  | 一般问题 | 页面展示异常、非核心功能失败      |
| P3  | 轻微问题 | 文案、样式、体验优化          |


---

## **15. 测试准入标准**

进入系统测试前必须满足：

1. 需求文档已评审；

3. 核心功能开发完成；

## **16. 测试准出标准**

4. ~~支付退款链路~~（V1.0 不验收）；
5. 相册案例链路通过；
6. ~~评价奖励链路~~（V1.0 不验收）；
7. 权限测试通过；
8. 隐私测试通过；
9. 回归测试通过；
10. 预发验证通过；

## **17. P0验收清单**（V2.0）

上线前必须逐项确认：

1. 用户能提交咨询/预约；
2. 商家能处理咨询线索；
3. 商家能创建服务相册并上传六阶段证据；
4. L2+ 配件未确认不可完工；
5. 用户能查看关联服务相册与配件确认；
6. 用户能授权/撤回公开案例；
7. 后台能审核商家、服务相册、公开案例；
8. 图片视频能上传；
9. 脱敏图能用于公开页；
10. 原图不能公开访问；
11. H5 只展示已审核内容；
12. 未授权案例不能公开；
13. 用户不能访问他人咨询/相册；
14. 商家不能访问其他门店线索/相册；
15. 消息通知可触发（咨询、配件、审核）；
16. 关键埋点可上报；
17. 生产配置不使用测试密钥；
18. 上线失败可回滚。
