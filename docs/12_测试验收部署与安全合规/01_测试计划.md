## **2. 测试目标**

测试目标包括：

1. 验证核心业务链路可用；
2. 验证订单、支付、退款链路稳定；
3. 验证相册上传、脱敏、案例公开链路可用；
4. 验证评价奖励规则正确；
5. 验证用户端、商家端、后台权限隔离；
6. 验证H5公开内容不泄露隐私；
7. 验证接口鉴权、幂等、错误码；
8. 验证消息通知和埋点上报；
9. 验证上线前无P0/P1阻塞缺陷；
10. 验证部署和回滚流程可执行。

## **4. 测试类型**


| 测试类型    | 说明              | 是否P0 |
| ------- | --------------- | ---- |
| 功能测试    | 验证业务功能          | 是    |
| 接口测试    | 验证API正确性        | 是    |
| 回归测试    | 验证修改未影响旧功能      | 是    |
| 兼容性测试   | 小程序、浏览器、设备适配    | 是    |
| 权限测试    | 验证角色隔离和越权       | 是    |
| 安全测试    | 鉴权、隐私、上传、回调安全   | 是    |
| 支付测试    | 微信支付、退款、回调、对账   | 是    |
| 上传测试    | 图片视频上传、格式、大小、归属 | 是    |
| 性能测试    | 核心接口和页面性能       | 是    |
| 异常测试    | 网络失败、重复提交、超时    | 是    |
| 埋点测试    | 事件是否准确上报        | 是    |
| 通知测试    | 消息触发、模板、失败重试    | 是    |
| SEO测试   | H5页面SEO字段和收录规则  | 是    |
| 灰度/上线测试 | 发布后验证           | 是    |

## **7. 核心业务链路测试**

## **7.1 标准交易闭环**

必须通过：

```
用户登录
→ 浏览服务
→ 创建订单
→ 发起支付
→ 支付回调成功
→ 商家接单
→ 商家开始服务
→ 商家上传相册
→ 商家提交完工
→ 用户确认完成
→ 用户评价
→ 奖励判断
→ 后台查看订单与评价

```

---

## **7.2 退款链路**

必须通过：

```
用户下单并支付
→ 商家拒单/用户取消
→ 生成退款申请
→ 后台或系统审核
→ 发起退款
→ 退款回调成功
→ 订单退款状态更新
→ 奖励冻结或撤销

```

---

## **7.3 案例公开链路**

必须通过：

```
订单完成
→ 生成维修相册
→ 图片脱敏
→ 生成案例草稿
→ 商家补充内容
→ 提交公开审核
→ 后台审核通过
→ H5公开案例页展示
→ sitemap收录

```

---

## **7.4 历史案例链路**

必须通过：

```
商家上传历史案例
→ 上传图片视频
→ 自动脱敏
→ 填写案例内容
→ 提交审核
→ 后台审核
→ 发布到H5

```

---

## **8. 接口测试要求**

接口测试必须覆盖：

1. 正常请求；
2. 参数缺失；
3. 参数类型错误；
4. 非法枚举值；
5. 未登录；
6. token过期；
7. 无权限；
8. 越权访问；
9. 资源不存在；
10. 状态不允许；
11. 重复提交；
12. 幂等校验；
13. 分页参数；
14. 错误码；
15. request_id 追踪。

---

## **9. 权限测试要求**

必须验证：

1. 用户只能查看自己的订单；
2. 用户不能查看其他用户相册；
3. 商家只能查看自己门店订单；
4. 商家不能审核自己的公开案例；
5. 商家不能修改平台审核结果；
6. 后台不同角色权限隔离；
7. 未授权H5不能访问未发布案例；
8. 已下线案例不能继续公开展示；
9. 支付回调不能被伪造；
10. 上传素材必须校验归属。

---

## **10. 支付退款测试**

必须覆盖：

1. 支付成功；
2. 用户取消支付；
3. 支付超时；
4. 重复支付请求；
5. 支付回调重复；
6. 支付金额不一致；
7. 支付成功但订单未更新；
8. 商家拒单自动退款；
9. 用户取消后退款；
10. 后台人工退款；
11. 退款回调重复；
12. 退款失败；
13. 部分退款；
14. 全额退款；
15. 退款后奖励冻结或撤销。

---

3. 超大文件拦截；

3. 已下线案例不可访问或 noindex；

11. 页面状态码正确；
12. 移动端展示正常；
13. 页面无隐私泄露；
14. 分享链接可访问。

## **14. 缺陷等级**


| 等级  | 定义   | 示例                  |
| --- | ---- | ------------------- |
| P0  | 阻塞上线 | 支付失败、越权、隐私泄露、订单无法完成 |
| P1  | 严重问题 | 核心链路异常、退款错误、审核失效    |
| P2  | 一般问题 | 页面展示异常、非核心功能失败      |
| P3  | 轻微问题 | 文案、样式、体验优化          |


---

## **15. 测试准入标准**

进入系统测试前必须满足：

1. 需求文档已评审；
2. 接口规范已确认；
3. 核心功能开发完成；
4. 开发自测通过；
5. 测试环境可用；
6. 测试账号准备完成；
7. 测试数据准备完成；
8. 支付沙箱或测试配置可用；
9. 日志可查看；
10. 版本范围明确。

---

## **16. 测试准出标准**

上线前必须满足：

1. P0 缺陷为 0；
2. P1 缺陷为 0 或有明确延期审批；
3. 核心交易链路通过；
4. 支付退款链路通过；
5. 相册案例链路通过；
6. 评价奖励链路通过；
7. 权限测试通过；
8. 隐私测试通过；
9. 回归测试通过；
10. 预发验证通过；
11. 上线回滚方案已准备；
12. 业务方验收通过。

---

## **17. P0验收清单**

上线前必须逐项确认：

1. 用户能完成下单支付；
2. 支付回调能正确更新订单；
3. 商家能接单并履约；
4. 用户能确认完成；
5. 用户能评价；
6. 奖励能正确生成或冻结；
7. 商家拒单能退款；
8. 后台能审核商家、案例、评价、退款；
9. 图片视频能上传；
10. 脱敏图能用于公开页；
11. 原图不能公开访问；
12. H5只展示已审核内容；
13. 未授权案例不能公开；
14. 用户不能访问他人订单；
15. 商家不能访问其他门店订单；
16. 支付退款支持幂等；
17. 消息通知可触发；
18. 关键埋点可上报；
19. 生产配置不使用测试密钥；
20. 上线失败可回滚。
