## **2. 基础原则**

监控、告警和日志必须遵守：

1. 核心链路必须可观测；
2. 异常必须可追踪；
3. 支付、退款、订单状态必须重点监控；
4. 隐私敏感信息不得明文写入日志；
5. 告警必须有责任人；
6. 告警必须区分等级；
7. 日志必须包含 request_id 或 trace_id；
8. 高危操作必须记录审计日志；
9. 监控指标必须可量化；
10. 告警处理必须闭环。

## **3. 监控范围**


| 类型   | 监控内容                   |
| ---- | ---------------------- |
| 系统监控 | CPU、内存、磁盘、网络、进程、容器状态   |
| 应用监控 | 接口耗时、错误率、QPS、依赖服务状态    |
| 业务监控 | 登录、下单、支付、退款、接单、上传、审核   |
| 数据监控 | 数据库连接、慢查询、数据积压、任务失败    |
| 支付监控 | 支付创建、支付回调、退款创建、退款回调    |
| 文件监控 | 上传成功率、脱敏成功率、原图访问异常     |
| 内容监控 | 案例审核、评价审核、违规内容         |
| H5监控 | 访问量、404、500、页面性能、SEO状态 |
| 安全监控 | 越权、异常登录、刷单、接口攻击、敏感操作   |
| 消息监控 | 微信订阅消息、站内消息、失败重试       |

## **4. 告警等级**


| 等级  | 说明              | 响应要求          | 示例              |
| --- | --------------- | ------------- | --------------- |
| P0  | 严重事故，影响交易、安全、隐私 | 立即响应，必要时回滚或降级 | 支付不可用、隐私泄露、越权访问 |
| P1  | 严重异常，影响核心链路部分用户 | 30分钟内响应       | 上传大面积失败、退款异常升高  |
| P2  | 一般异常，影响局部体验     | 4小时内响应        | 消息发送失败率升高       |
| P3  | 观察类问题           | 1个工作日内处理      | 慢接口、非核心页面错误     |

## **5. 核心业务监控指标**

## **5.1 用户链路指标**


| 指标        | 说明              | 建议告警      |
| --------- | --------------- | --------- |
| 登录成功率     | 登录成功数 / 登录请求数   | 低于 95% 告警 |
| 服务详情访问成功率 | 服务详情接口成功率       | 低于 98% 告警 |
| 创建订单成功率   | 创建订单成功数 / 提交数   | 低于 95% 告警 |
| 支付发起成功率   | 成功拉起支付数 / 支付请求数 | 低于 95% 告警 |
| 支付成功率     | 支付成功数 / 支付发起数   | 异常下降告警    |
| 订单详情错误率   | 订单详情接口错误率       | 超过 2% 告警  |


---

## **5.2 商家履约指标**


| 指标        | 说明         | 建议告警   |
| --------- | ---------- | ------ |
| 商家登录成功率   | 商家端登录成功率   | 低于 95% |
| 接单接口错误率   | 接单失败率      | 超过 2%  |
| 拒单退款触发成功率 | 拒单后退款是否生成  | 低于 99% |
| 开始服务成功率   | 状态流转成功率    | 低于 98% |
| 提交完工成功率   | 完工提交成功率    | 低于 98% |
| 相册上传成功率   | 图片/视频上传成功率 | 低于 95% |


---

## **5.3 后台审核指标**


| 指标       | 说明             | 建议告警    |
| -------- | -------------- | ------- |
| 后台登录失败次数 | 后台账号异常登录       | 连续失败告警  |
| 审核接口错误率  | 商家/评价/案例/退款审核  | 超过 2%   |
| 待审核积压量   | 长时间未处理审核任务     | 超阈值告警   |
| 高危操作次数   | 下线案例、查看明文、退款审批 | 异常升高告警  |
| 操作日志写入失败 | 审计日志是否落库       | 任意失败需告警 |

## **6. 支付退款监控**

支付退款属于 P0 监控对象。

## **6.1 支付监控**

必须监控：

1. 创建支付单成功率；
2. 微信支付参数生成失败数；
3. 支付回调接收数；
4. 支付回调验签失败数；
5. 支付回调处理失败数；
6. 支付金额不一致数；
7. 支付成功但订单未更新数；
8. 重复回调次数；
9. 支付单长时间未关闭数；
10. 待支付超时任务失败数。

---

## **6.2 退款监控**

必须监控：

1. 创建退款单成功率；
2. 退款申请审核失败数；
3. 退款发起失败数；
4. 退款回调接收数；
5. 退款回调验签失败数；
6. 退款回调处理失败数；
7. 退款金额超额拦截数；
8. 退款成功但订单状态未更新数；
9. 重复退款回调次数；
10. 退款长时间处于退款中数量。

---

## **6.3 支付退款P0告警场景**

以下必须触发 P0 或 P1 告警：

1. 支付回调连续失败；
2. 支付验签异常突然升高；
3. 支付金额不一致；
4. 订单已支付但状态未推进；
5. 退款金额超过可退金额；
6. 退款成功但订单未更新；
7. 大量支付单状态卡住；
8. 微信支付证书或密钥错误；
9. 生产环境误用测试商户号；
10. 支付退款接口 5xx 持续升高。

## **7. 上传与脱敏监控**

必须监控：


| 指标        | 说明          |
| --------- | ----------- |
| 上传凭证生成成功率 | 上传服务是否正常    |
| 上传确认成功率   | 文件元数据是否正常入库 |
| 上传失败率     | 对象存储或客户端异常  |
| 非法格式拦截数   | 风险素材上传情况    |
| 超大文件拦截数   | 客户端或攻击行为    |
| 脱敏任务成功率   | 图片脱敏是否正常    |
| 脱敏失败数     | 进入人工处理      |
| 原图公开访问尝试  | 高危安全事件      |
| 公开图生成失败数  | 影响H5公开案例    |
| 待脱敏任务积压   | 影响发布效率      |


P0场景：

```
原图可公网访问
未脱敏图片进入H5公开页
脱敏服务大面积失败且仍允许公开案例发布

```

## **8. H5公开页监控**

H5公开页需监控：

1. 页面访问量；
2. 404 数量；
3. 500 数量；
4. 页面首屏加载时间；
5. 公开案例页访问成功率；
6. 下线案例访问状态；
7. sitemap 生成状态；
8. sitemap URL 数量异常变化；
9. noindex/index 配置异常；
10. 页面隐私扫描结果。

建议定时扫描：

```
是否存在手机号
是否存在完整车牌
是否存在VIN
是否存在原图URL
是否存在未脱敏图片
是否存在未审核案例

```

## **9. 安全监控**

必须监控：

1. 用户访问他人订单失败次数；
2. 商家访问其他门店订单失败次数；
3. 后台越权访问次数；
4. 后台登录失败次数；
5. 敏感信息明文查看次数；
6. 批量导出数据次数；
7. 高频接口请求；
8. 高频下单行为；
9. 高频评价行为；
10. 高频奖励领取行为；
11. 支付回调非法请求；
12. 上传非法文件；
13. 黑名单账号行为；
14. IP异常请求；
15. 设备异常聚集。

## **10. 日志类型**


| 日志类型   | 说明                 |
| ------ | ------------------ |
| 访问日志   | HTTP请求、响应状态、耗时     |
| 应用日志   | 应用运行过程日志           |
| 错误日志   | 异常堆栈和错误上下文         |
| 业务日志   | 订单、支付、退款、评价、案例状态变化 |
| 审计日志   | 后台操作、高危操作、敏感信息访问   |
| 安全日志   | 越权、风控、异常请求         |
| 任务日志   | 定时任务、异步任务执行情况      |
| 回调日志   | 支付、退款、上传、消息回调      |
| 埋点日志   | 用户行为事件，注意脱敏        |
| 数据变更日志 | 核心状态流转记录           |


---

## **11. 日志字段规范**

核心日志建议包含：


| 字段                  | 说明           |
| ------------------- | ------------ |
| log_id              | 日志ID         |
| trace_id/request_id | 链路追踪ID       |
| user_id             | 用户ID，可空      |
| merchant_id         | 商家ID，可空      |
| store_id            | 门店ID，可空      |
| operator_id         | 后台操作人ID      |
| role                | 操作角色         |
| action              | 操作动作         |
| resource_type       | 资源类型         |
| resource_id         | 资源ID         |
| before_status       | 变更前状态        |
| after_status        | 变更后状态        |
| result              | success/fail |
| error_code          | 错误码          |
| error_message       | 错误信息，避免敏感明文  |
| ip                  | 请求IP         |
| user_agent          | 客户端信息        |
| created_at          | 日志时间         |


---

## **12. 禁止记录的敏感明文**

日志中禁止记录：

1. 手机号明文；
2. 完整车牌号；
3. 完整 VIN；
4. 身份证号；
5. 银行卡号；
6. 微信支付密钥；
7. token/session；
8. 登录密码；
9. 验证码；
10. 对象存储临时密钥；
11. 原图私有URL长期凭证；
12. 用户精确地址；
13. 证件照片URL；
14. 支付证书内容。

允许记录时必须脱敏：

```
手机号：138****8888
车牌：京A****8
VIN：LXXXX********1234

```

## **13. 审计日志规范**

以下操作必须记录审计日志：

1. 后台登录；
2. 后台退出；
3. 商家审核；
4. 服务审核；
5. 退款审核；
6. 评价审核；
7. 公开案例审核；
8. 案例下线；
9. 查看手机号明文；
10. 查看车牌明文；
11. 导出数据；
12. 修改权限；
13. 新增后台账号；
14. 禁用用户或商家；
15. 人工调整订单状态；
16. 人工发放或撤销奖励；
17. 修改支付退款配置。

## **15. 告警处理流程**

```
告警触发
→ 自动通知责任人
→ 判断告警等级
→ 排查日志和指标
→ 判断是否需要降级/回滚
→ 修复或临时止血
→ 验证恢复
→ 关闭告警
→ 记录原因和处理结果
→ P0/P1事故复盘

```

---

## **16. P0验收标准**

上线前必须满足：

1. 核心服务健康检查可用；
2. API错误率可监控；
3. 支付回调失败可告警；
4. 退款失败可告警；
5. 订单状态异常可监控；
6. 上传失败率可监控；
7. 脱敏失败可监控；
8. 原图公开访问异常可发现；
9. 后台高危操作有审计日志；
10. 越权访问有安全日志；
11. 日志不记录敏感明文；
12. request_id 可全链路追踪；
13. P0告警有明确责任人；
14. 告警通知渠道可用；
15. 上线后可查看生产日志。
