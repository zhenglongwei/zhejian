# **2. 模块定位**

风控与合规模块用于保障平台安全运营。

主要覆盖：

1. 商家入驻真实性；
2. 服务方案合规性；
3. 订单真实性；
4. 支付退款安全；
5. 维修内容真实性；
6. 案例公开隐私安全；
7. 评价与奖励反作弊；
8. H5 公开内容合规。

# **4. 风险等级**

## **4.1 风险等级定义**


| 等级      | 说明      | 处理方式       |
| ------- | ------- | ---------- |
| R0 无风险  | 正常业务    | 自动通过       |
| R1 低风险  | 轻微异常    | 正常通过，记录    |
| R2 中风险  | 需关注     | 进入人工审核     |
| R3 高风险  | 可能违规    | 先冻结/隐藏，再审核 |
| R4 严重风险 | 明确违规或违法 | 立即拦截/下架/封禁 |


---

## **4.2 风险处理策略**

```
低风险：记录日志
中风险：人工审核
高风险：先限制再审核
严重风险：直接拦截并通知平台

```

# **5. 用户风控**

## **5.1 用户异常规则**

命中以下情况进入风险队列：

1. 短时间内频繁下单又取消；
2. 多笔订单频繁申请退款；
3. 同一设备多个账号领取奖励；
4. 同一手机号关联多个异常账号；
5. 高频提交低质量评价；
6. 多次申请案例下架；
7. 恶意投诉商家；
8. 与商家存在疑似关联。

---

## **5.2 用户限制措施**


| 措施     | 说明          |
| ------ | ----------- |
| 奖励冻结   | 暂停奖励发放      |
| 评价待审核  | 评价不立即展示     |
| 下单限制   | 短期限制下单      |
| 公开授权限制 | 限制案例奖励      |
| 账号冻结   | 高风险用户禁用部分功能 |
| 平台客服介入 | 人工核查        |


---

# **6. 商家风控**

## **6.1 商家异常规则**

命中以下情况进入风险队列：

1. 营业执照或资质疑似伪造；
2. 服务项目超出资质范围；
3. 事故车服务承诺线上一口价；
4. 价格明显低于市场且无法解释；
5. 频繁拒单或接单后取消；
6. 短期大量好评且内容相似；
7. 被用户投诉诱导好评；
8. 上传案例疑似非本店案例；
9. 维修过程相册缺失但订单大量完成；
10. 退款率、投诉率异常高。

---

## **6.2 商家限制措施**


| 措施   | 说明          |
| ---- | ----------- |
| 服务下架 | 下架违规服务      |
| 限制接单 | 暂停新订单       |
| 案例下架 | 下架违规案例      |
| 奖励冻结 | 关联用户奖励暂停    |
| 门店降权 | 推荐排序降低      |
| 商家冻结 | 暂停商家工作台核心功能 |
| 清退   | 严重违规后退出平台   |


---

# **7. 订单风控**

## **7.1 异常订单规则**

以下订单标记异常：

1. 创建后立即支付又退款；
2. 用户和商家频繁重复交易；
3. 高金额订单无维修相册；
4. 复杂维修无检测记录；
5. 订单金额与服务价格严重不符；
6. 商家接单后极短时间完成；
7. 订单完成后用户立即差评并退款；
8. 同一用户短时间在多个门店下单；
9. 同一门店短时间大量新用户订单。

---

## **7.2 订单限制措施**


| 措施   | 说明        |
| ---- | --------- |
| 标记异常 | 运营后台可见    |
| 暂缓完成 | 需人工确认     |
| 暂停奖励 | 评价奖励冻结    |
| 暂缓结算 | 商家结算暂缓    |
| 进入售后 | 平台客服介入    |
| 强制关闭 | 高风险订单，需超管 |


---

# **8. 支付与退款风控**

## **8.1 支付风险**

关注：

1. 支付回调重复；
2. 支付金额与订单金额不一致；
3. 支付成功但订单未更新；
4. 同一订单多次支付；
5. 异常高金额支付；
6. 微信支付状态与平台状态不一致。

---

## **8.2 退款风险**

以下退款进入人工审核：

1. 超过订单实付金额；
2. 同一订单重复退款；
3. 订单已完成后立即退款；
4. 用户多次申请退款；
5. 商家频繁主动退款；
6. 退款原因与订单状态不匹配；
7. 高金额退款；
8. 售后争议退款。

---

## **8.3 退款限制**

1. 不允许超额退款；
2. 不允许重复成功退款；
3. 已结算订单退款需特殊处理；
4. 售后争议中暂停奖励；
5. 高金额退款需平台确认；
6. 微信退款失败需保留失败原因。

---

# **9. 评价与奖励风控**

## **9.1 评价风险**

以下评价进入审核：

1. 含敏感词；
2. 含手机号、车牌、VIN；
3. 恶意辱骂；
4. 与订单无关；
5. 内容明显复制粘贴；
6. 好评内容高度相似；
7. 商家回复诱导用户改评；
8. 用户短期评价过多。

---

## **9.2 奖励风险**

以下奖励冻结或驳回：

1. 订单未完成；
2. 订单退款中；
3. 售后中；
4. 用户异常；
5. 商家异常；
6. 评价内容无效；
7. 重复领取；
8. 单用户超过日/月封顶；
9. 单门店奖励异常集中；
10. 公开案例未审核通过。

---

## **9.3 禁止行为**

明确禁止：

1. 好评返现；
2. 差评不给奖励；
3. 商家私下返现；
4. 用户刷单领取奖励；
5. 多账号套利；
6. 发布虚假评价；
7. 删除差评换补偿。

---

# **10. 内容与隐私风控**

## **10.1 文本隐私检测**

检测内容包括：

1. 车牌号；
2. 手机号；
3. VIN；
4. 身份证号；
5. 真实姓名；
6. 详细住址；
7. 保险单号；
8. 微信号；
9. 银行卡号。

---

## **10.2 图片隐私检测**

图片中需重点识别：

1. 车牌；
2. VIN 铭牌；
3. 行驶证；
4. 驾驶证；
5. 身份证；
6. 保险单；
7. 手机号码；
8. 门店客户资料；
9. 人脸；
10. GPS 位置信息。

---

## **10.3 案例公开要求**

案例公开必须满足：

1. 用户授权或商家历史案例责任确认；
2. 原图已脱敏；
3. 文本无隐私；
4. 价格使用区间；
5. 明确案例来源；
6. 高风险车型谨慎公开；
7. 用户可申请下架；
8. H5 是否允许收录需单独判断。

---

# **11. H5 合规风控**

## **11.1 H5 禁止展示**

1. 未授权维修档案；
2. 未脱敏图片；
3. 用户手机号；
4. 车牌；
5. VIN；
6. 身份证、行驶证；
7. 精确订单金额；
8. 保险理赔个人信息；
9. 夸大宣传；
10. 虚假原厂授权。

---

## **11.2 noindex 风控规则**

以下内容应设置 noindex：

1. 隐私风险中等；
2. 稀有车型识别度高；
3. 用户不希望搜索引擎收录；
4. 案例内容质量不足；
5. 仅允许站内展示；
6. 审核人员判断不适合公开收录。

---

# **12. 风控处理流程**

## **12.1 自动风控流程**

```
业务动作触发
→ 风控规则检测
→ 计算风险等级
→ 执行动作：通过/记录/审核/冻结/拦截
→ 写入风控日志
→ 通知相关角色

```

---

## **12.2 人工审核流程**

```
进入风险队列
→ 平台审核人员查看详情
→ 查看关联订单/用户/商家/内容
→ 选择处理动作
→ 填写处理原因
→ 系统更新状态
→ 记录操作日志

```

# **15. MVP 风控规则清单**

P0 必须上线：

1. 订单未完成不发奖励；
2. 退款中不发奖励；
3. 售后中冻结奖励；
4. 重复评价不发奖励；
5. 用户日/月奖励封顶；
6. 商家短期评价异常集中进入审核；
7. 图片含车牌需脱敏；
8. 文本含手机号/VIN 进入审核；
9. 未授权案例不可公开；
10. 事故车不得线上承诺最终价；
11. 退款不得超过实付金额；
12. 重复退款拦截；
13. 高金额退款人工审核；
14. 商家资质不匹配服务不可上架；
15. H5 未脱敏内容不可发布。

## **16. 后台操作风控**

高危后台操作包括：

1. 审核通过商家；
2. 人工修改订单状态；
3. 审核退款；
4. 人工发起退款；
5. 查看敏感明文；
6. 导出数据；
7. 修改奖励状态；
8. 下线案例；
9. 修改权限；
10. 禁用账号。

要求：

1. 高危操作必须记录日志；
2. 部分操作需二次确认；
3. 财务相关操作需权限隔离；
4. 批量导出需审批；
5. 超级管理员操作需可审计。

---

## **17. 风控白名单与黑名单**

## **17.1 白名单**

白名单适用于：

1. 内部测试账号；
2. 合作商家；
3. 特殊运营活动账号；
4. 客服测试订单。

要求：

1. 白名单必须有有效期；
2. 白名单不能绕过隐私和支付安全；
3. 白名单操作需记录；
4. 不得用于生产刷单。

---

## **17.2 黑名单**

黑名单对象包括：

1. 用户ID；
2. 手机号；
3. 设备ID；
4. IP；
5. 商家ID；
6. 门店ID；
7. 支付账户；
8. 素材特征；
9. 评价文本特征。

黑名单动作：

1. 禁止登录；
2. 禁止下单；
3. 禁止评价；
4. 禁止提现；
5. 禁止上传；
6. 禁止发布案例；
7. 强制人工审核。

**18. 风控审核流程**

```
风险规则触发
→ 生成风控事件
→ 根据风险等级自动处理或进入人工审核
→ 审核员查看证据
→ 选择通过/驳回/冻结/封禁/下线等动作
→ 系统执行处理结果
→ 记录审核日志
→ 必要时通知用户或商家

```

# **19. P0 验收标准**

必须满足：

1. 支持用户、商家、订单、退款、内容、奖励的基础风控规则；
2. 风控规则支持配置开启/关闭；
3. 风险事件可记录；
4. 中高风险进入平台审核；
5. 高风险内容可先隐藏再审核；
6. 奖励风控可冻结或驳回；
7. 退款风控可阻止超额退款；
8. 案例公开前必须经过隐私检查；
9. H5 只展示脱敏和授权内容；
10. 平台可查看风险原因和命中规则；
11. 平台处理风险必须留痕；
12. 风控拦截不得导致主业务数据丢失。
