## **2. 模块定位**

商家审核管理用于平台对商家入驻、门店资料、资质能力、服务范围进行审核和管控。

主要解决：

1. 防止虚假商家入驻；
2. 防止无资质商家承接高风险维修；
3. 确保门店地址、联系方式、照片真实；
4. 确保商家符合平台辙见要求；
5. 为服务审核、案例审核、**服务相册**提供基础可信数据。

---

## **3. 审核对象**

商家审核对象包括：


| 对象     | 审核内容                 |
| ------ | -------------------- |
| 商家主体   | 营业执照、主体名称、统一社会信用代码   |
| 门店信息   | 门店名称、地址、坐标、门头照片      |
| 联系方式   | 负责人姓名、手机号、客服电话       |
| 资质能力   | 维修资质、新能源资质、专项资质      |
| 服务能力   | 擅长项目、工位、设备、技师能力      |
| 品牌授权   | 是否具备品牌授权证明           |
| 辙见要求 | 是否支持维修过程记录、相册上传、报价透明 |
| 风险状态   | 是否存在伪造资料、异常投诉、历史违规   |

## **5. 商家入驻审核流程**

### **5.1 标准审核流程**

```
商家提交入驻资料
→ 系统基础校验
→ 进入商家审核列表
→ 平台审核人员查看资料
→ 审核营业执照/门店照片/地址/联系方式/资质
→ 选择通过/要求修改/驳回
→ 记录审核日志
→ 通知商家结果

```

---

### **5.2 风险审核流程**

```
商家提交资料
→ 命中风控规则
→ 标记风险等级
→ 进入高优先级审核队列
→ 平台人工复核
→ 通过/要求修改/驳回/冻结
→ 写入风控日志和操作日志

```

---

## **6. 审核内容**

### **6.1 营业执照审核**

审核要点：

1. 营业执照图片是否清晰；
2. 主体名称是否完整；
3. 统一社会信用代码是否完整；
4. 经营状态是否看似正常；
5. 主体类型是否适合汽车维修服务；
6. 营业执照主体与门店名称是否存在明显不一致；
7. 是否存在涂改、拼接、伪造痕迹；
8. 是否上传无关图片。

不通过示例：

- 营业执照模糊无法识别；
- 主体名称缺失；
- 上传门头照代替营业执照；
- 明显 PS 或截图篡改；
- 主体业务与汽车维修完全无关。

---

### **6.2 门店照片审核**

审核照片包括：

1. 门头照片；
2. 门店外景；
3. 工位照片；
4. 维修设备照片；
5. 接待区照片，可选；
6. 新能源维修区域，可选；
7. 资质证书墙，可选。

审核要点：

1. 图片是否清晰；
2. 是否为真实门店；
3. 是否与填写门店名称一致；
4. 是否存在盗图或网络图痕迹；
5. 是否能体现实际维修能力；
6. 是否包含用户隐私信息；
7. 是否存在违规广告、外部联系方式、诱导交易信息。

---

### **6.3 地址真实性审核**

审核要点：

1. 门店地址是否完整；
2. 城市、区县、街道是否匹配；
3. 地图坐标是否与地址一致；
4. 门头照片是否能佐证地址真实性；
5. 地址是否为住宅、小区、虚拟地址；
6. 同一地址是否已有多个异常商家；
7. 是否存在明显定位偏移。

处理方式：


| 情况        | 处理      |
| --------- | ------- |
| 地址清晰且照片一致 | 可通过     |
| 地址不完整     | 要求修改    |
| 坐标偏差较大    | 要求重新定位  |
| 疑似虚拟地址    | 驳回或人工复核 |
| 同地址多商家异常  | 进入风控    |


---

### **6.4 联系方式审核**

审核内容：

1. 负责人手机号；
2. 门店客服电话；
3. 联系人姓名；
4. 微信号、外部联系方式展示情况。

审核规则：

1. 手机号必须为有效格式；
2. 负责人手机号需脱敏展示给平台普通审核人员；
3. 用户端不直接展示个人手机号，除非平台产品策略允许；
4. 商家资料不得引导用户添加微信绕过平台；
5. 一个手机号关联多个异常商家时进入风控。

---

### **6.5 服务能力审核**

审核内容：

1. 商家选择的擅长服务项目；
2. 维修资质类别；
3. 技师能力说明；
4. 工位与设备照片；
5. 是否支持标准维修流程；
6. 是否支持**服务相册**上传与六阶段留痕；
7. 是否支持到店报价说明（**禁止线上最终报价**，尤其事故车）。

服务能力需和资质匹配：


| 服务复杂度 | 示例          | 审核要求       |
| ----- | ----------- | ---------- |
| L1    | 洗车、美容、基础养护  | 基础门店照片即可   |
| L2    | 保养、刹车片、电瓶   | 需具备基础维修能力  |
| L3    | 钣喷、底盘、空调维修  | 需维修资质或充分材料 |
| L4    | 事故车、新能源高压维修 | 需专项资质，严格审核 |


---

### **6.6 平台辙见要求审核**

商家须接受：

1. **服务过程相册留痕**（六阶段 + 配件类型告知）；
2. L2+ 非原厂/同质件须走**车主确认**流程；
3. 服务项目与价格展示规则公开（参考区间/影响因素，非平台成交价）；
4. 不得虚假承诺、不得隐藏关键收费说明；
5. 复杂/事故车须说明「以到店检测为准」；
6. 公开案例须脱敏并经平台审核。

不符合：首次入驻要求修改或驳回；已入驻可限制服务上架、冻结门店或清退。

---

## **6.7 门店能力变更审核（卷十一）**

入驻通过后，商家修改下列内容须进入 **能力变更审核队列**，通过前公开面仍展示上一版已通过内容（无旧版则不展示该区块）：

| 对象 | 审核要点 |
| --- | --- |
| 技师公示卡 | 称呼是否得体；主修/年限/资质标签是否夸大、是否与门店资质匹配 |
| 设备/场标签及实景图 | 标签是否与照片相符；无盗图、无用户隐私车牌 |
| 品牌授权及有效期 | 凭证真实性；有效期是否填写；过期不得标「已授权」 |

**不进本队列（即时生效）**：营业时间、临时休息、门店电话、简介、擅长服务、擅长品牌、暂不承接文案。虚假内容依赖举报与事后处罚。

审核操作：通过 / 要求修改 / 驳回；须写操作留痕并回写「资料核实时间」。

授权或维修资质有效期到期：系统自动取消对应对外展示，并进入运营 **资质/授权催办列表**（卷十一二期 P2-04）：

| Tab | 口径 |
| --- | --- |
| 即将过期 | 有效期距今 ≤ 30 天（含当日） |
| 已过期 | 有效期早于今日 |

列表展示门店、商家、到期类型（品牌授权 / 维修资质）、到期日、剩余天数；运营人工外联催办。公开列表对已过期/即将过期门店做 **轻降权**（排序扣分），不对用户展示「过期」羞辱标签。

## **7. 风控规则**

商家审核阶段命中以下情况进入风险队列：

1. 营业执照疑似伪造；
2. 主体名称与门店信息严重不一致；
3. 门店照片疑似盗图；
4. 地址疑似虚假；
5. 同一手机号关联多个商家；
6. 同一地址存在多个异常商家；
7. 服务项目超出资质范围；
8. 事故车服务承诺线上最终价；
9. 虚假原厂授权；
10. 上传含隐私或违规宣传图片；
11. 历史投诉/举报率异常；
12. 曾被平台冻结或清退后重新入驻。

## **8. 审核操作**

平台审核人员可执行：


| 操作     | 说明            |
| ------ | ------------- |
| 通过     | 商家/门店可进入可用状态  |
| 要求修改   | 商家修改后重新提交     |
| 驳回     | 不允许入驻         |
| 冻结     | 已通过商家出现风险后冻结  |
| 修改服务权限 | 根据资质限制可上架服务范围 |
| 添加备注   | 内部备注，仅平台可见    |
| 标记风险   | 生成风险事件        |
| 查看审核记录 | 查看历史审核结论      |


---

## **9. 驳回 / 修改原因**

后台需提供标准原因：


| 类型      | 示例                 |
| ------- | ------------------ |
| 营业执照问题  | 图片模糊、主体不清、疑似伪造     |
| 门店照片问题  | 图片不清晰、疑似盗图、与门店不符   |
| 地址问题    | 地址不完整、坐标不准确、疑似虚假地址 |
| 联系方式问题  | 手机号无效、联系方式重复异常     |
| 资质不符    | 服务能力超出资质范围         |
| 品牌授权问题  | 声称授权但未提供证明         |
| 辙见不符合 | 不支持相册记录或报价透明       |
| 违规宣传    | 含外部联系方式、虚假承诺       |
| 风险异常    | 命中平台风控规则           |

## **13. 审核通过后的动作**

审核通过后：

1. 商家状态更新为 `APPROVED`；
2. 门店状态更新为可用；
3. 根据资质生成服务权限范围；
4. 商家可进入商家工作台；
5. 商家可提交服务方案审核；
6. 写入操作日志；
7. 通知商家审核通过。

---

## **14. 审核驳回后的动作**

审核驳回后：

1. 商家状态更新为 `REJECTED`；
2. 保存驳回原因；
3. 通知商家；
4. 商家不可发布服务；
5. 如存在严重违规，生成风险事件；
6. 保留审核记录。

---

## **15. P0 验收标准**

必须满足：

1. 支持商家提交入驻资料；
2. 支持平台查看营业执照；
3. 支持平台查看门店照片；
4. 支持平台查看地址和地图坐标；
5. 支持联系方式脱敏展示；
6. 支持服务能力审核；
7. 支持辙见要求确认；
8. 支持通过、驳回、要求修改；
9. 支持商家修改后重新提交；
10. 支持审核日志留痕；
11. 支持商家审核状态流转；
12. 支持资质不匹配时限制服务范围；
13. 支持高风险商家进入风控；
14. 非授权平台人员不可操作审核；
15. 审核结果可通知商家。
