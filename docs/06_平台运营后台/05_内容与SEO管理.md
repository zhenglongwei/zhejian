## **2. 模块定位**

内容与SEO管理用于平台运营人员管理公开内容资产，使已审核案例和聚合页能够安全、合规、可控地展示和被搜索引擎收录。

主要覆盖：

1. 案例标题编辑；
2. 案例摘要编辑；
3. FAQ 编辑；
4. SEO title 设置；
5. meta description 设置；
6. URL slug 设置；
7. 是否允许搜索引擎收录；
8. 推荐权重设置；
9. 收录状态查看；
10. sitemap 状态管理；
11. H5 页面上下线。

## **5. SEO 管理流程**

### **5.1 案例 SEO 流程**

```
公开案例审核通过
→ 生成默认 SEO 信息
→ 内容运营检查标题/摘要/FAQ
→ 设置 URL slug
→ 设置是否收录
→ 发布 H5 页面
→ 加入 sitemap
→ 查看收录状态

```

---

### **5.2 noindex 流程**

```
内容存在中等隐私或质量风险
→ 内容运营设置 noindex
→ 页面仍可访问
→ 搜索引擎不收录
→ sitemap 中不提交或标记不可索引

```

---

### **5.3 下线流程**

```
内容命中严重风险/用户撤回授权/商家异常
→ 平台下线页面
→ 移出 sitemap
→ 设置 noindex
→ 必要时返回 404/410 或保留不可访问状态
→ 记录操作日志

```

## **6. 功能要求**

## **6.1 编辑案例标题**

标题要求：

1. 准确描述维修内容；
2. 建议包含车型、服务项目、城市或门店；
3. 不得包含车牌、VIN、手机号；
4. 不得夸大宣传；
5. 不得堆砌关键词；
6. 长度建议 10-40 字。

示例：

- 推荐：`杭州宝马3系刹车片更换维修案例`
- 不推荐：`全杭州最低价宝马刹车片100%原厂维修`

---

## **6.2 编辑摘要**

摘要要求：

1. 简要说明车辆问题、维修过程和结果；
2. 不得暴露用户隐私；
3. 不得出现绝对化承诺；
4. 不得包含外部联系方式；
5. 长度建议 50-160 字。

示例：

```
本案例为杭州某门店完成的宝马3系刹车片更换服务，维修过程包含检测、旧件拆卸、新件安装和完工检查，价格以门店实际检测为准。

```

---

## **6.3 编辑 FAQ**

FAQ 要求：

1. 公开案例页至少 3 条；
2. 问题与维修项目相关；
3. 答案真实、清晰；
4. 不得诱导线下交易；
5. 不得虚假承诺；
6. 不得堆砌关键词。

FAQ 示例：


| 问题            | 回答                                |
| ------------- | --------------------------------- |
| 刹车片多久需要更换？    | 通常需要结合磨损程度、驾驶习惯和车辆提示判断，建议到店检测后确认。 |
| 更换刹车片需要多久？    | 常规更换一般约 1-2 小时，具体以车型和门店排期为准。      |
| 是否可以线上确定最终价格？ | 标准项目可参考价格区间，复杂情况需到店检测后确认。         |


---

## **6.4 设置 SEO title**

SEO title 要求：

1. 包含核心关键词；
2. 建议包含城市、车型、服务项目；
3. 不得包含隐私信息；
4. 不得堆砌关键词；
5. 长度建议 20-60 字。

示例：

```
杭州宝马3系刹车片更换案例_辙见过程参考

```

---

## **6.5 设置 meta description**

meta description 要求：

1. 概括页面内容；
2. 体现维修项目、车型、城市；
3. 不得夸大；
4. 不得包含外部联系方式；
5. 长度建议 60-160 字。

示例：

```
查看杭州宝马3系刹车片更换维修案例，包含维修前后图片、施工过程、常见问题和门店服务信息，内容已脱敏处理。

```

---

## **6.6 设置 URL slug**

URL slug 要求：

1. 简洁、可读；
2. 建议使用英文、拼音或标准编码；
3. 不得包含手机号、车牌、VIN；
4. 不得包含特殊符号；
5. 同站唯一；
6. 修改后需处理旧链接跳转。

示例：

```
hangzhou-bmw-3-brake-pad-replacement

```

## **6.7 设置是否收录**

可选项：


| 设置          | 说明          |
| ----------- | ----------- |
| allow_index | 允许搜索引擎收录    |
| noindex     | 页面可访问但不允许收录 |
| offline     | 页面下线不可访问    |


应设置 noindex 的情况：

1. 隐私风险中等；
2. 稀有车型识别度高；
3. 用户不希望搜索引擎收录；
4. 案例质量不足；
5. 仅允许站内展示；
6. 审核人员判断不适合公开收录。

不得允许收录的情况：

1. 未授权案例；
2. 未脱敏图片；
3. 含车牌/VIN/手机号；
4. 用户/商家投诉或**配件/隐私纠纷**未结案；
5. 商家资质异常；
6. 内容存在违规宣传。

---

## **6.8 设置推荐权重**

推荐权重用于影响内容在 H5 页面、门店主页、服务聚合页中的展示排序。

推荐权重可参考：


| 因素    | 说明        |
| ----- | --------- |
| 案例质量  | 图片清晰、节点完整 |
| 服务相关性 | 与当前页面主题匹配 |
| 城市相关性 | 与城市页匹配    |
| 门店内容质量 | 案例完整度、相册使用率（P1） |
| 用户授权  | 授权完整      |
| 内容新鲜度 | 新近案例优先    |
| 风险状态  | 有风险内容降低权重 |
| 转化效果  | 后续可参考访问转化 |


MVP 支持人工设置：


| 权重     | 说明   |
| ------ | ---- |
| 0      | 不推荐  |
| 1-30   | 低推荐  |
| 31-70  | 普通推荐 |
| 71-100 | 高推荐  |


---

## **6.9 查看收录状态**

后台需展示：

1. 是否允许收录；
2. noindex 状态；
3. sitemap 是否已提交；
4. 页面是否已发布；
5. 最近更新时间；
6. 搜索引擎收录状态，MVP 可手动维护；
7. H5 访问量；
8. 小程序跳转量；
9. 页面异常状态。

收录状态枚举：


| 状态            | 说明          |
| ------------- | ----------- |
| NOT_SUBMITTED | 未提交         |
| SUBMITTED     | 已加入 sitemap |
| INDEXED       | 已收录         |
| NOT_INDEXED   | 未收录         |
| NOINDEX       | 不允许收录       |
| OFFLINE       | 已下线         |
| ERROR         | 页面异常        |

## **7. 内容合规要求**

内容与 SEO 管理必须遵守：

1. 不得展示未授权服务相册；
2. 不得展示未脱敏图片；
3. 不得展示用户手机号；
4. 不得展示车牌；
5. 不得展示 VIN；
6. 不得展示身份证、行驶证、保单；
7. 不得展示**平台统一成交价**或成交承诺，复杂维修使用价格区间或影响因素；
8. 不得展示保险理赔个人信息；
9. 不得夸大宣传；
10. 不得虚假原厂授权；
11. 不得诱导线下交易；
12. 不得堆砌 SEO 关键词；
13. 不得生成误导性 FAQ。

## **10. 风控规则**

命中以下情况时，不允许发布或必须 noindex：

1. 文本含手机号；
2. 文本含 VIN；
3. 文本含车牌；
4. 图片未脱敏；
5. 用户未授权；
6. 用户/商家投诉或配件/隐私纠纷未结案；
7. 商家资质异常；
8. 案例质量不足；
9. SEO title 堆砌关键词；
10. meta description 含夸大宣传；
11. FAQ 含误导内容；
12. URL slug 含敏感信息；
13. 页面含外部联系方式；
14. 内容被用户申请下架。

## **13. 发布后的动作**

内容发布后：

1. 更新页面状态为 `PUBLISHED`；
2. 生成或更新 H5 页面；
3. 写入 SEO 元信息；
4. 生成或更新结构化数据；
5. 根据收录设置决定是否加入 sitemap；
6. 更新推荐索引；
7. 记录操作日志。

---

## **14. noindex 后的动作**

设置 noindex 后：

1. 页面仍可访问；
2. robots meta 设置为 `noindex`；
3. sitemap 中移除或不提交；
4. 页面不参与搜索引擎收录；
5. 内容仍可用于站内展示；
6. 记录操作日志。

---

## **15. 下线后的动作**

页面下线后：

1. 页面状态更新为 `OFFLINE`；
2. H5 页面不可访问或展示下线提示；
3. 从 sitemap 中移除；
4. 设置 noindex；
5. 从推荐位移除；
6. 如因风险下线，记录风控事件；
7. 记录操作日志。

---

## **16. P0 验收标准**

必须满足：

1. 支持编辑案例标题；
2. 支持编辑摘要；
3. 支持编辑 FAQ；
4. 案例 FAQ 至少 3 条；
5. 支持设置 SEO title；
6. 支持设置 meta description；
7. 支持设置 URL slug；
8. URL slug 不得包含隐私信息；
9. 支持设置是否收录；
10. 支持设置 noindex；
11. 支持设置推荐权重；
12. 支持查看收录状态；
13. 支持预览 H5 页面；
14. 支持页面发布和下线；
15. 未脱敏或未授权内容不得发布；
16. 风险内容不得允许收录；
17. SEO 字段修改必须留痕；
18. 高风险下线操作需二次确认；
19. 内容运营和超管权限需隔离；
20. H5 页面只能展示合规内容。
