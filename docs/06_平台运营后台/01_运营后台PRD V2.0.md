## **2. 模块定位**

平台运营工作台是平台内部**内容与合规管理入口**，覆盖：

1. 商家入驻与门店资质审核；
2. 服务方案展示内容审核；
3. **服务相册**隐私与合规审核；
4. 公开案例与历史案例审核；
5. 脱敏与隐私风险处理；
6. 用户/商家下架与投诉申请；
7. **咨询线索**统计与异常协助（非订单履约）；
8. H5 / GEO 内容与收录管理；
9. 运营数据概览；
10. 审核规则与类目配置（简版）。

细则见子文档：`02_商家审核管理.md`、`03_服务相册审核.md`、`04_公开案例审核.md`、`05_内容与SEO管理.md`、`00_风控与合规审核PRD V2.0.md`。

## **5. 商家审核管理**

入口：`平台运营工作台 → 商家审核`

摘要能力：入驻 Tab（待审核/要求修改/已通过/已驳回/已冻结）、资质与门店资料查看、通过/要求修改/驳回/冻结/限制服务复杂度。

V2.0 审核重点增加：**是否接受服务相册留痕、配件类型告知与车主确认**；删除「是否支持平台订单履约」类要求。

详情见 `02_商家审核管理.md`。

---

## **6. 服务方案审核**

入口：`平台运营工作台 → 服务审核`

### **6.1 审核重点**

1. 服务项目是否真实、复杂度（L1–L4）是否匹配资质；
2. 价格展示是否符合四型规范（一口价 / 参考区间 / 仅预约 / 事故车）；
3. 事故车、复杂维修是否声明「以到店检测为准」，**禁止线上最终报价**；
4. 是否存在虚假原厂授权、保证修好、全网最低等违规承诺；
5. 文案与图片是否合规，是否诱导线下绕过平台展示规则；
6. **删除** V1.0「退款规则、是否允许在线支付」等交易项。

### **6.2 审核操作**

| 操作 | 说明 |
| --- | --- |
| 通过 | 可在用户端展示 |
| 要求修改 | 商家修改后重新提交 |
| 驳回 | 不允许发布 |
| 平台下架 | 已上线服务违规 |
| 限制展示 | 仅门店页可见，不进聚合推荐 |
| 降级服务范围 | 限制可承接复杂度 |

## **7. 服务相册审核**

入口：`平台运营工作台 → 服务相册审核`

### **7.1 定位**

审核商家创建的**服务相册**（非平台订单相册）：隐私脱敏、六阶段完整性、配件告知与确认留痕、是否适合生成公开案例。

### **7.2 审核类型**

| 类型 | 说明 |
| --- | --- |
| 进行中相册 | 隐私检测、违规内容 |
| 待公开相册 | 脱敏完成度、授权前检查 |
| 风控命中 | 配件未确认即完工、图片重复、凭证缺失 |
| 投诉关联 | 用户隐私或配件纠纷 |

### **7.3 审核操作**

通过、要求修改、驳回、标记不可公开、退回重新脱敏、标记风控、生成案例草稿（仅当满足授权与脱敏条件）。

详情见 `03_服务相册审核.md` 与 `docs/04_维修过程相册/01_维修过程相册PRD V2.0.md`。

---

## **8. 公开案例审核**

入口：`平台运营工作台 → 案例审核`

### **8.1 案例类型**

1. **用户授权案例**（来自服务相册授权）；
2. **商家历史案例**；
3. 用户申请下架 / 撤回授权；
4. 高风险隐私案例；
5. H5 / GEO 待发布页。

**删除**：「平台订单案例」。

### **8.2 审核重点**

用户授权是否有效、脱敏是否完成、来源标签是否正确、价格是否为区间、SEO/FAQ 是否合规、是否适合收录。

### **8.3 审核操作**

通过并公开、通过仅门店展示、通过但 noindex、要求修改、驳回、禁止公开、下架、临时隐藏。

详情见 `04_公开案例审核.md`。

## **9. 咨询线索运营**

入口：`平台运营工作台 → 咨询线索`

> V2.0 平台**不管理订单履约**，本模块为**只读统计 + 异常协助**，不对接支付与退款。

### **9.1 列表与筛选**

按城市、门店、服务类型、提交时间、状态筛选；支持用户手机号（脱敏）、门店名称搜索。

### **9.2 线索状态（与商家端一致）**

已提交、门店已查看、门店已联系、用户已取消、已关闭。

### **9.3 平台可操作**

1. 查看线索详情（问题描述、图片、期望到店时间）；
2. 查看商家跟进状态与时间；
3. **标记异常**（垃圾咨询、虚假信息、骚扰、重复刷单式提交）；
4. 添加内部备注；
5. 联系商家/用户协助（电话外呼，非平台介入维修）；
6. 统计：今日新增、各门店线索数、超时未联系数（P1）。

### **9.4 不可做**

接单、改价、确认完工、发起退款、绑定订单状态。

## **10. 用户下架与投诉处理**

入口：`平台运营工作台 → 下架申请`

原则不变：**隐私相关先隐藏后审核**；处理用户撤回公开授权、隐私泄露投诉、案例举报。

操作：临时隐藏、确认下架、修改后恢复、驳回申请、通知用户/商家、记录日志。

## **17. 异常与边界**

1. **误操作**：审核留痕，超管可回滚高风险状态；
2. **小程序能力边界**：MVP 不做大批量导出、财务结算、复杂报表；
3. **审核积压**：按风险等级与提交时间排序，高风险优先；
4. **线索纠纷**：平台仅协助信息展示与转接，**不承担维修质量与资金责任**，引导用户与门店线下处理或行政/司法途径。

## **18. P0 验收标准**

1. 授权账号可进入运营工作台，非授权不可见；
2. 支持商家入驻与门店资质审核；
3. 支持服务方案展示内容审核；
4. 支持服务相册审核（原图/脱敏图、六阶段、配件确认留痕查看）；
5. 支持公开案例审核（用户授权 + 历史案例）；
6. 支持通过、驳回、要求修改、下架与临时隐藏；
7. 支持咨询线索只读查看与异常标记；
8. 支持用户下架/隐私投诉处理；
9. 支持 H5/GEO 基础内容与 noindex 管理；
10. 支持服务类目、复杂度、审核规则简版配置；
11. **不支持**订单查询、支付/退款、评价与奖励审核；
12. 关键操作有日志，高风险操作二次确认；
13. 角色权限隔离。
