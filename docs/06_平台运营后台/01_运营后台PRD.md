## **1. 文档信息**


| 项目   | 内容                                          |
| ---- | ------------------------------------------- |
| 文档名称 | 平台运营工作台PRD                                  |
| 当前版本 | V1.0                                        |
| 适用端  | 微信小程序内平台运营工作台                               |

# **2. 模块定位**

平台运营工作台是平台内部管理入口。

它需要覆盖：

1. 商家入驻审核；
2. 门店资料审核；
3. 服务方案审核；
4. 历史案例审核；
5. 案例脱敏审核；
6. 订单管理；
7. 支付退款协助；
8. 评价与奖励审核；
9. 用户下架申请处理；
10. 运营数据概览；
11. 基础配置管理。

---

# **6. 服务方案审核**

## **6.1 入口**

```
平台运营工作台
→ 服务审核

```

---

## **6.2 审核重点**

检查：

1. 服务项目是否真实；
2. 服务复杂度是否匹配；
3. 商家资质是否满足；
4. 价格是否明显异常；
5. 是否事故车一口价；
6. 是否存在虚假原厂授权；
7. 是否存在保证效果承诺；
8. 退款规则是否明确；
9. 图片是否合规；
10. 文案是否诱导或违规。

---

## **6.3 审核操作**


| 操作     | 说明         |
| ------ | ---------- |
| 通过     | 可上架        |
| 要求修改   | 商家修改后重新提交  |
| 驳回     | 不允许发布      |
| 平台下架   | 已上线服务违规    |
| 限制预约   | 只展示不可预约    |
| 降级服务范围 | 只能承接低复杂度项目 |


---

# **7. 案例审核管理**

## **7.1 入口**

```
平台运营工作台
→ 案例审核

```

---

## **7.2 案例类型**

1. 平台订单案例；
2. 商家历史案例；
3. 用户申请下架案例；
4. 高风险隐私案例；
5. H5 待发布案例。

---

## **7.3 审核重点**

必须检查：

1. 是否有用户授权；
2. 历史案例是否有商家责任确认；
3. 脱敏图是否安全；
4. 原图是否存在隐私；
5. 文字是否包含车牌、手机号、VIN；
6. 价格是否为区间；
7. 是否存在夸大宣传；
8. 是否标明案例来源；
9. 是否适合 H5 公开；
10. 是否允许搜索引擎收录。

---

## **7.4 审核操作**


| 操作          | 说明         |
| ----------- | ---------- |
| 通过并公开       | 小程序和 H5 可见 |
| 通过仅门店展示     | 不进入平台案例库   |
| 通过但 noindex | H5 可访问但不收录 |
| 要求修改        | 退回商家或重新脱敏  |
| 驳回          | 不公开        |
| 禁止公开        | 高隐私/严重违规   |
| 下架          | 已公开内容移除    |
| 临时隐藏        | 隐私投诉优先隐藏   |

# **10. 评价与奖励审核**

## **10.1 入口**

```
平台运营工作台
→ 评价与奖励

```

---

## **10.2 评价审核**

审核内容：

1. 是否含敏感词；
2. 是否含隐私；
3. 是否恶意辱骂；
4. 是否与订单相关；
5. 是否虚假评价；
6. 是否涉及商家诱导好评。

操作：

- 通过；
- 隐藏；
- 删除；
- 标记争议；
- 转售后处理。

---

## **10.3 奖励审核**

审核内容：

1. 订单是否完成；
2. 是否退款/售后；
3. 评价是否有效；
4. 用户是否异常；
5. 商家是否异常；
6. 奖励金额是否超过阈值；
7. 公开案例是否审核通过。

操作：

- 通过奖励；
- 冻结；
- 驳回；
- 修改金额，需权限；
- 发放；
- 标记风险。

# **16. 风险控制**

## **16.1 高风险操作二次确认**

以下操作必须二次确认：

1. 冻结商家；
2. 下架门店；
3. 强制退款；
4. 强制关闭订单；
5. 删除/下架案例；
6. 驳回奖励；
7. 修改奖励金额；
8. 修改佣金配置；
9. 修改服务复杂度规则。

---

## **16.2 超管权限限制**

仅超级管理员可操作：

1. 平台账号授权；
2. 商家冻结；
3. 门店永久关闭；
4. 佣金规则配置；
5. 奖励金额规则配置；
6. 高风险退款；
7. 权限变更；
8. 数据删除，原则不提供物理删除。

# **18. 异常与边界**

## **18.1 审核人员误操作**

处理：

1. 所有审核操作留痕；
2. 支持状态回滚，需超管；
3. 高风险操作二次确认；
4. 关键操作可要求填写原因。

---

## **18.2 小程序端操作不适合复杂后台**

由于 MVP 不做 PC 后台，小程序工作台需避免过重功能。

以下功能可后置：

1. 大批量导出；
2. 复杂财务结算；
3. 批量 SEO 管理；
4. 多维数据报表；
5. 大规模权限组织架构。

MVP 先保证关键审核和处理流程可用。

---

## **18.3 审核积压**

处理：

1. 按风险和提交时间排序；
2. 高风险优先；
3. 支持任务状态；
4. 后续支持分配审核人；
5. 超时提醒。

---

# **19. P0 验收标准**

必须满足：

1. 平台授权账号可进入运营工作台；
2. 非授权用户不可见；
3. 支持商家入驻审核；
4. 支持门店资质审核；
5. 支持服务方案审核；
6. 支持案例审核；
7. 支持查看原图和脱敏图；
8. 支持通过、驳回、要求修改；
9. 支持订单查询；
10. 支持支付状态查看；
11. 支持退款协助；
12. 支持评价审核；
13. 支持奖励审核；
14. 支持用户下架申请处理；
15. 支持基础服务类目和服务项目配置；
16. 支持奖励配置；
17. 所有关键操作必须记录日志；
18. 高风险操作必须二次确认；
19. 平台角色权限需隔离；
20. 小程序端可完成 MVP 核心运营闭环。
