# **2. 公开案例定义**

公开案例是指：

> 从维修相册、订单履约记录、商家历史案例中生成，并经过脱敏、授权、审核后，可以在 H5 或官网公开展示的维修案例内容。

公开案例可用于：

1. 用户查看维修参考；
2. 门店公开主页展示；
3. 服务项目聚合页展示；
4. 车型聚合页展示；
5. 城市服务页展示；
6. SEO 收录；
7. AI/GEO 内容引用；
8. 商家口碑资产沉淀。

---

# **3. 案例来源**

公开案例来源包括：


| 来源     | 说明                |
| ------ | ----------------- |
| 订单维修相册 | 用户在平台下单后形成的真实维修过程 |
| 独立相册   | 商家为历史维修上传的相册      |
| 历史案例   | 商家补充上传的过往维修案例     |
| 平台整理案例 | 平台基于合规素材整理生成      |


---

# **4. 公开前置条件**

案例进入公开审核前，必须满足：

1. 案例已生成；
2. 案例有标题；
3. 案例有服务项目；
4. 案例有车型或车辆信息；
5. 案例有维修前后或过程图片；
6. 所有公开图片已完成脱敏；
7. 不包含车牌、VIN、手机号、人脸等敏感信息；
8. 用户授权状态符合要求；
9. 商家主体和门店状态正常；
10. 案例不属于争议订单或严重售后订单；
11. 内容不涉及违法违规；
12. 图片和文字通过基础内容安全检测。

---

# **5. 审核对象**

审核对象包括：


| 对象    | 审核内容                      |
| ----- | ------------------------- |
| 案例标题  | 是否准确、真实、无夸大               |
| 案例摘要  | 是否描述清楚问题和维修结果             |
| 服务项目  | 是否选择正确                    |
| 车型信息  | 是否正确且不暴露隐私                |
| 城市门店  | 是否真实关联                    |
| 维修过程  | 是否符合逻辑                    |
| 图片视频  | 是否清晰、合规、已脱敏               |
| 价格信息  | 是否真实、是否允许展示               |
| FAQ   | 是否合理、无误导                  |
| SEO字段 | title、description、关键词是否合规 |
| 用户授权  | 是否允许公开展示                  |
| 商家资质  | 商家是否正常运营                  |
| 风险状态  | 是否存在投诉、纠纷、退款              |

# **7. 审核流程**

标准流程：

```
相册完成
→ 生成案例草稿
→ 自动脱敏
→ 内容安全检测
→ 商家补充标题/说明
→ 提交公开审核
→ 平台审核
→ 审核通过
→ 生成公开案例页
→ 加入索引和站点地图

```

驳回流程：

```
提交审核
→ 平台审核不通过
→ 填写驳回原因
→ 返回商家修改
→ 商家重新提交

```

风险流程：

```
提交审核
→ 命中隐私/违规/纠纷风险
→ 风险锁定
→ 平台人工复核
→ 通过/驳回/永久禁止公开

```

---

# **8. 审核规则**

## **8.1 真实性审核**

审核要点：

1. 案例是否来自真实维修；
2. 是否有关联订单或商家上传记录；
3. 维修过程是否前后一致；
4. 图片是否明显拼凑；
5. 文案是否夸大疗效；
6. 是否虚构价格、故障、车型；
7. 是否盗用其他平台图片。

不通过示例：

- “全网最低价”
- “100%保证修好”
- “原厂认证”但无资质证明
- 图片明显来自其他平台
- 文案与图片无关

---

## **8.2 隐私脱敏审核**

必须检查：

1. 车牌号是否隐藏；
2. VIN 是否隐藏；
3. 人脸是否隐藏；
4. 手机号是否隐藏；
5. 二维码是否隐藏；
6. 地址、证件是否隐藏；
7. 车内私人物品是否涉及隐私；
8. 行驶证、保单、维修单据是否可见。

不通过时：

```
退回脱敏 / 要求商家替换图片 / 平台人工涂抹

```

---

## **8.3 内容合规审核**

不允许出现：

1. 违法违规内容；
2. 色情、暴力、赌博；
3. 政治敏感内容；
4. 人身攻击；
5. 商家辱骂用户；
6. 虚假宣传；
7. 诱导线下交易；
8. 外部联系方式；
9. 非法改装宣传；
10. 侵犯第三方版权内容。

---

## **8.4 商业宣传审核**

案例可以体现门店能力，但不得过度营销。

允许：

- “本案例由 XX 门店完成”
- “维修项目：刹车片更换”
- “施工耗时约 2 小时”
- “使用品牌配件，具体以门店说明为准”

不允许：

- “全城最低”
- “不修包赔”
- “加微信报价”
- “绕过平台更便宜”
- “平台外转账优惠”

---

## **8.5 价格展示审核**

价格展示规则：

1. 标准服务可展示价格区间；
2. 复杂维修建议展示“以到店检测为准”；
3. 不得展示虚假低价引流；
4. 不得与实际服务商品价格冲突；
5. 不得承诺所有车型统一价格，除非真实支持。

---

## **8.6 SEO字段审核**

审核以下字段：


| 字段          | 要求              |
| ----------- | --------------- |
| title       | 清晰包含车型、项目、城市或门店 |
| description | 不夸大、不堆砌关键词      |
| H1          | 唯一且与案例主题一致      |
| FAQ         | 至少 3 条，内容真实     |
| 图片 alt      | 描述图片内容，不堆关键词    |
| URL slug    | 简洁、可读、无敏感信息     |
| schema      | 类型正确，信息真实       |


---

# **9. 用户授权规则**

订单相册生成公开案例时，需要满足授权要求。

## **9.1 必须授权的内容**

用户需要明确授权：

1. 维修过程图片可脱敏后展示；
2. 车辆局部图片可脱敏后展示；
3. 评价内容可公开展示；
4. 案例可用于平台公开网页和SEO收录。

---

## **9.2 不可公开的情况**

以下情况不得公开：

1. 用户明确拒绝公开；
2. 用户撤回授权；
3. 图片无法完成脱敏；
4. 涉及严重事故隐私；
5. 涉及保险、理赔、法律纠纷；
6. 订单处于投诉或售后中；
7. 平台判断公开有风险。

---

# **10. 驳回原因**

后台需要提供标准驳回原因：


| 驳回类型   | 示例                      |
| ------ | ----------------------- |
| 隐私未脱敏  | 图片中存在车牌/VIN/人脸          |
| 内容不真实  | 文案与图片不符                 |
| 图片质量差  | 图片模糊、无维修过程              |
| 违规宣传   | 包含外部联系方式                |
| 价格误导   | 价格与服务不一致                |
| 授权不足   | 用户未授权公开                 |
| 资质异常   | 商家状态异常                  |
| 订单争议   | 订单存在投诉或售后               |
| SEO不合规 | title/description 堆砌关键词 |
| FAQ不合规 | FAQ 内容空泛或误导             |


---

# **11. 后台功能要求**

运营后台需要支持：

1. 案例审核列表；
2. 按状态筛选；
3. 按城市筛选；
4. 按商家筛选；
5. 按服务项目筛选；
6. 查看案例详情；
7. 查看原始素材和脱敏素材；
8. 查看用户授权状态；
9. 查看关联订单；
10. 查看退款和售后状态；
11. 查看风控命中原因；
12. 审核通过；
13. 驳回修改；
14. 风险锁定；
15. 下线已发布案例；
16. 批量审核，后期；
17. 审核日志查看。

---

# **12. 审核通过后的动作**

审核通过后，系统执行：

1. 更新案例状态为 `APPROVED`；
2. 生成公开案例页；
3. 写入 SEO 元信息；
4. 生成结构化数据；
5. 加入站点地图；
6. 进入搜索索引；
7. 展示到门店主页；
8. 展示到相关服务项目页；
9. 展示到相关车型页；
10. 通知商家审核通过。

---

# **13. 审核驳回后的动作**

审核驳回后，系统执行：

1. 更新案例状态为 `REJECTED` 或 `NEED_MODIFY`；
2. 记录驳回原因；
3. 通知商家修改；
4. 保留审核日志；
5. 限制该案例公开；
6. 如涉及严重违规，记录风控事件。

---

# **14. 已发布案例下线规则**

以下情况需要下线案例：

1. 用户撤回授权；
2. 发现隐私泄露；
3. 商家资质异常；
4. 门店关闭；
5. 订单发生严重纠纷；
6. 案例内容被举报成立；
7. 违反平台内容规范；
8. 监管或法律要求；
9. SEO 页面质量严重不合格；
10. 商家请求下线且平台同意。

下线后：

```
case_status = OFFLINE
public_page_status = OFFLINE

```

并从站点地图中移除或标记不可索引。

# **16. P0 验收标准**

公开案例审核上线前必须满足：

1. 案例公开前必须审核；
2. 图片必须完成脱敏；
3. 用户未授权不得公开；
4. 订单争议中不得公开；
5. 退款售后严重异常不得公开；
6. 平台可审核通过；
7. 平台可驳回修改；
8. 平台可下线已发布案例；
9. 审核操作必须留痕；
10. 驳回原因必须可见；
11. H5公开页只能读取已审核案例；
12. SEO字段必须可审核；
13. FAQ 至少 3 条；
14. 图片 alt 不得为空；
15. 隐私泄露案例可紧急下线；
16. 商家可查看审核结果。
