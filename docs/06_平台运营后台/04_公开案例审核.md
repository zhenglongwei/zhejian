## **2. 公开案例定义**

> 从**服务相册（用户授权）**或**商家历史案例**生成，经脱敏、授权、审核后，在 H5 公开展示的维修案例内容。

用途：用户参考、门店主页、服务/城市/车型聚合页、SEO/GEO 收录、AI 可引用摘要。

---

## **3. 案例来源**

| 来源 | 说明 | 标签 |
| --- | --- | --- |
| **用户授权案例** | 车主授权服务相册脱敏公开 | 用户授权案例 |
| **商家历史案例** | 商家上传过往维修并责任确认 | 商家历史案例 |
| 平台整理案例 | 合规素材整理（少量） | 平台展示审核 |

**删除**：「订单服务相册 / 平台订单案例」。

---

## **4. 公开前置条件**

1. 案例已生成，有标题、服务项目、车型信息；
2. 有维修过程图片且**全部脱敏**；
3. 无车牌、VIN、手机号、人脸等敏感信息；
4. **用户授权案例**：授权有效且未撤回；
5. **历史案例**：商家责任确认已完成；
6. 商家/门店状态正常；
7. 无进行中的隐私投诉、配件纠纷举报；
8. 内容合法，通过基础安全检测。

---

## **5. 审核对象**

案例标题/摘要、服务项目、车型、门店、维修过程逻辑、脱敏图片、价格（区间/影响因素）、FAQ、SEO 字段、授权状态、商家资质、风险状态。

**删除**审核维度：订单关联、退款、售后、交易评价。

---

## **6. 案例审核状态**

DRAFT → WAIT_DESENSITIZE → DESENSITIZED → WAIT_AUDIT → AUDITING → APPROVED → PUBLISHED；以及 REJECTED、NEED_MODIFY、OFFLINE、RISK_LOCKED。

---

## **7. 审核流程**

```text
相册合规 + 授权
→ 生成案例草稿 → 自动脱敏 → 内容安全检测
→ 商家补充标题/FAQ → 提交公开审核
→ 平台审核 → 通过 → 生成 H5 页 → sitemap/索引
```

风险流程：命中隐私/违规/纠纷 → 风险锁定 → 人工复核。

---

## **8. 审核规则（摘要）**

### **8.1 真实性**

须有可追溯的服务相册或历史上传记录；禁止拼凑、盗图、虚构价格/故障。

### **8.2 隐私脱敏**

车牌、VIN、人脸、证件、保单、单据须隐藏；不通过则退回脱敏或替换图片。

**运营审核可见范围（定稿 2026-06-01）**：

| 审核员可见 | 审核员不可见 |
| --- | --- |
| 脱敏图 | 原图 |
| OCR / 风险摘要 | 原图 URL（Admin API **不返回** `rawUrl`） |
| 脱敏状态（已脱敏 / 需人工 / 失败） | — |

审核员依据 **脱敏图 + OCR 摘要** 判断；脱敏异常时可「重试脱敏」或「要求修改」，**不得在审核台查看原图**。

### **8.3 内容合规**

禁止违法违规、虚假宣传、外部联系方式、诱导违规导流。

### **8.4 商业宣传**

允许陈述门店与项目；禁止「全城最低」「不修包赔」「加微信报价」等。

### **8.5 价格展示**

标准项目可用区间；复杂/事故车须体现「以到店检测为准」；**不得暗示平台统一报价**。

### **8.6 SEO 字段**

title、description、H1、FAQ（≥3 条）、alt、slug、schema 须合规，不堆砌关键词。

---

## **9. 用户授权规则**

### **9.1 用户授权案例须授权**

1. 维修过程图片可脱敏后展示；
2. 车辆局部图片可脱敏后展示；
3. 案例可用于 H5 与 SEO/GEO 收录。

**删除**：评价内容公开授权（V1.0）。

### **9.2 不可公开**

用户拒绝/撤回授权、无法脱敏、严重事故/保险/法律纠纷、配件争议未结案、平台判断有风险。

---

## **10. 驳回原因（示例）**

| 类型 | 示例 |
| --- | --- |
| 隐私未脱敏 | 含车牌/VIN/人脸 |
| 授权不足 | 用户未授权或已撤回 |
| 配件留痕不足 | 非原厂案例缺少确认记录（若对外宣称已告知） |
| 内容不真实 | 文案与图片不符 |
| 违规宣传 | 外部联系方式、夸大承诺 |
| 门店/资质异常 | 商家冻结 |
| 纠纷/投诉 | 隐私或配件举报进行中 |
| SEO 不合规 | 关键词堆砌 |

---

## **11. 后台功能要求**

案例列表与筛选、详情、**脱敏图素材（不含原图）**、OCR/风险摘要、**授权记录**、**关联服务相册**、风控原因、通过/驳回/下线、审核日志。

**删除**：查看关联订单、退款/售后状态。

---

## **12. 审核通过后**

更新 APPROVED → 生成公开页 → SEO/schema → sitemap → 门店/服务/车型页展示 → 通知商家。

---

## **13. 已发布案例下线**

用户撤回授权、隐私泄露、商家/门店异常、配件/内容举报成立、监管要求、SEO 质量严重不合格等 → OFFLINE，移出 sitemap。

**删除**下线理由：「订单严重纠纷」→ 改为「配件/隐私纠纷举报成立」。

## **15. P0 验收标准**

1. 公开前必须审核；
2. 图片必须脱敏，用户未授权不得公开；
3. 配件/隐私争议中不得公开；
4. 支持通过、驳回、下线；
5. 操作留痕，H5 仅读已审核案例；
6. FAQ ≥3 条，SEO 字段可审；
7. 案例来源标签正确（用户授权 / 商家历史）；
8. **运营审核台仅展示脱敏图，不展示、不返回原图**。
