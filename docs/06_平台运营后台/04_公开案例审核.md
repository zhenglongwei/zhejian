## **2. 公开案例定义**

> 从**服务相册（用户确认发布到公开网站）**或**商家历史案例**生成，经脱敏、用户发布确认、**闸门 B 风险分流过审**后，在 H5 公开展示的维修案例内容。

用途：用户参考、门店主页、服务/城市/车型聚合页、SEO/GEO 收录、AI 可引用摘要。

| --- | --- | --- |
| **用户发布案例**（运营台可仍标「用户授权案例」过渡） | 车主确认发布服务相册脱敏内容到公开网站 | 用户发布案例 |

**删除**：「订单服务相册 / 平台订单案例」。

## **4. 公开前置条件**

1. 案例已生成，有标题、服务项目、车型信息；
2. **用户发布案例**：**不要求** 必有过程图；若有图则公开 payload **仅含已脱敏** URL（脱敏失败图不进入 H5）；
3. **商家历史 / 冷启动**：须有过程图且脱敏完成（冷启动另受 GEO 证据链 block）；
4. 无车牌、VIN、手机号、人脸等敏感信息（对已公开素材）；
5. **用户发布案例**：车主已确认发布且未撤回；
6. **历史案例**：商家责任确认已完成；
7. 商家/门店状态正常；
8. 无进行中的隐私投诉、配件纠纷举报；
9. 内容合法，通过基础安全检测。

---

## **5. 审核对象**

案例标题/摘要、服务项目、车型、门店、维修过程逻辑、脱敏图片、价格（区间/影响因素）、FAQ、SEO 字段、授权状态、商家资质、风险状态。

**删除**审核维度：订单关联、退款、售后、交易评价。

---

## **6. 案例审核状态**

DRAFT → WAIT_DESENSITIZE → DESENSITIZED → WAIT_AUDIT → AUDITING → APPROVED → PUBLISHED；以及 REJECTED、NEED_MODIFY、OFFLINE、RISK_LOCKED。

## **7. 审核流程（双闸门 · 2026-07-08 · 风险分流 2026-07-24）**

```text
商家正式提交完工 → pre-mask（后端，不变）
→ 闸门 A：相册完工合规（自动规则 + 抽检）→ 驳回则商家改相册
→ A 通过：用户端冻结展示「门店已提交，内容待您确认」
→ 用户确认「发布到公开网站」→ CaseSnapshot → 计算 gateBRisk
→ 闸门 B：
   · low → 自动过审并上线（H5 / sitemap）；约 10% 进入抽检队列（不挡上线）
   · high → pending_review 人审 → 通过后上线
→ 抽检不通过 → offline（保留 snapshot）
```

**运营后台**：分 **两个 Tab** —「相册完工合规（A）」与「案例公开审核（B）」。  
**B 列表**：待审核（高风险人审）+ **自动过审抽检** + 已通过 / 已驳回。

**B 驳回**：须告知原因；用户可重试/手工脱敏或改评价；**不解锁商家**（除非用户主动撤回）。  
**抽检下架**：不改 snapshot；公开链不可见；留痕 `spot_check_fail`。

风险流程：命中隐私/违规/纠纷 → 记为 high → 人审；抽检发现风险 → 下架复核。

## **8. 审核规则（摘要）**

### **8.0 闸门分工（2026-07-08）**

| | 闸门 A（完工） | 闸门 B（公开网站） |
| --- | --- | --- |
| 对象 | 服务相册留档 | 公开案例 + 用户发布确认包 |
| 重点 | 合法合规 | 脱敏 + 用户评价/配图 |
| 驳回回到 | 商家 | 用户 |

### **8.1 真实性**

须有可追溯的服务相册或历史上传记录；禁止拼凑、盗图、虚构价格/故障。

### **8.2 隐私脱敏**

车牌、VIN、人脸、证件、保单、单据须隐藏；不通过则退回脱敏或替换图片。

**运营审核可见范围（定稿 2026-06-01）**：

| 审核员可见 | 审核员不可见 |
| --- | --- |
| 脱敏图 | 原图 |
| OCR / 风险摘要 | 原图 URL（Admin API **不返回** `rawUrl`） |
| 脱敏状态（已脱敏 / 需人工 / 失败） | — |

审核员依据 **脱敏图 + OCR 摘要** 判断；脱敏异常时可「重试脱敏」或「要求修改」，**不得在审核台查看原图**。

### **8.3 内容合规**

禁止违法违规、虚假宣传、外部联系方式、诱导违规导流。

### **8.4 商业宣传**

允许陈述门店与项目；禁止「全城最低」「不修包赔」「加微信报价」等。

### **8.5 价格展示**

标准项目可用区间；复杂/事故车须体现「以到店检测为准」；**不得暗示平台统一报价**。

### **8.6 SEO 字段**

title、description、H1、FAQ（≥3 条）、alt、slug、schema 须合规，不堆砌关键词。

### **8.7 快照冻结与运营编辑边界（定稿 2026-07-09 · 卷九）**

| 区域 | 公示后（`snapshotVersion ≥ 1`） | API / UI |
| --- | --- | --- |
| **快照正文**（title/body/nodes/过程图） | **只读** | `PUT` 改 snapshot 域 → `409 SNAPSHOT_FROZEN` |
| **提炼层**（aiSummary/seo/faq/topic） | 可编辑 | `PUT /admin/cases/:id/enrichment` |
| **运营 LLM 润色** | **禁止 adopt 写 snapshot** | `CaseGeoLlmReview` 在 `snapshotFrozen` 时隐藏 |
| **代商家改相册** | **禁止** | 商家授权后相册 409；B 驳回不解锁 |

**审核通过**：仅更新 `status`、`published_h5`、enrichment 发布字段；**不** rebuild snapshot。  
**合规-only**：B 闸仍只审脱敏 + 用户侧内容；不因六阶段缺项驳回（§9.3 不变）。

## **9. 用户发布规则**（修订 2026-07-18 · 风险分流 2026-07-24）

> 历史用语「用户授权」= 车主确认 **发布到公开网站**。口径 §4.5：用户点一下才上线；闸门 B 为 **风险分流**（**废止**「必须人工审核才公开」全量要求）。

### **9.1 用户发布案例须有发布确认**

1. 维修过程图片可脱敏后展示；
2. 车辆局部图片可脱敏后展示；
3. 案例可用于 H5 与 SEO/GEO 收录。

**删除**：评价内容公开授权（V1.0）。

### **9.2 不可公开**

用户拒绝/撤回发布、**已展示素材**无法合规脱敏、严重事故/保险/法律纠纷、配件争议未结案、平台判断有 **合规/隐私** 风险。

**删除 / 不适用于用户发布案例**：因「六阶段不全 / 无过程图 / GEO 证据 block」 alone 驳回（留档质量归商家，车主发布用于监督与分享）。

### **9.3 用户发布案例 · 闸门 B 口径（定稿 2026-07-08 · 文案 2026-07-18 · 风险分流 2026-07-24）**

| 审核项 | 是否作为 B 门槛 |
| --- | --- |
| 发布确认有效、未撤回 | **是** |
| 已展示脱敏图的 OCR / 隐私风险 | **是**（非 low / 需人工 → **high**，人审） |
| 用户评价/配图合规 | **是**（PII / 绝对化 → **high**） |
| 进行中隐私/配件纠纷、事故模板 | **是** → **high** |
| 违规宣传、外部导流（商家留档） | **否**（应在 **闸门 A** 拦截） |
| 六阶段 / 关键节点 / 过程图数量 | **否** |
| 全部图片脱敏任务完成 | **否**（失败图不公开；用户可重试脱敏） |

**分流结果**：

| `gateBRisk` | 结果 |
| --- | --- |
| **low** | 自动 `approved`/`published`；可选 `spotCheck=pending`（默认约 10%） |
| **high** | `pending_review`，人审通过后才上网 |

**B 驳回 UX**：须返回结构化 `rejectReason` + 用户可操作项（`retry_desensitize` / `manual_desensitize` / `edit_review`）。  
**抽检 UX**：抽检队列仅针对已自动过审案；不通过 → 下架；**不得**把抽检做成再次全量人审。

## **10. 驳回原因（示例）**

| 类型 | 示例 |
| --- | --- |
| 隐私未脱敏 | 含车牌/VIN/人脸 |
| 发布确认不足 | 用户未确认发布或已撤回 |
| 配件留痕不足 | 非原厂案例缺少确认记录（若对外宣称已告知） |
| 内容不真实 | 文案与图片不符 |
| 违规宣传 | 外部联系方式、夸大承诺 |
| 门店/资质异常 | 商家冻结 |
| 纠纷/投诉 | 隐私或配件举报进行中 |
| SEO 不合规 | 关键词堆砌 |

## **12. 审核通过后**

更新 APPROVED → 生成公开页 → SEO/schema → sitemap → 门店/服务/车型页展示 → 通知商家。

---

## **13. 已发布案例下线**

用户撤回授权、隐私泄露、商家/门店异常、配件/内容举报成立、监管要求、 SEO 质量严重不合格等 → OFFLINE，移出 sitemap。

**举报下线（2026-07-09 · CASE-OPS-06）**：仅改 `status=offline` + 清空 `published_at`；**不得**删除或改写 `content_json.snapshot` / `enrichment_json`。须写入 `case_review_log`（`review_action=report_offline`，含 `snapshotVersion` / `snapshotPreserved` 摘要）。

**删除**下线理由：「订单严重纠纷」→ 改为「配件/隐私纠纷举报成立」。

## **15. P0 验收标准**

1. 公开前必须审核；
2. 图片必须脱敏，用户未授权不得公开；
3. 配件/隐私争议中不得公开；
4. 支持通过、驳回、下线；
5. 操作留痕，H5 仅读已审核案例；
6. FAQ ≥3 条，SEO 字段可审；
7. 案例来源标签正确（用户发布 / 商家历史；过渡期可显示「用户授权」）；
8. **运营审核台仅展示脱敏图，不展示、不返回原图**。
