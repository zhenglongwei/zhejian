---

## **2. 模块定位**

维修相册审核用于检查商家在订单履约过程中上传的维修图片和说明是否符合平台要求。

主要服务于：

1. 订单履约可信记录；
2. 用户查看维修过程；
3. 售后争议处理；
4. 商家服务质量评估；
5. 公开案例生成；
6. 风控识别异常订单；
7. 平台透明维修能力建设。

---

## **3. 相册来源**

维修相册来源包括：


| 来源     | 说明         |
| ------ | ---------- |
| 平台订单相册 | 商家在订单履约中上传 |
| 历史维修相册 | 商家补充历史案例上传 |
| 售后补充相册 | 售后争议中补充上传  |
| 平台整理素材 | 平台人工整理后补充  |


---

## **4. 审核对象**


| 对象    | 审核内容            |
| ----- | --------------- |
| 图片    | 清晰度、真实性、隐私、违规内容 |
| 节点    | 维修前、维修中、维修后是否完整 |
| 文案说明  | 是否真实、是否夸大、是否含隐私 |
| 订单关联  | 是否和订单服务项目一致     |
| 车辆信息  | 是否暴露车牌、VIN、人脸等  |
| 公开适用性 | 是否适合生成公开案例      |
| 风险状态  | 是否命中风控规则        |


---

## **5. 相册节点要求**

根据服务类型不同，维修相册应包含不同节点。

### **5.1 通用节点**


| 节点    | 是否必填   | 说明              |
| ----- | ------ | --------------- |
| 维修前   | 必填     | 展示车辆/部件维修前状态    |
| 检测过程  | 复杂服务必填 | 展示检测、拆检、诊断过程    |
| 维修中   | 建议必填   | 展示施工或更换过程       |
| 维修后   | 必填     | 展示维修完成效果        |
| 结算/交付 | 可选     | 展示交付状态，不得包含隐私单据 |


---

### **5.2 不同服务的节点要求**


| 服务类型  | 节点要求                   |
| ----- | ---------------------- |
| 基础养护  | 维修前、维修后                |
| 配件更换  | 旧件、新件、安装过程、完成效果        |
| 钣喷维修  | 损伤前、打磨/修复中、喷涂后、完成效果    |
| 事故车维修 | 检测、拆解、维修中、完成效果，需谨慎公开   |
| 新能源维修 | 检测、绝缘/高压安全处理、维修过程、完成效果 |
| 美容清洗  | 施工前、施工中、施工后            |

## **7. 审核流程**

### **7.1 标准流程**

```
商家上传维修相册
→ 系统校验图片数量和节点
→ 自动隐私检测
→ 命中风险则进入审核
→ 平台审核图片和说明
→ 通过/要求修改/驳回
→ 可生成公开案例草稿

```

---

### **7.2 公开案例前流程**

```
维修相册审核通过
→ 自动脱敏
→ 生成案例草稿
→ 商家补充标题/摘要/FAQ
→ 提交公开案例审核

```

---

### **7.3 风险流程**

```
上传相册
→ 命中隐私/虚假/违规风险
→ 风险锁定
→ 平台人工复核
→ 退回脱敏/要求修改/禁止公开/记录风控

```

---

## **8. 审核重点**

### **8.1 图片是否清晰**

审核要点：

1. 图片主体是否清楚；
2. 维修部位是否可辨认；
3. 是否严重模糊；
4. 是否过暗、过曝；
5. 是否被遮挡；
6. 是否为重复图片；
7. 是否上传无关图片。

处理方式：


| 情况       | 处理    |
| -------- | ----- |
| 轻微模糊但可识别 | 可通过   |
| 关键节点模糊   | 要求重传  |
| 无法判断维修内容 | 驳回    |
| 无关图片     | 删除或驳回 |


---

### **8.2 是否包含敏感信息**

敏感信息包括：

1. 车牌号；
2. VIN；
3. 人脸；
4. 手机号；
5. 身份证；
6. 行驶证；
7. 驾驶证；
8. 保险单；
9. 维修单据；
10. 客户资料；
11. 门店客户名单；
12. GPS 位置信息。

处理方式：


| 风险        | 处理        |
| --------- | --------- |
| 轻微隐私，支持脱敏 | 退回脱敏或平台脱敏 |
| 明显车牌/VIN  | 不得公开，需脱敏  |
| 证件/保单完整露出 | 风险锁定      |
| 用户强隐私泄露   | 先隐藏，后审核   |


---

### **8.3 节点是否完整**

审核要点：

1. 是否包含维修前图片；
2. 是否包含维修后图片；
3. 复杂维修是否包含检测过程；
4. 更换配件是否有旧件和新件；
5. 钣喷是否有施工过程；
6. 事故车是否有拆检记录；
7. 图片顺序是否合理。

不通过示例：

- 只有完工图，无维修前图；
- 高金额维修无过程图；
- 复杂维修没有检测记录；
- 图片与订单服务项目不匹配。

---

### **8.4 内容是否真实可信**

审核要点：

1. 图片是否与订单服务一致；
2. 图片前后是否连续；
3. 是否存在拼凑图片；
4. 是否疑似盗用网络图片；
5. 是否与门店环境明显不一致；
6. 是否同一图片重复用于多个订单；
7. 维修耗时和图片内容是否合理；
8. 高金额订单是否有足够过程证据。

命中以下情况进入风控：

1. 多个订单使用相同图片；
2. 高金额订单无相册；
3. 复杂维修相册节点缺失；
4. 商家大量订单快速完成但相册雷同；
5. 图片元信息异常，后续可做。

---

### **8.5 是否存在夸大描述**

不允许出现：

1. “100%修好”；
2. “全网最低”；
3. “原厂认证”但无资质；
4. “不修包赔”；
5. “永久质保”但无规则；
6. “事故车一口价修复”；
7. “绕过平台更便宜”；
8. 加微信、电话私下成交。

允许表达：

- “本次更换刹车片”；
- “维修前后效果对比”；
- “具体价格以检测结果为准”；
- “施工耗时约 2 小时”。

---

### **8.6 是否适合公开**

适合公开的相册需满足：

1. 图片清晰；
2. 节点完整；
3. 维修过程真实；
4. 已完成脱敏；
5. 无车牌、VIN、人脸；
6. 无单据、证件、客户资料；
7. 订单无严重售后或纠纷；
8. 用户授权符合要求；
9. 文案无夸大宣传；
10. 价格可使用区间表达。

不适合公开的情况：

1. 用户拒绝公开；
2. 事故隐私较重；
3. 保险理赔相关；
4. 法律纠纷相关；
5. 高识别度稀有车型且用户未明确授权；
6. 图片无法完全脱敏；
7. 内容质量过低。

---

## **9. 审核操作**


| 操作     | 说明              |
| ------ | --------------- |
| 通过     | 相册审核通过，可用于订单留痕  |
| 要求修改   | 商家补充或替换图片       |
| 驳回     | 相册不符合要求         |
| 退回脱敏   | 图片存在隐私但可处理      |
| 标记不可公开 | 可用于订单，但不能生成公开案例 |
| 标记风险   | 生成风险事件          |
| 临时隐藏   | 隐私投诉或高风险时先隐藏    |
| 生成案例草稿 | 相册符合公开条件时可生成案例  |


---

## **10. 驳回 / 修改原因**


| 类型    | 示例           |
| ----- | ------------ |
| 图片不清晰 | 关键维修部位无法识别   |
| 节点缺失  | 缺少维修前或维修后    |
| 隐私未脱敏 | 含车牌、VIN、人脸   |
| 内容不真实 | 图片与订单服务不一致   |
| 重复图片  | 多节点使用相同图片    |
| 违规宣传  | 含外部联系方式或夸大描述 |
| 不适合公开 | 涉及事故隐私、保险理赔  |
| 订单异常  | 订单存在退款、投诉或售后 |

## **13. 与风控的关系**

维修相册审核需与风控联动：


| 风控场景      | 处理      |
| --------- | ------- |
| 高金额订单无相册  | 标记订单异常  |
| 复杂维修无检测图  | 进入人工审核  |
| 图片含车牌/VIN | 进入内容风险  |
| 多订单图片重复   | 标记商家风险  |
| 相册缺失但订单完成 | 暂缓奖励或结算 |
| 相册涉及隐私投诉  | 先隐藏后审核  |


---

## **14. P0 验收标准**

必须满足：

1. 商家可上传维修相册；
2. 相册支持按节点上传；
3. 平台可查看维修相册；
4. 平台可查看原图和脱敏图；
5. 支持图片清晰度人工判断；
6. 支持隐私信息审核；
7. 支持节点完整性审核；
8. 支持真实性审核；
9. 支持是否适合公开判断；
10. 支持通过、驳回、要求修改；
11. 支持退回脱敏；
12. 支持标记不可公开；
13. 支持生成公开案例草稿；
14. 审核操作必须留痕；
15. 高风险相册可进入风控；
16. 含隐私图片不得直接公开。
