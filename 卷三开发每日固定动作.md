# 卷三开发 · 每日固定动作

> **这是什么**：卷三阶段的**每日开发 checklist**，告诉你「今天开工前看什么、写代码时注意什么、收工前勾什么」。  
> **不是**：一份要一次性读完的 PRD；**不用背**，按当天任务查对应小节即可。  
> **进度看哪里**：`docs/00_开发计划.md` §7.1（任务 ID 如 B-MASK-03）

---

## 怎么用（3 分钟读懂）

### 你每天只需要做 4 件事

| 时机 | 你要做什么 | 看文档哪一节 |
| --- | --- | --- |
| **早上开工** | 定今天做哪个任务 ID → 打开对应 PRD → 让 AI 先出方案 | [§1 开做前](#1-开做前每天第一件事) |
| **写代码中** | 对照 checklist，别踩坑（假脱敏、原图泄露等） | [§2 开发中](#2-开发中按任务类型) |
| **今天这块做完** | 让 AI 跑对应检查 skill | [§3 写完一块](#3-写完一块当天或-pr-前) |
| **下班 / 提交前** | 勾开发计划、写明天第一条任务 | [§6 每日收尾](#6-每日收尾5-分钟) |

**发体验版、合并 PR** 时才需要 [§4 合并前](#4-合并前)、[§5 发体验版](#5-发体验版--上预发前)——不是每天都要做。

## 0. 本阶段与卷一的差异

| 维度 | 卷一（已完成） | 卷三（当前） |
| --- | --- | --- |
| 主战场 | 小程序用户端 + 商家相册 + 读 API | **后端脱敏引擎** + **运营审核后台** |
| 脱敏 | 流程闸门 + URL 隔离（MVP 复制原图） | **真实 OCR/打码**（B-MASK-03） |
| 必读 PRD | `02_用户端` / `04_维修过程相册` | **`06_平台运营后台`** + `06_自动脱敏规则` |
| 实现目录 | `pages/`、`components/`、`packageMerchant/` | **`backend/`**、**`/admin/`**（待建） |
| 发版门槛 | `release-checklist` 全端冒烟 | 脱敏准确率 + 审核台 + 关闭自动通过入驻 |

## 1. 开做前（每天第一件事）

1. **Read 开发计划**：打开 `docs/00_开发计划.md`，确认今日任务 ID（§7.1.1 优先 B-MASK-03 → B-MASK-04 → B-MASK-06 → OPS-MASK-01）。
2. **Read 对应 PRD**（按任务选读，不要凭记忆）：

   | 任务域 | 必读 PRD |
   | --- | --- |
   | B-MASK-03/04/06 | `docs/04_维修过程相册/06_自动脱敏规则.md`、`08_图片脱敏工具PRD.md`、`docs/10_技术架构与接口/05_图片视频上传API.md` §12 |
   | OPS-MASK-01 / A-PUB-06 | `docs/06_平台运营后台/04_公开案例审核.md`、`03_维修相册审核.md` |
   | B-MERCH-04 | `docs/06_平台运营后台/02_商家审核管理.md` |
   | 风控/下架 | `docs/06_平台运营后台/00_风控与合规审核PRD V2.0.md` |

3. **Read 现有实现**（改 backend 前必做）：
   - `backend/src/services/media.service.js`（当前复制 stub）
   - `backend/src/services/desensitize.service.js`
   - `backend/src/routes/desensitize.js`、`backend/src/routes/media.js`

4. **非 trivial 变更**：先输出「需求理解 → 涉及文件 → 方案 → 是否改表/改契约」，再写代码。

5. **指定 skill**（见 §7）：
   - backend 脱敏 → `desensitize-engine`
   - **运营 Web 新页** → **`ops-admin-scaffold`**（先出计划，确认后再写 `admin-web/`）
   - 小程序/H5 新页 → `page-scaffold`

## 2. 开发中（按任务类型）

### 2.1 脱敏引擎（B-MASK-03/04/06）

- [ ] 替换 `runMediaDesensitize` 复制逻辑，**禁止**静默把原图当脱敏图
- [ ] 单张失败 → `NEED_MANUAL` / 可重试，不得写 `SUCCESS`
- [ ] OCR/检测结果落库（B-MASK-04），供 OPS-MASK-01 展示
- [ ] 跑通：`service_pre_mask` → 工作台一键脱敏 → `maskingConfirmed` 闸门
- [ ] 用 **真实含车牌/人脸测试图** 验收，不用 seed 假图糊弄
- [ ] 第三方 API 密钥只放 `backend/.env`，勿提交仓库

### 2.2 运营审核台（OPS-MASK-01 及后续）

**开页必用 `ops-admin-scaffold`**：先输出列表/详情/审核操作计划，确认后再编码。

- [ ] 列表 + 筛选 + 分页；详情含**脱敏图**、OCR 摘要、风险等级（**不展示原图**）
- [ ] 运营 PRD §10 检查项可对照操作（通过 / 驳回 / 要求修改 / 下线）
- [ ] 仅 **审核角色** `admin_token` 可访问；**Admin API 不返回 rawUrl**；原图 URL 不下发 C 端
- [ ] Tag/合规文案与设计体系一致（已脱敏、已审核；禁「平台订单案例」）
- [ ] loading / empty / error / 防重复提交
- [ ] 构建 `base: '/admin/'`，本地 preview 路径正确

### 2.3 衔接卷一遗留

- [ ] **A-PUB-06**：无车主冷启动 → 须 B-MASK-03 + 审核通过才可公开
- [x] **B-MERCH-04**：`MERCHANT_AUTO_APPROVE=false` 与商家审核台同批上线
- [ ] 改 C 端契约时同步检查：`privacy-desensitization-check`

## 3. 写完一块（当天或 PR 前）

按改动类型触发检查（**只报告 vs 改代码** 与 skill 约定一致）：

| 改动类型 | 必跑 skill |
| --- | --- |
| 脱敏引擎 / media API | `desensitize-engine` → `privacy-desensitization-check` |
| 公开案例 / 审核流转 | `privacy-desensitization-check` |
| **运营 Web 新页**（`admin-web/`） | **`ops-admin-scaffold`** → `api-integration` → `privacy-desensitization-check` → `design-system-check`（文案/Tag） |
| 接新 API（小程序/运营台） | `api-integration` |
| 涉及价格展示 | `price-compliance-check` |
| 图片 URL/封面异常 | `miniprogram-image-debug` |

**卷三额外自查**（backend 向）：

1. 脱敏失败是否会被误标为已脱敏？
2. `media_assets` / 任务表是否有版本号或审计字段？
3. 生产 `deploy:verify` / health 是否仍通过？
4. 是否误暴露 `rawUrl` 给公开读 API？

---

## 4. 合并前

1. **`ops-admin-scaffold` 自查**（若改 admin-web）：鉴权、原图隔离、审核留痕、无 V1 订单/评价入口
2. `component-usage-review`（若有新 UI）
3. `design-system-check`（若有新 UI）
4. 更新 `docs/00_开发计划.md`：完成项 `[ ]` → `[x]`，§8 版本记录一行
5. PR 描述写清：任务 ID、验收口径、是否依赖 env/迁移

---

## 5. 发体验版 / 上预发前

1. **`release-checklist`**（全端；卷三新增 backend 脱敏真实验收项）
2. **`privacy-desensitization-check`**（必须用真实打码图抽测公开案例/H5）
3. 确认 `MERCHANT_AUTO_APPROVE` 与环境变量文档一致
4. 运营台仅内网或鉴权后可访问

---

3. 明日第一条任务写在日历/Issue 标题里（带 ID，如 `B-MASK-03`）
