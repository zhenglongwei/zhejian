# 卷三开发 · 每日固定动作

> **适用阶段**：卷一 P0 验收通过后 → 卷三 ops（优先 B-MASK-03）→ 卷二/卷四  
> **进度唯一来源**：`docs/00_开发计划.md` §7.1

## 0. 本阶段与卷一的差异

| 维度 | 卷一（已完成） | 卷三（当前） |
| --- | --- | --- |
| 主战场 | 小程序用户端 + 商家相册 + 读 API | **后端脱敏引擎** + **运营审核后台** |
| 脱敏 | 流程闸门 + URL 隔离（MVP 复制原图） | **真实 OCR/打码**（B-MASK-03） |
| 必读 PRD | `02_用户端` / `04_维修过程相册` | **`06_平台运营后台`** + `06_自动脱敏规则` |
| 实现目录 | `pages/`、`components/`、`packageMerchant/` | **`backend/`**、**`/admin/`**（待建） |
| 发版门槛 | `release-checklist` 全端冒烟 | 脱敏准确率 + 审核台 + 关闭自动通过入驻 |

## 1. 开做前（每天第一件事）

1. **Read 开发计划**：打开 `docs/00_开发计划.md`，确认今日任务 ID（§7.1.1 优先 B-MASK-03 → B-MASK-04 → B-MASK-06 → OPS-MASK-01）。
2. **Read 对应 PRD**（按任务选读，不要凭记忆）：

   | 任务域 | 必读 PRD |
   | --- | --- |
   | B-MASK-03/04/06 | `docs/04_维修过程相册/06_自动脱敏规则.md`、`08_图片脱敏工具PRD.md`、`docs/10_技术架构与接口/05_图片视频上传API.md` §12 |
   | OPS-MASK-01 / A-PUB-06 | `docs/06_平台运营后台/04_公开案例审核.md`、`03_维修相册审核.md` |
   | B-MERCH-04 | `docs/06_平台运营后台/02_商家审核管理.md` |
   | 风控/下架 | `docs/06_平台运营后台/00_风控与合规审核PRD V2.0.md` |

3. **Read 现有实现**（改 backend 前必做）：
   - `backend/src/services/media.service.js`（当前复制 stub）
   - `backend/src/services/desensitize.service.js`
   - `backend/src/routes/desensitize.js`、`backend/src/routes/media.js`

4. **非 trivial 变更**：先输出「需求理解 → 涉及文件 → 方案 → 是否改表/改契约」，再写代码。

5. **指定 skill**（见 §5）：卷三 backend/审核任务先 Read 对应 skill，再动手。

## 3. 写完一块（当天或 PR 前）

按改动类型触发检查（**只报告 vs 改代码** 与 skill 约定一致）：

| 改动类型 | 必跑 skill |
| --- | --- |
| 脱敏引擎 / media API | `desensitize-engine`（待建，见 §5）→ `privacy-desensitization-check` |
| 公开案例 / 审核流转 | `privacy-desensitization-check` |
| 运营台新页面 | `ops-admin-scaffold`（Web `/admin/`）→ `design-system-check`（文案/Tag） |
| 接新 API（小程序/运营台） | `api-integration` |
| 涉及价格展示 | `price-compliance-check` |
| 图片 URL/封面异常 | `miniprogram-image-debug` |

**卷三额外自查**（backend 向）：

1. 脱敏失败是否会被误标为已脱敏？
2. `media_assets` / 任务表是否有版本号或审计字段？
3. 生产 `deploy:verify` / health 是否仍通过？
4. 是否误暴露 `rawUrl` 给公开读 API？
